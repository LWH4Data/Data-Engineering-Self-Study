<h1>초기 설정에 대한 자세한 부분은 도서를 참고.</h1>

<h1>1. Build system의 그레이들과 메이븐의 차이</h1>
<ul>
  <li>
    우리가 작성한 코드는 바로 사용할 수는 없는 코드이고 사용자가 바로 사용하게 하기 위해서는 <strong>빌드</strong>의 과정을 거쳐야 한다..
  </li>
  <li>
    빌드 과정에 <strong>필요한 것들을 관리</strong>하고 <strong>자동화</strong>하는 도구를 빌드 관리 도구ㄹ ㅏ한다.
  </li>
    <ul>
      <li>
        의존성 내려받기, 코드 패키징, 테스트 실행 등이 포함된다.
      </li>
    </ul>
  <li>
    과거에는 메이븐이 주로 사용되었으나 최근에는 <strong>그레이들</strong>을 많이 사용하며 다음과 같은 특징이 있다.
  </li>
    <ul>
      <li>
        메이븐에 비하여 가독성이 좋고 설정이 간단하다.
      </li>
      <li>
        자바, 코틀린, 그루비 등 <strong>다양한 언어</strong>를 지원하며 원하는 대로 빌드 스크립트를 작성할 수 있다.
      </li>
      <li>
        빌드 테스트 속도가 메이븐에 비하여 더 빠르다.
      </li>
    </ul>
</ul>

<br><br>

<h1>2. 스프링 부트 4 프로젝트 실행하기</h1>
<ul>
  <li>
    <strong>build.gradle 파일</strong>은 프로젝트가 어떻게 빌드하고 어떤 의존성을 가져오며 어떻게 실행할지 등을 정의하는 <strong>설계도 역할</strong>을 한다.
  </li>
  <li>
    자세한 내용은 실습 저장소의 주석을 확인.
  </li>
    <ul>
      <li>
        localhost:8080으로 접속하면 당연히 404가 반환된다. 이는 localhost:8080으로 요청을 보내면 <strong>index.html</strong>을 찾도록 되어 있는데 index.html이 없기 때문이다.
      </li>
    </ul>
</ul>

<h1>3. 포스트맨 설치하기</h1>
<ul>
  <li>
    포스트맨은 <strong>HTTP 요청</strong>을 보낼 수 있는 <strong>클라이언트 프로그램</strong>이다.
  </li>
  <li>
    API 개발을 완료하였을 때 매번 웹 브라우저를 켜고 URL을 입력해 요청하는 작업은 불편하며 포스트맨을 사용하면 <strong>몇 번의 클릭</strong>만으로 테스트를 해볼 수 있다.
  </li>
</ul>
