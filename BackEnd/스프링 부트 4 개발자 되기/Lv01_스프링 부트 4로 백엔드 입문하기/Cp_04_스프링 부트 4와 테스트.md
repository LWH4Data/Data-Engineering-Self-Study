<ul>

  </li>
    <ul>
      <li>
        본 개발 공부로 공부를 미루느 경우가 있지만 유지보수와 코드 수정 시에 유리하기에 권장한다.
      </li>
    </ul>

<h2>1-1. 테스트 코드란?</h2>
<ul>
  <li>
    테스트 코드는 <strong>test 디렉터리</strong>에서 작업한다.
  </li>
  <li>
    다양한 방법이 존재하며 도서에서는 <strong>given-when-then 패턴</strong>을 활용한다. (세 단계로 구분하여 작성하는 방식).
  </li>
    <ul>
      <li>
        <strong>given</strong>: 테스트 실행을 <strong>준비</strong>하는 단계이다.
      </li>
      <li>
        <strong>when</strong>: 테스트를 <strong>진행</strong>하는 단계이다.
      </li>
      <li>
        <strong>then</strong>: 테스트 <strong>결과를 검증</strong>하는 단계이다.
      </li>
    </ul>
</ul>

```java
// 1. given-when-then 패턴 사례.
@DisplayName("새로운 메뉴를 저장한다.");
@Test
public void saveMenuTest() {
  // given: 메뉴를 저장하기 위한 준비 과정.
  final String name = "아메리카노";
  final int price = 2000;
  final Menu americano = new Menu(name, price);
  
  // when: 실제로 메뉴를 저장.
  final long saveId = menuService.save(americano);

  // then: 메뉴가 잘 추가 되었는지 검증.
  final Menu saveMenu = menuService.findById(savedId).get();
  assertThat(saveMenu.getName()).isEqualTo(name);
  assertThat(saveMenu.getPrice()).isEqualTo(price);
}
```

      </li>
      <li>
        Spring Test & Spring Boot Test: 스프링 부트 애플리케이션을 위한 통합 테스트 지원.

      <li>

      <li>
        JSONassert: JSON용 어설션 라이브러리.

      </li>
    </ul>
</ul>

<h2>2-1. JUnit이란?</h2>
<ul>
  <li>
    JUnit은 자바 언어를 위한 <strong>단위 테스트 프레임워크</strong>이다.
  </li>
    <ul>
      <li>
        단위 테스트란 작성한 코드가 <strong>의도대로 작동</strong>하는지 <strong>작은 단위로 검증</strong>하는 것을 의미한다.
      </li>
      <li>
        단위는 보통 <strong>메소드</strong>가 된다.
      </li>
    </ul>
  <li>
    JUnit의 특징은 다음과 같다.
  </li>
    <ul>
      <li>
        <strong>테스트 방식을 구분</strong>할 수 있는 애너테이션 제공.
      </li>
      <li>
        <strong>@Test 애너테이션</strong>으로 메소드를 호출할 때마다 새 <strong>인스턴스를 생성, 독립 테스트</strong> 가능.
      </li>
      <li>
        <strong>예상 결과를 검증</strong>하는 어설션 메소드 제공.
      </li>
      <li>
        사용 방법이 단순하고 테스트 코드 작성 시간이 적음.
      </li>
      <li>
        자동 실행, 자체 결과를 확인하고 즉각적인 피드백 제공.
      </li>
    </ul>
</ul>

<h3>2-1-1. Junit으로 단위 테스트 코드 만들기</h3>
<ul>
  <li>
    실습과 함께 참고.
  </li>
  <li>
    <strong>@DisplayName 애너테이션</strong>은 <strong>테스트 이름</strong>을 명시한다.
  </li>
  <li>
    <strong>@Test 애너테이션</strong>이 붙은 메소드는 <strong>테스트를 수행하는 메소드</strong>가 된다.
  </li>
  <li>
    JUnit은 <strong>테스트끼리 영향을 주지 않도록</strong> 각 테스트가 실행할 때마다 테스트를 위한 실행 객체를 만들고 테스트가 종료되면 실헹 객체를 삭제한다.
  </li>
    <ul>
      <li>
        assertEquals()는 첫 번째 인수에 기대하는 값, 두 번째 인수에 실제로 검증할 값ㅇ르 넣어 확인한다.
      </li>
    </ul>
  <li>
    <strong>실패를 위한 테스트 코드</strong>를 작성하여 실패한 경우를 대비할 수도 있다.
  </li>
  <li>
    JUnit은 하나의 케이스가 실패하면 전체 테스트가 실패한 것으로 표시한다.
  </li>
  <li>
    테스트는 애너테이션마다 순서가 정해진다.
  </li>
  <li>
    @BeforeEach: 각 테스트 인스턴스마다 실행 → 인스턴스 상태 초기화 → non-static
  </li>
  <li>
    @BeforeAll: 테스트 클래스 전체에서 1회 실행 → 공통 자원 초기화 → static (아직 생성된 인스턴스가 없기 때문).
  </li>
</ul>

<h3>2-1-2. AssertJ로 검증문 가독성 높이기</h3>
<ul>
  <li>
    AssertJ는 JUnit의 부족한 가독성을 높여주는 라이브러리이다.
  </li>
  <li>
    AssertJ의 다양한 메소드에 대해서는 p107 참고.
  </li>
</ul>

```java
// 1. AsssertJ 전후비교.
// 1-1. JUnit만 사용할 때
Assertions.assertEquals(sum, a + b);
// 1-2. AssertJ를 쓸 때
assertTaht(a + b).isEqualTo(sum);
```
