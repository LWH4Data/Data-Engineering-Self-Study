<h1>1. 사전 지식: 스프링 시큐리티</h1>
<ul>
  <li>
    스프링 시큐리티(spring security)는 스프링 기반의 애플리케이션 <strong>보안(인증, 인가, 권한)</strong>을 담당하는 스프링 하위 프레임워크이다.
  </li>
</ul>

<h3>1-1-1. 인증과 인가</h3>
<ul>
  <li>
    <strong>인증(authentication)</strong>은 사용자의 <strong>신원을 입증</strong>하는 과정이다.
  </li>
  <li>
    <strong>인가(authorization)</strong>는 사이트의 특정 부분에 <strong>접근할 수 있는 권한</strong>을 확인하는 작업이다.
  </li>
</ul>

<h3>1-1-2. 스프링 시큐리티</h3>
<ul>
  <li>
    스프링 시큐리티는 스프링 기반 애플리케이션의 보안을 담당하는 스프링 하위 프레임워크이다.
  </li>
  <li>
    애너테이션으로 설정이 쉬운 편이며 CSRF 공격, 세션 고청(session fixation) 공격을 방어하고 요청 헤더도 보안처리를 해준다.
  </li>
    <ul>
      <li>
        CSRF 공격: 사용자의 권한을 가지고 특정 동작을 수행하도록 유도하는 공격.
      </li>
      <li>
        세션 고정 공격: 사용자의 인증 정보를 탈취하거나 변조하는 공격.
      </li>
    </ul>
</ul>

<h4>1-1-3. 필터 기반으로 동작하는 스프링 시큐리티</h4>
<ul>
  <li>
    스프링 시큐리티는 <strong>필터 기반</strong>으로 동작하며 다양한 필터가 존재한다. 스프링 시큐리티 사용에 있어서 이를 알아두는 것이 좋다.
  </li>
    <ul>
      <li>
        각 필터에서 인증, 인가와 관련된 작업을 처리한다.
      </li>
      <li>
        SecurityCOntextPersistenceFilter부터 시작하여 FilterSecurityInterceptor까지 순서대로 이어진다.
      </li>
      <li>
        특정 필터를 제거하거나 필터 뒤에 커스텀 필터를 넣는 등의 설정이 가능하다.
      </li>
      <li>
        각 필터들의 자세한 설명은 p291 참고.
      </li>
    </ul>
  <li>
    눈여겨볼 필터는 두 가지이다. 하나는 UsernamePasswordAuthenticationFilter이고 다른 하나는 FilterSecurityInterceptor 이다.
  </li>
    <ul>
      <li>
        UsernamePasswordAuthenticationFilter는 아이디와 패스워드가 넘어오면 인증 요청을 위임하는 <strong>인증 관리자 역할</strong>을 한다.
      </li>
      <li>
        FilterSecurityInterceptor는 권한 부여 처리를 위임해 접근 제어 결정을 귑헤나느 <strong>접근 결정 관리자 역할</strong>을 한다.
      </li>
    </ul>
  <li>
    사용자가 아디이와 패스워드 입력.
    <br>→ HTTPServletRequest에 아이디와 패스워드 정보 전달.
    <br>→ AuthenticationFilter는 넘어온 아이디와 비밀번호의 유효성을 검사.
    <br>→ 유효성 검사 후 실제 구현체인 UsernamePasswordAuthenticationToken을 만들어 넘긴다.
    <br>→ 전달 받은 인증용 객체를 AuthenticationManager와 AuthenticationProvider에 보낸다.
    <br>→ 사용자 아이디는 UserDetailService에 보낸다.
    <br>→ UserDetailService는 사용자 아이디로 찾은 정보를 UserDetails 객체로 만들어 AuthenticationProvider에 전달한다.
    <br>→ DB에 있는 사용자 정보를 가져온다.
    <br>→ 입력 정보와 UserDetails 정보를 비교해 실제 인증 처리를 한다.
    <br>→ SecurityContectHolder에 Authentication을 저장한다.
    <br>→ 인증 성공 시에는 AuthenticationSuccessHandler를, 실패 시에는 AuthenticationFailureHandler를 실행한다.
  </li>
</ul>
