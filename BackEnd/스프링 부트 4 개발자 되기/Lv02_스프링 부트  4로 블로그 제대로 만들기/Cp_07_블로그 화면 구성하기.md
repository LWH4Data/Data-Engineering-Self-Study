<h1>1. 사전 지식: 타임리프</h1>
<ul>
  <li>
    타임리프는 템플릿 엔진이다.
  </li>
    <ul>
      <li>
        템플릿 엔진이란 스프링 서버에서 데이터를 받아 우리가 보는 <strong>웹 페이지(HTML) 상에 데이터를 넣어 보여주는 도구</strong>이다.
      </li>
    </ul>
  <li>
    템플릿 엔진은 HTML과 템플릿 엔진을 위한 문법을 조금 섞어 사용해야 한다.
  </li>
</ul>

<br>

<h2>1-1. 템플릿 엔진 개념 잡기</h2>
<ul>
  <li>
    템플릿 엔진은 문법이 조금씩 다르지만 한 번 배워두면 다른 템플릿 엔진은 금방 활용할 수 있다.
  </li>
</ul>

```html
<!-- 1. 템플릿 문법 예 -->
<!--   - 서버에서 받은 값으로 {}내의 값을 대체하여 동적으로 활용할 수 있다. --> 
<h1 text=${이름}>
<p text=${나이}>
```

<h3>1-3-1. 모델의 역할 살펴보기</h3>
<ul>

    컨트롤러는 <strong>모델</strong>을 통해 데이터를 설정한다.
    <br>→ 모델은 <strong>설정된 데이터를 전달</strong>해 키에 맞는 데이터를 뷰에서 조회할 수 있도록 한다.
  </li>
  <li>
    즉, 모델은 컨트롤러와 뷰의 <strong>중간다리</strong> 역할을 한다.
  </li>

<h2>3-1. 엔티티에 생성, 수정 시간 추가하기</h2>
<ul>
  <li>
    Article.java 파일에 필드 추가.
  </li>
    <ul>
      <li>
        @CreatedDate: 엔티티가 생성될 때 생성 시간을 저장한다.
      </li>
      <li>
        @LastModifiedDate: 엔티티가 수정될 때 마지막으로 수정된 시간을 저장한다.
      </li>
      <li>
        @EntityListeners(AuditingEntityListener.class): 엔티티의 생성 및 수정 시간을 자동으로 감시하고 기록한다.
      </li>
    </ul>
  <li>
    import.sql을 수정하여 섭서가 시작될 때 create_at과 update_at이 바뀌도록 한다.
  </li>
  <li>
    마지막으로 SpringBootDeveloperApplication.java 파일에 created_at과 udpate_at을 자동으로 업데이트하기 위한 애너테이션을 추가한다.
  </li>
    <ul>
      <li>
        @EnableJpaAuditing: created_at과 update_at을 자동으로 업데이트.
      </li>
    </ul>
</ul>

<h1>5. 수정/생성 기능 추가하기</h1>
<h2>수정/생성 뷰 컨트롤러 작성하기</h2>
<ul>
  <li>
    생성과 수정은 버튼과 기존 값의 유무가 다르기에 <strong>URL의 설계와 흐름</strong>도 다르다.
  </li>
    <ul>
      <li>
        글을 생성할 때에는 별도의 <strong>쿼리 파라미터</strong>가 없지만 수정을 할 때에는 URL에 ?id=123 등 쿼리 파라미터가 있다.
      </li>
        <ul>
          <li>
            쿼리 파라미터란 HTTP 요청에서 <strong>'?'로 시작</strong>하는 키 값으로 이루진 문자열이며 <strong>'&'</strong>로 구분한다.
          </li>
        </ul>
    </ul>
  <li>
    <strong>쿼리 파라미터가 있는 경우</strong> 수정 요청이기에 글 항목을 보여주고 <strong>쿼리 파라미터가 없는 경우</strong>에는 새 글이기에 화면에 아무것도 보여줄 필요가 없다.
  </li>
    <ul>
      <li>
        더하여 뷰에서는 쿼리 파라미터의 id 여부에 따라 [수정]과 [생성] 중 적절한 버튼을 보여주어야 한다.
      </li>
    </ul>
  <li>
    수정 화면을 보여주기 위해 컨트롤러 메소드를 추가한다. (BlogViewController.java → newArticle()).
  </li>
</ul>
