<h2>2-3. 가상 환경</h2>
<ul>
  <li>
    pip을 통해 다운로드 된 패키지는 표준 파이썬과 그 안에 담긴 모든 라이브러리를 보통 <strong>운영 체제의 표준 위</strong>에 설치된다.
  </li>
    <ul>
      <li>
        운영 체제 표준 위에 설치된 파일은 변경할 수 없지만 pip 표준 파이썬 파일에 영향을 미치지 않는 <strong>기본 디렉터리</strong>를 사용하기에 <strong>변경이 가능</strong>하다.
      </li>
    </ul>
  <li>
    파이썬 가상 환경을 사용하는 경우에는 venv란 모듈을 사용한다.
  </li>
    <ul>
      <li>
        가상 환경(virtual environment)은 pip가 다운로드한 <strong>패키지를 저장하기 위한 디렉터리</strong>이다. (가상 환경이 활성화 된 경우 파이썬 모듈을 로드할 때 이곳을 <strong>먼저 확인</strong> 함). 
      </li>
    </ul>
</ul>

```bash
# 1. 가상 환경 설정 코드

# venv 독립 프로그램
venv venv1

# python 모듈로 실행
python -m venv venv1

# 가상 환경 실행
source venv1/bin/activate

# 가상 환경 종료
deactivate
```

<h2>2-4. Poetry</h2>
<ul>
  <li>
    pip과 venv를 결합한 방법을 고안한 것이 poetry이다.
  </li>
  <li>
    poetry는 <strong>pip install poetry</strong>로 설치하며 패키지를 가상 환경에 추가하는 <strong>poetry add</strong>와 패키지를 다운로드해 설치하는 <strong>poetry install</strong>과 같은 여러 하위 명령어가 있다.
  </li>
  <li>
    pip은 <strong>requirements.txt</strong>로, Poetry는 <strong>pyproject.toml</strong>로 의존성을 관리한다.
  </li>
</ul>

<h1>4. 변수</h1>
<ul>
  <li>
    소프트웨어의 세계에서 <strong>객체(object)</strong>는 무수히 많은 의미로 통용되며 파이썬에서 객체는 <strong>모든 개별 데이터 조각을 감싸는 데이터 구조</strong>이다.
  </li>
  <li>
    객체는 기타 관리 정보화 함께 다음과 같은 정보를 지닌다.
  </li>
    <ul>
      <li>
        고유 <strong>ID</strong>
      </li>
      <li>
        하드웨어와 일치하는 저수준의 <strong>type</strong>
      </li>
      <li>
        특정 <strong>value(물리적 비트)</strong>
      </li>
      <li>
        객체를 참조하는 변수의 수를 나타내는 <strong>number of references</strong>
      </li>
    </ul>
  <li>
    파이썬은 객체 수준에서 값을 변경할 수 있으면 <strong>변경 가능(mutable)</strong>, 반대로 변경할 수 없으면 <strong>변경 불능(immutable)</strong>이라 한다.
  </li>
  <li>
    <strong>변수 수준</strong>에서 파이썬과 다른 언어에 차이가 있다.
  </li>
    <ul>
      <li>
        많은 언어에서 변수는 비트 단위로 저장된 원시 값이 담긴 메모리 영역을 직접 가리키는 <strong>포인터</strong>이다.
      </li>
      <li>
        파이썬의 변수는 객체와 일시적으로 연결된 <strong>이름</strong>일 뿐이다. 따라서 새 값을 할당하면 해당 값을 포함하는 <strong>새로운 객체를 생성</strong>하고 <strong>새로운 객체를 참조</strong>한다.
      </li>
    </ul>
  <li>
    <strong>스코프(scope)</strong>는 이름이 살아 있고 접근 가능한 코드 범위이며 파이썬에서는 <strong>LEGB 규칙</strong>에 따라 변수를 탐색한다.
  </li>
    <ul>
      <li>
        <strong>L (Local)</strong>: 지역 스코프, 현재 실행 중인 함수 내부
      </li>
      <li>
        <strong>E (Enclosing)</strong>: 외부 함수 스코프, 중첩 함수의 경우 바깥 함수의 변수
      </li>
      <li>
        <strong>G (Global)</strong>: 전역 스코프, 현재 모듈 파일 전체
      </li>
      <li>
        <strong>B (Built-in)</strong>: 내장 스코프, 파이썬이 기본 제공하는 이름
      </li>
    </ul>
</ul>

<h1>7. 웹 프레임워크</h1>
<ul>
  <li>
    웹 프레임워크의 주요 역할은 <strong>HTTP로 전달되는 바이트</strong>와 <strong>파이썬에서 사용되는 데이터 구조</strong>를 <strong>서로 변환</strong>하는 것이다.
  </li>
  <li>
    기존의 동기식 통신은 디스크나 네트워크와 같이 CPU보다 훨씬 느리기에 동시성이 필요해졌으며 따라서 파이썬의 <strong>비동기 서버 게이트웨이 인터페이스(asynchronous server gateway interface, ASGI)</strong>가 탄생했다.
  </li>
</ul>
