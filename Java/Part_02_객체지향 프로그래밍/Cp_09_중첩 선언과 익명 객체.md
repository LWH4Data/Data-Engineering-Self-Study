<h1>1. 중첩 클래스</h1>
<ul>
  <li>
    객체지향 프로그램에서는 <strong>클래스 간에 서로 긴밀한 관계</strong>를 맺고 상호작용한다.
  </li>
  <li>
    클래스가 여러 클래스와 관계를 맺는 경우 독립적으로 선언하는 것이 좋지만, <strong>특정 클래스</strong>만 관계를 맺을 경우 <strong>중첩 클래스</strong>로 선언하는 것이 유지보수에 도움이 된다.
  </li>
  <li>
    중첩 클래스(Nested Class)란 <strong>클래스 내부에 선언한 클래스</strong>를 의미한다.
  </li>
    <ul>
      <li>
        <strong>클래스의 멤버</strong>를 쉽게 사용할 수 있다.
      </li>
      <li>
        <strong>외부</strong>에는 중첩 관계 클래스를 <strong>감추어</strong> 코드의 복잡성을 줄일 수 있다.
      </li>
    </ul>
  <li>
    중첩 클래스는 <strong>선언하는 위치</strong>에 따라 두 가지로 분류된다.
  </li>
    <ul>
      <li>
        <strong>멤버 클래스</strong>: 클래스의 <strong>멤버</strong>로 선언된다.
      </li>
        <ul>
          <li>
            <strong>인스턴스 멤버 클래스</strong>: 상위 클래스 객체를 생성해야만 생성이 가능하다.
          </li>
          <li>
            <strong>정적 멤버 클래스</strong>: 상위 클래스 객체를 생성하지 않아도 생성이 가능하다.
          </li>
        </ul>
      <li>
        <strong>로컬 클래스</strong>: <strong>메소드</strong> 내부에 선언되는 중첨 클래스
      </li>
        <ul>
          <li>
            <strong>method</strong>가 실행 될 때만 객체를 생성할 수 있다.
          </li>
        </ul>
    </ul>
  <li>
    중첩 클래스도 클래스이기에 컴파일을 할 시 <strong>바이트코드 파일(.class)</strong>이 생성된다.
  </li>
    <ul>
      <li>
        멤버 클래스의 경우 <strong>'바깥 클래스$멤버 클래스.class'</strong>
      </li>
      <li>
        로컬 클래스의 경우 <strong>'바깥 클래스$1로컬 클래스.class'</strong>
      </li>
    </ul>
</ul>

<br><br>

<h1>2. 인스턴스 멤버 클래스</h1>
<ul>
  <li>
    <strong>인스턴스 멤버 클래스</strong>란 특정 클래스의 <strong>멤버</strong>로 선언된 클래스를 의미한다.
  </li>
  <li>
    접근 제한자에 따른 인스턴스 멤버 클래스의 접근 범위
  </li>
    <ul>
      <li>
        <strong>public class B {}</strong>: <strong>다른 패키지</strong>에서 B 클래스 사용 가능.
      </li>
      <li>
        <strong>class B {}</strong>: <strong>같은 패키지</strong>에서만 B 클래스 사용 가능.
      </li>
      <li>
        <strong>private class B {}</strong>: <strong>상위 클래스</strong>에서만 클래스 사용 가능.
      </li>
    </ul>
  <li>
    인스턴스 멤버 클래스는 주로 상위 클래스 내부에서 사용되기에 <strong>private 접근 제한</strong>을 갖는 것이 일반적이다.
  </li>
    <ul>
      <li>
        인스턴스 멤버 클래스는 상위 클래스의 <strong>인스턴스 필드값, 생성자, 인스턴스 메소드</strong>에서 생성이 가능하다.
      </li>
    </ul>
  <li>
    내부 클래슨 객체를 외부 클래스의 <strong>외부에서 생성</strong>하려면 <strong>default</strong> 또는 <strong>public</strong> 접근 제한을 가져야 하며 <strong>외부 클래스 객체를 먼저 생성</strong>하여야 한다.
  </li>
</ul>

```java
// 1. 인스턴스 멤버 클래스 형태
/* 
[public] class A {
  [public | private] class B {
  }
} 
*/

// 2. 외부에서 내부 클래스 객체 생성
// 외부 클래스를 먼저 생성한다.
A a = new A();
// 이후 내부 클래스를 생성한다.
A.B b = a.new B();
```

<h1>3. 정적 멤버 클래스</h1>
<ul>
  <li>
    정적 멤버 클래스란 <strong>static 키워드</strong>와 함께 특정 클래스의 <strong>멤버</strong>로 선언된 클래스를 의미한다.
  </li>
  <li>
    접근 제한자에 따른 정적 메소드의 범위
  </li>
    <ul>
      <li>
        <strong>public static class B {}</strong>: <strong>다른 패키지</strong>에서 B 클래스 사용 가능.
      </li>
      <li>
        <strong>static class B {}</strong>: <strong>같은 패키지</strong>에서만 B 클래스 사용 가능.
      </li>
      <li>
        <strong>private static class B {}</strong>: <strong>A 클래스(외부) 내부</strong>에서만 B 클래스 사용 가능.
      </li>
    </ul>
  <li>
    정적 멤버 클래스는 외부 클래스의 외부에서 함께 사용되는 경우가 많기 때문에 주로 <strong>default</strong> 혹은 <strong>public</strong> 접근 제한을 갖는다.
  </li>
  <li>
    내부 객체는 외부 객체의 내부 어디든 객체를 생성할 수 있다.
  </li>
  <li>
    A 클래스 <strong>외부</strong>에서 B 객체를 생성하려면 <strong>A 객체 생성 없이</strong> A 클래스로 접근하여 B 객체를 생성할 수 있다.
  </li>
</ul>

```java
// 1. 정적 멤버 클래스 선언 형식
/*
[public] class A {
  [public | private] static class B {
  }
}
*/

// 2. A 클래스 외부에서 B 객체를 생성.
A.B b = new A.B();
```

<h1>4. 로컬 클래스</h1>
<ul>
  <li>
    <strong>생성자</strong> 또는 <strong>메소드 내부</strong>에 선언된 클래스를 <strong>로컬(local) 클래스</strong>라 한다.
  </li>
  <li>
    로컬 클래스는 생성자와 메소드가 <strong>실행될 동안</strong>에만 객체를 생성할 수 있다.
  </li>
  <li>
    로컬 클래스 내부에는 일반 클래스와 동일하게 <strong>필드, 생성자, 메소드 선언</strong>이 올 수 있다.
  </li>
    <ul>
      <li>
        정적 필드와 정적 메소드는 Java 17부터 선언이 가능하다.
      </li>
    </ul>
  <li>
    로컬 변수(생성자, 메소드의 매개변수 또는 내부에 선언된 변수)를 로컬 클래스에 사용할 경우 <strong>읽을 수만 있고 수정할 수는 없다</strong>.
  </li>
    <ul>
      <li>
        로컬 클래스는 <strong>final 특성</strong>을 갖기 때문이다.
      </li>
    </ul>
  <li>
    Java 8 이후부터는 명시적으로 final 키워드를 붙이지 않아도 되지만, 로컬 변수에 final 키워드를 명시할 수도 있다.
  </li>
</ul>

```java
// 1. 로컬 클래스 선언.
/*
[public] class A {
  // 생성자.
  public A() {
    class B { }
  }

  // 메소드.
  public void method() {
    class B { }
  }
}
*/
```

<h1>5. 바깥 멤버 접근</h1>
<ul>
  <li>
    중첩 클래스는 바깥 클래스와 긴말한 관계를 맺고 바깥 클래스의 멤버(필드, 메소드)에 접근할 수 있으나 <strong>선언 방법</strong>에 따라 <strong>접근이 제한</strong>될 수 있다.
  </li>
</ul>

<br>

<h2>5-1. 바깥 클래스 멤버 접근 제한</h2>
<ul>
  <li>
    정적 멤버 클래스는 <strong>바깥 객체가 없어도</strong> 사용할 수 있어야 하기에 바깥 클래스의 인스턴스 필드와 인스턴스 메소드는 사용하지 못한다.
  </li>
</ul>
