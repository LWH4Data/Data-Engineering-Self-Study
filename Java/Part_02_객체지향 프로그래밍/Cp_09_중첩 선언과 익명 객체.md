<h1>1. 중첩 클래스</h1>
<ul>
  <li>
    객체지향 프로그램에서는 <strong>클래스 간에 서로 긴밀한 관계</strong>를 맺고 상호작용한다.
  </li>
  <li>
    클래스가 여러 클래스와 관계를 맺는 경우 독립적으로 선언하는 것이 좋지만, <strong>특정 클래스</strong>만 관계를 맺을 경우 <strong>중첩 클래스</strong>로 선언하는 것이 유지보수에 도움이 된다.
  </li>
  <li>
    중첩 클래스(Nested Class)란 <strong>클래스 내부에 선언한 클래스</strong>를 의미한다.
  </li>
    <ul>
      <li>
        <strong>클래스의 멤버</strong>를 쉽게 사용할 수 있다.
      </li>
      <li>
        <strong>외부</strong>에는 중첩 관계 클래스를 <strong>감추어</strong> 코드의 복잡성을 줄일 수 있다.
      </li>
    </ul>
  <li>
    중첩 클래스는 <strong>선언하는 위치</strong>에 따라 두 가지로 분류된다.
  </li>
    <ul>
      <li>
        <strong>멤버 클래스</strong>: 클래스의 <strong>멤버</strong>로 선언된다.
      </li>
        <ul>
          <li>
            <strong>인스턴스 멤버 클래스</strong>: 상위 클래스 객체를 생성해야만 생성이 가능하다.
          </li>
          <li>
            <strong>정적 멤버 클래스</strong>: 상위 클래스 객체를 생성하지 않아도 생성이 가능하다.
          </li>
        </ul>
      <li>
        <strong>로컬 클래스</strong>: <strong>메소드</strong> 내부에 선언되는 중첨 클래스
      </li>
        <ul>
          <li>
            <strong>method</strong>가 실행 될 때만 객체를 생성할 수 있다.
          </li>
        </ul>
    </ul>
  <li>
    중첩 클래스도 클래스이기에 컴파일을 할 시 <strong>바이트코드 파일(.class)</strong>이 생성된다.
  </li>
    <ul>
      <li>
        멤버 클래스의 경우 <strong>'바깥 클래스$멤버 클래스.class'</strong>
      </li>
      <li>
        로컬 클래스의 경우 <strong>'바깥 클래스$1로컬 클래스.class'</strong>
      </li>
    </ul>
</ul>
