<h1>1. 멀티 스레드 개념</h1>
<ul>
  <li>
    운영체제는 실행 중인 프로그램을 <strong>프로세스(process)</strong>로 관리한다.
  </li>
    <ul>
      <li>
        운영체제는 <strong>멀티 프로세스</strong>를 생성하여 멀티 태스킹(multi tasking)을 처리한다.
      </li>
    </ul>
  <li>
    <strong>하나의 프로세스</strong>가 멀티 태스킹을 처리할 때에는 <strong>멀티 스레드(multi thread)</strong>를 사용한다.
  </li>
    <ul>
      <li>
        <strong>스레드(thread)</strong>란 <strong>코드의 실행 흐름</strong>을 의미한다.
      </li>
    </ul>
  <li>
    즉, <strong>멀티 프로세스 → 프로그램 단위</strong>의 멀티 테스킹 / <strong>멀티 스레드 → 프로그램 내부</strong> 멀티 테스킹이라 볼 수 있다.
  </li>
  <li>
    멀티 프로세스는 서로 독립적이기에 문제가 발생하여도 다른 프로세스에 영향을 주지 않지만, 스레드는 <strong>다른 스레드에 영향을 미친다</strong>.
  </li>
    <ul>
      <li>
        따라서 멀티 스레드를 사용하는 경우 <strong>예외 처리</strong>를 신경써야 한다.
      </li>
    </ul>
  <li>
    멀티 스레드는 데이터 분할 병렬 처리, 안드로이드 앱 네트워크 통신, 다수의 클라이언트 요청 처리 등에 사용된다.
  </li>
  <li>
    <strong>가상 스레드(virtual thread)</strong> 소수의 실제 스레드에서 스케줄링하여 사용할 수 있다.
  </li>
</ul>

<br><br>

<h1>2. 메인 스레드</h1>
<ul>
  <li>
    모든 자바 프로그램은 <strong>메인 스레드(main thread)</strong>가 <strong>main() 메소드를 실행</strong>하면서 시작된다.
  </li>
  <li>
    메인 스레드는 필요에 따라 <strong>추가 작업 스레드</strong>들을 만들어 실행시킬 수 있다.
  </li>
  <li>
    <strong>싱글 스레드</strong>는 <strong>메인 스레드가 종료</strong>되면 프로세스도 종료된다, 하지만 <strong>멀티 스레드</strong>는 실행 중인 스레드가 <strong>하나라도</strong> 있다면 프로세스가 종료되지 않는다.
  </li>
</ul>

<br><br>

<h1>3. 작업 스레드 생성과 실행</h1>
<ul>
  <li>
    멀티 스레드로 실행하는 프로그램을 개발하기 위해서는 우선 <strong>몇 개의 작업</strong>을 <strong>병렬</strong>로 실행할지 결정하고 <strong>작업별로 스레드를 생성</strong>해야 한다.
  </li>
    <ul>
      <li>
        자바 프로그램은 메인 스레드가 반드시 존재하기에 <strong>메인 작업 이외에 추가적인 작업 수</strong>만큼 스레드를 생성한다.
      </li>
    </ul>
  <li>
    자바는 작업 스레드도 객체로 관리하기에 클래스가 필요하다.
  </li>
    <ul>
      <li>
        Thread 클래스로 <strong>직접 객체를 생성</strong>할 수도 있고, <strong>하위 클래스</strong>를 만들어 생성할 수도 있다.
      </li>
    </ul>
</ul>

<br>

<h2>3-1. Thread 클래스로 직접 생성</h2>
<ul>
  <li>
    <strong>java.lang 패키지</strong>에 있는 <strong>Thread 클래스</strong>로부터 작업 스레드 객체를 직접 생성할 수 있다. 
  </li>
    <ul>
      <li>
        <strong>Runnable 구현 객체</strong>를 매개값으로 갖는 생성자를 호출한다.
      </li>
    </ul>
  <li>
    Runnable은 스레드가 작업을 실행할 때 사용하는 <strong>인터페이스</strong>이다.
  </li>
    <ul>
      <li>
        Runnable에는 <strong>run() 메소드</strong>가 정의되어 있으며 구현 클래스는 <strong>run()을 재정의</strong>한다.
      </li>
      <li>
        재정의된 run() 메소드는 <strong>스레드가 실행할 코드</strong>를 가지고 있어야 한다.
      </li>
    </ul>
  <li>
    Runnable 구현 클래스는 작업 내용을 재정의 한 것이기에 결과적으로 <strong>스레드에 전달</strong>을 해야한다.
  </li>
    <ul>
      <li>
        Thread <strong>생성자 매개값</strong>으로 Runnable 객체를 전달한다.
      </li>
      <li>
        명시적으로 Runnable 구현 클래스를 작성하지 않고 Thread 생성자 호출 시에 <strong>Runnable 익명 구현 객체</strong>를 매개값으로 사용할 수 있으며 이 방법을 더 많이 사용한다.
      </li>
    </ul>
  <li>
    작업 스레드를 실행하기 위해서는 스래드 객체의 <strong>start() 메소드</strong>를 호출해야 한다.
  </li>
</ul>

```java
// 1. java.lang 패키지의 Thread 클래스를 활용해 작업 스레드 객체 직접 생성.
Thread thread = new Thread(Runnable target);

// 2. Runnable 구현 클래스 작성.
class Task implements Runnable {

  // Runnable의 run() 메소드를 재정의하여 사용.
  @Override
  public void run() {
    // 스레드가 실행할 코드.
  }
}

// 3. Thread에 Runnable로 재정의한 작업 전달.
// Runnable 구현 객체 생성.
Runnable task = new Task();
// Thread 생성자 매개값으로 Runnable 객체 전달.
Thread thread = new Thread(task);

// 4. Runnable 익명 구현 객체를 매개값으로 활용하여 전달.
Thread thread = new Thread(new Runnable) {
  @Override
  public void run() {
    // 스레드가 실행할 코드.
  }
}

// 5. start() 메소드를 호출하여 작업 스레드 실행.
thread.start();
```

<br>

<h2>3-2. Thread 자식 클래스로 생성</h2>
<ul>
  <li>
    Thread를 자식 클래스로 생성하는 경우 <strong>Thread 클래스를 상속</strong>한 다음 <strong>run() 메소드를 재정의</strong>해서 스레드가 실행할 코드를 작성하고 객체를 생성한다.
  </li>
  <li>
    실행 방법은 동일하게 start() 메소드를 호출하면 된다.
  </li>
</ul>

```java
// 1. Thread 클래스를 상속받아 run() 메소드 재정의
public class WorkerThread extends Thread {
  @Override
  public void run() {
    // 스레드가 실행할 코드
  }
}

// 스레드 객체 생성.
Thread thread = new WorkerThread();
thread.start();


// 2. 익명 자식 객체 활용.
// - Thread()로 끝나면 그냥 객체 생성.
// - Thread() {}로 정의하면 익명 자식 객체.
Thread thread = new Thread() {
  @Override
  public void run() {
    // 스레드가 실행할 코드
  }
};
thread.start();
```

<h1>4. 스레드 이름</h1>
<ul>
  <li>
    스레드는 <strong>자신의 이름</strong>을 가지고 있다.
  </li>
    <ul>
      <li>
        메인 스레드는 main이라는 이름을, 작업 스레드는 Thread-n이라는 이름을 가진다.
      </li>
      <li>
        Thread-n 이라는 이름 대신 다른 이름을 설정하고 싶다면 <strong>setName() 메소드</strong>를 사용하면 된다.
      </li>
    </ul>
  <li>
    스레드 이름은 <strong>디버깅</strong>할 때 어떤 스레드가 작업을 하는지 조사할 목적으로 사용된다.
  </li>
    <ul>
      <li>
        현재 코드를 <strong>어떤 스레드</strong>가 실행하고 있는지 확인할 때에는 <strong>정적 메소드인 currnetThread()</strong>로 <strong>스레드 객체</strong>를 얻고, <strong>getName()</strong>으로 <strong>메소드 이름</strong>을 출력한다.
      </li>
    </ul>
</ul>

```java
// 1. 스레드 이름 설정
/*
thread.setName("스레드 이름");
*/

// 2. 스레드 이름 얻기.
Thread thread = Thread.currentThread();
System.out.println(thread.getName());
```

<h1>5. 스레드 상태</h1>
<ul>
  <li>
    스레드 객체를 생성(NEW)하고 start() 메소드를 호출하면 곧바로 스레드가 실행되는 것이 아니라 <strong>실행 대기 상태(REUNNABLE)</strong>가 된다.
  </li>
    <ul>
      <li>
        대기 상태란 실행을 기다리고 있는 상태를 의미한다.
      </li>
    </ul>
  <li>
    실행 대기하는 스레드는 <strong>CPU 스케줄링</strong>에 따라 CPU를 점유하고 <strong>run() 메소드를 실행</strong>한다. 이때 실행 상태를 RUNNING 상태라 한다.
  </li>
    <ul>
      <li>
        실행 스레드는 run() 메소드를 모두 실행하기 전에 스케줄링에 의해 <strong>실행 대기 상태</strong>로 돌아갈 수 있다.
      </li>
        <ul>
          <li>
            이때 다른 스레드가 실행 상태가 된다.
          </li>
        </ul>
      <li>
        스레드는 실행 대기 상태와 실행 상태를 번갈아 가며 자신의 run() 메소드를 조금씩 실행한다.
      </li>
    </ul>
  <li>
    실행 상태에서 run() 메소드가 종료되면 더 이상 실행할 코드가 없기 때문에 스레드의 실행은 멈춘다. 이 상태를 <strong>종료 상태(TERMINATED)</strong>라 한다. 
  </li>
  <li>
    실행 상태에서 <strong>일시 정지 상태</strong>로 가기도 하는데 이는 스레드가 <strong>실행할 수 없는 상태</strong>를 의미한다.
  </li>
  <li>
    스레드가 다시 실행 상태로 가기 위해서는 일시 정지 상태에서 실행 대기 상태로 가야하며 이는 <strong>메소드</strong>를 할용하여 수행한다.
  </li>
    <ul>
      <li>
        일시 정지로 보내는 메소드
      </li>
        <ul>
          <li>
            Thread.sleep(long millis)
          </li>
          <li>
            Thread.join()
          </li>
          <li>
            Object.wait()
          </li>
        </ul>
      <li>
        일시 정지에서 벗어나는 메소드
      </li>
        <ul>
          <li>
            Thread.interrupt()
          </li>
          <li>
            Object.notify()
          </li>
          <li>
            Object.notifyAll()
          </li>
        </ul>
      <li>
        실행 대기로 보냄
      </li>
        <ul>
          <li>
            Thread.yield()
          </li>
        </ul>
</ul>

<br>

<h2>5-1. 주어진 시간 동안 일시 정지</h2>
<ul>
  <li>
    실행 중인 스레드를 <strong>일시 정지</strong>하고 싶다면 Thread 클래스의 <strong>정적 메소드 sleep()</strong>을 이용하면 된다.
  </li>
    <ul>
      <li>
        매개값으로는 얼마 동안 정지를 할 것인지를 <strong>밀리세컨드(1/1000) 단위</strong>로 시간을 준다.
      </li>
    </ul>
  <li>
    알사 정지 동안ㅇ <strong>InterruptedException</strong>이 발생할 수 있기 때문에 <strong>예외 처리</strong>를 해주어야 한다.
  </li>
</ul>

```java
// 1. sleep() 활용 사례.
try {
  Thread.sleep(1000);
} catch(InterruptedException) {
  // Interrupt() 메소드가 호출되면 실행.
}
```

<h2>5-2. 다른 스레드의 종료를 기다림</h2>
<ul>
  <li>
    예를 들어 계산 스레드의 작업을 기다렸다가 계산 결과를 받아 처리하는 스레드는 계산 스레드를 기다려야 한다.
  </li>
  <li>
    다른 스레드를 기다릴 때에는 <strong>join() 메소드</strong>를 사용한다.
  </li>
    <ul>
      <li>
        <strong>ThreadA → ThreadB</strong>로 join() 메소드를 사용할 경우 ThreadA는 T<strong>hreadB가 끝날 때까지</strong> 대기한다.
      </li>
    </ul>
</ul>

```java
// 1. join() 활용 사례.
// threadB를 시작한다.
threadB.start();
// threadB가 완료될 때까지 대기한다.
threadB.join();
```

<h2>5-2. 다른 스레드에게 실행 양보</h2>
<ul>
  <li>
    큰 의미는 없지만 어쩔 수 없이 확인이 필요한 코드가 있다면 <strong>yield()</strong>를 통해 <strong>다른 스레드에 실행을 양보</strong>하고 <strong>자신은 실행 대기 상태</strong>로 가는 것이 도움이 될 수 있다.
  </li>
    <ul>
      <li>
        아예 필요 없는 경우에는 sleep을 사용해도 된다.
      </li>
    </ul>
</ul>

```java
// 1. yield 활용 사례
public void run() {
  while(true) {
    if(work) {
      System.out.println("ThreadA 작업 내용");
      // else인 경우 아무 것도 출력하지 않는 의미 없는 작업이기 때문에
      // 다른 스레드에 양도한다.
    } else {
      Thread.yield();
    }
  }
}
```

<h1>6. 스레드 동기화</h1>
<ul>
  <li>
    멀티 스레드는 하나의 객체를 <strong>공유</strong>하여 작업할 수 있다. 단, 다른 스레드에 의해 <strong>객체 내부의 데이터가 쉽게 변경</strong>될 수 있는 점을 주의해야 한다.
  </li>
  <li>
    스레드가 사용 중인 객체를 다른 스레드가 변경할 수 없도록 하려면 스레드 작업이 끝날 때까지 객체에 <strong>잠금</strong>을 걸면 된다. 자바는 <strong>동기화(aynchronized) 메소드와 블록</strong>으로 잠금을 한다.
  </li>
    <ul>
      <li>
        스레드가 동기화 메소드 혹은 블록을 실행할 때 다른 스레드는 <strong>해당 메소드를 포함한 다른 동기화 메소드 및 블록</strong>도 실행할 수 없다. (일반 메소드는 실행할 수 있다).
      </li>
    </ul>
</ul>

<br>

<h2>6-1. 동기화 메소드 및 블록 선언</h2>
<ul>
  <li>
    동기화 메소드를 선언할 때에는 <strong>synchronized 키워드</strong>를 붙이면 된며 인스턴스 메소드와 정적 메소드 모두 붙일 수 있다.
  </li>
    <ul>
      <li>
        동기화 메소드가 실행되는 즉시 객체는 잠금이 일어나고, 메소드 실행이 끝나면 잠금이 풀린다.
      </li>
    </ul>
  <li>
    메소드 전체가 아닌 일부 영역을 실행할 때만 객체 잠금을 걸고 싶다면 동기화 블록을 만들면 된다.
  </li>
</ul>

```java
// 1. 동기화 메소드 선언.
public synchonized void mehtod() {
  // 단 하나의 스레드만 실행하는 영역
}

// 2. 동기화 블록 선언.
public void method() {
  // 여러 스레드가 실행할 수 있는 영역.

  synchronized(공유객체) {
    // 단 하나의 스레드만 실행하는 영역
  }

  // 여러 스레드가 실행할 수 있는 영역.
}
```

<h2>6-2. wait()과 notify()를 이용한 스레드 제어</h2>
<ul>
  <li>
    경우에 따라 <strong>두 개의 스레드를 번갈아</strong> 실행할 때도 있다. 이때에는 자신의 작업이 끝나면 <strong>상대방 스레드를 일시 정지 상태에서 해제</strong>하고 <strong>자신을 일시 정지 상태</strong>로 만든다.
  </li>
  <li>
    교대로 실행하는 방법의 핵심은 <strong>공유 객체</strong>에 있다.
  </li>
    <ul>
      <li>
        공유 객체는 두 스레드가 작업할 내용을 각각 <strong>동기화 메소드</strong>로 정한다.
        <br>→ 한 스레드가 작업을 완료하면 <strong>notify() 메소드</strong>를 호출해 일시 정지 상태에 있는 다른 스레드를 <strong>실행 대기 상태</strong>로 만든다.
        <br>→ 기존에 실행 중이던 스레드는 두 번 작업을 하지 않도록 <strong>wait() 메소드</strong>를 호출하여 <strong>일시 정지</strong>한다.
      </li>
    </ul>
  <li>
    <strong>notify()</strong>는 wait()에 의해 일시 정지된 <strong>스레드 하나</strong>를 실행 대기 상태로 만든다. 반면 <strong>notifyAll()</strong>은 <strong>모든 정지된 스레드</strong>를 실행 대기 상태로 만든다.
  </li>
    <ul>
      <li>
        단, notify()와 notifyAll() 모두 <strong>동기화 메소드</strong> 또는 <strong>동기화 블록</strong> 내에서만 사용할 수 있다.
      </li>
    </ul>
</ul>

<h1>7. 스레드 안전 종료</h1>
<ul>
  <li>
    스레드는 자신의 run() 메소드가 모두 실행되면 자동으로 종료되지만, 경우에 따라 실행 중인 스레드를 <strong>즉시 종료</strong>할 필요가 있다.
  </li>
  <li>
    스레드를 안전하게 종료하는 방법은 <strong>사용하던 리소스들을 정리</strong>하고 run() 메소드를 빨리 종료하는 것으로, 주로 <strong>조건 이용 방법</strong>과 <strong>interrupt() 메소드</strong>를 활용한다.
  </li>
    <ul>
      <li>
        과거에는 stop() 메소드를 사용하였으나 stop()은 리소스들이 불안전한 상태로 남겨지기 때문에 사용하지 않는다.
      </li>
    </ul>
</ul>

<br>

<h2>7-1. 조건 이용</h2>
<ul>
  <li>
    stop 필드 조건에 딸 run() 메소드 종료 유도.
  </li>
</ul>

```java
public class XXXThread extends Thread {
  private boolean stop;

  public void run() {
    while( !stop ) {
      // 스레드가 반복 실행하는 코드;
    }
    // 스레드가 사용한 리소스 정리
  }
}
```

<br>

<h2>7-2. interrupt() 메소드 이용</h2>
<ul>
  <li>
    <strong>interrupt() 메소드</strong>는 스레드가 <strong>일시 정지 상태</strong>에 있을 때 <strong>InterruptedException 예외</strong>를 발생시키는 역할을 한다.
  </li>
  <li>
    스레드는 <strong>실행 대기 / 실행 상태</strong>일 때에는 interrupt() 메소드가 호출 되어도 InterruptedException을 발생하지 않기에 sleep() 등을 활용하여야 한다.
  </li>
  <li>
    일시 정지 상태를 만들지 않고 interrupt() 메소드 호출 여부를 결정하기 위해 Thread의 <strong>interrupted()</strong>와 <strong>isInterrupted()</strong> 메소드를 사용할 수 있다.
  </li>
    <ul>
      <li>
        interrupted()와 isInterrupted() 메소드는 <strong>interrupt() 메소드 호출 여부</strong>를 리턴한다.
      </li>
      <li>
        interrupted()는 정적 메소드이고, isInterrupted()는 인스턴스 메소드이다.
      </li>
    </ul>
</ul>

```java
// 1. interrupted() 메소드와 isInterrupted() 메소드.
boolean status = Thread.interrupted();
boolean status = objThread.isInterrupted();
```

<h1>8. 데몬 스레드</h1>
<ul>
  <li>
    <strong>데몬(daemon) 스레드</strong>는 주 스레드의 작업을 <strong>보조</strong>하는 스레드이다. 주 스레드가 종료되면 데몬 스레드로 종료된다.
  </li>
  <li>
    스레드를 데몬으로 만들기 위해서는 주 스레드가 <strong>데몬이 될 스레드의 setDaemon(true)</strong>를 호출하면 된다.
  </li>
</ul>

```java
// 1. 데몬 스레드 사례.
public static void main(String[] args) {
  
  // 데몬으로 사용할 스레드 객체를 생성한다.
  AutoSaveThread thread = new AutoSaveThread();

  // setDaemon() 메소드를 통해 AutoSaveThread를 데몬 스레드로 만든다.
  thread.setDaemon(true);

  // 스레드 실행.
  Thread.start();
  ...
}
```

<h1>9. 스레드풀</h1>
<ul>
  <li>
    병렬 작업이 많아지면 CPU와 메모리가 과열된다. 따라서 병렬 작업 증가로 인한 스레드의 폭증을 막기 위해서는 <strong>스레드풀(ThreadPool)</strong>을 사용하는 것이 좋다.
  </li>
  <li>
    스레드 풀이란 작업 처리에 사용되는 스레드를 <strong>제한된 개수</strong>만큼 정해 놓고 <strong>작업 큐(Queue)</strong>에 들어오는 작업들을 <strong>스레드가 하나씩 맡아</strong> 처리하는 방식이다.
  </li>
    <ul>
      <li>
        작업량이 증가하여도 스레드의 개수가 늘지 않는다.
      </li>
    </ul>
</ul>

<br>

<h2>9-1. 스레드풀 생성</h2>
<ul>
  <li>
    자바는 스레드풀을 생성하고 사용할 수 있도록 <strong>java.util.concurrent</strong> 패키지의 <strong>ExecutorService 인터페이스</strong>와 <strong>Executors 클래스</strong>를 제공한다.
  </li>
    <ul>
      <li>
        Executors의 두 메소드 <strong>newCachedThreadPool()</strong>과 <strong>newFixedThreadPool(int n Threads)</strong>를 사용하면 ExecutorSerive 구현 객체를 만들 수 있다. (628p 참고).
      </li>
    </ul>
  <li>
    기본적인 파라미터는 다음과 같다.
  </li>
    <ul>
      <li>
        <strong>초기 수</strong>: 스레드풀이 생성될 때 <strong>기본적으로 생성되는 스레드의 수</strong>.
      </li>
      <li>
        <strong>코어 수</strong>: 스레드가 증가된 후 사용되지 않는 스레드를 제거할 때, <strong>최소한 풀에서 유지하는 스레드 수</strong>.
      </li>
      <li>
        <strong>최대 수</strong>: 증가되는 스레드의 <strong>한도 수</strong>.
      </li>
    </ul>
</ul>

```java
// 1. newCachedThreadPool() 메소드로 스레드풀 초기화.
//   - 초기 수와 코어 수는 0 이다.
//   - 작업 개수가 많아지면 새 스레드를 생성하여 작업을 처리한다.
//   - 60초 동안 스레드가 아무 작업을 하지 않으면 스레드를 풀에서 제거한다.
ExecutorService executorService = Executors.newCachedThreadPool();

// 2. newFixedThreadPool() 메소드로 스레드풀 초기화.
//   - 초기 수 0, 작업 개수가 많아지면 5개까지 증가한다.
//   - 생성된 스레드를 제거하지 않는다는 특징이 있다.
ExecutorService executorService = Executos.newFixedThreadPool(5);

// 3. ThreadPoolExecutor을 통한 스레드 초기화.
ExecutorService threadPool = new ThreadPoolExecutor(
  3,                                 // 코어 스레드 개수
  100,                               // 최대 스레드 개수
  120L,                              // 놀고 있는 시간
  TimeUnit.SECONDS,                  // 놀고 있는 시간 단위
  new SynchronousQueue<Runnable>()   // 작업 큐
);
```

<br>

<h2>9-2. 스레드풀 종료</h2>
<ul>
  <li>
    스레드풀의 스레드는 기본적으로 데몬 스레드가 아니기 때문에 main 스레드가 종료되더라도 <strong>계속 실행 상태</strong>로 남아있는다.
  </li>
  <li>
    스레드풀의 모든 스레드를 종료하려면 ExecutorService의 두 메소드 <strong>shutdown()</strong> 혹은 <strong>shutdownNow()</strong> 중 하나를 사용해야 한다.
  </li>
    <ul>
      <li>
        <strong>shutdown()</strong>: 리턴 타입은 void이며 현재 처리 중인 작업과 작업 큐의 대기 작업 <strong>모두 처리</strong>한 뒤에 스레드풀을 종료한다.
      </li>
      <li>
        <strong>shutdownNow()</strong>: 리턴 타입은 List<Runnable>로 작업 큐에 있는 <strong>미처리된 작업(Runnable)을 반환</strong>하고, 현재 진행 중인 작업은 <strong>Interrupt하여 중지</strong>한다.
      </li>
    </ul>
  <li>
    즉, 남아있는 작업을 마무리할 때에는 shutdown()을, 마무리가 필요 없을 때에는 shutdownNow()를 사용한다.
  </li>
</ul>
