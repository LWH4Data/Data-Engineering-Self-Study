<h1>1. 멀티 스레드 개념</h1>
<ul>
  <li>
    운영체제는 실행 중인 프로그램을 <strong>프로세스(process)</strong>로 관리한다.
  </li>
    <ul>
      <li>
        운영체제는 <strong>멀티 프로세스</strong>를 생성하여 멀티 태스킹(multi tasking)을 처리한다.
      </li>
    </ul>
  <li>
    <strong>하나의 프로세스</strong>가 멀티 태스킹을 처리할 때에는 <strong>멀티 스레드(multi thread)</strong>를 사용한다.
  </li>
    <ul>
      <li>
        <strong>스레드(thread)</strong>란 <strong>코드의 실행 흐름</strong>을 의미한다.
      </li>
    </ul>
  <li>
    즉, <strong>멀티 프로세스 → 프로그램 단위</strong>의 멀티 테스킹 / <strong>멀티 스레드 → 프로그램 내부</strong> 멀티 테스킹이라 볼 수 있다.
  </li>
  <li>
    멀티 프로세스는 서로 독립적이기에 문제가 발생하여도 다른 프로세스에 영향을 주지 않지만, 스레드는 <strong>다른 스레드에 영향을 미친다</strong>.
  </li>
    <ul>
      <li>
        따라서 멀티 스레드를 사용하는 경우 <strong>예외 처리</strong>를 신경써야 한다.
      </li>
    </ul>
  <li>
    멀티 스레드는 데이터 분할 병렬 처리, 안드로이드 앱 네트워크 통신, 다수의 클라이언트 요청 처리 등에 사용된다.
  </li>
  <li>
    <strong>가상 스레드(virtual thread)</strong> 소수의 실제 스레드에서 스케줄링하여 사용할 수 있다.
  </li>
</ul>

<br><br>

<h1>2. 메인 스레드</h1>
<ul>
  <li>
    모든 자바 프로그램은 <strong>메인 스레드(main thread)</strong>가 <strong>main() 메소드를 실행</strong>하면서 시작된다.
  </li>
  <li>
    메인 스레드는 필요에 따라 <strong>추가 작업 스레드</strong>들을 만들어 실행시킬 수 있다.
  </li>
  <li>
    <strong>싱글 스레드</strong>는 <strong>메인 스레드가 종료</strong>되면 프로세스도 종료된다, 하지만 <strong>멀티 스레드</strong>는 실행 중인 스레드가 <strong>하나라도</strong> 있다면 프로세스가 종료되지 않는다.
  </li>
</ul>

<br><br>

<h1>3. 작업 스레드 생성과 실행</h1>
<ul>
  <li>
    멀티 스레드로 실행하는 프로그램을 개발하기 위해서는 우선 <strong>몇 개의 작업</strong>을 <strong>병렬</strong>로 실행할지 결정하고 <strong>작업별로 스레드를 생성</strong>해야 한다.
  </li>
    <ul>
      <li>
        자바 프로그램은 메인 스레드가 반드시 존재하기에 <strong>메인 작업 이외에 추가적인 작업 수</strong>만큼 스레드를 생성한다.
      </li>
    </ul>
  <li>
    자바는 작업 스레드도 객체로 관리하기에 클래스가 필요하다.
  </li>
    <ul>
      <li>
        Thread 클래스로 <strong>직접 객체를 생성</strong>할 수도 있고, <strong>하위 클래스</strong>를 만들어 생성할 수도 있다.
      </li>
    </ul>
</ul>

<br>

<h2>3-1. Thread 클래스로 직접 생성</h2>
<ul>
  <li>
    <strong>java.lang 패키지</strong>에 있는 <strong>Thread 클래스</strong>로부터 작업 스레드 객체를 직접 생성할 수 있다. 
  </li>
    <ul>
      <li>
        <strong>Runnable 구현 객체</strong>를 매개값으로 갖는 생성자를 호출한다.
      </li>
    </ul>
  <li>
    Runnable은 스레드가 작업을 실행할 때 사용하는 <strong>인터페이스</strong>이다.
  </li>
    <ul>
      <li>
        Runnable에는 <strong>run() 메소드</strong>가 정의되어 있으며 구현 클래스는 <strong>run()을 재정의</strong>한다.
      </li>
      <li>
        재정의된 run() 메소드는 <strong>스레드가 실행할 코드</strong>를 가지고 있어야 한다.
      </li>
    </ul>
  <li>
    Runnable 구현 클래스는 작업 내용을 재정의 한 것이기에 결과적으로 <strong>스레드에 전달</strong>을 해야한다.
  </li>
    <ul>
      <li>
        Thread <strong>생성자 매개값</strong>으로 Runnable 객체를 전달한다.
      </li>
      <li>
        명시적으로 Runnable 구현 클래스를 작성하지 않고 Thread 생성자 호출 시에 <strong>Runnable 익명 구현 객체</strong>를 매개값으로 사용할 수 있으며 이 방법을 더 많이 사용한다.
      </li>
    </ul>
  <li>
    작업 스레드를 실행하기 위해서는 
  </li>
</ul>

```java
// 1. java.lang 패키지의 Thread 클래스를 활용해 작업 스레드 객체 직접 생성.
Thread thread = new Thread(Runnable target);

// 2. Runnable 구현 클래스 작성.
class Task implements Runnable {

  // Runnable의 run() 메소드를 재정의하여 사용.
  @Override
  public void run() {
    // 스레드가 실행할 코드.
  }
}

// 3. Thread에 Runnable로 재정의한 작업 전달.
// Runnable 구현 객체 생성.
Runnable task = new Task();
// Thread 생성자 매개값으로 Runnable 객체 전달.
Thread thread = new Thread(task);

// 4. Runnable 익명 구현 객체를 매개값으로 활용하여 전달.
Thread thread = new Thread(new Runnable) {
  @Override
  public void run() {
    // 스레드가 실행할 코드.
  }
}
```
