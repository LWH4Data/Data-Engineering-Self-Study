<h1>1. 스트림이란?</h1>
<ul>
  <li>
    지금까지는 컬렉션 및 배열에 저장된 요소를 반복 처리하기 위해 for 문 혹은 Iterator(반복자)를 이용하였다.
  </li>
  <li>
    Java 8부터는 컬렉션 및 배열 요소를 반복 처리하기 위해 <strong>스트림(Stream)</strong>을 사용할 수 있다.
  </li>
  <li>
    Stream과 Iterator의 세 가지 차이점.
  </li>
    <ul>
      <li>
        1. Stream은 <strong>내부 반복자</strong>로 처리 속도가 빠르고 병렬 처리에 효율적이다.
      </li>
      <li>
        2. <strong>람다식</strong>으로 다양한 요소 처리를 정의할 수 있다.
      </li>
      <li>
        3. 중간 처리와 최종 처리를 수행하도록 <strong>파이프 파인</strong>을 형성할 수 있다.
      </li>
    </ul>
</ul>

```java
// 1. 기존의 for와 iterator를 통한 반복.
// 1-1. for loop
List<String> list = ...;
for(int i=0; i<list.size();>; i++) {
  String item = list.get(i);
  // item 처리.
}

// 1-2. Iterator
Set<String> set = ...;
Iterator<String> iterator = set.iterator();
while(iterator.hasNext()) {
  String item = iterator.next();
  // 요소 처리
}

// 2. Stream 활용.
//   - List 컬레션의 stream() 메소드로 Stream 객체를 읽는다.
//   - forEach() 메소드로 요소를 어떻게 처리할지 람다식으로 제공한다. 
Stream<String> stream = list.stream();
stream.forEach( item -> // item 처리 );
```

<h1>2. 내부 반복자</h1>
<ul>
  <li>
    for loop과 Iterator는 컬렉션 요소를 <strong>컬렉션 바깥쪽</strong>으로 반복해서 가져와 처리하며 <strong>외부 반복자</strong>라 한다.
  </li>
    <ul>
      <li>
        컬렉션 요소를 <strong>외부로 가져오는 코드</strong>와 <strong>처리하는 코드</strong>를 모두 <strong>개발자 코드</strong>가 가지고 있어야 한다.
      </li>
    </ul>
  <li>
    Stream은 요소 처리 방법을 <strong>컬렉션 내부로 주입</strong>하여 반복 처리를 하며 이를 <strong>내부 반복자</strong>라 한다.
  </li>
    <ul>
      <li>
        개발자 코드에서 제공한 <strong>데이터 처리 코드(람다식)</strong>을 가지고 <strong>컬렉션 내부</strong>에서 요소를 반복 처리한다.
      </li>
      <li>
        <strong>멀티 코어 CPU</strong>를 최대한 활용하기 위해 요소들을 분배시켜 <strong>병렬 작업</strong>을 할 수 있다.
      </li>
        <ul>
          <li>
            따라서 하나씩 처리하는 외부 반복자 보다 효율적으로 요소를 반복시킬 수 있다.
          </li>
        </ul>
    </ul>
</ul>

<h1>3. 중간 처리와 최종 처리</h1>
<ul>
  <li>
    스트림은 하나 이상 연결될 수 있다. 여러 스트림이 연결되어 있는 것을 <strong>스트림 파이프라인(pipeline)</strong>이라 한다.
  </li>
    <ul>
      <li>
        <strong>오리지널 스트림</strong>과 <strong>집계 처리</strong> 사이의 중간 스트림들은 다양한 처리를 한다.
      </li>
        <ul>
          <li>
            최종 처리를 위해 요소를 걸러내기(필터링)
          </li>
          <li>
            요소를 변환시키기(매핑)
          </li>
          <li>
            정렬 작업 수행 및 중간 처리에서 정제된 요소들을 반복
          </li>
          <li>
            집계(카운팅, 총합, 평균) 등
          </li>
        </ul>
    </ul>
</ul>


```java
// 1. 스트림 파이프라인 표현.
// Student 스트림.
Stream<Student> studentStream = list.stream();
// score 스트림.
//   - mapToInt(): 객체를 int 값으로 매핑하여 IntStream으로 변환시킨다.
//     - 어떤 int 값으로 매핑할 것인지는 람다식으로 제공한다.
IntStream scoreStream = studentStream.mapToInt( student -> student.getScore() );
// 평균 계산.
//   - IntStream의 average() 메소드는 평균 값을 계산한다.
double avg = scoreStream.average().getAsDouble();

// 2. 메소드 체이닝 패턴으로 표현.
double avg = list.stream()
  .mapToInt(student -> student.getScore())
  .average()
  .getAsDouble();
```

<h1>4. 리소스로부터 스트림 얻기</h1>
<ul>
  <li>
    <strong>java.util.stream 패키지</strong>에는 스트림 인터페이스들이 있으며 <strong>BaseStream 인터페이스</strong>에는 다음의 자식 인터페이스들이 있다.
  </li>
    <ul>
      <li>
        Stream: <strong>객체 요소</strong>를 처리.
      </li>
      <li>
        IntStream, LongStream, DoubleStream: 각 <strong>기본 타입</strong>인 int, long, double 요소를 처리하는 스트림.
      </li>
    </ul>
  <li>
    BaseStream은 모든 스트렘에서 사용할 수 있는 공통 메소드들이 있으며 이를 포함한 각 Stream의 메소드는 <strong>p731</strong>을 참고.
  </li>
</ul>

        자식 인터페이스인 List와 Set 인터페이스를 구현한 모든 컬렉션에서 객체 스트림을 얻을 수 있다.
      </li>
    </ul>
</ul>

  </li>
</ul>

<br>

<h2>4-3. 숫자 범위로부터 스트림 얻기</h2>
<ul>
  <li>
    <strong>IntStream</strong> 또는 <strong>LongStream</strong>의 정적 메소드 <strong>range()</strong>와 <strong>rangeClosed()</strong> 메소드를 사용하면 <strong>특정 범위의 정수 스트림</strong>을 얻을 수 있다.
  </li>
    <ul>
      <li>
        첫 번째 매개값은 시작 수이고, 두 번째 매개값은 끝 수이다.
      </li>
      <li>
        끝 수를 포함할 때에는 range(), 포함하지 않을 때에는 rangeClosed()를 사용한다.
      </li>
    </ul>
</ul>
