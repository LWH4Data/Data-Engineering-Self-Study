<h1>1. 제네릭이란?</h1>
<ul>
  <li>
    <strong>제네릭(Generic)</strong>이란 <strong>결정되지 않은 타입</strong>을 파라미터로 처리하고 <strong>실제 사용할 때</strong> 파라미터를 구체적인 타입으로 대체시키는 기능이다.
  </li>
    <ul>
      <li>
        다양한 내용을 저장할 경우 특정 클래스 타입으로 선언이 불가하다.
      </li>
      <li>
        Object 타입은 최상위 부모 클래스로 자동 타입 변환을 활용할 수 있으나 내용물을 얻을 때 Object 타입이기에 정확한 타입을 알기 어렵다.
      </li>
      <li>
        instanceof로 타입을 조사할 수도 있지만, 모든 종류의 클래스를 대상으로 조사가 불가하다.
      </li>
    </ul>
  <li>
    타입 파라미터를 대체하는 타입은 <strong>클래스 및 인터페이스</strong>이다. <strong>기본 타입</strong>은 타입 파라미터의 대체 타입이 될 수 없다.
  </li>
</ul>

```java
// 1. 타입 T라는 타입 파라미터로 결정되지 않은 content 타입 선언 (T는 임의의 명칭)
public class Box <T> {
  public T content;
}

// 2. 객체를 생성하는 시점에 사용할 타입을 선언한다.
Box<String> box = new Box<String>();
box.content = "안녕하세요";
// 강제 타입 변환이 필요 없이 "안녕하세요"를 바로 얻을 수 있다.
String content = box.content;

// 3. 100이라는 정수를 저장할 때에는 <T> 대신 Integer로 대체한다.
Box<Integer> box = new Box<Integer>();
box.content = 100;
// 강제 타입 변환 없이 100을 바로 얻을 수 있다.
int content = box.content;

// 4. 변수를 선언할 때 동일한 타입으로 호출하고 싶다면 호출 시 
//    생성자 타입을 명시하지 않고 <>만 붙일 수 있다.
Box<String> box = new Box<String>(); // Box<String> box = new Box<>();
Box<Integer> box = new Box<Integer>(); // Box<Integer> box = new Box<>();
```

<h1>2. 제네릭 타입</h1>
<ul>
  <li>
    제네릭 타입은 <strong>결정되지 않은 타입</strong>을 파라미터로 가지는 <strong>클래스</strong>와 <strong>인터페이스</strong>를 의미한다.
  </li>
  <li>
    타입 파라미터는 변수명과 동일한 규칙에 따라 작성할 수 있지만, 일반적으로 <strong>대문자 알파벳 한 글자</strong>로 표현한다.
  </li>
  <li>
    외부에서 제네릭 타입을 사용하려면 타입 파라미터에 <strong>구체적인 타입</strong>을 지정해야 한다.
  </li>
  <li>
    타입 파라미터는 기본적으로 Object 타입으로 간주되기에 <strong>Object가 가지고 있는 메소드</strong>를 호출 할 수 있다.
  </li>
</ul>

```java
// 1. 제네릭 타입을 사용하는 Class와 Interface 예.
/*
public class 클래스명<A, B, ...> { ... }
public interface 인터페이스명<A, B, ...> { ... }
*/
```

<br><br>

<h1>3. 제네릭 메소드</h1>
<ul>
  <li>
    제네릭 메소드는 <strong>타입 파라미터</strong>를 가지고 있는 메소드를 의미한다.
  </li>
  <li>
    제네릭 메소드는 <strong>리턴 타입 앞</strong>에 <> 기호를 추가하고 타입 파라미터를 정의한 뒤, 리턴 타입과 매개변수 타입에서 사용한다.
  </li>
</ul>

```java
// 1. 제네릭 메소드 선언.
/*
public <A, B, ...>  리턴타입  메소드명(매개변수, ...) { ... }
*/
public <T> Box<T> boxing(T t) { ... }
// - <T>: 타입 파라미터로 <T> 정의
// - Box<T>: 리턴 타입을 T로 설정.
// - (T t): 매개 변수 타입도 T로 설정.

// 2. 타입 파라미터 T는 매개값이 어떤 타입이냐에 따라 컴파일 과정에서 구체적인 타입으로 대체된다.
Box<Integer> box1 = boxing(100);
Box<String> box2 = boxing("안녕하세요"); 
```

<h1>4. 제한된 타입 파라미터</h1>
<ul>
  <li>
    경우에 따라 타입 파라미터를 <strong>대체</strong>하는 구체적인 타입을 제한할 필요가 있다.
  </li>
  <li>
    위와 같이 모든 타입으로 대체할 수 없고 <strong>특정 타입</strong>과 <strong>자식 또는 구현 관계</strong>에 있는 타입만 대체할 수 있는 타입 파라미터를 <strong>제한된 타입 파라미터(bounded type parameter)</strong>라 한다.
  </li>
    <ul>
      <li>
        상위 타입은 <strong>클래스</strong>뿐만 아니라 <strong>인터페이스</strong>도 가능하며, 인터페이스의 경우 <strong>implements</strong>를 사용하지 않는다.
      </li>
    </ul>
</ul>

```java
// 1. 제한된 타입 파라미터의 정의
/*
public <T extends 상위타입> 리턴타입 메소드(매개변수, ...) { ... }
*/

// 2. Number 타입과 자식 클래스(Byte, Short, Integer, Long, Double)에만 대체 가능한 타입 파라미터 정의.
public <T extends Number> boolean compare(T t1, T t2) {
  double v1 = t1.doubleValue();    // Number의 double() 메소드 사용.
  double v2 = t2.doubleValue();    // Number의 double() 메소드 사용.
  return (v1 == v2);
}
```

<h1>5. 와일드카드 타입 파라미터</h1>
<ul>
  <li>
    제네릭 타입을 매개값이나 리턴 타입으로 사용할 때 타입 파라미터로 <strong>?(와일드카드)</strong>를 사용할 수 있다.
  </li>
  <li>
    ?는 <strong>범위에 있는 모든 타입</strong>으로 대체할 수 있다는 표시이다.
  </li>
</ul>
