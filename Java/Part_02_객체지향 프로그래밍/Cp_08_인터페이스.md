<h1>1. 인터페이스의 역할</h1>
<ul>
  <li>
    <strong>인터페이스(interface)</strong>는 사전적 의미로 <strong>두 장치를 연결하는 접속기</strong>를 의미한다.
  </li>
  <li>
    두 장치를 개체로 본다면 인터페이스는 <strong>두 객체를 연결</strong>하는 역할을 한다.
  </li>
  <li>
    객체가 서로를 호출할 때 호출되는 객체가 변할 수 있으며 이런 경우의 <strong>다형성을 관리</strong>한다.
  </li>
    <ul>
      <li>
        상속으로 다형성을 관리하는 경우도 있지만 실제로는 인터페이스를 더 많이 사용한다.
      </li>
    </ul>
</ul>

<br><br>

<h1>2. 인터페이스와 구현 클래스 선언</h1>
<ul>
  <li>
    인터페이스는 <strong>'*.java'</strong> 형태의 <strong>소스 파일</strong>로 작성되고, <strong>'*.class'</strong> 형태로 <strong>컴파일</strong> 되기에 물리적 형태는 클래스와 동일하다.
  </li>
  <li>
    단, 소스를 작성할 때 선언하는 방법과 구성 멤버가 클래스와 다르다.
  </li>
</ul>

<h2>2-1. 인터페이스 선언</h2>
<ul>
  <li>
    인터페이스 선언은 <strong>interface</strong> 키워드를 사용한다.
  </li>
  <li>
    접근 제한자는 class와 동일
  </li>
    <ul>
      <li>
        <strong>default</strong>: 같은 패키지 내에서만 사용 가능.
      </li>
      <li>
        <strong>public</strong>: 패키지와 상관없이 사용 가능.
      </li>
    </ul>
</ul>

```java
// 1. interface의 접근 제한자.
//   - default
interface /** 인터페이스명 { ... } **/
//   - public
public interface /** 인터페이스명 {...} **/
```

```java
// 2. 인터페이스가 갖는 멤버들에는 다음과 같은 종류가 있다.
public interface /** 인터페이스명 **/ {
    // public 상수 필드
    // public 추상 메소드
    // public 디폴트 메소드
    // public 정적 메소드
    // private 메소드
    // private 정적 메소드
}
```

<h2>2-2. 구현 클래스 선언</h2>
<ul>
  <li>
    <strong>객체 A</strong>가 인터페이스의 추상 메소드를 호출 → <strong>인터페이스</strong>는 객체 B의 메소드를 실행으로 연결된다.
  </li>
    <ul>
      <li>
        이때 객체 B는 인터페이스에 선언된 추상 메소드와 <strong>동일한 선언부</strong>를 가진 메소드를 갖고 있어야 한다.
      </li>
      <li>
        이때 객체 B를 <strong>인터페이스를 구현한(implement) 객체</strong>라 한다.
      </li>
    </ul>
  <li>
    <strong>implements 키워드</strong>는 해당 클래스가 <strong>인터페이스를 통해 사용할 수 있음</strong>과 <strong>추상 메소드를 재정의한 메소드</strong>가 있다는 의미이다.
  </li>
</ul>

```java
// 1. implements를 통해 인터페이스를 구현하고 있음을 선언부에 명시한다.
public class B implements /** 인터페이스명 **/ { ... }
```

<h2>2-3. 변수 선언과 구현 객체 대입</h2>
<ul>
  <li>
    인터페이스도 하나의 <strong>타입</strong>이다.
  </li>
    <ul>
      <li>
        인터페이스는 <strong>참조 타입</strong>에 속하기에 객체를 참조하지 않고 있다는 의미로 <strong>null</strong>을 대입할 수 있다.
      </li>
    </ul>
  <li>
    인터페이스를 통해 구현 객체를 사용하려면, 인터페이스 변수에 <strong>구현 객체의 번지</strong>를 대입해야 한다.
  </li>
</ul>

```java
// 인터페이스는 참조 타입.
RemoteControl rc;
// 객체를 참조하지 않는다는 의미로 null 대입 가능.
RemoteControl rc = null;

// Television 객체를 생성하고 번지를 대입.
rc = new Televison();

// 대입된 구현 객체를 변수를 통해 인터페이스의 추상 메소드를 호출.
// 이후 메소드 호출 시 인터페이스의 추상 메소드를 호출할 수 있다.
rc.turnOn();

// 변수 선언과 동시에 객체 대입도 가능.
RemoteControl rc = new Television();
```

<h1>3. 상수 필드</h1>
<ul>
  <li>
    인터페이스는 <strong>public static final</strong> 특성을 갖는 <strong>불변의 상수 필드</strong>를 멤버로 가질 수 있다.
  </li>
  <li>
    인터페이스에 선언된 <strong>필드</strong>는 모두 <strong>public static final</strong> 특성을 갖기 때문에 public static final을 생략하여도 자동으로 <strong>컴파일 과정</strong>에서 붙는다.
  </li>
  <li>
    상수명은 <strong>대문자</strong>로 작성하되, 서로 다른 단어로 구성되어 있을 경우 <strong>언더바(_)</strong>로 연결하는 것이 관례이다.
  </li>
  <li>
    상수는 구현 객체와 관련 없는 <strong>인터페이스 소속 멤버</strong>이기에 <strong>인터페이스로 바로 접근</strong>해 상수값을 읽을 수 있다.
  </li>
</ul>

<h1>4. 추상 메소드</h1>
<ul>
  <li>
    인터페이스는 구현 클래스가 재정의해야하는 <strong>public 추상 메소드(abstract method)</strong>를 멤버로 가질 수 있다.
  </li>
    <ul>
      <li>
        추상 메소드란 <strong>리턴타입, 메소드명, 매개변수</strong>만 기술되고 중괄호({})</strong>를 붙이지 않는 메소드를 의미한다.
      </li>
    </ul>
  <li>
    추상 메소드는 객체 A가 인터페이스를 통해 어떻게 메소드를 호출할 수 있는지 <strong>호출 방법</strong>을 알려주는 역할을 한다.
  </li>
  <li>
    구현 클래스에서 <strong>추상 메소드를 재정의</strong>할 때에는 <strong>public보다 더 낮은 접근 제한</strong>으로 재정의할 수가 없다.
  </li>
    <ul>
      <li>
        재정의되는 모든 메소드는 public이 추가되어 있다.  
      </li>
    </ul>
  <li>
    인터페이스로 구현 객체를 사용할 때에는 <strong>인터페이스 변수</strong>를 선언하고, <strong>구현 객체</strong>를 대입해야 한다.
  </li>
    <ul>
      <li>
        인터페이스 변수는 <strong>참조 타입</strong>이기 때문에 구현 객체가 대입되면 <strong>구현 객체의 번지</strong>를 저장한다.
      </li>
    </ul>
  <li>
    구현 객체가 대입되면 인터페이스 변수로 추상 메소드를 호출할 때 <strong>어떤 구현 객체가 대입</strong>되었는가에 따라 실행 내용이 달라진다.
  </li>
</ul>

```java
// 추상 메소드 활용.
/** public abstract 리턴타입 메소드명(매개변수, ...); **/

// 인터페이스 변수 선언 및 구현 객체 대입
RemoteControl rc;
rc = new Television();
RemoteControl rc = new Audio();
```

<h1>5. 디폴트 메소드</h1>
<ul>
  <li>
    인터페이스에는 <strong>완전한 실행 코드를 갖는 디폴트 메소드</strong> 선언이 가능하다.
  </li>
    <ul>
      <li>
        추상 메소드에는 실행부({})가 없지만, 디폴트 메소드에는 <strong>실행부</strong>가 있다. 
      </li>
      <li>
        선언 방법은 클래스 메소드와 동일하지만, <strong>default 키워드</strong>가 리턴 타입 앞에 붙는다.
      </li>
    </ul>
  <li>
    디폴트 메소드의 실행부({})에는 상수 필드를 읽거나 추상 메소드를 호출하는 코드를 작성할 수 있다.
  </li>
  <li>
    디폴트 메소드는 <strong>구현 객체</strong>가 필요한 메소드이다.
  </li>
    <ul>
      <li>
        따라서 메소드를 호출하기 위해서는 구현 객체(Television)를 <strong>인터페이스 변수에 대입한 뒤</strong>에 호출해야 한다.
      </li>
    </ul>
  <li>
    구현 클래스는 <strong>디폴트 메소드</strong>를 재정의해서 자신에게 맞게 수정할 수도 있다.
  </li>
    <ul>
      <li>
        재정의를 할 때에는 <strong>public 접근 제한자</strong>를 반드시 붙이고, <strong>default 키워드를 생략</strong>해야 한다.
      </li>
    </ul>
</ul>

```java
// 디폴트 메소드 선언
/** [public] defualt 리턴타입 메소드명(매개변수, ...) { ... } **/
```
