<h1>1. 예외와 예외 클래스</h1>
<ul>
  <li>
    컴퓨터 <strong>하드웨어의 고장</strong>으로 인해 응용프로그램 실행 오류가 발생하는 것을 자바에서는 <strong>에러(error)</strong>라 한다.
  </li>
  <li>
    자바에는 에러 외에 <strong>예외(exception)</strong>라 부르는 오류가 있다. 예외란 <strong>잘못된 사용 또는 코딩</strong>으로 인한 오를 의미한다. 예외에는 다음 두 가지가 있다.
  </li>
    <ul>
      <li>
        <strong>일반 예외(Exception)</strong>: 컴파일러가 예외 처리 코드 여부를 <strong>검사하는 예외</strong>를 의미한다.
      </li>
      <li>
        <strong>실행 예외(Runtime Exception)</strong>: 컴파일러가 예외 처리 코드 여부를 <strong>검사하지 않는</strong> 예외를 의미한다.
      </li>
    </ul>
  <li>
    자바는 예외가 발생하면 <strong>예외 클래스</strong>로부터 <strong>객체</strong>를 생성한다.
  </li>
  <li>
    자바의 모든 에러와 예외 클래스는 <strong>Throwable</strong>을 상속받아 만들어진다.
  </li>
    <ul>
      <li>
        <strong>예외 클래스</strong>는 추가적으로 <strong>java.lang.Exception 클래스</strong>를 상속받는다.
      </li>
      <li>
        <strong>실행 예외</strong>는 <strong>RuntimeException</strong>과 그 <strong>자식 클래스</strong>에 해당한다.
      </li>
      <li>
        위의 예외를 제외한 모든 예외는 모두 일반 예외이다.
      </li>
    </ul>
  <li>
    자바는 자주 사용되는 예외 클래스를 표준 라이브러리로 제공한다.
  </li>
</ul>

<br><br>

<h1>2. 예외 처리 코드</h1>
<ul>
  <li>
    예외가 발생했을 때 프로그램의 <strong>갑작스러운 종료를 막고 정상 실행을 유지</strong>할 수 있도록 처리하는 코드를 <strong>예외 처리 코드</strong>라 한다.
  </li>
  <li>
    예외 처리 코드는 <strong>try-catch-finally 블록</strong>으로 구성된다.
  </li>
    <ul>
      <li>
        try-catch-finally 블록은 <strong>생성자 내부</strong>와 <strong>메소드 내부</strong>에서 작성된다.
      </li>
      <li>
        try 블록에서 작성한 코드가 <strong>예외없이</strong> 작동하면 catch 블록은 실행되지 않고 <strong>finally 블록</strong>이 실행된다. 반면 <strong>예외가 발생</strong>하면 catch와 finally 블록 <strong>모두 실행</strong>된다.
      </li>
        <ul>
          <li>
            finally 블록은 return 문을 사용하더라도 실행된다(항상 실행됨).
          </li>
          <li>
            finally 블록은 옵션으로 생략 가능하다.
          </li>
        </ul>
    </ul>
</ul>

<br><br>

<h1>3. 예외 종류에 따른 처리</h1>
<ul>
  <li>
    try 블록에서 <strong>다양한 종류의 예외</strong>가 발생할 수 있는데 이때에는 <strong>다중 catch</strong>를 통해 발생하는 예외에 따라 다른 예외 처리 코드를 작성할 수 있다.
  </li>
  <li>
    하나의 예외가 발생하면 즉시 실행을 멈추고 해당 catch 블록으로 이동하기 때문에 <strong>단 하나만 실행</strong>된다.
  </li>
  <li>
    상속 관계의 예외 처리를 할 때에는 <strong>하위 클래스의 catch 블록을 먼저 작성</strong>한 뒤 이후 상위 클래스의 catch 블록을 작성해야 한다.
  </li>
    <ul>
      <li>
        예외가 발생하면 catch 블록을 위에서부터 차례로 검사하는데 하위 예외도 <strong>상위 클래스 타입</strong>이기 때문이다.
      </li>
    </ul>
  <li>
    <strong>두 개 이상의 예외를 하나의 catch 블록</strong>으로 동일하게 예외 처리하고 싶을 때에는 catch 블록에 예외 클래스를 <strong>기호 |</strong>로 연결한다.
  </li>
</ul>

<br><br>

<h1>4. 리소스 자동 닫기</h1>
<ul>
  <li>
    리소스(resource)란 <strong>데이터를 제공하는 객체</strong>를 의미하며 리소스를 사용하기 위해서는 <strong>열고(open)</strong> 사용 후에는 <strong>닫아야(close)</strong> 한다.
  </li>
    <ul>
      <li>
        닫아야 다른 프로그램에서 사용할 수 있다.
      </li>
    </ul>
  <li>
    리소스를 사용하다가 예외가 발생할 경우에도 안전하게 닫는 것이 중요하다.
  </li>
  <li>
    <strong>try-with-resources 블록</strong>을 사용하여 더 쉽게 구현할 수도 있다.
  </li>
    <ul>
      <li>
        try 블록이 정상적으로 실행 완료되거나 예외가 발생하면 <strong>자동으로 리소스의 close() 메소드를 호출</strong>한다.
      </li>
    </ul>
  <li>
    try-with-resource 블록은 <strong>java.lang.AutoCloseable 인터페이스</strong>를 구현한 뒤 <strong>AutoCloseable 인퍼페이스의 close() 메소드를 재정의</strong> 해야한다.
  </li>
  <li>
    <strong>복수 개의 리소스</strong>를 사용해야 하는 경우 try() 괄호 안에 <strong>세미콜론(;)</strong>으로 구분해서 리소스를 여는 코드를 작성한다.
  </li>
</ul>

```java
// 1. FileInputStream을 통한 파일 읽기.
FileInputStream fis = null;
try {
  // 파일 열기
  fis = nwe FileInputStream("file.txt");
  ... 
} catch(IOException e) {
  ...
} finally {
  // 파일 닫기
  fis.close();
}

// 2. try-with-resources 블록을 통해 좀 더 쉽게 읽기.
// - try 블록이 정상적으로 실행을 완료하였거나 도중에 예외가 발생하면 close() 메소드가 호출된다.
public class FileInputStream implements AutoCloseable {
  ...
  @Override
  public void close() throws Exception { ... }
}

// 3. 복수 개의 리소스 사용

FlieInputStream fis1 = new FileInputStream("file1.txt");
FileInputStream fis2 = new FileInputStream("file2.txt");
try(fis1, fis2) {
  ...
} catch(IOException e) {
  ...
}
```

<h1>5. 예외 떠넘기기</h1>
<ul>
  <li>
    메소드 내부에서 예외가 발생할 때 try-catch 블록으로 예외를 처리하는 것이 기본이지만, <strong>throws</strong>를 통해 예외를 <strong>메소드를 호출</strong>한 곳으로 떠넘길 수 있다.
  </li>
    <ul>
      <li>
        throws는 <strong>메소드 선언부 끝</strong>에 작성하며 떠넘길 예외 클래스를 <strong>쉼표로 구분</strong>하여 나열해 주면 된다.
      </li>
      <li>
        throws 키워드가 있는 메소드가 예외를 처리하지 않고 떠넘겼기 때문에 <strong>메소드를 호출하는 곳</strong>에서 예외를 받아 처리해야 한다.
      </li>
    </ul>
  <li>
    나열해야할 클래스가 많을 때에는 <strong>throws Exception</strong> 또는 <strong>throws Throwable</strong> 만으로 모든 예외를 간단히 떠넘길 수도 있다.
  </li>
  <li>
    <strong>main() 메소드</strong>에서도 throws 키워드를 사용해 예외를 떠넘길 수 있다. 이때에는 <strong>JVM</strong>이 최정족으로 예외를 처리한다.
  </li>
    <ul>
      <li>
        JVM은 예외의 내용을 <strong>콘솔에 출력</strong>하는 것으로 예외 처리를 한다.
      </li>
    </ul>
</ul>

```java
// 1. throws를 통한 예외 떠넘기기 형식
/*
리턴타입 메소드명(매개변수, ...) throws 예외클래스1, 예외클래스2, ... {
}
*/

// 2. 떠넘겨진 예외 처리 예.
public void method1() {
  try {
    // method2() 호출.
    method2() {
      // throws된 예외를 받아 처리.
      catch(ClassNotFoundException e) {
        System.out.println("예외 처리: " + e.getMessage());
      }
    }
    
    // method2()에서 예외를 throws
    public void method2() throws ClassNotFoundException {
      Class.forName("java.lang.String2");
    }
  }
}

// 3. 많은 예외 클래스 한 번에 넘기기.
/*
리턴타입 메소드명(매개변수, ...) throws Exception {
}
*/

// 4. main 메소드 내에서 throws 활용.
public static void main(String[] args) throws Exception {
  ...
}
```

<h1>6. 사용자 정의 예외</h1>
<ul>
  <li>
    <strong>표준 예외 클래스에 존재하지 않는 예외 클래스</strong>는 <strong>직접 정의</strong>해서 사용해야하며 이를 사용자 정의 예외라 한다.
  </li>
</ul>

<br>

<h2>6-1. 사용자 정의 예외</h2>
<ul>
  <li>
    사용자 정의 예외는 컴파일러가 체크하는 <strong>일반 예외</strong>로 선언할 수도 있고, 컴파일러가 체크하지 않는 <strong>실행 예외</strong>로 선언할 수도 있다.
  </li>
    <ul>
      <li>
        통상적으로 일반 예외는 <strong>Exception의 자식 클래스</strong>로 선언하고, 실행 예외는 <strong>RuntimeException의 자식 클래스</strong>로 선언한다.
      </li>
    </ul>
  <li>
    사용자 정의 예외 클래스에는 <strong>기본 생성자</strong>와 <strong>에외 메시지를 입력받는 생성자</strong>를 선언한다.
  </li>
    <ul>
      <li>
        예외 메시지는 <strong>부모 생성자 매개값</strong>으로 넘겨준다. 
      </li>
        <ul>
          <li>
            예외 객체의 <strong>공통 메소드인 getMessage()의 리턴값</strong>으로 사용하기 위해서이다.
          </li>
        </ul>
    </ul>
</ul>

```java
// 1. 사용자 정의 예외 클래스
/*
public class XXXException extends | Exception | RuntimeException {
  // 기본 생성자.
  public XXXException() {
  }
  
  // 예외 메시지를 입력받는 생성자.
  public XXXException(String message) {
    super(message);
  }
}
*/
```

<br>

<h2>6-2. 예외 발생 시키기</h2>
<ul>
  <li>
    사용자 정의 예외를 직접 코드에서 발생시키려면 throw 키워드와 함께 <strong>예외 객체를 제공</strong>하면 된다.
  </li>
    <ul>
      <li>
        예외의 원인에 해당하는 메시지를 제공하고 싶다면 <strong>생성자 매개값</strong>으로 전달한다.
      </li>
    </ul>
  <li>
    throws된 예외는 try-catch 블록으로 예외를 처리할 수도 있지만, 대부분 메소드를 호출한 곳에서 예외를 처리하도록 throws 키워드로 예외를 넘긴다.
  </li>
</ul>

```java
// 1. 사용자 정의 예외 발생 시키기.
throw new Exception();
throw new RuntimeException();
throw new InsufficientException();

// 2. 사용자 정의 예외 발생 + 메시지 전달.
throw new Exception("예외메시지");
throw new RuntimeException("예외메시지");
throw new InsufficientException("예외메시지");
```

```java
// 1. 예외 처리 방법.
// 1-1. try-catch 문으로 throw 직접 처리.
void method() {
  try {
    ...
    throw new Exception("예외메시지");
    ...
  } catch(Exception e) {
    String message = e.getMessage();
  }
}

// 1-2. 메시지를 전달.
void method() throws Exception {
  ...
  throw new Exception("예외메시지");
  ...
}
```
