<h1>1. 람다식이란?</h1>
<ul>
  <li>
    <strong>함수형 프로그래밍(functional programming)</strong>이란 <strong>함수를 정의</strong>하고, 함수를 <strong>데이터 처리부</strong>로 보내 데이터를 처리하는 기법을 의미한다.
  </li>
    <ul>
      <li>
        데이터 처리부는 <strong>데이터만</strong> 가지고 있을 뿐, 처리 방법은 정해져 있지 않다.
      </li>
      <li>
        데이터 처리부는 제공된 함수의 입력값으로 데이터를 넣고 함수에 정의된 처리 내용을 실행한다. 따라서 동일한 데이터라도 <strong>제공되는 함수에 따라 결과가 다를 수 있다</strong>.
      </li>
      <li>
        제공된 함수에 따라 결과가 다른 것을 <strong>데이터 처리의 다형성</strong>이라 볼 수도 있다.
      </li>
    </ul>
  <li>
    자바는 함수형 프로그래밍을 위해 Java 8부터 <strong>람다식(Lambda Expression)</strong>을 지원한다.
  </li>
    <ul>
      <li>
        람다식은 데이터 처리부에 제공되는 <strong>함수 역할을 하는 매개변수</strong>를 가진 <strong>중괄호 블록</strong>이다.
      </li>
        <ul>
          <li>
            데이터 터리부는 람자식을 받아 매개변수에 데이터를 대입하고 중괄호를 실행한다.
          </li>
        </ul>
    </ul>
  <li>
    자바는 람다식을 <strong>익명 구현 객체</strong>로 변환한다.
  </li>
    <ul>
      <li>
        익명 구현 객체란 이름이 없는 인터페이스 구현 객체를 의미한다.
      </li>
    </ul>
  <li>
    람다식은 인터페이스의 익명 구현 객체이기에 인터페이스 타입의 매개변수에 대입될 수 있다.
  </li>
    <ul>
      <li>
        그러나 인터페이스의 익명 구현 객체를 람다식으로 표현하려면 <strong>단 하나의 추상 메소드</strong>만 가져야 한다.
      </li>
    </ul>
  <li>
    인터페이스가 단 하나의 추상 메소드를 가질 때, 이를 <strong>함수형 인터페이스(functional interface)</strong>라 한다.
  </li>
    <ul>
      <li>
        함수형 인터페이스임을 보장하기 위해서는 <strong>@FunctionalInterface 어노테이션</strong>을 붙이면 된다.
      </li>
    </ul>
</ul>

```java
// 1. 람다식 형태.
/*
람다식: (매개변수, ...) → { 처리 내용 }
*/


// 2. Calculable 인터페이스 사례로 보는 람다식.
// 2-1. 사용할 Calculable 인터페이스.
public interface Calculable {
  // 추상 메소드
  void calculate(int x, int y);
}

// 2-2. 인터페이스의 익명 구현 객체 생성.
new Calculable() {
  @Override
  public void calculate(int x, int y) { 처리내용 }
};
// 람다식 표현
(x, y) -> { 처리내용 }


// 3. Calculable 매개변수를 갖는 action() 메소드
public void action(Calculable calculate) {
  int x = 10;
  int y = 4;
  calculable.calculate(x, y);    // 데이터를 제공하고 추상 메소드를 호출.
}

// 3-1. action() 메소드를 호출할 때 매개값을 람다식으로 제공.
action( (x, y) -> {
  int result = x + y; // 람다식
  System.out.println(result);
});


// 4. 함수형 인터페이스(인터페이스에 단 하나의 추상 메소드)
// ( ) -> { ... }
public interface Runnable {
  void run();
}

// ( x, y ) -> { ... }
@FunctionalInterface
public interface Calculable {
  void calculate(int x, int y);
}
```

<h1>2. 매개변수가 없는 람다식</h1>
<ul>
  <li>
    함수형 인터페이스의 추상 메소드에 매개변수가 없는 경우 <strong>매개변수를 생략</strong>하고 람다식을 작성할 수 있다.
  </li>
    <ul>
      <li>
        실행문이 하나라면 실행문의 <strong>중괄호를 생략</strong>할 수 있다.
      </li>
    </ul>
</ul>

```java
// 1. 매개변수가 없는 람다식
/*
( ) -> {
  실행문;
  실행문;
}
*/

// 2. 매개변수가 없으면서 실행문이 하나인 람다식 (중괄호 생략).
/*
( ) -> 실행문
*/
```
