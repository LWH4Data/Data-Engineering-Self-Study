<h1>1. 입출력 스트림</h1>
<ul>
  <li>
    데이터는 키보드, 파일, 프로그램 등에서 입력되고, 모니터, 파일 저장, 다른 프로그램 전송 등으로 출력될 수 있다. 이를 총칭하여 <strong>데이터 입출력</strong>이라 한다.
  </li>
  <li>
    자바는 <strong>입력 스트림</strong>과 <strong>출력 스트림</strong>을 통해 데이터를 입출력한다.
  </li>
    <ul>
      <li>
        스트림(stream)은 <strong>단방향</strong>으로 데이터가 흐르는 것을 의미한다.
      </li>
      <li>
        데이터는 <strong>출발지 → 도착지</strong>로 흘러간다.
      </li>
      <li>
        프로그램 관점에서 데이터가 들어오면 입력 스트림, 데이터가 나가면 출력 스트림이 된다.
      </li>
      <li>
        프로그램이 다른 프로그램과 데이터를 교환하려면 양쪽 모두 입력 스트림과 출력 스트림이 필요하다.
      </li>
    </ul>
  <li>
    어떤 데이터를 입출력하는가에 따라 스트림은 <strong>바이트 스트림</strong>과 <strong>문자 스트림</strong>으로 나뉜다.
  </li>
    <ul>
      <li>
        바이트 스트림: 그림, 멀티미디어, 문자 등 <strong>모든 종류의 데이터</strong>를 입출력할 때 사용한다.
      </li>
      <li>
        문자 스트림: <strong>문자</strong>만 입출력할 때 사용한다.
      </li>
    </ul>
  <li>
    자바는 데이터 입출력과 관련된 라이브러리를 <strong>java.io</strong> 패키지에서 제공한다.
  </li>
    <ul>
      <li>
        바이트 입출력 스트림의 최상위 클래스는 <strong>InputStream</strong>과 <strong>OutputStream</strong>이다.
      </li>
        <ul>
          <li>
            이를 상속받는 자식 클래스는 <strong>접미사</strong>로 InputStream과 OutputStream을 받는다. (FileInputStream, ...)
          </li>
        </ul>
      <li>
        문자 입출력 스트림의 최상위 클래스는 <strong>Reader</strong>와 <strong>Writer</strong>이다.
      </li>
        <ul>
          <li>
            이를 상속받는 자식 클래스는 <strong>접미사</strong>로 Reader 또는 Writer가 붙는다. (FileReader, ...)
          </li>
        </ul>
    </ul>
</ul>

<br><br>

<h1>2. 바이트 출력 스트림</h1>
<ul>
  <li>
    OutputStream은 바이트 출력 스트림의 최상위 클래스로 추상 클래스이다. 주요 자식 클래스와 메소드는 p784 참고.
  </li>
</ul>

<br>

<h2>2-1. 1 바이트 출력</h2>
<ul>
  <li>
    <strong>write(int b)</strong> 메소드는 매개값 int(4 byte)에서 <strong>마지막 1 byte</strong>만을 출력한다.
  </li>
    <ul>
      <li>
        매개변수가 int 타입이지만 4 byte를 모두 보내는 것은 아니다.
      </li>
    </ul>
  <li>
    FileOutputStream 생성자는 주어진 파일을 생성할 수 없으면 IOException을 발생시키며 write(), flush(), close() 또한 동일하다. 따라서 <strong>IOException 예외 처리</strong>를 해야한다.
  </li>
  <li>
    OutputStream은 내부에 <strong>작은 버퍼(buffer)</strong>를 가지고 있다.
  </li>
    <ul>
      <li>
        <strong>write()</strong>: 버퍼에 바이트를 <strong>우선 저장</strong>한다. 이후 <strong>버퍼가 차면</strong> 순서대로 바이트를 출력한다.
      </li>
      <li>
        <strong>flush()</strong>: 내부 버퍼에 잔류하는 <strong>모든 바이트를 출력</strong>하고 <strong>버퍼를 비운다</strong>.
      </li>
      <li>
        <strong>close()</strong>: 출력 스트림을 더이상 사용하지 않을 때 <strong>출력 스트림이 사용했던 메모리를 해제</strong>한다.
      </li>
    </ul>
  <li>
    내부 버퍼를 사용하는 이유는 <strong>출력 성능</strong>을 향상시키기 위해서이다.
  </li>
    <ul>
      <li>
        버퍼를 <strong>배치 단위</strong>로 처리하여 매번 처리하지 않고 묶어서 처리하여 성능을 올린다.
      </li>
      <li>
        배치 처리를 통해 <strong>OS 호출 횟수</strong>와 <strong>디스크 접근 횟수</strong>를 줄이기 때문이다.
      </li>
    </ul>
</ul>

<br>

<h2>2-2. 바이트 배열 출력</h2>
<ul>
  <li>
    일반적으로는 1 바이트를 출력하지는 않고 <strong>wrtie(byte[] b) 메소드</strong>를 통해 <strong>주어진 배열의 모든 바이트</strong>를 출력한다.
  </li>
</ul>

<br><br>

<h1>3. 바이트 입력 스트림</h1>
<ul>
  <li>
    InputStream은 <strong>바이트 입력 스트림의 최상위 클래스</strong>이며 <strong>추상 클래스</strong>이다. 
  </li>
    <ul>
      <li>
        모든 바이트 입력 스트림은 <strong>InputStream 클래스</strong>를 상속받아 만들어진다.
      </li>
    </ul>
  <li>
    InputStream 클래스에는 바이트 입력 스트림이 기본적으로 가져야 할 메소드가 정의되어 있다. (p788 참고).
  </li>
</ul>

<br>

<h2>3-1. 1 바이트 읽기</h2>
<ul>
  <li>
    read() 메소드는 입력 스트림에서 <strong>1byte</strong>를 읽고 <strong>int(4byte)</strong> 타입으로 리턴한다.
  </li>
    <ul>
      <li>
        4byte 중 <strong>끝 1byte</strong>에만 데이터가 들어가 있다.
      </li>
    </ul>
  <li>
    더 이상 입력 스트림으로부터 바이트를 <strong>읽을 수 없다면</strong> read() 메소드는 <strong>-1을 반환</strong>한다.
  </li>
    <ul>
      <li>
        이를 활용해 읽을 수 있는 마지막 바이트까지 반복해서 한 바이트씩 읽을 수 있다.
      </li>
    </ul>
  <li>
    FileInputStream 생성자는 주어진 파일이 존재하지 않을 경우 <strong>FileNoteFoundException</strong>을 발생시킨다.
  </li>
    <ul>
      <li>
        read(), close() 메소드에서는 <strong>IOException</strong>이 발생할 수 있다.
      </li>
      <li>
        따라서 <strong>두 가지 예외를 모두 처리</strong>해야 한다.
      </li>
    </ul>
</ul>

```java
// 1. read() 메소드가 -1을 반환할 때까지 데이터 읽기.
InputStream is = ...;
while (true) {

    // 1 바이트를 읽고 리턴
    int data = is.read();

    // -1을 리턴한 경우 데이터를 다 읽은 것이기에 while 문 종료.
    if (data == -1) break;
}
```

<br>

<h2>3-2. 바이트 배열로 읽기</h2>
<ul>
  <li>
    <strong>read(byte[] b) 메소드</strong>는 입력 스트림으로부터 <strong>주어진 배열의 길이</strong>만큼 바이트를 읽고 <strong>배열에 저장</strong>한 다음 <strong>읽은 바이트 수를 리턴</strong>한다.
  </li>
  <li>
    read(byte[] b) 또한 입력 스트림으로부터 바이트를 더 이상 읽을 수 없다면 <strong>-1을 리턴</strong>한다.
  </li>
    <ul>
      <li>
        이를 활용해 마지막 바이트까지 반복해서 읽을 수 있다.
      </li>
    </ul>
  <li>
    한 번만 원하는 수의 입력을 읽을 수 있기에 <strong>많은 양의 바이트</strong>를 읽을 때 좋다.
  </li>
  <li>
    Java 9부터 좀 더 편리하게 <strong>입력 스트림에서 출력 스트림으로 바이트를 복사</strong>하는 <strong>transferTo()</strong> 메소드가 InputStream에 추가되었다.
  </li>
</ul>

```java
// 1. read(byte[] b)가 -1을 반환할 때까지 읽기.
InputStream is = ...;
byte[] data = new[100];
while (true) {
  // 최대 100 byte를 읽고 읽은 바이트는 배열 data에 저장.
  // 읽은 수를 리턴.
  int is.read(data);
  // -1을 리턴하면 while 문 종료.
  if(num == -1) break;
}
```

<h1>4. 문자 입출력 스트림</h1>
<ul>
  <li>
    바이트 입출력 스트림에서는 InputStream과 OutputStream을 사용했다면 문자 입출력은 <strong>Reader</strong>와 <strong>Writer</strong>를 사용한다.
  </li>
</ul>

<br>
<h2>4-1. 문자 출력</h2>
<ul>
  <li>
    Writer는 문자 출력 스트림의 <strong>최상위 클래스</strong>로 <strong>추상 클래스</strong>이다. 모든 문자 출력 스트림 클래는 Wrtier 클래스를 <strong>상속</strong>받아 만들어진다.
  </li>
  <li>
    Writer 클래스에는 모든 문자 출력 스크림이 기본적으로 가져야할 메소드가 정의되어 있다. (p794 참고).
  </li>
  <li>
    Wrtier는 OutputStream과 사용 방법이 동일하지만, 출력 단위가 <strong>문자(char)</strong>이며 문자열을 출력하는 <strong>write(String str) 메소드</strong>를 추가로 제공한다.
  </li>
</ul>

<br>

<h2>4-2. 문자 읽기</h2>
<ul>
  <li>
    Reader는 문자 입력 스트림의 <strong>최상위 클래스</strong>이며 <strong>추상 클래스</strong>이다. 모든 문자 입력 스트림 클래스는 Reader 클래스를 <strong>상속</strong>받아 만들어진다.
  </li>
  <li>
    Reader 클래스에는 문자 입력 스트림이 기본적으로 가져야 할 메소드가 정의되어 있다. (p796 참고).
  </li>
  <li>
    Reader와 InputStream은 사용 방법이 동일하나, Reader는 출력 단위가 <strong>문자(char)</strong> 이라는 차이가 있다.
  </li>
</ul>

<br><br>

<h1>5. 보조 스트림</h1>
<ul>
  <li>
    보조 스트림은 <strong>다른 스트림과 연결</strong>되어 여러 가지 <strong>편리한 기능을 제공</strong>하는 스트림이다.
  </li>
  <li>
    보조 스트림은 자체적으로 <strong>입출력이 불가</strong>하여 <strong>입출력 소스로부터 직접 생성된 스트림에 연결</strong>해 사용해야 한다.
  </li>
  <li>
    입출력 스트림에 보조 스트림을 연결하려면 보조 스트림을 생성할 때 <strong>생성자 매개값</strong>으로 입출력 스트림을 제공하면 된다.
  </li>
    <ul>
      <li>
        보조스트림 변수 = new 보조스트림(입출력스트림);
      </li>
    </ul>
  <li>
    보조 스트림은 또 다른 보조 스트림과 연결되어 <strong>스트림 체인</strong>으로 구성할 수 있다.
  </li>
</ul>

```java
// 1. 입력 스트림인 FileInputStream에 InputStreamReader 보조 스트림 연결.
InputStream is = new FileInputStream("...");
InputStreamReader reader = new InputStreamReader( is ); 

// 2. InputStreamReader에 BufferedReader 보조 스트림 연결.
InputStream is = new FileInputStream("...");
// FileInputStream is → InputStreamReader reader
InputStreamReader reader = new InputStreamReader( is );
// InputStreamReader reader → BufferedReader br
BufferedReader br = new BufferedReader( reader );
```

<h1>6. 문자 변환 스트림</h1>
<ul>
  <li>
    바이트 스트림(InputStream, OutputStream)에서 입출력할 데이터가 <strong>문자</strong>라면 <strong>문자 스트림(Reader, Writer)</strong>로 변환해 사용하는 것이 좋다.
  </li>
    <ul>
      <li>
        문자로 입출력하는 <strong>편리함</strong>과 <strong>문자셋의 종류</strong>를 지정할 수 있기 때문이다.
      </li>
    </ul>
  <li>
    FileInputStream에 InputStreamReader를 연결하지 않고 FileReader를 직접생성할 수 있다.
  </li>
    <ul>
      <li>
        FileReader는 <strong>InputStreamReader의 자식 클래스</strong>이다.
      </li>
    </ul>
</ul>

<br>

<h2>6-1. InputSteram을 Reader로 변환</h2>
<ul>
  <li>
    InputStream을 Reader로 변환하려면 InputStreamReader 보조 스트림을 연결하면 된다.
  </li>
</ul>

```java
// 1. InputStream을 Reader로 변환.
InputStream is = new FileInputStream("C:/Temp/test.txt");
Reader reader = new InputStreamReader(is);
```

<br>

<h2>6-2. OutputStream을 Writer로 변환</h2>
<ul>
  <li>
    OutputStream을 Writer로 변환하려면 <strong>OutputStreamWriter 보조 스트림</strong>을 연결한다.
  </li>
  <li>
    마찬가지로 FileOutputStream에 OutputStreamWriter를 연결하지 않고 FileWriter를 직접 생성할 수 있다.
  </li>
    <ul>
      <li>
        FileWriter는 OutputStreamWriter의 자식 클래스이다.
      </li>
      <li>
        즉, FileWriter가 내부적으로 FileOutputStream에 OutputStreamWriter 보조 스트림을 연결한 것으로 볼 수 있다.
      </li>
    </ul>
</ul>

```java
// 1. OutputStream을 Writer로 변환.
OutputStream os = new FileOutputStream("C:/Temp/test.txt");
Wrtier writer = new OutputStreamWriter(os);
```

<h1>7. 성능 향상 스트림</h1>
<ul>
  <li>
    CPU, 메모리와는 별개로 <strong>하드 디스크의 입출력</strong>이 늦거나 <strong>네트워크에 문제</strong>가 있으면 실행 성능에 문제가 발생한다.
  </li>
  <li>
    프로그램의 입출력 소스와 직접 작업하지 않고 중간에 <strong>메모리 버퍼(buffer)</strong>와 작업을 하면 실행 성능을 향상시킬 수 있다.
  </li>
    <ul>
      <li>
        입출력에 있어 직접 하드 디스크에 접근하는 것이 아니라 메모리 버퍼에 일을 넘기면 <strong>디스크 접근을 줄여</strong> 성능이 올라간다.
      </li>
    </ul>
  <li>
    메모리 버퍼를 제공하여 프로그램의 실행 성능을 향상시키는 <strong>보조 스트림</strong>이 있다.
  </li>
    <ul>
      <li>
        바이트 스트림에는 BufferedInputStream과 BufferedOutputStream이 있다.
      </li>
      <li>
        문자 스트림에는 BufferedReader와 BufferedWriter가 있다.
      </li>
    </ul>
  <li>
    문자 입력 스트림 Reader에 BufferedReader를 연결하면 성능 향상 외에도 <strong>행 단위로 문자열을 읽는 readLine() 메소드</strong>를 사용할 수 있다.
  </li>
</ul>

```java
// 1. readLine() 메소드 활용 예.
BufferedReader br = new BufferedReader(new FileReader("..."));
while(true) {
    // 파일에서 한 행씩 읽음.
    String str = br.readLine()    
    // 더 이상 읽을 행이 없을 경우(파일 끝) while 문 종료.
    if(str == null) break;        
}
```

<h1>8. 기본 타입 스트림</h1>
<ul>
  <li>
    바이트 스트림에 <strong>DataInputStream</strong>과 <strong>DataOutputStream</strong> 보조 스트림을 연결하면 <strong>기본 타입</strong>인 boolean, char, short, int, long, float, double 값을 입출력할 수 있다.
  </li>
  <li>
    자세한 메소드는 p807 참고.
  </li>
  <li>
    단, 데이터 타입의 크기가 모두 다르기에 <strong>입력과 출력 순서가 동일</strong>해야 한다.
  </li>
    <ul>
      <li>
        출력 순서가 int → Boolean → double이라면 입력 순서 또한 int → Boolean → double이 되어야 한다.
      </li>
    </ul>
</ul>

```java
// 1. DataInputStream, DataOutputStream 보조 스트림 연결.
DataInputStream dis = new DataInputStream(바이트 입력 스트림);
DataOutputStream dos = new DataOutputStream(바이트 출력 스트림);
```

<h1>9. 프린트 스트림</h1>
<ul>
  <li>
    PrintStream과 PrintWrtier는 프린터와 유사하게 출력하는 print(), println(), printf() 메소드를 갖는 보조 스트림이다.
  </li>
    <ul>
      <li>
        콘솔에 System.out.println()이라고 표기한 것을 out이 PrintStream 타입이기 때문이다. (System 클래스 소속).
      </li>
    </ul>
  <li>
    PrintStream은 바이트 출력 스트림과, PrintWriter는 문자 출력 스트림과 연결된다.
  </li>
  <li>
    PrintStream과 PrintWriter는 거의 같은 메소드를 가지고 있다. (p809 참고).
  </li>
</ul>

<h1>10. 객체 스트림</h1>
<ul>
  <li>
    자바는 메모리에 생성된 객체를 파일 또는 네트워크로 출력할 수 있다.
  </li>
    <ul>
      <li>
        객체를 출력할 때에는 필드값을 <strong>일렬로 늘어선 바이트</strong>로 변경해야하며 이를 <strong>직렬화(serialization)</strong>라고 한다.
      </li>
      <li>
        반대로 직렬화된 바이트를 <strong>객체의 필드값으로 복원</strong>하는 것을 <strong>역직렬화(deserialization)</strong>라 한다.
      </li>
    </ul>
  <li>
    ObjectInputStream과 ObjectOutputStream은 객체를 입출력할 수 있는 보조 스트림이다.
  </li>
    <ul>
      <li>
        <strong>ObjectInputStream</strong>은 바이트 입력 스트림과 연결되어 객체를 <strong>역직렬화</strong> 한다.
      </li>
      <li>
        <strong>ObjectOutputStream</strong>은 바이트 출력 스트림과 연결되어 객체를 <strong>직렬화</strong> 한다.
      </li>
    </ul>
  <li>
    대규모 데이터를 바이트로 처리하는 등의 작업에 필요하다.
  </li>
</ul>

```java
// 1. ObjectInputStream과 ObjectOutputStream 보조 스트림 연결.
ObjectInputStream ois = new ObjectInputStream 입력 스트림;
ObjectOutputStream oos = new ObjectOutputStream 출력 스트림;

// 2. ObjectOutputStream으로 객체를 직렬화하기 위해서는 writeObject() 메소드 사용.
oos.writeObject(객체);

// 3. readObject으로 역질렬화. 단, 이때에는 목적에 맞게 Object 리턴 
//    타입을 구체적인 타입으로 강제 타입 변한해야 한다.
객체타입 변수 = (객체타입) ois.readObject();
```

<h2>10-1. Serializable 인터페이스</h2>
<ul>
  <li>
    자바는 <strong>Serializable 인터페이스</strong>를 구현한 클래스만 직렬화할 수 있다.
  </li>
    <ul>
      <li>
        Serializable 인터페이스는 멤버가 없는 빈 인터페이스이다. 단지 객체를 <strong>직렬화 할 수 있다는 표시 역할</strong>을 한다.
      </li>
    </ul>
  <li>
    객체가 직렬화될 때 인스턴스 필드값은 직렬화 대상이지만 <strong>정적 필드값</strong>과 <strong>transient로 선언된 필드 값</strong>은 직렬화에서 제외된다.
  </li>
    <ul>
      <li>
        static은 객체 상태가 아닌 <strong>클래스 전체의 상태</strong>이기에 제외한다.
      </li>
      <li>
        transient는 사용자가 <strong>직렬화 대상이 아님을 명시</strong>한 것이기에 제외한다.
      </li>
    </ul>
</ul>

```java
// 1. Serializable 구현체 사례.
public class XXX implements Serializable {
  // 직렬화 대상 네
  public int field1;
  protected int field2;
  int field3;
  private int field4;
  // 정적 필드 제외
  public static int field5;
  // transient 제외.
  transient int field6;
}
```

<br>

<h2>10-2. serialVersionUID 필드</h2>
<ul>
  <li>
    직렬화할 때 사용된 클래스와 역직렬화할 때 사용된 클래스는 기본적으로 <strong>동일한 클래스</strong>여야 한다.
  </li>
  <li>
    <strong>동일한 serialVersionUID</strong>를 가지면서 <strong>클래스 이름</strong>이 동일하고 <strong>필드 구조가 호환</strong>되면 직렬화/역직렬화가 가능하다.
  </li>
  <li>
    serialVersionUID는 가능한한 유일한 값으로 설정하는 것이 좋다.
  </li>
</ul>

```java
// 1. 서로 다른 클래스로는 직렬화와 역직렬화 불가.
//   - 두 번째 클래스에는 field3이 있어 동일한 클래스가 아니다.
// 1-1. 첫 번째 클래스.
public class Member implements Serializable {
  int field1;
  int field2;
}

// 1-2. 두 번째 클래스.
public class Member implements Serializable {
  int field1;
  int field2;
  // 위와 다른 필드가 추가됨.
  int field3;
}

// 2. 동일한 serialVersionUID를 갖는 경우 가능.
// 2-1. 첫 번째 클래스.
public class Member implements Serializable {
  // serialVersionUID 1로 설정.
  static final long
    serialVersionUID = 1;
  int field1;
  int field2;
}

// 2-2. 두 번째 클래스.
public class Member implements Serializable {
  // 동일한 serialVersionUID를 사용하여 직렬화/역직렬화 가능.
  static final long
    serialVersionUID = 1;
  int field1;
  int field2;
  int field3;
}
```

<h1>11. File과 Files 클래스</h1>
<ul>
  <li>
    java.io 패키지와 java.nio.file 패키지는 <strong>파일과 디렉토리 정보</strong>를 가지고 있는 <strong>File과 Files 클래스</strong>를 제공한다.
  </li>
    <ul>
      <li>
        Files는 File을 개선한 클래스이다.
      </li>
    </ul>
</ul>

<br>

<h2>11-1. File 클래스</h2>
<ul>
  <li>
    File 객체는 생성을 하였다 하여 파일이나 데렉토리가 생성되지는 않는다.
  </li>
    <ul>
      <li>
        실제로 파일이나 디렉토리가 없는 경우에도 예외가 발생하지 않는다.
      </li>
      <li>
        따라서 실제로 파일이나 디렉토리의 존재 여부를 확인하고 싶다면 객체 생성후 <strong>exists() 메소드</strong>를 호출하면 된다.
      </li>
      <li>
        자세한 메소드 정보는 p816 확인.
      </li>
    </ul>
  <li>
    <strong>입출력 스트림을 생성</strong>할 때 File 객체를 활용할 수 있다.
  </li>
</ul>

```java
// 1. File 클래스로 File 객체 생성.
File file = new File("경로");

// 2. 윈도우에서 File 객체 생성.
File file = new File("C:/Temp/file.txt");
File file = new File("C:\\Temp\\file.txt");

// 3. File 객체를 통해 파일 입출력 생성.
// 3-1. 첫 번째 방법. (직접 경로 전달).
FileInputStream fis = new FileInputStream("C:/Temp/image.gif");

// 3-2. 두 번째 방법. (file 객체를 생성하고 전달).
FileInputStream fis = new FileInputStream(file);
```

<br>

<h2>11-2. Files 클래스</h2>
<ul>
  <li>
    Files 클래스는 <strong>정적 메소드</strong>로 구성되어 있어 File 클래스처럼 <strong>객체로 만들 필요가 없다</strong>.
  </li>
    <ul>
      <li>
        Files의 정적 메소드는 <strong>운영체제의 파일 시스템</strong>에게 파일 작업을 수행하도록 <strong>위임</strong>한다.
      </li>
    </ul>
  <li>
    Files 클래스가 갖는 정적 메소드는 p819 참고.
  </li>
    <ul>
      <li>
        메소드들은 매개값으로 <strong>Path 객체</strong>를 받는다. Path 객체는 파일이나 <strong>디렉토리를 찾기위한 경로 정보</strong>를 갖고 있다.
      </li>
      <li>
        Path 객체의 경로 정보는 <strong>정적 메소드 get()</strong>을 통해 얻을 수 있다.
      </li>
        <ul>
          <li>
            get() 메소드의 매개값은 <strong>파일 경로</strong>이다.
          </li>
          <li>
            파일 경로는 전체 경로를 <strong>한 번에 지정</strong>할 수도 있고 원할 경우 상위/하위 디렉터리를 <strong>나누어 지정</strong>할 수도 있다.
          </li>
          <li>
            파일 경로는 <strong>절대 경로</strong>와 <strong>상대 경로</strong> 모두 활용할 수 있다.
          </li>
        </ul>
    </ul>
</ul>

```java
// 1. 정적 메소드 get()을 통해 경로 정보 얻기.
Path path = Paths.get(String first, String ... more)

// 2. "C:\Temp\dir\file.txt" 경로를 통해 Path 객체 얻기.
Path path = Paths.get("C:/Temp/dir/file.txt");
Path path = Paths.get("C:/Temp/dir", "file.txt");
Path path = Paths.get("C:", "Temp", "dir", "file.txt");

// 3. 절대 경로와 상대 경로를 통한 경로 지정.
// 3-1. 절대 경로.
Path path = Paths.get("dir/file.txt");
// 3-2. 상대 경로.
Path path = Paths.get("./dir/file.txt");
// 3-3. 상위 디렉토리를 통한 경로 지정.
Path path = Paths.get("../dir2/file.txt");
```
