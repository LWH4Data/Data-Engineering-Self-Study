# 🌐 네트워크 기초 정리 (IP, DNS, 프록시, ISP)

---

## 1. 📡 데이터 전송 방식

### ✅ 유선 vs 무선

- **유선**
  - 전기/빛으로 데이터 전달
  - 안정적, 빠름
- **무선**
  - 전파(전자기파)로 전달
  - 자유롭지만 간섭 있음

👉 데이터는 항상 0과 1, 전달 방식만 다름

---

## 2. 💻 네트워크 어댑터 vs 공유기

### ✅ 네트워크 어댑터 (LAN 카드)
- 내 컴퓨터 내부 장치
- 네트워크 연결 역할
- 어댑터마다 IP 할당

### ✅ 공유기
- 여러 기기를 인터넷에 연결
- 내부 IP 분배 (DHCP)
- 외부와 연결하는 출구 역할

---

## 3. 🌐 IP 주소 구조

### ✅ 공인 IP (Public IP)
- 외부 인터넷에서 보이는 IP
- 공유기 기준 1개

### ✅ 사설 IP (Private IP)
- 내부 네트워크에서 사용하는 IP
- 기기마다 다름

구조:
내 PC (192.168.x.x)
   ↓
공유기 (공인 IP)
   ↓
인터넷

---

## 4. 🔢 서브넷 마스크

### ✅ 정의
👉 같은 네트워크 범위를 구분하는 기준

예:
- IP: 192.168.31.85
- 서브넷: 255.255.252.0

👉 같은 네트워크 범위:
192.168.28.0 ~ 192.168.31.255

### ✅ 핵심 규칙
- 255 → 고정 (네트워크)
- 0 → 자유 (기기)
- 252 → 일부만 고정 (묶임)

👉 계산:
256 - 252 = 4 → 4개 단위로 묶임

---

## 5. 🚪 기본 게이트웨이

👉 인터넷으로 나가는 출구 (공유기 IP)

예:
- 게이트웨이: 192.168.28.1

---

## 6. 🌍 DNS (Domain Name System)

### ✅ 정의
👉 도메인 → IP로 변환하는 시스템

예:
google.com → 142.xxx.xxx.xxx

---

## 7. 🧠 DNS 캐시 구조

DNS는 여러 단계에 저장됨:

1. 브라우저 캐시
2. OS 캐시
3. 공유기
4. ISP DNS
5. 루트 DNS → 권한 DNS

구조:
브라우저 → OS → 공유기 → ISP → 루트 DNS

---

## 8. ⏱ DNS 속도

- 캐시 없으면 → 약간 느림
- 캐시 있으면 → 매우 빠름

👉 대부분 캐시 덕분에 빠름

---

## 9. 🏢 ISP (Internet Service Provider)

### ✅ 정의
👉 인터넷 연결을 제공하는 회사

예:
- KT
- SK Broadband
- LG U+

### ✅ 역할
- 인터넷 연결
- 공인 IP 제공
- DNS 제공

구조:
내 PC → 공유기 → ISP → 인터넷

---

## 10. 🔁 프록시 (Proxy)

### ✅ 정의
👉 중간에서 요청을 대신 처리하는 서버

구조:
사용자 → 프록시 → 실제 서버

---

## 11. 🌐 프록시 IP

👉 프록시 서버의 공인 IP

- 외부에는 프록시 IP만 보임
- 실제 서버 IP는 숨겨짐

---

## 12. ⚠️ DNS vs 프록시 차이

- DNS
  - 도메인 → IP 변환
  - 단순 안내 역할

- 프록시
  - 요청을 대신 처리
  - 보안, 캐싱, 로드밸런싱 수행

👉 DNS = 네비게이션  
👉 프록시 = 대리인

---

## 🔥 최종 핵심 요약

- IP는 기기가 아니라 **어댑터 기준**
- 공유기는 **공인 IP 1개 + 사설 IP 여러 개**
- 서브넷은 **네트워크 범위 정의**
- DNS는 **이름 → IP 변환 시스템**
- 프록시는 **중간 서버**
- ISP는 **인터넷 제공자**

👉 전체 흐름:
내 PC → 공유기 → ISP → 인터넷 → 서버
