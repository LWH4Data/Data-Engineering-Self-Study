<h1>2. 로컬(지역) 변수 타입 추론</h1>
<ul>
  <li>
    자바 11부터는 <strong>로컬(지역) 변수</strong>를 위한 <strong>타입 추론(type inference for local variable)</strong> 기능을 사용할 수 있다.
  </li>
  <li>
    변수를 선언할 때 예약된 타입(reserved type)인 <strong>var</strong>를 사용하면 컴파일 시에 로컬 변수에 <strong>대입되는 값</strong>을 보고 컴파일러가 <strong>추론(inference)하여 타입을 결정</strong>한다.
  </li>
  <li>
    로컬 변수 타입 추론 기능은 개발자들 사이에서 <strong>코드의 간결성</strong>과 <strong>가독성</strong>을 기준으로 의견이 나뉜다.
  </li>
  <li>
    예약된 타입 var는 <strong>생성자(constructor)</strong> 혹은 <strong>메소드(method)</strong> 안에서 로컬 변수로 선언할 때에만 활용이 가능하다.
  </li>
    <ul>
      <li>
        클래스나 인터페이스에서는 사용이 불가하다.
      </li>
    </ul>
</ul>

```java
// 1. 전통적인 방식의 변수 선언 (타입 명시).
void method() {
  String name = "홍길동";
  int age = 25;
}

// 2. var를 활용한 변수 선언.
void method() {
  var name = "홍길동";
  var age = 25;
}
```

<h1>3. switch 문의 null 처리</h1>
<ul>
  <li>
    switch 문에는 다양한 표현식이 존재한다.
  </li>
</ul>

```java
// 1. 전통적 방식.
switch(표현값) {
  case 레이블: ...
    실행문; ...
    [ break; ]
...
default:
  실행문; ...
  [ break ]
}

// 2. 강화된 방식 1(자바 17부터)
switch(표현값) {
  case 레이블 [, 레이블] -> {
    실행문; ...
  }
  ...
  default -> {
    실행문, ...
  }
}

// 3. 강화된 방식 2(자바 17부터)
타입 변수 = switch(표현값) {
  case 레이블 [, 레이블] -> 리턴값;
  ...
  case 레이블 [, 레이블] -> {
    실행문; ...
    yield 리턴값;
  }
    ...
    default -> 리턴값
};

// 4. 자바 17부터는 레이블에 null을 지정하여 NullPointerExcepton 예외를 방지할 수 있다.
switch(object) {
  ...
  case null          -> { ... } // object가 null일 경우 선택.
  case null, default -> { ... } // object가 null이거나 위의 case가 선택되지 않은 경우 선택.
}
```

<h1>4. switch 문의 패턴 매칭</h1>
<ul>
  <li>
    자바 21부터는 switch 레이블에 <strong>패턴과 가드</strong>를 작성하여 <strong>표현값과 매칭</strong>시킬 수도 있다.
  </li>
  <li>
    표현식이 <strong>객체를 참조하는 변수</strong>일 경우에만 가능하다.
  </li>
</ul>

```java
// 1. 강화된 방식 1(자바 21부터)
switch(표현값) {
  case 패턴 [가드] -> {
    실행문; ...
  }
  ...
  case null [, default] -> {
    실행문; ...
  }
  default -> {
    실행문; ...
  }
}

// 2. 강화된 방식 2(자바 21부터)
타입 변수 = switch(표현값) {
  case 패턴 [가드] -> 리턴값;
  case 패턴 [가드] -> {
    ...;
    yield 리턴값;
  }
  ...
  case null [, default] -> 리턴값;
    default -> 리턴값;
};
```

<br>

<h2>4-1. 레이블에 패턴 사용</h2>
<ul>
  <li>
    표현값이 <strong>참조 타입 변수</strong>일 경우 <strong>패턴</strong>을 사용하여 <strong>타입 검사</strong>를 수행하고, <strong>자동 타입 변환</strong>하여 <strong>패턴 변수를 초기화</strong>한다.
  </li>
    <ul>
      <li>
        패턴 변수는 <strong>중괄호({}) 블록</strong>에서 사용할 수 있다.
      </li>
    </ul>
  <li>
    표현값은 패턴 중 하나와 <strong>반드시 매칭</strong>되도록 해야 한다.
  </li>
    <ul>
      <li>
        매칭할 패턴이 없는 경우 나머지 매칭을 위해 <strong>default</strong>를 포함해야 하며 이를 표현값과 실행문의 <strong>완전성(exhaustiveness)</strong>이라 한다.
      </li>
    </ul>
</ul>

```java
// 1. 패턴 변수 사용.
switch(object) {
  // Integer와 String은 참조 타입(클래스)이므로, i와 s는 객체를 참조하는 변수이다.
  case Integer i -> { // i 변수 사용 }    // object가 Integer 타입인 경우 매칭(자동 타입 변환).
  case String s -> { // s 변수 사용 }     // obeject가 String 타입인 경우 매칭
}

// 2. 패턴 매칭의 예시.
switch(object) (
  case Integer i        -> { ... }
  case String s         -> { ... }
  case null, default    -> { ... }
)
```

<br>

<h2>4-2. 가드 사용</h2>
<ul>
  <li>
    패턴과 함께 좀 더 상세한 입력 조건을 만들기 위해 <strong>when</strong>으로 시작하는 <strong>가드(guard)</strong>를 사용할 수 있다.
  </li>
    <ul>
      <li>
        when의 뒤에는 <strong>패턴 변수</strong>가 사용되며 <strong>boolean을 리턴하는 조건식</strong> 또는 <strong>메소드 호출 코드</strong>가 올 수 있다. (true를 리턴하면 레이블이 선택되고 중괄호가 실행된다).
      </li>
    </ul>
</ul>

```java
// 1. when 사용 사례
Switch(object) {
  case Integer i when i > 0 {
    // object가 Integer 타입이면서 양수일 경우 선택.
  }

  case String s when s.equals("a") {
    // object가 String 타입이면서 "a"일 경우 선택.
  }
}
```

<br>

<h2>4-3. 레이블 통과</h2>
<ul>
  <li>
    레이블에 <strong>패턴</strong>이 사용되면 기본적으로 다음 레이블로 통과가 금지된다.
  </li>
    <ul>
      <li>
        다음이 <strong>default</strong>라면 통과할 수 있다.
      </li>
      <li>
        <strong>"->"</strong>를 사용하면 무조건 통과가 금지된다.
      </li>
    </ul>
</ul>

```java
// 1. 레이블 통과 사례.
// 1-1. 기본적으로 통과 불가.
switch (object) {
  case String s:
    ...
    break;
  
  // 1-2. 다음이 default이기에 통과 가능.
  case Integer i:
    ...
    break; // 명시적 통과 금지.

  default:
    ...
}

// 2. "->"을 사용할 경우 절대 통과 금지.
switch(object) {
  case Integer i -> { ... } 
  case String s -> { ... }
  default -> { ... }
}
```

<h2>4-4. 레이블 작성 순서</h2>
<ul>
  <li>
    레이블이 패턴인 경우 <strong>좁은 범위의 패턴</strong>을 먼저 작성하고, <strong>넓은 범위의 패턴</strong>을 나중에 작성한다.
  </li>
  <li>
    switch문은 <strong>위에서부터 순차적</strong>으로 표현값과 패턴을 매칭하기 때문에 <strong>위쪽 패턴이 먼저 매칭되면 아래쪽 패턴은 검사하지 않는다.
  </li>
  <li>
    <strong>부모</strong>는 자식 클래스보다 항상 넓은 범위를 가지고, <strong>인터페이스</strong>는 구현 클래스보다 항상 넓은 범위를 가진다.
  </li>
</ul>

```java
// 1. 아래로 갈수록 정수 범위가 넓어지도록 작성한다.
Integer data = 3;
switch (data) {
  case 0 -> System.out.println("0");
  case Integer i when i > 0 -> System.out.println("0 or positive number");
  case Integer i -> System.out.println("negative number");
}

// 2. 부모-작식의 작성 순서.
class A {}
class B extends A {}
Object obj = new A();
switch obj {
  case B b -> System.out.println("B type");        // 자식 패턴 작성.
  case A a -> System.out.println("A type");        // 부모 패턴 작성.
  default -> System.out.println("unknown type");   // 이외의 패턴.
}
```

<h1>5. 레코드 패턴</h1>
<ul>
  <li>
    레코드(record)는 <strong>불변의 값 객체(value Object)</strong>이다.
  </li>
    <ul>
      <li>
        레코드가 생성될 때 전달받은 매개값은 <strong>필드값으로 세팅</strong>되며 사용 중일 때에는 필드값을 바꿀 수 없다.
      </li>
      <li>
        레코드는 주로 <strong>DTO(Data Transfer Object)용</strong>으로 사용한다.
      </li>
    </ul>
  <li>
    자바 21부터 지원하는 레코드 패턴(Record Patterns)은 레코드의 필드값을 분해해서 <strong>변수에 대입</strong>하는 기능을 제공한다.
  </li>
    <ul>
      <li>
        레코드의 필드값을 얻기 위해 <strong>Getter를 호출할 필요 없이</strong> 매개변수로 바로 받을 수 있어 편리하다.
      </li>
    </ul>
</ul>

```java
// 1. Switch 문의 레이블에 레코드 패턴 사용 사례.
if(obj instanceof 레코드(타입 변수, 타입 변수)) {
  // 변수 사용.
}

switch(obj) {
  case 레코드(타입 변수, 타입 변수) {
    // 변수 사용.
  }
}
```

<h1>6. 가상 스레드</h1>
<ul>
  <li>
    <strong>가상(virtual) 스레드</strong>는 자바 21부터 사용할 수 있는 기능으로 <strong>처리량이 높은 동시 애플리케이션</strong>을 개발할 때 사용할 수 있는 경량(lightweight) 스레드이다.
  </li>
</ul>

<br>

<h2>6-1. 가상 스레드 개요</h2>
<ul>
  <li>
    자바 17까지는 스레드는 <strong>운영체제가 제공하는 플랫폼(platform) 스레드</strong>를 래핑했기 때문에 스레드와 플랫폼 스레드가 <strong>1:1로 매핑</strong> 되었다.
  </li>
    <ul>
      <li>
        CPU 코어와 플랫폼 스레드가 1:1 매핑인 것은 아니다.
      </li>
      <li>
        플랫폼 스레드는 비용(CPU 또는 메모리)이 많이 들기 때문에 <strong>스레드 수를 제한</strong>해서 사용해야 했다.
      </li>
    </ul>
  <li>
    자바 21부터는 스레드가 플랫폼 스레드와 <strong>n:1로 매핑</strong>된다. 따라서 플랫폼 스레드 부족 문제를 완하한다.
  </li>
    <ul>
      <li>
        가상 스레드는 CPU에서 <strong>계산</strong>을 수행하는 동안에만 플랫폼 스레드를 사용한다.
      </li>
        <ul>
          <li>
            가상 스레드가 블로킹 I/O 작업(파일 입풀력, 네트워킹)을 수행하는 경우 가상 스레드는 일시 정지 되지만, 플랫폼 스레드는 <strong>다른 가상 스레드의 작업을 처리</strong>한다. 
          </li>
        </ul>
    </ul>
</ul>

<br>

<h2>6-2. 가상 스레드풀 생성</h2>
<ul>
  <li>
    가상 스레드는 메모리가 부족하지 않다면 개수를 무제한으로 사용할 수 있기 때문에 스레드풀에서 <strong>최대 개수를 제한할 필요가 없다</strong>. 
  </li>
    <ul>
      <li>
        실제로 가상 스레드를 생성하는 Executor의 메소드는 최대 개수를 입력받지 않는다.
      </li>
    </ul>
  <li>
    가상 스레드는 플랫폼 스레드에 비해 비교적 생성 비용이 저렴하고 개수에 제한을 받지 않기 때문에 스레드풀로 생성하여 관리하지 않는다. 대신 <strong>작업 건수별</strong>로 새로운 가상 스레드를 생성해 처리한다.
  </li>
    <ul>
      <li>
        개발자가 가상 스레드들이 몇 개의 플랫폼 스레드를 사용하고 있는지 알 필요는 없다.
      </li>
    </ul>
</ul>

```java
// 1. 플랫폼 스레드풀.
ExecutorService platformExecutor = Executors.newFixedThreadPool(최대 개수);

// 2. 가상 스레드풀
ExecutorService virtualExecutor = Executors.newVirtualThreadPerTaskExecutor();
```
