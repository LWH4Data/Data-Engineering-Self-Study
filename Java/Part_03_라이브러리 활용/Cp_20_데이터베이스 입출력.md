<h1>1. JDBC 개요</h1>
<ul>
  <li>
    자바는 데이터베이스(DB)와 연결해서 데이터 입출력 작접을 할 수 있도록 <strong>JDBC(Java Database Connectivity) 라이브러리(java.sql 패키지)</strong>를 제공한다.
  </li>
    <ul>
      <li>
        DBMS의 종류와 상관없이 동일하게 사용할 수 있도록 클래스와 인터페이스가 구현되어 있다.
      </li>
    </ul>
  <li>
    JDBC 인터페이스를 통해 실제로 DB와 작업하는 것은 <strong>JDBC Driver</strong>이다. 
  </li>
    <ul>
      <li>
        JDBC Driver는 <strong>JDBC 인터페이스를 구현</strong>한 것으로 <strong>DBMS마다 다운로드</strong> 받아 사용한다.
      </li>
    </ul>
  <li>
    연결 옵션들은 다음과 같다.
  </li>
    <ul>
      <li>
        DriverManager: JDBC Driver 관리, DB와 연결 및 Connection 구현 객체 생성.
      </li>
      <li>
        Connection: Statement, PreparedStatement, CallableStatement 구현 객체 생성, 트랜잭션(Transaction) 처리 및 DB 연결 해제.
      </li>
        <ul>
          <li>
            Statement: SQL의 DDL(Data Definition Language)와 DML(Data Manipulation Language)를 실행할 때 주로 사용. 주로 변경되지 않는 정적 SQL 문을 실행할 때 사용.
          </li>
          <li>
            PreparedStatement: SQL의 DDL과 DML 문을 실행할 때 사용. 매개변수화된 SQL 문을 사용할 수 있어 편리성과 보안성이 좋음. 따라서 Statement보다 주로 사용됨.
          </li>
          <li>
            CallableStatement: DB에 저장된 프로시저(procuder)와 함수(function)를 호출할 때 사용한다.
          </li>
        </ul>
      <li>
        ResultSet: DB에서 가져온 데이터 읽기.
      </li>
    </ul>
</ul>

<h1>5. DB 연결</h1>
<ul>
  <li>
    클라이언트 프로그램에서 DB와 연결하려면 해당 DBMS의 JDBC Driver와 다음 네 가지 정보가 필요하다.
  </li>
    <ul>
      <li>
        DMBS가 설치된 컴퓨터의 IP 주소
      </li>
        <ul>
          <li>
            DB가 있는 컴퓨터를 찾아간다.
          </li>
        </ul>
      <li>
        DBMS가 허용하는 포트(Port) 번호
      </li>
        <ul>
          <li>
            찾은 컴퓨터에서 DBMS로 연결하는데 사용한다.
          </li>
        </ul>
      <li>
        사용자(DB 계정) 및 비밀번호
      </li>
        <ul>
          <li>
            어떤 사용자인지 식별하기 위해 사용한다.
          </li>
        </ul>
      <li>
        사용하고자 하는 DB 이름
      </li>
        <ul>
          <li>
            DBMS에는 여러 개의 DB가 존재하기 때문에 어떤 DB를 사용할지 판단한다.
          </li>
        </ul>
    </ul>
  <li>
    클라이언트 프로그램을 DB에 연결하기 위해 가장 먼저 하는 작업은 <strong>JDBC Driver를 메모리로 로딩</strong>하는 것이다.
  </li>
    <ul>
      <li>
        문자열로 주어진 JDBC Driver 클래스의 <strong>Build Path</strong>를 찾아 <strong>메로리</strong>로 로딩.
      </li>
      <li>
        JDBC Driver 클래스의 static 블록이 실행되며 <strong>DriverManager</strong>에 DJBC Driver 객체를 등록한다.
      </li>
      <li>
        클래스를 찾지 못하는 경우 <strong>ClassNotFoundException이 발생</strong>하기 때문에 예외 처리가 필요하다.
      </li>
    </ul>
  <li>
    DriverManager에 JDBC Driver가 등록되면 <strong>getConnection() 메소드</strong>로 <strong>DB와 연결</strong>할 수 있다.
  </li>
    <ul>
      <li>
        첫 번째 매개값은 연결 문자열로 DBMS마다 다른 형식을 가지고 있다.
      </li>
      <li>
        예를 들어 MySQL은 "jdbc:mysql://localhost:3306/thisisjava"로 localhost에 설치되면 MySQL에 접근을 의미한다.
      </li>
        <ul>
          <li>
            원격 DB의 경우 localhost 대신 IP 주소를 기술한다.
          </li>
          <li>
            3306은 Port 번호, thisisjava는 DB명이다.
          </li>
        </ul>
      <li>
        연결이 성공하면 <strong>Connection 객체</strong>를 리턴하고, 반면 실패하면 <strong>SQLException</strong>이 발생하기에 예외처리가 필요하다.
      </li>
    </ul>
</ul>

```java
// 1. JDBC Driver를 메모리로 로딩.
Class.forName("com.mysql.cj.jdbc.Driver");

// 2. getConnection() 메소드로 DB와 연결.
Connection conn = DriverManager.getConnection("연결 문자열", "사용자", "비밀번호");
```

<h1>9. 데이터 읽기</h1>
<ul>
  <li>
    Preparedstatement를 생성할 때 SQL 문이 INSERT, UPDATE, DELETE일 경우 executeUpdate() 메소드를 호출하지만, <strong>SELECT 문</strong>일 경우 <strong>executeQuery() 메소드</strong>를 호출한다.
  </li>
    <ul>
      <li>
        executeQuery() 메소드는 가져온 데이터를 <strong>ResultSet에 저장하고 리턴</strong>한다.
      </li>
    </ul>
</ul>

```java
// 1. ResultSet에 결과를 저장.
ResultSet rs = pstmt.executeQuery();
```

<br>

<h2>9-1. ResultSet 구조</h2>
<ul>
  <li>
    ResultSet은 SELECT 문에 기술된 컬럼으로 구성된 <strong>행(row)의 집합</strong>이다.
  </li>
  <li>
    ResultSet은 <strong>커서(cursor)</strong>가 있는 행의 데이터만 읽을 수 있다. 커서란 <strong>행을 가리키는 포인터</strong>를 의미한다.
  </li>
    <ul>
      <li>
        ResultSet은 실제 가져온 데이터 <strong>행의 앞에는 beforeFirst 행</strong>이, <strong>뒤에는 afterLast 행</strong>이 붙으며 <strong>최초 커서는 beforeFirst</strong>를 가리킨다.
      </li>
      <li>
        따라서 첫 번째 데이터 행인 first를 읽기 위해서는 <strong>커서를 이동</strong>해야 하며 <strong>next() 메소드</strong>를 사용한다.
      </li>
        <ul>
          <li>
            next() 메소드는 <strong>커서를 다음 행으로 이동</strong> 시키며 이동한 행에 <strong>데이터가 있으면 true, 없으면 false를 반환</strong>한다.
          </li>
        </ul>
      <li>
        <strong>1개의 데이터 행</strong>만 가져오는 경우 if 조건식에 next() 메소드를 <strong>1회 호출</strong>한다.
      </li>
        <ul>
          <li>
            true와 false에 대한 처리를 적절히 작성한다.
          </li>
          <li>
            주로 SELECT 문이 <strong>기본키(primary key)를 조건</strong>으로 데이터를 가져오는 경우에 해당한다.
          </li>
        </ul>
      <li>
        <strong>n개의 데이터 행</strong>을 가져오는 경우 <strong>while 문</strong>을 이용해서 <strong>next() 메소드</strong>를 반복호출한다.
      </li>
    </ul>
  <li>
    SELECT 문에 따라 ResultSet에 많은 데이터 행이 저장될 수 있기에 다 사용했다면 close() 메소드를 통해 메모리를 해제하는 것이 좋다.
  </li>
</ul>

```java
// 1. userid, username, userage로 구성된 ResultSet 리턴.
SELECT userid, username, userage FROM users

// 2. next() 메소드를 통한 커서 이동.
boolean result = rs.next();

// 3. next()를 통해 행 가져오기.
// 3-1. 1개의 데이터 행.
ResultSet rs = pstmt.executeQuery();
if(rs.next()) {
    // 첫 번째 데이터 행 처리.
} else {
    // afterLast 행으로 이동했을 경우.
}
// 3-2. n개의 데이터 행을 가져올 경우.
ResultSet rs = pstmt.executeQuery();
while(rs.next()) {
  // last 행까지 이동하면서 데이터 행 처리.
}
  // afterLast 행으로 이동했을 경우.
```

<h2>9-2. 데이터 행 읽기</h2>
<ul>
  <li>
    커서가 있는 데이터 행에서 각 컬럼의 값은 <strong>Getter 메소드</strong>로 읽을 수 있다.
  </li>
    <ul>
      <li>
        <strong>컬럼의 데이터 타입</strong>에 따라 <strong>getXxx() 메소드</strong>가 사용된다.
      </li>
      <li>
        ResultSet의 컬럼 순번은 <strong>1부터 시작</strong>하며 <strong>SELECT 쿼리에서 작성된 컬럼 순서</strong>대로 userid=1, username=2, userage=3, ... 이 된다.
      </li>
    </ul>
  <li>
    SELECT 문에 <strong>연산식</strong>이나 <strong>함수 호출</strong>이 포함되어 있다면 컬럼 이름 대신 <strong>컬럼 순번</strong>으로 읽어야 한다.
  </li>
    <ul>
      <li>
        예를 들어 <strong>"userage - 1" 연산식</strong>을 사용하면 컬럼 순번으로만 읽을 수 있다.
      </li>
      <li>
        <strong>"(userage -1) as userage"</strong>와 같은 <strong>별명(alias)</strong>이 있다면 별명이 컬럼 이름이 된다.
      </li>
    </ul>
</ul>

```java
// 1. 연산식 활용.
SELECT userid, userage - 1
FROM users
// ResultSet 처리
String userId = rs.getString("userid"); // 컬럼명 가능
int userAge = rs.getInt(2);             // 연산식은 순번으로 접근

// 2. 컬럼명 활용.
SELECT userid, userage - 1 AS userage
FROM users
// ResultSet 처리
String userId = rs.getString("userid");
int userAge = rs.getInt("userage"); // alias로 접근 가능
```

<h1>10. 트랜잭션 처리</h1>
<ul>
  <li>
    <strong>트랜잭션(transaction)</strong>은 <strong>기능 처리의 최소 단위</strong>를 의미한다. 트랜잭션은 소작업들이 <strong>모두 성공하거나 모두 실패</strong>해야 한다.
  </li>
    <ul>
      <li>
        하나의 기능은 여러 가지 <strong>소작업</strong>들로 구성될 수 있다.
      </li>
      <li>
        최소 단위란 소작업들을 <strong>더이상 분리할 수 없으며 전체를 하나</strong>로 본다는 개념이다.
      </li>
    </ul>
  <li>
    DB는 트랜잭션을 처리하기 위해 <strong>커밋(commit)</strong>과 <strong>롤백(rollback)</strong>을 제공한다.
  </li>
    <ul>
      <li>
        커밋(commit): 내부 작업들을 모두 성공 처리한다.
      </li>
      <li>
        롤백(rollback): 실행 전으로 돌아간다.
      </li>
    </ul>
  <li>
    JDBC에서는 <strong>INSERT, UPDATE, DELETE</strong> 문을 실행할 때마다 자동 커밋이 일어난다.
  </li>
    <ul>
      <li>
        계좌 이체와 같이 <strong>두 가지 UPDATE 문</strong>을 실행할 때에는 문제가 발생할 수 있다.
      </li>
    </ul>
  <li>
    JDBC에서 트랜잭션을 코드로 제어하기 위해서는 우선 <strong>자동 커밋 기능</strong>을 꺼야 한다.
  </li>
    <ul>
      <li>
        자동 커밋 기능을 끄면 Connection의 <strong>commit()</strong>과 <strong>rollback()</strong> 메소드 제어가 가능하다.
      </li>
    </ul>
  <li>
    트랜잭션을 처리한 이후에는 원래대로 <strong>자동 커밋 기능</strong>을 켜두어야 한다.
  </li>
    <ul>
      <li>
        특히 <strong>커넥션 풀(Connection Pool)</strong>을 사용할 경우 다른 비즈니스 로직과도 <strong>커넥트를 공유</strong>하기 때문이다.
      </li>
    </ul>
</ul>

```java
// 1. JDBC 자동 커밋 기능 끄기. (Connection의 setAutoCommit() 메소드 활용).
conn.setAutoCommit(false);

// 2. 트랜잭션을 위한 일반적인 코드 작성.
Connection conn = null;
try {
  // 트랜잭션 시작  ------------------------------------
    // 자동 커밋 기능 끄기.
    conn.setAutoCommit(false);
    // 소작업 처리.
    
    // 소작업 처리.

    // 커밋 -> 모두 성공 처리.
    conn.commit();
  // 트랜잭션 종료 -------------------------------------
} catch (Exception e) {
  try {
    // 롤백 -> 모두 실패 처리.
    conn.rollback();
  } catch (SQLException e1) {}
} finally {
  if(conn != null) {
    try {
      // 원래대로 자동 커밋 기능 켜기.
      conn.setAutoCommit(true);
      // 연결 끊기.
      conn.close();
    } catch (SQLException e) {}
  }
}
```

<h2>10-1. 커넥션 풀</h2>
<ul>
  <li>
    <strong>다수의 클라이언트의 요청</strong>을 처리하는 서버 프로그램은 대부분 <strong>커넥션 풀(Connection Pool)</strong>을 사용한다.
  </li>
    <ul>
      <li>
        이는 <strong>미리 일정량의 Connection을 생성</strong>해 두고, 클라이언트의 요청이 들어오면 Connection을 제공하고 <strong>다시 반환</strong>받는 역할을 한다.
      </li>
    </ul>
  <li>
    커넥션 풀을 사용하면 생성된 Connection을 <strong>재사용</strong>할 수 있기에 DB <strong>연결 시간</strong>을 줄일 수 있으며 <strong>전체 Connection 수를 관리</strong>할 수도 있다.
  </li>
</ul>
