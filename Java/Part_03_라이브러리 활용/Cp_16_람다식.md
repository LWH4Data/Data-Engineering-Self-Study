<h1>1. 람다식이란?</h1>
<ul>
  <li>
    <strong>함수형 프로그래밍(functional programming)</strong>이란 <strong>함수를 정의</strong>하고, 함수를 <strong>데이터 처리부</strong>로 보내 데이터를 처리하는 기법을 의미한다.
  </li>
    <ul>
      <li>
        데이터 처리부는 <strong>데이터만</strong> 가지고 있을 뿐, 처리 방법은 정해져 있지 않다.
      </li>
      <li>
        데이터 처리부는 제공된 함수의 입력값으로 데이터를 넣고 함수에 정의된 처리 내용을 실행한다. 따라서 동일한 데이터라도 <strong>제공되는 함수에 따라 결과가 다를 수 있다</strong>.
      </li>
      <li>
        제공된 함수에 따라 결과가 다른 것을 <strong>데이터 처리의 다형성</strong>이라 볼 수도 있다.
      </li>
    </ul>
  <li>
    자바는 함수형 프로그래밍을 위해 Java 8부터 <strong>람다식(Lambda Expression)</strong>을 지원한다.
  </li>
    <ul>
      <li>
        람다식은 데이터 처리부에 제공되는 <strong>함수 역할을 하는 매개변수</strong>를 가진 <strong>중괄호 블록</strong>이다.
      </li>
        <ul>
          <li>
            데이터 터리부는 람자식을 받아 매개변수에 데이터를 대입하고 중괄호를 실행한다.
          </li>
        </ul>
    </ul>
  <li>
    자바는 람다식을 <strong>익명 구현 객체</strong>로 변환한다.
  </li>
    <ul>
      <li>
        익명 구현 객체란 이름이 없는 인터페이스 구현 객체를 의미한다.
      </li>
    </ul>
  <li>
    람다식은 인터페이스의 익명 구현 객체이기에 인터페이스 타입의 매개변수에 대입될 수 있다.
  </li>
    <ul>
      <li>
        그러나 인터페이스의 익명 구현 객체를 람다식으로 표현하려면 <strong>단 하나의 추상 메소드</strong>만 가져야 한다.
      </li>
    </ul>
  <li>
    인터페이스가 단 하나의 추상 메소드를 가질 때, 이를 <strong>함수형 인터페이스(functional interface)</strong>라 한다.
  </li>
    <ul>
      <li>
        함수형 인터페이스임을 보장하기 위해서는 <strong>@FunctionalInterface 어노테이션</strong>을 붙이면 된다.
      </li>
    </ul>
</ul>

```java
// 1. 람다식 형태.
/*
람다식: (매개변수, ...) → { 처리 내용 }
*/


// 2. Calculable 인터페이스 사례로 보는 람다식.
// 2-1. 사용할 Calculable 인터페이스.
public interface Calculable {
  // 추상 메소드
  void calculate(int x, int y);
}

// 2-2. 인터페이스의 익명 구현 객체 생성.
new Calculable() {
  @Override
  public void calculate(int x, int y) { 처리내용 }
};
// 람다식 표현
(x, y) -> { 처리내용 }


// 3. Calculable 매개변수를 갖는 action() 메소드
public void action(Calculable calculate) {
  int x = 10;
  int y = 4;
  calculable.calculate(x, y);    // 데이터를 제공하고 추상 메소드를 호출.
}

// 3-1. action() 메소드를 호출할 때 매개값을 람다식으로 제공.
action( (x, y) -> {
  int result = x + y; // 람다식
  System.out.println(result);
});


// 4. 함수형 인터페이스(인터페이스에 단 하나의 추상 메소드)
// ( ) -> { ... }
public interface Runnable {
  void run();
}

// ( x, y ) -> { ... }
@FunctionalInterface
public interface Calculable {
  void calculate(int x, int y);
}
```

<h1>2. 매개변수가 없는 람다식</h1>
<ul>
  <li>
    함수형 인터페이스의 추상 메소드에 매개변수가 없는 경우 <strong>매개변수를 생략</strong>하고 람다식을 작성할 수 있다.
  </li>
    <ul>
      <li>
        실행문이 하나라면 실행문의 <strong>중괄호를 생략</strong>할 수 있다.
      </li>
    </ul>
</ul>

```java
// 1. 매개변수가 없는 람다식
/*
( ) -> {
  실행문;
  실행문;
}
*/

// 2. 매개변수가 없으면서 실행문이 하나인 람다식 (중괄호 생략).
/*
( ) -> 실행문
*/
```

<br><br>

<h1>3. 매개변수가 있는 람다식</h1>
<ul>
  <li>
    함수형 인터페이스의 추상 메소드에 <strong>매개변수가 있는 경우</strong> 여러 방법의 작성 방법이 있다.
  </li>
    <ul>
      <li>
        매개변수를 선언할 때 <strong>타입은 생략</strong>할 수 있으며 주로 이 방식을 사용한다.
      </li>
      <li>
        구체적인 타입 대신 var 타입을 사용할 수도 있다.
      </li>
    </ul>
  <li>
    매개변수가 하나인 경우 <strong>괄호를 생략</strong>할 수 있다.
  </li>
    <ul>
      <li>
        단, 이 경우 <strong>타입 또는 var</strong>를 붙일 수 없다.
      </li>
    </ul>
</ul>

```java
// 1. 매개변수가 여럿 일 때 작성 방법.
// 1-1. 타입 지정.
/*
(타입 매개변수, ... ) -> {
  실행문;
  실행문;
}
*/

// 1-2. var 타입 활용.
/*
(var 매개변수, ...) -> {
  실행문;
  실행문;
}
*/

// 1-3. 타입 생략.
/*
(매개변수, ...) -> {
 실행문;
 실행문;
}
*/


// 2. 매개변수가 하나인 경우.
// 2-1. 실행문 두 개 이상.
/*
매개변수 -> {
  실행문;
  실행문;
}
*/

// 2-2. 실행문 하나.
// 매개변수 -> 실행문.
```

<h1>4. 리턴값이 있는 람다식</h1>
<ul>
  <li>
    함수형 인터페이스의 추상 메소드에 <strong>리턴값이 있는 경우</strong>에도 몇 가지 방식으로 람다식을 작성할 수 있다.
  </li>
    <ul>
      <li>
        return문이 하나인 경우 중괄호와 return 키워드를 <strong>생략</strong>할 수 있다.
      </li>
      <li>
        리턴값은 연산식 또는 리턴값이 있는 메소드 호출로 <strong>대체</strong>할 수 있다.
      </li>
    </ul>
</ul>

```java
// 1. 리턴값이 있는 경우.
// 1-1. return 명시.
/*
(매개변수, ...) -> {
  실행문;
  return 값;
}
*/

// 1-2. return 명시 X
// (매개변수, ...) -> 값
```

<h1>5. 메소드 참조</h1>
<ul>
  <li>
    메소드르 참조하여 <strong>매개변수의 정보 및 리턴 타입</strong>을 알아내 람다식에서 <strong>불필요한 매개변수를 제거</strong>하는 것을 목적으로 한다.
  </li>
</ul>

```java
// 1. max() 정적 메소드를 람다식으로 호출.
(left, right) -> Math.max(left, right);
// 람다식은 단순히 두 개의 값을 전달만하기에 다음처럼 줄일 수 있다.
Math :: max
```

<br>

<h2>5-1. 정적 메소드와 인스턴스 메소드 참조</h2>
<ul>
  <li>
    정적(static) 메소드를 참조할 경우 "클래스 이름 + :: + 정적 메소드 이름" 으로 기술한다.
  </li>
  <li>
    인스턴스 메소드의 경우 먼저 <strong>객체를 생성</strong>하고, <strong>"참조 변수 + :: + 인스턴스 메소드 이름"</strong> 으로 기술한다.
  </li>
</ul>

```java
// 1. 정적 메소드 참조.
/* 
클래스 :: 메소드
*/

// 2. 인스턴스 메소드 참조.
/*
참조변수 :: 메소드
*/
```

<br>

<h2>5-2. 매개변수의 메소드 참조</h2>

```java
// 1. 매개변수 메소드 참조 예
(a, b) -> { a.instanceMethod(b); }
// 참조로 표현. (클래스 이름 :: 메소드)
a :: instanceMethod
```

<h1>6. 생성자 참조</h1>
<ul>
  <li>
    생성자 참조란 <strong>객체를 생성</strong>하는 것을 의미한다.
  </li>
  <li>
    람다식이 단순히 <strong>객체를 생성</strong>하고 <strong>리턴</strong>하도록 구성된다면 람다식을 생성자로 대치할 수 있다.
  </li>
  <li>
    생성자가 <strong>오버로딩</strong>되어 여러 개가 있는 경우 컴파일러는 함수형 인터페이스의 추상 메소드와 <strong>동일한 매개변수 타입과 개수</strong>를 가진 생성자를 찾아 실행한다.
  </li>
    <ul>
      <li>
        생성자가 존재하지 않으면 컴파일러 오류가 발생한다.
      </li>
    </ul>
</ul>

```java
// 1. 단순히 객체를 생성하고 리턴만하는 생성자.
(a, b) -> { return new 클래스(a, b); }
// 람다식 표현 (생성자 참조)
/*
클래스 :: new
*/
```
