<h1>1. 스트림이란?</h1>
<ul>
  <li>
    지금까지는 컬렉션 및 배열에 저장된 요소를 반복 처리하기 위해 for 문 혹은 Iterator(반복자)를 이용하였다.
  </li>
  <li>
    Java 8부터는 컬렉션 및 배열 요소를 반복 처리하기 위해 <strong>스트림(Stream)</strong>을 사용할 수 있다.
  </li>
  <li>
    Stream과 Iterator의 세 가지 차이점.
  </li>
    <ul>
      <li>
        1. Stream은 <strong>내부 반복자</strong>로 처리 속도가 빠르고 병렬 처리에 효율적이다.
      </li>
      <li>
        2. <strong>람다식</strong>으로 다양한 요소 처리를 정의할 수 있다.
      </li>
      <li>
        3. 중간 처리와 최종 처리를 수행하도록 <strong>파이프 파인</strong>을 형성할 수 있다.
      </li>
    </ul>
</ul>

```java
// 1. 기존의 for와 iterator를 통한 반복.
// 1-1. for loop
List<String> list = ...;
for(int i=0; i<list.size();>; i++) {
  String item = list.get(i);
  // item 처리.
}

// 1-2. Iterator
Set<String> set = ...;
Iterator<String> iterator = set.iterator();
while(iterator.hasNext()) {
  String item = iterator.next();
  // 요소 처리
}

// 2. Stream 활용.
//   - List 컬레션의 stream() 메소드로 Stream 객체를 읽는다.
//   - forEach() 메소드로 요소를 어떻게 처리할지 람다식으로 제공한다. 
Stream<String> stream = list.stream();
stream.forEach( item -> // item 처리 );
```

<h1>2. 내부 반복자</h1>
<ul>
  <li>
    for loop과 Iterator는 컬렉션 요소를 <strong>컬렉션 바깥쪽</strong>으로 반복해서 가져와 처리하며 <strong>외부 반복자</strong>라 한다.
  </li>
    <ul>
      <li>
        컬렉션 요소를 <strong>외부로 가져오는 코드</strong>와 <strong>처리하는 코드</strong>를 모두 <strong>개발자 코드</strong>가 가지고 있어야 한다.
      </li>
    </ul>
  <li>
    Stream은 요소 처리 방법을 <strong>컬렉션 내부로 주입</strong>하여 반복 처리를 하며 이를 <strong>내부 반복자</strong>라 한다.
  </li>
    <ul>
      <li>
        개발자 코드에서 제공한 <strong>데이터 처리 코드(람다식)</strong>을 가지고 <strong>컬렉션 내부</strong>에서 요소를 반복 처리한다.
      </li>
      <li>
        <strong>멀티 코어 CPU</strong>를 최대한 활용하기 위해 요소들을 분배시켜 <strong>병렬 작업</strong>을 할 수 있다.
      </li>
        <ul>
          <li>
            따라서 하나씩 처리하는 외부 반복자 보다 효율적으로 요소를 반복시킬 수 있다.
          </li>
        </ul>
    </ul>
</ul>

<br><br>

<h1>3. 중간 처리와 최종 처리</h1>
<ul>
  <li>
    스트림은 하나 이상 연결될 수 있다. 여러 스트림이 연결되어 있는 것을 <strong>스트림 파이프라인(pipeline)</strong>이라 한다.
  </li>
    <ul>
      <li>
        <strong>오리지널 스트림</strong>과 <strong>집계 처리</strong> 사이의 중간 스트림들은 다양한 처리를 한다.
      </li>
        <ul>
          <li>
            최종 처리를 위해 요소를 걸러내기(필터링)
          </li>
          <li>
            요소를 변환시키기(매핑)
          </li>
          <li>
            정렬 작업 수행 및 중간 처리에서 정제된 요소들을 반복
          </li>
          <li>
            집계(카운팅, 총합, 평균) 등
          </li>
        </ul>
    </ul>
</ul>


```java
// 1. 스트림 파이프라인 표현.
// Student 스트림.
Stream<Student> studentStream = list.stream();
// score 스트림.
//   - mapToInt(): 객체를 int 값으로 매핑하여 IntStream으로 변환시킨다.
//     - 어떤 int 값으로 매핑할 것인지는 람다식으로 제공한다.
IntStream scoreStream = studentStream.mapToInt( student -> student.getScore() );
// 평균 계산.
//   - IntStream의 average() 메소드는 평균 값을 계산한다.
double avg = scoreStream.average().getAsDouble();

// 2. 메소드 체이닝 패턴으로 표현.
double avg = list.stream()
  .mapToInt(student -> student.getScore())
  .average()
  .getAsDouble();
```

<h1>4. 리소스로부터 스트림 얻기</h1>
<ul>
  <li>

  </li>
    <ul>
      <li>
        Stream: <strong>객체 요소</strong>를 처리.
      </li>
      <li>
        IntStream, LongStream, DoubleStream: 각 <strong>기본 타입</strong>인 int, long, double 요소를 처리하는 스트림.
      </li>
    </ul>
  <li>
    BaseStream은 모든 스트렘에서 사용할 수 있는 공통 메소드들이 있으며 이를 포함한 각 Stream의 메소드는 <strong>p731</strong>을 참고.
  </li>
</ul>

<h1>5. 요소 걸러내기(필터링)</h1>
<ul>
  <li>
    필터링은 요소를 걸러내는 <strong>중간 처리 기능</strong>으로 <strong>distinct()</strong>와 <strong>filter()</strong>가 있다.
  </li>
  <ul>
    <li>
      distinct() 메소드는 <strong>요소의 중복</  strong>을 제거한다.
    </li>
      <ul>
        <li>
          distinct() 메소드는 <strong>equals() 메소드</strong>의 리턴 값이 true이면 동일한 요소로 판단하고 제거한다.
        </li>
        <li>
          IntStream, LongStream, DoubleStream은 <strong>같은 값</strong>일 경우 중복을 제거한다.
        </li>
      </ul>
    <li>
      filter() 메소드는 매개값으로 주어진 <strong>Predicate가 true</strong>를 리턴하는 요소만 필터링한다.
    </li>
      <ul>
        <li>
          Predicate는 매개값을 조사한 후 boolean을 리턴하는 <strong>test() 메소드</strong>를 가지고 있다.
        </li>
      </ul>
  </ul>
</ul>

```java
// 1. RPredicate<T> 람다식
// T -> { ... return true }
// T -> true;
```

<h1>6. 요소 변환(매핑)</h1>
<ul>
  <li>
    매핑(mapping)은 스트림의 요소를 <strong>다른 요소로 변환</strong>하는 기능이다.
  </li>
    <ul>
      <li>
        mapXxx(), asDoubleStream(), asLongStream(), boxed(), flatMapXxx() 등이 있다.
      </li>
    </ul>
</ul>

<br>

<h2>6-1. 요소를 다른 요소로 변환</h2>
<ul>
  <li>
    mapXxx() 메소드는 요소를 다른 요소로 변환한 새로운 스트림을 리턴한다.
  </li>
  <li>
    mapXxx()의 매개타입인 Function은 p740에서 자세히 볼 수 있다.
  </li>
  <li>
    모든 Function은 매개값을 리턴값으로 매핑(변환)하는 applyXxx() 메소드를 가지고 있다.
  </li>
  <li>
    기본 타입 간의 변환이거나 기본 타입 요소를 래터(Wrapper) 객체 요소로 변환하려면 간편화 메소드를 사용할 수 있다. (p742).
  </li>
</ul>

```java
// 1. Function<T, R>을 람다식으로 표현.
// T -> { ... return R; }
// T -> R; return 문만 있을 경우 중괄호와 return 키워드는 생략 가능.
```

<br>

<h2>6-2. 요소를 복수 개의 요소로 변환</h2>

<h2>7-1. Comparable 구현 객체의 정렬</h2>
<ul>
  <li>
    스트림의 요소가 <strong>객체</strong>일 경우 객체가 <strong>Comparable</strong>을 구현하고 있어야만 <strong>sorted() 메소드</strong>를 사용할 수 있다.
  </li>
    <ul>
      <li>
        그렇지 않은 경우 ClassCastException이 발생한다.
      </li>
    </ul>
  <li>
    <strong>내림차순 정렬</strong>의 경우 <strong>Comparator.reverseOrder() 메소드</strong>가 리턴하는 Comparator를 매개값으로 제공한다.
  </li>
</ul>

```java
// 1. Comparable의 예.
public class Xxx implements Comparable {
  ...
}
List<Xxx> list = new ArrayList<>();
Stream<Xxx> stream = list.stream();
stream<Xxx> orderedStream = stream.sorted();

// 2. 내림차순 정렬의 예.
Stream<Xxx> reverseOrderedStream = stream.sorted(Comparator.reverseOrder());
```

<br>

<h2>7-2. Comparator를 이용한 정렬</h2>
<ul>
  <li>
    요소 객체가 Comparable을 구현하고 있지 않은 경우 <strong>비교자</strong>를 제공하여 요소를 정렬할 수 있다.
  </li>
    <ul>
      <li>
        비교자는 Comparator 인터페이스를 구현한 객체를 의미한다.
      </li>
    </ul>
</ul>

```java
// 1. 간단한 람다식으로 비교자 구현.
sorted((o1, o2) -> { 
  /*
    o1 < o2인 경우 음수,
    o1 == o2인 경우 0,
    o1 > o2인 경우 양수를 리턴하게 구현.

    o1과 o2가 정수인 경우 Integer.compare(o1, o2)를,
    실수인 경우 Double.compare(o1, o2)를 호출해서 리턴값을 리턴해도 좋다.
  */
})
```

<h1>8. 요소를 하나씩 처리(루핑)</h1>
<ul>
  <li>
    루핑(looping)은 스트림에서 요소를 <strong>하나씩 반복해서 가져와 처리</strong>하는 것을 의미한다.
  </li>
  <li>
    루핑 메소드에는 <strong>peek()</strong>과 <strong>forEach()</strong>가 있다.
  </li>
    <ul>
      <li>
        peek()과 forEach() 메소드에서 대한 설명은 p750 참고.
      </li>
      <li>
        peek()은 <strong>중간 처리 메소드</strong>이고, forEach()는 <strong>최종 처리 메소드</strong>이다.
      </li>
        <ul>
          <li>
            따라서 peek() 메소드 위에는 최종 처리 메소드가 필요하다.
          </li>
        </ul>
      <li>
        매개타입인 Consumer는 함수형 인터페이스이다. 자세한 종류는 p751참고.
      </li>
      <li>
        모든 Consumer는 매개값을 처리(소비)하는 <strong>accep() 메소드</strong>를 가지고 있다.
      </li>
    </ul>
</ul>

```java
// 1. Consumer<? super T>를 람다식으로 표현.
// T -> { ... }
// T -> 실행문;    // 하나의 실행문만 있을 경우 중괄호 생략.
```

<h1>10. 요소 기본 집계</h1>
<ul>
  <li>
    <strong>집계(Aggregate)</strong>는 대량의 데이터를 가공하여 하나의 값으로 축소하는 <strong>리덕션(reduction)</strong>이라 볼 수 있다.
  </li>
</ul>

<br>

<h2>10-1. 스트림이 제공하는 기본 집계</h2>
<ul>
  <li>
    스트림은 카운팅, 최대, 최소, 평균, 합계 등을 처리하는 최종 처리 메소드를 제공한다. (p754 참고).
  </li>
  <li>
    집계 메소드는 OptionalXXX, Optional, OptionalDouble, OptionalInt, OptionalLong 클래스를 리턴한다.
  </li>
    <ul>
      <li>
        리턴된 클래스에는 get(), getAsDouble(), getAsInt(), getAsLong() 을 호출하면 최종값을 얻을 수 있다.
      </li>
    </ul>
</ul>

<br>

<h2>10-2. Optional 클래스</h2>
<ul>
  <li>
    Optional, OptionalDouble, OptionalInt, OptionalLong 클래스는 집계값이 존재하지 않을 경우 <strong>디폴트 값을 설정</strong>하거나 집계값을 처리하는 <strong>Consumer</strong>를 등록할 수 있다.
  </li>
  <li>
    자세한 메소드의 설명은 p756 참고.
  </li>
</ul>

```java
// 1. 컬력션에 요소가 없어 NoSuchElementException이 발생할 수 있다.
//    이를 방지하기위해 하단에 요소가 없는 average()를 처리하는 
//    세 가지 방법의 사례를 본다.
// 1-1. isPresent() 메소드를 통해 true를 리턴할 때만 집계값을 얻는다.
OptionalDouble optional = stream
  .average();
if(optional.isPresent()) {
  System.out.println("평균: " + optinal.getAsDouble());
} else {
  System.out.println("평균: 0.0");
}

// 1-2. orElse() 메소드로 집계값이 없을 경우를 대비하여 디폴트 값을 정한다.
double avg = stream
  .average()
  .orElse(0.0);
System.out.println("평균: " + avg);

// 1-3. ifPresent() 메소드로 집계값이 있을 경우에만 동작하는 Consumer
//      람다식을 제공한다.
stream
  .average()
  .ifPresent(a -> System.out.println("평균: " + a));
```

<h1>11. 요소 커스텀 집계</h1>
<ul>
  <li>
    스트림은 기본 집계 메소드 외에도 <strong>다양한 집계 결과물</strong>을 만들 수 있도록 <strong>reduce() 메소드를</strong> 제공한다. (p750 참고).
  </li>
  <li>
    매개값인 <strong>BinaryOperator</strong>는 함수형 인터페이스이다.
  </li>
    <ul>
      <li>
        BinaryOperator는 <strong>두 개의 매개값을 받아 하나의 값을 리턴하는 apply() 메소드</strong>를 가지고 있다.
      </li>
    </ul>
  <li>
    reduce()는 <strong>스트림에 요소가 없을 경우</strong> 예외가 발생하지만, <strong>identity 매개값</strong>이 주어지면 이 값을 <strong>디폴트</strong>로 리턴한다.
  </li>
</ul>

```java
// 1. BinaryOperator의 apply() 메소드 람다식 예.
// (a, b) -> { ... return 값; }
// (a, b) -> 값    // return 문만 있을 경우 중괄호와 return 키워드 생략 가능.

// 2. identity 활용 사례.
// 2-1. 스트림에 요소가 없을 경우 NosuchElementException 발생.
int sum = stream
  .reduce((a, b) -> a+b)
  .getAsInt();

// 2-2. identity 값을 두어 디폴트 값 출력.
int sum = stream
  .reduce(0, (a, b) -> a+b);
```

<h1>12. 요소 수집</h1>
<ul>
  <li>
    스트림은 요소들을 <strong>필터링 또는 매핑한 후 수집</strong>하는 최종 처리 메소드 <strong>collect()</strong>를 제공한다.
  </li>
    <ul>
      <li>
        collect()를 통해 <strong>필요한 요소만 컬렉션</strong>에 담거나 요소들을 <strong>그루핑한 후 집계</strong>할 수도 있다.
      </li>
    </ul>
</ul>

<br>

<h2>12-1. 필터링한 요소 수집</h2>
<ul>
  <li>
    Stream의 <strong>collect(Collector<T, A, R> collector) 메소드</strong>는 필터링 또는 매핑된 요소들을 <strong>새로운 컬렉션에 수집</strong>하고 <strong>리턴</strong>한다.
  </li>
    <ul>
      <li>
        매개값인 Collector는 <strong>어떤 요소를 어떤 컬렉션</strong>에 수집할 것인지를 결정한다.
      </li>
      <li>
        T 요소를 A 누적기가 R에 저장한다. A가 "?"인 경우는 Collector가 List, Set, Map 컬레션에 요소를 저장하는 방법을 알고 있어 누적기가 필요 없는 경우이다.
      </li>
        <ul>
          <li>
            T: 요소
          </li>
          <li>
            A: 누적기(accumulator)
          </li>
          <li>
            R: 요소가 저장될 컬렉션
          </li>
        </ul>
      <li>
        Collector의 구현 객체는 <strong>Collectors 클래스의 정적 메소드</strong>로 얻을 수 있다.
      </li>
    </ul>
</ul>

```java
// 1. Student 스트림에서 남학생만 필터링하여 List로 생성.
List<Student> maleList = totalList.stream()
  .filter(s->s.getSex().equals("남"))    // 남학생만 필터링.
  .collect(Collectors.toList());


// 2. 이번에는 이름을 key로, 점수를 value로 갖는 Map 컬렉션을 생성.
Map<String, Integer> map = totalList.stream()
  .collect(
    Collectors.toMap(
      s -> s.getName(),    // Student 객체에서 키가 될 부분 리턴
      s -> s.getScore()    // Student 객체에서 값이 될 부분 리턴
  )
)

// 2-1. Java 16 부터는 요소 스트림에서 List 컬렉션 얻기 가능.
List<Student> maleList = totalList.stream()
  .filter(s->s.getSex().equals("남"))
  .toList();
```

<br>

<h2>12-2. 요소 그루핑</h2>
<ul>
  <li>
    collect() 메소드는 단순 요소 수집 외에도 <strong>컬렉션의 요소들을 그룹핑하여 Map 객체를 생성</strong>하는 기능도 제공한다.
  </li>
    <ul>
      <li>
        <strong>Collectors.groupingBy() 메소드</strong>에서 얻은 Collector를 collect() 메소드를 호출할 때 제공하면 된다.
      </li>
    </ul>
  <li>
    Collectors.groupingBy() 메소드는 <strong>그룹핑 후 매핑 및 집계</strong>를 수행할 수 있도록 두 번째 매개값 <strong>Collector</strong>를 가질 수 있다.
  </li>
    <ul>
      <li>
        Collector를 얻을 수 있는 Collectors의 정적 메소드들은 p766 참고.
      </li>
    </ul>
</ul>

```java
// 1. "남", "여"를 키로 설정하고 List<Student>를 값으로 갖는 Map을 생성.
Map<String, List<Student>> map = totalList.stream()
  .collect(
    Collectors.groupingBy(s -> s.getSex())  // 그룹핑 키 리턴.
  )

// 2. 학생들을 성별로 그룹핑하고 각각의 평균 점수를 구해 Map으로 얻기.
Map<String, Double> map = totalList.stream()
  .collect(
    Collectors.groupingBy(
      s -> s.getSex(),
      Collectors.averagingDouble(s -> getScore())
    )
  );
```

<h1>13. 요소 병렬 처리</h1>
<ul>
  <li>
    <strong>요소 병렬 처리(Parallel Operation)</strong>란 <strong>멀티 코어 CPU 환경</strong>에서 전체 요소를 분할하여 각 코어가 <strong>병렬적으로 처리</strong>하는 것을 의미한다.
  </li>
    <ul>
      <li>
        요소 병렬 처리의 목적은 <strong>작업 처리 시간 단축</strong>이다.
      </li>
      <li>
        자바는 요소 병렬 처리를 위해 <strong>병렬 스트림</strong>을 제공한다.
      </li>
    </ul>
</ul>

<br>

<h2>13-1. 동시성과 병렬성</h2>
<ul>
  <li>
    <strong>동시성(Concurrency)</strong>은 멀티 작업을 위해 멀티 스레드가 <strong>하나의 코어</strong>에서 <strong>번갈아 가며 실행</strong>하는 것을 의미한다.
  </li>
    <ul>
      <li>
        동시성은 <strong>한 시점에 하나</strong>의 작업만 실행한다.
      </li>
      <li>
        번갈아 작업을 시행하는 것이 빨라 동시에 처리되는 것 처럼 보이는 것 뿐이다.
      </li>
    </ul>
  <li>
    <strong>병렬성(Parallel)</strong>이란 멀티 작업을 위해 <strong>멀티 코어</strong>를 <strong>각각 이용</strong>하여 병렬로 실행하는 것을 의미한다.
  </li>
    <ul>
      <li>
        병렬성은 <strong>한 시점에 여러 개</strong>의 작업을 병렬로 실행하기 때문에 동시성보다 성능이 좋다.
      </li>
    </ul>
  <li>
    병렬성에는 <strong>데이터 병렬성(Data parallelism)</strong>과 <strong>작업 병렬성(Task parallelism)</strong>으로 구분된다.
  </li>
</ul>

<br>

<h3>13-1-1. 데이터 병렬성</h3>
<ul>
  <li>
    데이터 병렬성은 전체 데이터를 분할하여 <strong>서브 데이터셋</strong>으로 만들고, <strong>서브 데이터셋들을 병렬 처리</strong>하여 작업을 빨리 끝내는 것을 의미한다.
  </li>
  <li>
    자바의 병렬 스트림은 <strong>데이터 병렬성</strong>을 구현한 것이다.
  </li>
</ul>

<h3>13-1-2. 작업 병렬성</h3>
<ul>
  <li>
    작업 병렬성은 <strong>서로 다른 작업</strong>을 병렬 처리하는 것을 의미한다.
  </li>
  <li>
    작업 병렬성의 대표적인 예는 <strong>서버 프로그램</strong>이며 서버는 각 클라이언트의 요청을 <strong>개별 스레드</strong>에서 <strong>병렬</strong>로 처리한다.
  </li>
</ul>

<br>

<h2>13-2. 포크조인 프레임워크</h2>
<ul>
  <li>
    자바 병렬 스트림은 요소들을 병렬 처리하기 위해 <strong>포크조인 프레임워크(Forkjoin Framework)</strong>를 사용한다.
  </li>
    <ul>
      <li>
        <strong>포크 단계</strong>에서 전체 요소들을 <strong>서브 요소셋</strong>으로 분할한다.
        <br>→ 분할된 서브 요소셋을 <strong>멀티 코어에서 병렬</strong>로 처리한다.
        <br>→ <strong>조인 단계</strong>에서는 <strong>서브 결과를 결합</strong>하여 <strong>최종 결과</strong>를 만들어낸다. 
      </li>
    </ul>
  <li>
    병렬 처리 스트림은 포크 단계에서 요소를 순서대로 분할하지 않으며 내부적으로 요소들을 나누는 알고리즘에 의해 분할된다. (신경쓸 필요 X).
  </li>
  <li>
    포크조인 프레임워크는 병렬 처리를 위해 <strong>스레드풀</strong>을 사용한다.
  </li>
    <ul>
      <li>
        각 코어에서 서브 요소셋을 처리하는 것은 <strong>작업 스레드</strong>가 하기에 <strong>스레드 관리</strong>가 필요하다.
      </li>
      <li>
        포크조인 프레임워크는 ExecutorService의 구현 객체인 <strong>ForkJoinPool</strong>을 사용해 작업 스레드를 관리한다.
      </li>
    </ul>
</ul>

<br>

<h2>13-3. 병렬 스트림 사용</h2>
<ul>
  <li>
    자바 병렬 스트림은 백그라운드에서 포크조인 프레임워크가 사용되기 때문에 쉽게 이용할 수 있다.
  </li>
  <li>
    병렬 스트림은 <strong>parallelStream()</strong>과 <strong>parallel() 메소드</strong>로 얻을 수 있다. (p770 참고).
  </li>
    <ul>
      <li>
        parallelStream() 메소드는 <strong>컬렉션(List, Set)</strong>으로부터 <strong>병렬 스트림을 바로 리턴</strong>한다.
      </li>
      <li>
        parallel() 메소드는 <strong>기존 스트림</strong>을 병렬 처리 스트림으로 <strong>변환</strong>한다.
      </li>
    </ul>
</ul>

<br>

<h2>13-4. 병렬 처리 성능</h2>
<ul>
  <li>
    스트림 병렬 처리가 항상 스트림 순차 처리보다 속도가 빠른 것은 아니다.
  </li>
</ul>

<h3>13-4-1. 요소의 수와 요소당 처리 시간</h3>
<ul>
  <li>
    컬렉션 전체 <strong>요소의 수가 적고, 요소당 처리 시간이 짧다면</strong> 일반 스트림이 더 빠를 수 있다.
  </li>
    <ul>
      <li>
        병렬 처리에는 포크 및 조인 단계, 스레드 풀 생성 등 <strong>오버헤드</strong>가 존재하기 때문이다.
      </li>
    </ul>
</ul>

<h3>13-4-2. 스트림 소스의 종류</h3>
<ul>
  <li>
    ArrayList와 배열은 인덱스로 요소를 관리하기에 포크 단계에서 요소를 쉽게 분리할 수 있다.
  </li>
    <ul>
      <li>
        그러나 HashSet, TreeSet은 요소 분리가 쉽지 않고, LinkedList의 경우 링크를 따라가야하기에 상대적으로 병렬 처리가 늦다.
      </li>
    </ul>
</ul>

<h3>13-4-3. 코어의 수</h3>
<ul>
  <li>
    CPU 코어 수가 많은 경우 병렬 스트림의 성능이 좋으나, 코어의 수가 적다면 일반 스트림이 더 빠를 수 있다.
  </li>
    <ul>
      <li>
        병렬 스트림은 <strong>스레드 수가 증가</strong>하여 동시성이 많이 일어나기 때문이다.
      </li>
    </ul>
</ul>
