<h1>1. 네트워크 기초</h1>
<ul>
  <li>
    네트워크(network)는 <strong>여러 컴퓨터</strong>들을 통신 회선으로 연결한 것을 의미한다.
  </li>
    <ul>
      <li>
        <strong>LAN(Local Area Network)</strong>는 가정, 회사, 건물 등 <strong>특정 영역</strong>에 존재하는 컴퓨터를 연결한 것이다.
      </li>
      <li>
        <strong>WAN(Wide Area Network)</strong>은 <strong>LAN을 연결</strong>한 것이다.
      </li>
    </ul>
</ul>

<br>

<h2>1-1. 서버와 클라이언트</h2>
<ul>
  <li>
    서비스를 제공하는 프로그램을 일반적으로 <strong>서버(server)</strong>라 하고, 서비스를 요청하는 프로그램을 <strong>클라이언트(client)</strong>라 한다.
  </li>
  <li>
    클라이언트 응답 요청 → 서버 처리 → 응답 전송
  </li>
</ul>

<br>

<h2>1-2. IP 주소</h2>
<ul>
  <li>
    IP(Internet Protocol)는 <strong>컴퓨터의 고유한 주소</strong>이다.
  </li>
  <li>
    IP 주소는 네트워크 <strong>어댑터(LAN 카드)</strong>마다 할당된다.
  </li>
    <ul>
      <li>
        컴퓨터에 두 개의 네트워크 어댑터가 장착되어 있다면 두 개의 IP 주소를 할당받을 수 있다.
      </li>
    </ul>
  <li>
    <strong>공유기</strong>는 <strong>외부 인터넷과 연결되는 장치</strong>이며, 외부에서는 공유기의 <strong>공인 IP</strong>(예: 203.xxx.xxx.xxx)로 인식된다.
  </li>
    <ul>
      <li>
        공유기 내부에서 각 기기는 <strong>사설 IP</strong>를 가진다. (예: 192.168.0.2, 192.168.0.3 ...).
      </li>
      <li>
        만약 하나의 디바이스가 <strong>여러 어댑터</strong>를 갖는다면 해당 디바이스는 <strong>어댑터의 수만큼</strong> IP를 가질 수 있다.
      </li>
    </ul>
  <li>
    윈도우에서는 ipconfig로, 맥OS에서는 ifconfig로 현재 디바이스 기기의 네트워크 IP 주소를 확인할 수 있다.
  </li>
    <ul>
      <li>
        <strong>연결별 DNS 접미사</strong>: 도메인 이름 뒤에 자동으로 붙는 주소 (옵션).
      </li>
      <li>
        <strong>링크-로컬 IPv6 주소</strong>: 같은 네트워크 안에서만 쓰는 임시 IPv6 주소.
      </li>
      <li>
        <strong>IPv4 주소</strong>: 내 디바이스의 내부 IP(사설 IP).
      </li>
      <li>
        <strong>서브넷 마스크</strong>: 같은 네트워크 범위를 정하는 기준.
      </li>
      <li>
        <strong>기본 게이트웨이</strong>: 인터넷으로 나갈 때 거치는 출구 (공유기).
      </li>
    </ul>
  <li>
    연결할 상대의 컴퓨터 IP 주소를 모르면 프로그램들은 서로 통신할 수 없다. 따라서 프로그램들은 <strong>DNS(Domain Name System)</strong>를 활용해 컴퓨터의 <strong>IP 주소를 검색</strong>한다.
  </li>
    <ul>
      <li>
        DNS는 <strong>IP를 등록하는 저장소</strong>이다. 대중에게 서비스를 제공하는 대부분의 컴퓨터는 도메인 이름으로 IP를 DNS에 미리 등록해 둔다.
      </li>
    </ul>
  <li>
    <strong>웹 브라우저</strong>는 <strong>웹 서버와 통신하는 클라이언트</strong>로 사용자가 입력한 도메인 이름으로 DNS에서 IP 주소를 찾아 웹 서버와 연결한다.
  </li>
</ul>

<br>

<h2>1-3. Port 번호</h2>
<ul>
  <li>
    한대의 컴퓨터에는 다양한 서버 프로그램(웹, DB, FTP 등)들이 실행될 수 있으며 따라서 <strong>하나의 IP 안에 동시에 여러 서버</strong>가 존재할 수 있다.
  </li>
    <ul>
      <li>
        IP는 컴퓨터의 <strong>네트워크 어댑터까지의 접근</strong>만 가능하기에 컴퓨터에 도착한 후 <strong>어느 서버에 접근</strong>할지 알 수 없다.
      </li>
      <li>
        이때 클라이언트가 하나의 컴퓨터 내에서 어떤 서버에 접근할지를 선택하기 위해서는 <strong>Port 번호</8strong>가 필요하다.
      </li>
    </ul>
  <li>
    Port는 <strong>운영체제</strong>가 관리하는 서버 프로그램의 <strong>연결 번호</strong>이며 서버는 시작할 때 <strong>특정 Port 번호에 바인딩</strong>한다.
  </li>
  <li>
    <strong>클라이언트</strong> 또한 서버에서 보낸 정보를 받기 위해 Port가 필요하다.
  </li>
    <ul>
      <li>
        그러나 서버처럼 고정적인 Port에 바인딩하지 않고 운영체제가 <strong>자동으로 부여하는 번호</strong>를 사용한다.
      </li>
    </ul>
  <li>
    프로그램에서 사용할 수 있는 전체 Port 번호의 범위는 <strong>0 ~ 65535</strong>이며 사용 목적에 따라 세 가지 범위를 가진다 (권장 사안임).
  </li>
    <ul>
      <li>
        <strong>Well Know Port Numbers (0 ~ 1023)</strong>
      </li>
        <ul>
          <li>
            <strong>국제인터넷주소관리기구(CANN)</strong>가 특정 애츨리케이션용으로 미리 예약한 Port.
          </li>
        </ul>
      <li>
        <strong>Registered Port Numbers (1024 ~ 49151)</strong>
      </li>
        <ul>
          <li>
            <strong>회사</strong>에서 등록해서 사용할 수 있는 Port.
          </li>
        </ul>
      <li>
        <strong>Dynamic Or Private Port Numbers (49152 ~ 65535)</strong>
      </li>
        <ul>
          <li>
            <strong>운영체제</strong>가 부여하는 동적 Port 또는 <strong>개인용</strong> 목적 Port
          </li>
        </ul>
    </ul>
</ul>

<br><br>

<h1>2. IP 주소 얻기</h1>
<ul>
  <li>
    자바는 IP 주소를 <strong>java.net 패키지</strong>의 <strong>InetAddress</strong>로 표현한다.
  </li>
    <ul>
      <li>
        InetAddress를 활용하면 <strong>로컬 컴퓨터의 IP 주소</strong>와 도메인 이름으로 <strong>DNS를 검색하여 IP 주소</strong>를 가져올 수 있다.
      </li>
    </ul>
  <li>
    하나의 도메인 이름으로 여러 IP가 등록되어 있는 이유는 클라이언트가 많이 연결될 경우 <strong>서버 부하를 나누기 위해서</strong>이다.
  </li>
  <li>
    메소드로 얻은 InetAddress 객체의 IP는 <strong>getHostAddress() 메소드</strong>를 통해 얻을 수 있다.
  </li>
    <ul>
      <li>
        리턴값은 <strong>문자열로 된 IP 주소</strong>이다.
      </li>
    </ul>
</ul>

```java
// 1. IP 가져오기.
// 1-1. getLocalHost() 사용.
InetAddress ia = InetAdress.getLocalHost();


// 2. 컴퓨터의 도메인을 알고 있을 경우 IP 가져오기.
// 2-1. getByName()
//   - DNS에서 도메인 이름으로 등록된 단 하나의 IP 주소를 가져온다.
InetAddress ia = InetAddress.getByName(String domainName);
// 2-2. getAllByName()
//   - DNS에 등록된 모든 IP 주소를 배열로 가져온다.
InetAddress[ ] iaArr = InetAddress.getAllByName(String domainName);

// 3. 메소드로 얻은 IP 얻기.
String ip = InetAddress.getHostAddress();
```

<h1>3. TCP 네트워킹</h1>
<ul>
  <li>
    IP 주소로 프로그램들이 통신을 할 때에는 <strong>데이터 전송 규약</strong>이 있으며 이를 <strong>전송용 프로토콜</strong>이라 한다.
  </li>
    <ul>
      <li>
        인터네 전송용 프로토콜은 <strong>TCP(Transmission Control Protocol)</strong>와 <strong>UDP(User Datagram Protocol)</strong>가 있다.
      </li>
    </ul>
  <li>
    TCP는 <strong>연결형 프로토콜</strong>이다. 따라서 상대방이 연결된 상태에서 데이터를 주고 받는다.
  </li>
    <ul>
      <li>
          클라이언트가 연결을 요청하고 서버가 연결을 수락하면 <strong>통신 회선이 고정</strong>되고 데이터는 고정회선을 통해 전달된다.
      </li>
      <li>
          TCP는 보낸 데이터가 <strong>순서대로</strong> 전달되며 <strong>손실</strong>이 발생하지 않는다.
      </li>
    </ul>
  <li>
    TCP는 IP와 함께 사용되기 때문에 TCP/IP 라고도 한다.
  </li>
    <ul>
      <li>
        TCP는 <strong>웹 브라우저가 웹 서버에 연결</strong>할 때 사용되며 이메일 전송, 파일 전송, DB 연동 등에도 사용된다.
      </li>
    </ul>
  <li>
    java는 TCP 네트워킹을 위해 java.net 패키지에서 <strong>ServerSocket</strong>과 <strong>Socket 클래스</strong>를 제공한다.
  </li>
    <ul>
      <li>
        <strong>ServerSocket</strong>: 클라이언트의 <strong>연결을 수락</strong>하는 서버 쪽 클래스.
      </li>
      <li>
        <strong>Socket</strong>: 클라이언트에서 <strong>연결 요청</strong>할 때 + 서버 양쪽에서 <strong>데이터를 주고 받을 때</strong> 사용한다.
      </li>
    </ul>
  <li>
    <strong>ServerSocket</strong>을 생성할 때 <strong>바인딩할 Port 번호</strong>를 지정해야 한다.
  </li>
    <ul>
      <li>
        클라이언트는 <strong>Socket</strong>을 통해 <strong>서버의 IP 주소</strong>와 <strong>Port 번호</strong>로 요청을 한다.
      </li>
      <li>
        ServerSocket은 <strong>accept() 메소드</strong>로 연결을 수락하고 <strong>통신용 Socket을 생성</strong>한다.
      </li>
      <li>
        클라이언트와 서버는 <strong>양쪽의 Socket</strong>을 통해 데이터를 주고받는다.
      </li>
    </ul>
</ul>

<br>

<h2>3-1. TCP 서버</h2>
<ul>
  <li>
    TCP 서버 프로그램을 개발하기 위해서는 우선 <strong>ServerSocket 객체</strong>를 생성해야 한다.
  </li>
  <li>
    ServerSocket이 생성되었다면 연결 요청을 수락할 <strong>accept() 메소드</strong>를 실행해야 한다.
  </li>
    <ul>
      <li>
        accept()는 클라이언트가 연결 요청하기 전까지 <strong>블로킹</strong>된다. 블로킹이란 실행이 멈춘 상태를 의미한다.
      </li>
      <li>
        클라이언트의 연결 요청이 들어오면 블로킹이 해제되고 통신용 Socket을 리턴한다.
      </li>
    </ul>
</ul>

```java
// 1. 50001 Port에 바인딩하는 ServerSocket 코드.
ServerSocket serverSocket = new ServerSocket(50001);

// 2. 기본 생성자로 객체를 생성하고 Port 바인딩을 위해 bind() 메소드 호출.
ServerSocket serverSocket = new ServerSocket();
serverSocket.bind(new InetSocketAddress(50001));

// 3. 서버에 여러 대의 IP가 할당되었을 때 특정 IP에만 서비스를 하고 싶은 겨우.
ServerSocket serverSocket = new ServerSocket();
// InetSocketAddress의 매개값으로 사용할 IP를 전달한다.
serverSocket.bind( new InetSocketAddress("xxx.xxx.xxx.xxx.", 50001) );

// 4. 요청을 수락하기 위해 accept() 메소드 실행.
Socket socket = serverSocket.accept();

// 5. 연결된 Socket에서 IP와 Port 번호 얻기.
// 5-1. 우선 getRemoteSocketAddress()를 통해 InetSocketAddress 정보 얻기.
InetSocketAddress isa = (InetSocketAddress) socket.getRemoteSocketAddress();
// 5-2. getHostToString()으로 IP 얻기.
String clientIp = isa.getHostToString()l
// 5-3. getPort()로 포트 얻기.
String portNo = isa.getPort();

// 6. close() 메소드를 호출하여 서버 종료.
//   - 이때 Port 번호가 언바인딩 된다.
serverSocket.close();
```

<h2>3-2. TCP 클라이언트</h2>
<ul>
  <li>
    클라이언트가 서버에 연결 요청을 하는 경우 Socket 객체를 생성할 때 생성자 매개값으로 <strong>서버 IP 주소</strong>와 <strong>Port 번호</strong>를 제공하면 된다.
  </li>
    <ul>
      <li>
        로컬 컴퓨터에서 실행하는 경우 IP 주소 대신 <strong>localhost</strong>를 사용할 수 있다.
      </li>
    </ul>
  <li>
    IP 대신 <strong>DNS</strong>에서 IP 주소를 검색할 때에는 생성자 매개값으로 <strong>InetAddress</strong>를 제공해야 한다.
  </li>
  <li>
    연결 요청시에는 두 가지 예외가 발생할 수 있다.
  </li>
    <ul>
      <li>
        <strong>IP 주소가 잘못 표기</strong>되었을 때에는 <strong>UnknownHostException</strong>이 발생한다.
      </li>
      <li>
        IP와 Port 번호로 <strong>연결할 수 없을 때</strong>에는 <strong>IOException</strong>이 발생한다.
      </li>
    </ul>
  <li>
    서버와 연결된 후 클라이언트에서 연결을 끊고 싶다면 Socket의 <strong>close() 메소드</strong>를 호출한다.
  </li>
</ul>

```java
// 1. 클라이언트에서 서버로 연결 요청 보내기.
Socket socket = new Socket( "IP", 50001 );

// 2. DNS로 IP 주소 검색.
// 2-1. 매개값으로 넘기기.
Socket socket = new Socket(InetAddress.getByName("domainName"), 50001);
// 2-2. 기본 생성자로 Socket 생성 후 connect() 메소드로 연결 요청하기.
socket = new Socket();
socket.connect( new InetSocketAddress("domainName", 50001) );

// 3. UnknownHostException과 IOException 처리.
try {
  Socket socket = new Socket("IP", 50001);
} catch (UnknownHostException e) {
  // IP 표기 방법이 잘못되었을 경우.
} catch (IOException e) {
  // IP와 Port로 서버에 연결할 수 없는 경우.
}

// 4. 연결 종료.
socket.close();
```

<h2>3-3. 입출력 스트림으로 데이터 주고받기</h2>
<ul>
  <li>
    클라이언트가 연결 요청(connect())을 하고 서버가 연결을 수락(accept())한 순간부터 양쪽의 Socket 객체로부터 각각 <strong>입력 스트림(InputStream)</strong>과 <strong>출력 스트림(OutputStream)</strong>을 얻을 수 있다.
  </li>
  <li>
    상대에게 데이터를 보낼 때에는 <strong>byte[] 배열을 생성</strong>하고 이를 매개값으로 하여 OutputStream의 <strong>write() 메소드를 호출</strong>한다.
  </li>
    <ul>
      <li>
        문자열을 조금 더 간결하게 보내고 싶을 때에는 보조 스트림인 <strong>DataOutputStream</strong>을 연결하여 사용한다.
      </li>
    </ul>
  <li>
    데이터를 받을 때에는 데이터를 받기 위한 <strong>byte[] 배열을 하나 생성</strong>한다.
  </li>
    <ul>
      <li>
        생성한 byte[] 배열을 매개값으로 하여 <strong>InputStream의 read() 메소드를 호출</strong>한다.
      </li>
      <li>
        read() 메소드는 <strong>읽은 데이터를 byte[] 배열에 저장</strong>하고 <strong>읽은 바이트 수를 리턴</strong>한다.
      </li>
    </ul>
  <li>
    마찬가지로 문자열을 조금 더 편하게 받고 싶다면 보조 스트림인 <strong>DataInputStream</strong>을 연결해서 사용하면 된다.
  </li>
</ul>

```java
// 1. Socket으로 입력 스트림과 출력 스트림 얻기.
InputStream is = socket.getInputStream();
OutputStream os = socket.getOutputStream();

// 2. UTF-8로 인코딩한 배열을 얻고, write() 메소드로 전송.
String data = "보낼 데이터";
byte[ ] bytes = data.getBytes("UTF-8");
// 데이터를 보내기 위한 스트림을 생성.
OutputStream os = socket.getOutputStream();
os.write(bytes);
os.flush();


// 3. 보조 스트림 DataOutputStream을 통해 좀 더 간결하게 데이터 보내기.
String data = "보낼 데이터";
// 바이트로 보내는 stream을 만들고 이를 DataOutputStream으로 
// 래핑하여 기본형 String으로 보낼 수 있도록 준비.
DataOutputStream dos = new DataOutputStream(socket.getOutputStream());
// 내보낼 때에는 writeUTF로 바이트로 변환해서 실제 전송.
dos.writeUTF(data);
dos.flush();

// 4. byte[] 배열을 UTF-8로 디코딩하여 문자열 얻기.
byte[ ] bytes = new byte[1024];
InputStream is = socket.getInputStream();
int num = is.read(bytes);
// bytes를 0부터 num까지 읽고 UTF-8로 해석하겠다.
String data = new String(bytes, 0, num, "UTF-8");

// 5. DataInputStream 보조 스트림을 통해 좀 더 쉽게 데이터 얻기.
DataInputStream dis = new DataInputStream(socket.getInputStream());
String data = dis.readUTF();
```

<h1>4. UTP 네트워킹</h1>
<ul>
  <li>
    <strong>UDP(User Datagram Protocol)</strong>는 발신자가 <strong>일방적으로 수신자에게 데이터를 보내는 구조</strong>로 TCP 처럼 요청/수락 과정이 없어 <strong>상대적으로 빠르다</strong>.
  </li>
  <li>
    UDP는 <strong>여러 회선</strong>을 통해 데이터가 전송되기 때문에 사용되는 회선에 따라 <strong>속도 차이</strong>가 발생한다.
  </li>
    <ul>
      <li>
        또한 이러한 차이로 인해 속도에 따라 데이터가 <strong>순서대로</strong> 전달되지 않거나 <strong>데이터 손실</strong>이 발생할 수 있다.
      </li>
    </ul>
  <li>
    연속적인 데이터의 전달은 일부 손실을 감안할 수 있기에 <strong>데이터의 속성</strong>에 따라 UDP와 TCP를 선택한다.
  </li>
    <ul>
      <li>
        UDP: 데이터 전달 속도 > 신뢰성
      </li>
      <li>
        TCP: 데이터 전달 속도 < 신뢰성
      </li>
    </ul>
  <li>
    자바는 UDP 네트워킹을 위해 java.net 패키지에서 <strong>DatagramSocket</strong>과 <strong>DatagramPacket 클래스</strong>를 제공한다.
  </li>
    <ul>
      <li>
        DatagramSocket: 발신점과 수신점.
      </li>
      <li>
        DatagramPacket: 주고 받는 데이터.
      </li>
    </ul>
</ul>

<br>

<h2>4-1. UDP 서버</h2>
<ul>
  <li>
    UDP는 TCP와 달리 <strong>연결(Connection)</strong>의 개념이 없기 때문에, accept()와 같은 <strong>연결 수락 과정</strong>이 필요 없다.
  </li>
  <li>
    UDP 서버를 위한 <strong>DatagramSocket 객체</strong>를 생성할 때 <strong>바인딩할 Port</strong>를 생성자 매개값으로 제공한다.
  </li>
  <li>
    UDP 서버는 클라이언트가 보낸 DatagramPacket을 <strong>항상 받을 준비</strong>를 해야한다.
  </li>
    <ul>
      <li>
        <strong>receive() 메소드</strong>는 데이터를 수신할 때까지 블로킹되고, 데이터가 수신되면 <strong>매개값으로 주어진 DatagramPacket에 저장</strong>한다.
      </li>
    </ul>
  <li>
    UDP 서버가 클라이언트에게 처리 내용을 보낼 때 필요한 IP 주소와 Port 번호는 <strong>receive()로 받은 DatagramPacket</strong>에서 얻을 수 있다.
  </li>
    <ul>
      <li>
        <strong>getSocketAddress() 메소드</strong>를 호출하고 정보가 담긴 <strong>SocketAddress 객체</strong>를 얻어 활용한다.
      </li>
    </ul>
</ul>

```java
// 1. DatagramSocket 생성.
DatagramSocket datagramSocket = new DatagramSocket(50001);

// 2. 데이터를 받아 저장.
//   - DatagramPacket(수신된 데이터를 저장할 배열, 수신할 수 있는 최대 바이트).
DatagramPacket receivePacket = new DatagramPacket(new byte[1024], 1024);
datagramSocket.receive(receivePacket);

// 3. receive() 메소드 실행 후 수신된 데이터와 바이트 수 얻기.
byte[] bytes = receivePacket.getData();
int num = receivePacket.getLength();
// 3-1. 읽은 데이터가 문자열인 경우.
String data = new String(bytes, 0, num, "UTF-8");

// 4. UDP 서버가 클라이언트에게 반환하기 위해 IP 주소와 Port 번호 얻기.
// 4-1. 우선 호출 정보를 받는다.
SocketAddress socketAddress = receivePacket.getSocketAddress();
// 4-2. 얻은 정보를 매개값으로 하여 클라이언트에 전달할 객체 생성.
String data = "처리 결과";
byte[] bytes = data.getBytes("UTF-8");
DatagramPacket sendPacket = new DatagramPacket( bytes, 0, bytes.length, socketAddress );
// 4-3. send() 메소드를 통해 데이터 전송.
datagramSocket.send( sendPacket );

// 5. UDP 서버 종료.
datagramSocket.close();
```

<br>

<h2>4-2. UDP 클라이언트</h2>
<ul>
  <li>
    UDP 클라이언트는 서버에 <strong>요청 내용을 보내고 결과를 받는 역할</strong>을 한다.
  </li>
    <ul>
      <li>
        UDP 클라이언트를 위한 DatagramSocket 객체는 <strong>기본 생성자</strong>로 생성한다.
      </li>
      <li>
        Port 번호는 <strong>자동으로 부여</strong>된다.
      </li>
    </ul>
  <li>
    UDP 클라이언트는 처리 결과가 언제 올지 모르기 때문에 <strong>항상 받을 준비</strong>를 하기 위해 <strong>receive() 메소드</strong>를 호출한다.
  </li>
    <ul>
      <li>
        receive() 메소드는 데이터를 수신할 때까지 <strong>블로킹</strong>되고, 데이터가 수신되면 매개값으로 주어진 <strong>DatagramPacket에 저장</strong>한다.
      </li>
    </ul>
  <li>
    UDP 서버와 통신할 필요가 없다면 DatagramSocket을 닫기 위해 <strong>close() 메소드</strong>를 호출한다.
  </li>
</ul>

```java
// 1. UDP 클라이언트를 위한 DatagramSocket 객체 생성.
DatagramSocket datagramSocket = new DatagramSocket();

// 2. 요청을 보내기 위한 DatagramPacket 생성.
String data = "요청 내용";
byte[ ] bytes = data.getBytes("UTF-8");
// (바이트 배열, 보내고자하는 바이트 수, UDP 서버의 IP와 Port정보).
DatagramPacket sendPacket = new DatagramPacket(
  bytes, bytes.length, new InetSocketAddress("localhost", 50001)
);

// 3. send()를 통한 전송.
datagramSocket.send(sendPacket);
```

<h1>5. 서버의 동시 요청 처리</h1>
<ul>
  <li>
    일반적으로 서버는 다수의 클라이언트로부터 <strong>동시에 요청</strong>을 받아 처리한다. TCP와 UDP 또한 동일한다.
  </li>
    <ul>
      <li>
        accept()와 receive()를 제외한 요청을 처리하는 코드를 <strong>별도의 스레드</strong>로 분리하여 성능을 올릴 수 있다.
      </li>
        <ul>
          <li>
            즉, 데이터를 받거나 요청을 수락하는 부분과 처리하는 부분을 <strong>분리</strong>하는 것이다.
          </li>
        </ul>
    </ul>
  <li>
    스레드로 처리할 때 주의할 점은 클라이언트가 폭증할 때 <strong>과도한 스레드 생성을 방지</strong>해야 한다는 점이다.
  </li>
    <ul>
      <li>
        과도한 스레드 생성을 방지하는 데에는 <strong>스레드풀</strong>을 사용하는 것이 바람직하다. 스레드풀은 <strong>작업 처리 스레드 수를 제한</strong>한다.
      </li>
      <li>
        단, 제한된 스레드 수로 인해 작업 큐의 대기 작업이 증가되어 응답을 늦게 반환할 수 있다.
      </li>
    </ul>
</ul>

```java
// 1. TCP 서버의 분리.
while(true) {
  Socket socket = serverSocket.accept();
  
  // 스레드로 처리.
  /*
  데이터 받기
  ...
  데이터 보내기
  */
}

// 2. UDP 서버의 분리.
while(true) {
  DatagramPacket = receivePacket = ...datagramSocket.receive(receivePacket);
  ...

  // 스레드로 처리.
  /*
  10 개의 뉴스를 클라이언트로 전송.
  */
}
```

<h1>6. JSON 데이터 형식</h1>
<ul>
  <li>
    네트워크 통신에서 가장 많이 사용되는 데이터 형식은 <string>JSON(JavaScript Object Notation)</string>이다.
  </li>
    <ul>
      <li>
        <string>두 개 이상의 속성</string>이 있는 경우 <string>객체 {}</string>로 표기한다.
      </li>
      <li>
        <string>두 개 이상의 값</string>이 있는 경우 <string>배열 []</string>로 표기한다.
      </li>
    </ul>
  <li>
    https://github.com/stleary/JSON-java 에서 다운을 받고 별도로 IntellJ에 추가해 주어야 함.
  </li>
  <li>
    <string>JSONObject 클래스</string>는 <string>JSON 객체 표기</string>를 생성하거나 파싱할 때, <string>JSONArray 클래스</string>는 <string>JSON 배열 표기</string>를 생성하거나 파싱할 때 사용한다.
  </li>
  <li>
    생성된 JSON은 속성 순서가 중요하지 않기 때문에 <strong>추가한 순서대로</strong> 작성되지 않아도 된다. 또한 <strong>줄바꿈 처리</strong>도 되지 않는다.
  </li>
    <ul>
      <li>
        하지만 이런 점이 <strong>네트워크 전송량</strong>을 줄여주기 때문에 더 좋다.
      </li>
    </ul>
</ul>

```java
// 1. JSON 형식 예.
{
  "id": "winter",
  "name": "한겨울",
  "age": 25,
  "student": true,
  "tel": { "home": "02-123-1234", "mobile": "010-123-1234" },
  "skill": { "java", "c", "c++" }
}
```
