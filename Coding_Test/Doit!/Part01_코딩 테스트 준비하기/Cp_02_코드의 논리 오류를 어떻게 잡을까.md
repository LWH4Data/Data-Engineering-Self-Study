<h1>1. 디버깅은 왜 중요할까?</h1>
<ul>
  <li>
    프로그램에서 발생하는 문법 오류나 논리 오류를 찾아 바르게 수정하는 과정을 <strong>디버깅(debugging)</strong>이라 한다.
  </li>
</ul>
<h4>디버깅 하는 법</h4>
<ul>
  <li>
    코드에서 디버깅하고자 하는 <strong>중단점</strong>을 설정한다. (여러개 가능)<br>
    → IDE의 디버깅 기능을 실행하면 코드 1줄씩 실행 하거나 중단점 까지 실행할 수 있다. (변숫값이 의도한 대로 변하는지 확인한다).<br>
    → 변숫값 이외에도 원하느 수식을 입력하여 논리 오류를 파악할 수 있다.
  </li>
  <li>
    변숫값 추적은 <strong>PyCharm</strong>의 <strong>Variable 기능</strong>을 활용하면 된다.
  </li>
</ul>

<h1>2. 디버깅 활용 사례 살펴보기</h1>
<h2>2-1. 코딩 테스트를 진행하며 실수하기 쉬운 4가지 오류 찾아보기</h2>

```python
# 1. 디버깅 대상 코드
import random

testcase = int(input())
answer = 0
A = [0] * (100001)

# 인덱스 범위 지정 오류
for i in range(0, 10001):
    A[i] = random.randrange(1, 101)

for t in range(1, testcase+1):
    # answer를 초기화하지 않아 오류 발생.
    start, end = map(int, input().split())

    for i in range(start, end + 1):
        answer = answer + A[i]
    
    # 잘못된 변수 사용 오류: t가 아닌 testcase 사용.
    # 파이썬의 자동형 변환 조심. (/, //, %).
    print(str(testcase) + " " + str(answer/2))
```
