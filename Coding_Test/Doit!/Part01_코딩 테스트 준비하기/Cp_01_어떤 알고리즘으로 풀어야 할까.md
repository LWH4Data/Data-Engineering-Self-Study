<h1>시간 복잡도 표기법 알아보기</h1>
<ul>
  <li>
    시간 복잡도는 주어진 문제를 해결하기 위한 <strong>연산 횟수</strong>를 의미한다.
  </li>
  <li>
    파이썬은 1초에 <strong>2,000만 번 ~ 1억 번</strong>의 연산을 1초의 수행 시간으로 예측할 수 있다.
  </li>
  <li>
    시간 복잡도를 정의하는 세 가지 유형
  </li>
    <ul>
      <li>
        <strong>빅-오메가(Ω(n))</strong>: <strong>최선일 때(best case)</strong>의 연산 횟수를 나타낸다.
      </li>
      <li>
        <strong>빅-세타(Θ(n))</strong>: <strong>보통일 때(average case)</strong>의 연산 회수를 나타낸다.
      </li>
      <li>
        <strong>빅-오(O(n))</strong>: <strong>최악일 때(worst-case)</strong>의 연산 횟수를 나타낸다.
      </li>
    </ul>
  <li>
    코딩 테스트에서는 다양한 테스트 케이스가 있기에 최악을 염두한 <strong>O(n)</strong> 방식의 시간 복잡도를 사용하는 것이 좋다.
  </li>
</ul>

```python
# 1. 시간 복잡도 예제
import random
findNumber = random.randrange(1, 101) # 1 ~ 100 사이 무작위값 생성.

for i in range(1, 101):
    if i == findNumber:
        print(i)
        break

"""
< 시간 복잡도 >
- Ω(1)
- Θ(N/2)
- O(n)
"""
```

<h1>2. 시간 복잡도 활용하기</h1>
<h4>알고리즘 선택의 기준으로 사용하기</h4>
<ul>
  <li>
    각 알고리즘의 시간 복잡도를 알고 있을 때, 이를 활용해서 <strong>특정 문제에 어떤 알고리즘을 사용할 수 있을지 판단</strong>할 수 있다.
  </li>
</ul>

<h4>시간 복잡도를 바탕으로 코드 로직 개선하기</h4>
<ul>
  <li>
    시간 복잡도를 통해 코드 로직을 개선할 때에는 코드의 시간 복잡도를 먼저 도출한다.
  </li>
    <ul>
      <li>
        <strong>상수</strong>는 시간 복잡도에서 제외한다.
      </li>
      <li>
        <strong>가장 많이 중첩된 반복문의 횟수</strong>가 시간 복잡도의 기준이 된다.
      </li>
    </ul>
</ul>

```python
# 1. 연산 횟수 차이.

# N---------------------------------
N = 100000
cnt = 1

for i in range(N):
    print("연산 횟수:" + str(cnt))
    cnt += 1

# N---------------------------------
N = 100000
cnt = 1

for i in range(N):
    print("연산 횟수:" + str(cnt))
    cnt+=1

for i in range(N):
    print("연산 횟수:" + str(cnt))
    cnt += 1

for i in range(N):
    print("연산 횟수:" + str(cnt))
    cnt += 1

# N^2-------------------------------
#   ● 최대 중첩된 for loop이 2중 중첩이기에 시간 복잡도의 기준이 된다.
N = 100000
cnt = 1

for i in range(N):
    for j in range(N):
        print("연산 횟수:" + str(cnt))
        cnt += 1
```
