<h1>1. 콘텐츠 프로바이더 이해하기</h1>
<ul>
  <li>
    콘텐츠 프로바이더는 <strong>앱끼리 데이터를 연동</tstrong>하는 컴포넌트이다.
  </li>
    <ul>
      <li>
        데이터는 대상 <strong>앱의 데이터베이스</strong>나 <strong>파일</strong> 또는 <strong>앱에 할당된 메모리</strong>에 있다.
      </li>
    </ul>
  <li>
    앱의 데이터는 기본적으로 해당 앱의 구성 요소에서 이용 가능하지만 <strong>외부 앱</strong>에서는 보안상의 이유로 <strong>접근이 불가</strong>하다.
  </li>
    <ul>
      <li>
        외부 앱과 데이터를 공유하기 위해서는 <strong>데이터를 공유하는 앱</strong>에서 콘텐츠 프로바이더와 접근 방법을 제공하고, <strong>외부 앱</strong>에서 콘텐츠 프로바이더를 이용해 공개한 데이터 접근할 수 있다.
      </li>
    </ul>
</ul>

<br>

<h2>1-1. 콘텐츠 프로바이더 작성하기</h2>
<ul>
  <li>
    콘텐츠 프로바이더는 <strong>ContentProvider 클래스</strong>를 상속받아 작성한다.
  </li>
  <li>
    ContentProvider를 상속받는 클래스에 <strong>onCreate(), getType(), query(), insert(), update(), delete()</strong> 함수를 재정의해서 작성한다.
  </li>
    <ul>
      <li>
        onCreate() 함수는 콘텐츠 프로바이더의 생명주기 함수로 시스템이 <strong>콘텐츠 프로바이더 객체를 생성</strong>할 때 자동으로 호출된다.
      </li>
        <ul>
          <li>
            외부 앱에서 query(), insert(), update(), delete() 함수 등을 호출해 <strong>데이터를 조작</strong>한다.
          </li>
        </ul>
      <li>
        query(), insert(), update(), delete() 콘텐츠는 프로바이더를 만들 때 클래스에 꼭 정의해 주어야 하는데 외부 요청으로 위의 기능을 허용하지 않으려면 <strong>함수 내부를 구현하지 않으면</strong> 된다.
      </li>
    </ul>
  <li>
    콘텐츠 프로바이더도 안드로이드 컴포넌트이기 때문에 <strong>매니페스트에 등록</strong>해야 한다.
  </li>
    <ul>
      <li>
        다른 컴포넌트와 달리 <strong>name 속성</strong> 뿐만 아니라 <strong>authorities 속성</strong>도 반드시 선언해야 한다.
      </li>
      <li>
        콘텐츠 프로바이더는 provider 태그로 등록하며 <strong>name 속성</strong>은 등록하고자 하는 <strong>클래스명</strong>, <strong>authorities 속성</strong>은 이 콘텐츠 프로바이더를 이용할 때 <strong>식별값으로 사용되는 문자열</strong>이다.
      </li>
        <ul>
          <li>
            authorities 속성은 <strong>개발자가 지정하는 고유한 값</strong>이어야 한다.
          </li>
        </ul>
    </ul>
</ul>

```kotlin

    }

    }
```

<h2>1-2. 콘텐츠 프로바디어 이용하기</h2>
<ul>
  <li>
    콘텐츠 프로바이더는 <strong>인텐트</strong>와 상관없다.
  </li>
    <ul>
      <li>
        필요한 순간에 시스템에서 자동으로 생성하여 query(), insert(), update(), delete() 함수만 호출해 주면 된다.
      </li>
    </ul>
  <li>
    외부 앱에서 콘텐츠 프로바이더를 사용하려면 먼저 매니페스트에 해당 앱에 관한 <strong>패키지 공개 설정</strong>을 해주어야 한다.
  </li>
    <ul>
      <li>
        콘텐츠 프로바이더를 사용하는 앱의 매니페스트에 <strong>대상 앱의 패키지명</strong>을 <strong>package 태그</strong>로 명시한다. 혹은 사용하려는 <strong>콘텐츠 프로바이더의 authoriteis 속성</strong>을 <strong>provider 태그</strong>로 선언한다.
      </li>
    </ul>
  <li>
    시스템에 등록된 콘텐츠 프로바이더를 사용할 때는 <strong>ContentResolver 객체</strong>를 이용한다.
  </li>
    <ul>
      <li>
        contentResolver 속성으로 ContentResolver 객체를 얻고 다음과 같은 함수로 데이터를 조작한다.
      </li>
        <ul>
          <li>
            public final int delete(Uri url, String where, String[] selectionArgs)
          </li>
          <li>
            public final Uri insert(Uri url, ContentValues values)
          </li>
          <li>
            public final Cursor query(Uri uri, String[] projection, String selection, String[] selectArgs, String sortOrder)
          </li>
          <li>
            public final int update(Uri, uri, ContentValues values, String where, String[] selectionArgs)
          </li>
        </ul>
    </ul>
  <li>
    함수들의 첫 번째 매개변수는 대상 콘텐츠 프로바이더를 식별하는 <strong>Uri 객체</strong>이다. 
  </li>
    <ul>
      <li>
        Uri 객체의 URL 문자열은 프로토콜명과 콘텐츠 프로바이더의 식별자로 등록된 <strong>authorities 값</strong>이어야 한다.
      </li>
      <li>
        content://com.example.test_provider
      </li>
        <ul>
          <li>
            content://는 프로토콜(scheme)
          </li>
          <li>
            com.example.test_provider는 호스트(host)로 authorities이다.
          </li>
        </ul>
      <li>
        콘텐츠 프로바이더를 이용할 때 URL 문자열에서 호스트 뒤에 <strong>경로(path)</strong>를 설정할 수도 있다.
      </li>
        <ul>
          <li>
            경로는 선택사항이며 이를 통해 <strong>조건</strong>을 명시할 수 있다.
          </li>
            <ul>
              <li>
                경로에는 단어나 숫자를 사용한다. 단어는 <strong>단어에 해당하는 모든 데이터</strong>를, 숫자는 <strong>숫자로 식별되는 데이터</strong>를 의미하는 조건으로 주로 사용한다.
              </li>
              <li>
                content://com.example.test_provider/user/1
              </li>
                <ul>
                  <li>
                    user/1이 경로에 해당한다. user 데이터 중 1로 식별되는 데이터를 가져오겠다는 의미이다.
                  </li>
                </ul>
            </ul>
        </ul>
    </ul>
  <li>
    update(), insert() 함수의 매개변수로 지정되는 ContentValues는 <strong>Map 형태의 집합 객체</strong>이다.
  </li>
    <ul>
      <li>
        <strong>키-값</strong>으로 여러 건의 데이터를 ContentValues에 지정하고 이 객체를 insert(), update() 함수의 매개변수로 넘겨 데이터를 저장하거나 수정한다.
      </li>
    </ul>
  <li>
    <strong>query() 함수</strong>의 반환 타입인 <strong>Cursor</strong>도 가져올 데이터의 <strong>Map 객체</strong>이다.
  </li>
</ul>

```xml

    <package android:name="com.example.test_outter" />
</queries>
```

<ul>
  <li>
    우선 주로록 앱에서 데이터를 가져오기 위해 <strong>퍼미션을 설정</strong>해야 한다.
  </li>
  <li>

        <ul>

<h2>2-2. 갤러리 앱 연동하기</h2>
<ul>
  <li>
    갤러리 앱 연동은 <strong>인텐트</strong>로 갤러리 앱의 목록 화면을 띄우거나 갤러리 앱의 <strong>콘텐츠 프로바이더</strong>로 데이터를 가져오는 작업이다.
  </li>
</ul>

<h3>2-2-1. 아미지 작업 시 고려 사항</h3>
<ul>
  <li>
    안드로이드 앱에서 이미지 데이터를 이용할 때에는 다음을 고려해야 한다.
  </li>
    <ul>
      <li>
        안드로이드에서 이미지는 <strong>Drawable</strong>이나 <strong>Bitmap 객체</strong>로 표현한다.
      </li>
        <ul>
          <li>
            Drawable은 주로 <strong>리소스 이미지</strong>를 표현할 때 사용한다.
          </li>
          <li>
            Bitmap은 <strong>파일에서 읽은 이미지</strong>나 <strong>네트워크에서 내려받은 이미지</strong>를 표현할 때 사용한다.
          </li>
          <li>
            Bitmap과 Drawable은 서로 <strong>호환</strong>되기 때문에 서로 타입을 바꿀 수 있다.
          </li>
          <li>
            Bitmap 이미지는 BitmapFactory 클래스의 <strong>decode로 시작</strong>하는 다음과 같은 함수로 생성한다.
          </li>
            <ul>
              <li>
                BitmapFactory.decodeByteArray(): byte[] 배열의 데이터로 비트맵 생성.
              </li>
              <li>
                BitmapFactory.decodeFile(): 파일 경로를 매개변수로 지정하면 해당 파일에서 데이터를 읽을 수 있는 FileInputStream을 만들어 decodeStream() 함수를 이용한다.
              </li>
              <li>
                BitmapFactory.decodeResource(): <strong>리소스 이미지</strong>로 비트맵을 생성한다.
              </li>
              <li>
                BitmapFactory.decodeStream(): <strong>InputStream</strong>으로 읽은 데이터로 비트맵 생성.
              </li>
            </ul>
        </ul>
      <li>
        Bitmap 객체는 <strong>BitmapFactory</strong>로 생성한다.
      </li>
      <li>
        BitmapFactory로 이미지를 생성할 때에는 <strong>OOM 오류</strong>를 고려해야 한다.
      </li>
        <ul>
          <li>
            BitmapFactory를 이용해 비트맵을 만들 때 카메라로 찍은 사진이나 서버에서 내려받은 <strong>크기가 큰 이미지</strong>를 불러올 때에는 OOM(out of memeory) 오류가 발생할 수 있다.
          </li>
            <ul>
              <li>
                이미지의 크기를 줄여 해결할 수 있다.
              </li>
            </ul>
        </ul>
      <li>
        Glide나 Picasso 같은 <strong>이미지 처리 라이브러리</strong>를 이용하는 것이 호율적일 수 있다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 옵션을 지정하지 않고 비트맵 생성.
//   - 매개변수에 옵션을 지정하지 않으면 원본을 그래도 불러와 OOM 오류가 발생할 수 있다.
val bitmap = BitmapFactory.decodeStream(inputStream)

// 2. 옵션을 지정해 비트맵 생성.
val option = BitmapFactory.Options()

// BitmapFactory.Option 객체의 inSampleSize 속성을 이용해 사이즈를 줄인다.
// inSampleSize에 적용된 값만큼의 비율로 데이터를 줄여서 불러온다.
option.inSampleSize = 4
val bitmap = BitmapFactory.decodeStream(inpoutStream, null, option)
```

<h3>2-2-2. 갤러리 앱 연동 방법</h3>
<ul>
  <li>
    인텐트로 갤러리 앱의 사진 목록을 띄우고 사용자가 선택한 사진을 읽어 화면에 출력하는 방법을 소개한다.
  </li>
  <li>
    인텐트로 갤러리 앱의 사진 목록을 출력하는 코드를 작성한다.
  </li>
    <ul>
      <li>
        인텐트의 액션 문자열은 <strong>Intent.ACTION_PICK</strong>, 데이터는 <strong>MediaStore.Images.Media.EXTERNAL_CONTENT_URI</strong>, 타입을 <strong>image/*</strong>로 지정한 인텐트를 전달한다.
      </li>
    </ul>
</ul>


```kotlin
// 2. inSampleSize에 비율을 지정하여 적절한 비율로 이미지 크기 줄이기
private fun calculateInSampleSize(fileUri: Uri, reqWidth: Int, reqHeight: Int): Int {
    val options = BitmapFactory.Options()
    
    // 옵션만 설정하고자 true로 지정한다.
    options.inJustDecodeBounds = true

    try {
        var inputStream = contentResolver.openInputStream(fileUri)
        
        // options의 inJustDecodeBounds가 위에서 true로 설정되었기 때문에 Bitmap 객체가 생성되지 않는다.
        // → 대신 읽은 이미지의 각종 정보가 option 객체에 설정된다.
        BitmapFactory.decodeStream(inputStream, null, options)
        inputStream!!.close()
        inputSteram = null
    } catch (e: Exception) {
        e.printStackTrace()
    }
    val (height: Int, width: Int) = options.run {outHeight to outWidth }
    var inSampleSize = 1

    // inSampleSize 비율 계산.
    if (height > reqHeight || width > reqWidth) {
        val halfHeight: Int = height / 2
        val halfWidth: Int = width / 2
        while (halfHeight / inSampleSize >= reqHeight && halfWidth / inSampleSize >= reqWidth) {
            inSampleSize *= 2
        }
    }
    return inSampleSize
}
```

<h2>2-3. 카메라 앱 연동하기</h2>
<ul>
  <li>
    카메라 앱을 연동하여 사진을 촬영하고 결과를 돌려받는 방식은 다음 두 가지이다.
  </li>
    <ul>
      <li>
        <strong>사진 데이터</strong>를 가져오는 방법
      </li>
        <ul>
          <li>
            카메라 앱으로 사진을 촬영한 후 파일로 저장하지 않고 <strong>데이터</strong>만 넘겨준다.
          </li>
          <li>
            데이터만 넘겨주기에 구현이 쉽지만 넘어오는 <strong>사진 데이터의 크기가 작다</strong>는 단점이 있다.
          </li>
        </ul>
      <li>
        사진 <strong>파일을 공유</strong>하는 방법
      </li>
        <ul>
          <li>
            촬영한 사진을 <strong>파일에 저장</strong>한 후 <strong>성공인지 실패</strong>인지만 넘겨준다.
          </li>
          <li>
            휴대폰의 카메라 성능 만큼의 <strong>큰 크기의 사진</strong>을 공유할 수 있지만 앱이 파일 정보를 공유하기 때문에 몇 가지 <strong>준비 작업</strong>이 필요하다.
          </li>
        </ul>
    </ul>
</ul>

<h3>2-3-2. 사진 파일을 공유하는 방법</h3>
<ul>
  <li>
    카메라 앱으로 촬영한 사진 파일을 공유하는 방법은 다음 절차를 따른다.
  </li>
    <ul>
      <li>
        앱에서 <strong>사진을 저장할 파일</strong>을 만든다.
        <br>→ 사진 파일 정보를 포함한 <strong>인텐트</strong>를 전달해 <strong>카메라 앱을 실행</strong>한다.
        <br>→ 카메라 앱으로 사진을 촬영하여 <strong>공유된 파일에 저장</strong>한다.
        <br>→ 카메라 앱을 종료하면서 <strong>성공 또는 실패</strong>를 반환한다.
        <br>→ 카메라 앱이 저장한 사진 파일을 앱에서 이용한다.
      </li>
    </ul>
  <li>
    사진 파일을 공유하는 방법을 이용하기 위해서는 먼저 앱에서 <strong>외장 메모리에 파일</strong>을 만들어 줘야한다.
  </li>
    <ul>
      <li>
        파일을 만들 때에는 <strong>getExternalFilesDir()</strong>을 이용한다.
      </li>
    </ul>
  <li>
    하지만 API 레벨 24 버전부터는 file:// 프로토콜로 구성된 URI를 외부에 노출하지 못하도록 하는 <strong>엄격 모드(strict mode)</strong>가 적용되어 앱끼리 파일을 공유하기 위해서는 <strong>content:// 프로토콜을 이용</strong>하고 임시로 접근할 수 있는 <strong>권한</strong>을 부여해야한다.
  </li>
    <ul>
      <li>
        권한을 부여할 때에는 <strong>FileProvider 클래스</strong>를 이용한다.
      </li>
        <ul>
          <li>
            FileProvider 클래스는 <strong>androidx 라이브러리</strong>에서 제공하며 XML 설정을 기반으로 <strong>content:// 프로토콜로 구성된 URI</strong>를 생성한다. 따라서 FileProvider를 이용하려면 <strong>공유할 파일의 URI</strong>값을 만들어야 한다.
          </li>
            <ul>
              <li>
                프로젝트의 res/xml 디렉터리에 파일 프로바이더용 XML 파일을 만든다.
              </li>
            </ul>
        </ul>
    </ul>
  <li>
    파일 프로바이더용 XML 파일은 path 속성에 지정한 경로의 <strong>파일 권한</strong>을 설정한다.
  </li>
    <ul>
      <li>
        이곳에 지정한 경로는 getExternalFilesDir() 함수로 파일을 만들었을 때 <strong>파일이 저장되는 위치</strong>이다.
      </li>
      <li>
        이후 매니페스트 파일에 파일 프로바이더용 XML 파일을 등록한다.
      </li>
    </ul>
</ul>

```xml
<!-- 1. res/xml에 파일 프로바이더용 XML 파일을 생성한다. -->
<paths xmlns:android="http://schemas.android.com/apk/res/android">
    <external-path name="myfiles" path="Android/data/com.example.test16/files/Pictures" />
</paths>
```

```xml
<!-- 2. 매니페스트 파일 프로바이더용 XML 파일 등록. -->
<provider
    android:name="androidx.core.content.FileProvider"
    android:authorities="com.example.test16.fileprovider"
    android:exported="false"
    android:grantUriPermissions="true">

    <!-- resource 속성으로 res/xml에 만들어 놓은 XML 파일을 지정한다. -->
    <meta-data
        android:name="android.support.FILE_PROVIDER_PATHS"
        android:resource="@xml/file_paths">
    </meta-data>
</provider>
```

```kotlin
// 3. 코드에서 카메라 앱을 연동하기 위해 파일을 생성한다.

    ".jpg",
    storageDir
)

// 나중에 파일 내용을 읽을 때 사용하기 위해 파일 경로를 filePath에 저장한다.

```


<h2>2-5. 전화 앱 연동하기</h2>
<ul>
  <li>
    전화를 거는 기능은 전화 앱의 액티비티에 이미 구현되어 있기 때문에 전화 앱과 연동해서 전화번호 데이터만 넘겨주면 된다.
  </li>
  <li>

  </li>
  <li>
    전화 앱과 연동하려면 인텐트의 액션 문자열을 <strong>Intent.ACTION_CALL</strong>로 지정한다.
  </li>
  <li>
    데이터의 정보는 <strong>tel:</strong>로 시작해야 하고 이어서 <strong>전화번호를 명시</strong>한다.
  </li>
</ul>

```xml
<!-- 1. 전화를 거는 퍼미션 설정. -->
<uses-feature
    android:name="android.hardware.telephony"
    android:required="false" />
<uses-permission android:name="android.permission.CALL_PHONE" />
```

```kotlin
// 2. 전화 앱을 실행하는 인텐트를 선언하여 시스템에 전달.

val intent = Intent(Intent.ACTION_CALL, Uri.parse("tel:02-120"))
```
