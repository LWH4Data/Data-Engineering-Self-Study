<h1>1. 브로드캐스트 리시버 이해하기</h1>
<ul>
  <li>
    브로드캐스트 리시버는 흔히 <strong>이벤트 모델</strong>로 실행되는 컴포넌트라고 정의한다. 
  </li>
    <ul>
      <li>
        여기서 이벤트란 사용자 이벤트가 아닌 부팅이 완료되는 것과 같은 <strong>시스템의 특정한 상황</strong>을 의미한다.
      </li>
    </ul>
  <li>
    브로드캐스트 리시버(broadcat receiver)는 줄여서 리시버라고도 하며 직역하면 방송 수신기라고도 할 수 있다.
  </li>
    <ul>
      <li>
        시스템에서 특정한 상황을 알리는 방송을 할 때 이를 받아서 처리하는 <strong>수신기를 앱에 장착</strong>한다고 생각할 수 있다.
      </li>
      <li>
        위의 수신기도 액티비티처럼 <strong>안드로이드의 컴포넌트</strong>이므로 <strong>인텐트</strong>를 시스템에 전달하여 실행한다.
      </li>
    </ul>
</ul>

<br>

<h2>1-1. 브로드캐스트 리시버 만들기</h2>
<ul>
  <li>
    브로드캐스트 리시버를 만들려면 BroadcastReceiver를 상속받는 클래스</strong>를 선언해야 한다.
  </li>
  <li>
    브로드캐스트 리시버의 생명주기 함수는 onReceiver() 함수가 자동으로 호출된다.
  </li>
    <ul>
      <li>
        어디선가 이 리시버를 실행하려고 인텐트를 시작하면 <strong>onReceive() 함수가 자동 호출</strong>된다. 이후 호출한 <strong>인텐트 객체</strong>를 <strong>매개변수로 전달받는다</strong>.
      </li>
    </ul>
  <li>
    onReceive() 함수는 실행한 후 <strong>10초 이내로 완료</strong>할 것을 권장하기 때문에 오래 걸리는 작업을 담기에 부적절하다.
  </li>
  <li>
    onReceive() 함수가 실행을 마치면 브로드캐스트 리시버 객체는 <strong>소멸</strong>한다.
  </li>
  <li>
    브로드캐스트 리시버도 컴포넌트이기 때문에 <strong>매니페스트 파일</strong>에 등록한다.
  </li>
    <ul>
      <li>
        브로드캐스트 리시버를 등록하는 태그는 <strong>receiver</strong>이며 필수 속성은 <strong>클래스명을 지정하는 name</strong> 이다.
      </li>
      <li>
        브로드캐스트 리시버를 명시적 인탠트로 실행하려면 <strong>클래스명</strong>만 등록하고, 암시적 인텐트로 실행하려면 <strong>intent-filter</strong>를 선언해 주어야 한다.
      </li>
    </ul>
  <li>
    브로드캐스트 리시버 객체는 <strong>액티비티</strong>나 <strong>서비스</strong>에서 생성할 수 있다.
  </li>
</ul>

```kotlin
// 1. 브로드캐스트 리시버 만들기.
class MyReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
    }
}
```

```xml
<!-- 2. 브로드캐스트 리시버 등록. -->
<receiver
    android:name=".MyReceiver"
    android:enabled="true"
    android:exported="true">
</receiver>
```

<h2>1-2. 동적 등록과 해제</h2>
<ul>
  <li>
    안드로이드 컴포넌트는 <strong>매니페스트에</strong> 등록해야 시스템에서 인지하고 어디선가 인텐트가 발생할 때 컴포넌트를 실행하는 것이 일반이지만 브로드캐스트 리시버는 매니페스트에 등록하지 않고도 <strong>코드에서 필요한 순간 동적으로 등록</strong>할 수도 있다.
  </li>
  <li>
    브로드캐스트 리시버 객체는 <strong>액티비티</strong>나 <strong>서비스</strong>에서 생성할 수 있다.
  </li>
    <ul>
      <li>
        이후 필요한 순간에 <strong>registerReceiver() 함수</strong>를 통해 시스템에 등록한다.
      </li>
      <li>
        registerReceiver()로 브로드캐스트 리시버를 등록하려면 <strong>등록하고자 하는 브로드캐스트 리시버 객체</strong>와 <strong>인텐트 필터 정보</strong> 그리고 <strong>공개상태 정보</strong>를 명시해야 한다.
      </li>
        <ul>
          <li>
            공개상태 정보는 <strong>RECEIVER_EXPORTED</strong>와 <strong>RECEIVER_NOT_EXPORTED</strong> 중 하나를 지정한다.
          </li>
            <ul>
              <li>
                암시적 인텐트에 의해 실행되려면 <strong>RECEIVER_EXPORTED</strong>를 지정한다.
              </li>
            </ul>
        </ul>
    </ul>
  <li>
    registerReceiver() 함수로 등록한 리시버는 사용한 후 필요 없으면 <strong>해제</strong>해 줘야 한다. 해제할 때에는 <strong>unregisterReciever() 함수</strong>를 이용한다.
  </li>
  <li>
    특정한 상황에 <strong>항상 브로드캐스트 리시버</strong>를 실행해야 한다면 <strong>매니페스트 파일</strong>에 등록해서 사용한다. 하지만 <strong>특정한 액티비티나 서비스</strong>가 동작할 때만 브로드캐스트 리시버를 실행해야 한다면 컴포넌트가 실행될 때 <strong>동적으로 등록</strong>하는 방법을 사용한다.
  </li>
</ul>

```kotlin
// 1. 액티비티나 서비스 컴포넌트에서 리시버 객체 생성.
val receiver = object : BroadcastReceiver() {
    override fun onReceive(context: Context?, intent: Intent?) {
    }
}
```

```kotlin
// 2. 동적 등록.
registerReceiver(
    receiver, 
    // 암시적 인텐트로 선언.
    IntentFilter("ACTION_RECEIVER"),
    // 암시적 인텐트이기 때문에 RECEIVER_EXPORTED로 지정.
    RECEIVER_EXPORTED)
```

<h2>1-3. 브로드캐스트 리시버 실행하기</h2>
<ul>
  <li>
    브로드캐스트 리시버를 실행하기 위해서는 <strong>인텐트</strong>가 필요하다.
  </li>
    <ul>
      <li>
        리시버의 <strong>클래스명만</strong> 등록했으면 클래스 타입 레퍼런스를 이용해 <strong>명시적 인텐트</strong>로 실행한다.
      </li>
      <li>
        인텐트 필터로 등록했으면 <strong>암시적 인텐트</strong>로 실행한다.
      </li>
    </ul>
  <li>
    만약 리시버를 매니페스트 파일에 등록하고 <strong>intent-filter 태그</strong>를 선언했다면 암시적 인텐트로는 실행할 수 없다.
  </li>
    <ul>
      <li>
        코드에서 registerReceiver() 함수로 등록한 리시버는 암시적 인텐트로도 잘 실행된다.
      </li>
    </ul>
  <li>
    브로드캐스트 리시버를 실행하는 인텐트는 <strong>sendBroadcast() 함수</strong>로 시스템에 전달한다.
  </li>
  <li>
    액티비티 인텐트는 시스템에 해당 인텐트로 실행될 액티비티가 없으면 인텐트를 시작한 곳에서 <strong>오류</strong>가 발생하고, 실행될 액티비티가 여러 개가 있다면 사용자 선택으로 <strong>하나만 실행</strong>된다.
  </li>
    <ul>
      <li>
        반면 브로드캐스트 리시버는 인텐트로 실행될 리시버가 없으면 <strong>아무런 일도</strong> 일어나지 않고, 리시버가 여러 개 있으면 <strong>모두 실행</strong>된다.
      </li>
        <ul>
          <li>
            위와 같은 독특한 동작 때문에 브로드캐스트 리시버를 사용한다.
          </li>
        </ul>
    </ul>
</ul>

```kotlin
// 1. 리시버 실행.
val intent = Intent(this, MyReceiver::class.java)
// 인텐트를 sendBroadcast() 함수로 전달.
sendBroadcast(intent)
```

<h1>2. 시스템 상태 파악하기</h1>
<ul>
  <li>
    시스템에서 발생하는 인텐트는 부팅 완료, 화면 켬/끔, 배터리 상태 등이 대표적이다. 
  </li>
</ul>

<br>

<h2>2-1. 부팅 완료</h2>
<ul>
  <li>
    앱에서 부팅이 완료될 때 특정한 작업을 수행하고 싶다면 브로드캐스트 리시버를 만들고 <strong>매니페스트 파일</strong>에 <strong>인텐트 필터</strong>를 구성해 등록한다.
  </li>
  <li>
    부팅이 완료되면 시스템에서 <strong>android:intent.action.BOOT_COMPLETED 액션 문자열</strong>을 포함하는 <strong>인텐트</strong>가 발생한다. 
  </li>
    <ul>
      <li>
        실행할 리시버에는 <strong>action 태그</strong>의 <strong>name 속성</strong>에 동일한 액션 문자열을 등록한다.
      </li>
    </ul>
  <li>
    리시버를 실행하려면 권한이 필요하기에 매니페스트 파일에 퍼미션을 추가한다.
  </li>
</ul>

```xml
<!-- 1. 브로드캐스트 리시버와 인텐트 필터 등록. -->
<receiver
    android:name=".MyReceiver"
    android:enabled="true"
    android:exported="true">
    
    <!-- name 속성에 동일한 액션 문자열을 등록한다. -->
    <intent-filter>
        <action android:name="android.intent.action.BOOT_COMPLETED" />
    </intent-filter>
</receiver>
```

```xml
<!-- 2. 리시버 실행을 위한 퍼미션 설정. -->
<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
```

<h2>2-2. 화면 켬/끔</h2>
<ul>
  <li>
    화면을 켜거나 끄는 상황을 감지하는 브로드캐스트 리시버는 매니페스트에 등록하면 실행되지 않기 때문에 <strong>액티비티</strong>나 <strong>서비스 컴포넌트 코드</strong>에서 <strong>registerReceiver() 함수</strong>를 이용해 동적으로 등록해야 한다.
  </li>
  <li>
    화면을 끄는 것과 켜는 것의 브로드캐스트 리시버를 따로 만들 수 있지만 하나의 리시버에서 켜고 끄는 상황을 모두 감지하도록 할 수도 있다.
  </li>
</ul>

```kotlin
// 1. 화면 켬/끔 리시버.
receiver = object : BroadcastReceiver() {
    override fun onReceive(context: Context?, intent: Intent?) {
        when (intent?.action) {
            
            // android.intent.action.SCREEON_ON을 통해 켤 때 설정.
            Intent.ACTION_SCREEN_ON -> Log.d("kkang", "screen on")

            // android.intent.action.SCREEN_OFF을 통해 끌 때 설정.
            Intent.ACTION_SCREEN_OFF -> Log.d("kkang", "screen off")
        }
    }
}
```

```kotlin
// 2. 1에서 정의한 브로드캐스트 리시버를 등록.
val filter = IntentFilter(Intent.ACTION_SCREEN_ON).apply {
    addAction(Intent.ACTION_SCREEN_OFF)
}

// 액션 문자열 정보로 구성한 인텐트 필터와 함께 registerReceiver() 함수로 시스템에 등록한다.
registerRecevier(receiver, filter)

// registerReceiver() 함수로 등록한 브로드캐스트 리시버는 필요 없는 순간이 되면 unregisterReceiver() 함수로 등록을 해제해 줘야 한다.
unregesterReceiver(receiver)
```

<h2>2-3. 배터리 상태</h2>
<ul>
  <li>
    안드로이드 시스템에서 배터리 상태가 변경되면 다음 액션 문자열로 인텐트가 발생한다.
  </li>
    <ul>
      <li>
        BATTERY_LOW: 배터리가 낮은 상태로 변경되는 순간.
      </li>
      <li>
        BATTERY_OKAY: 배터리가 정산 상태로 변경되는 순간.
      </li>
      <li>
        BATTERY_CHANGED: 충전 상태가 변경되는 순간.
      </li>
      <li>
        ACTION_POWER_CONNECTED: 전원이 공급되기 시작한 순간.
      </li>
      <li>
        ACTION_POWER_DISCONNECTED: 전원 공급을 끊은 순간.
      </li>
    </ul>
  <li>
    브로드캐스트 리시버는 시스템에서 상태가 변경되었을 때 인텐트를 발생시켜 줘야만 실행된다.
  </li>
    <ul>
      <li>
        만약 시스템 상태가 변경되지 않았지만 배터리 정보가 필요한 경우에는 배터리와 관련된 정보를 registerReceiver() 함수의 반환값인 인텐트 객체 안에 존재하는 <strong>extra 정보를 꺼내서 사용</strong>한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 배터리 상태 리시버 작성.
receiver = object : BroadcastReceiver() {
    override fun onReceive(context: Context?, intent: Intent?) {
        when(intent?.action) {
            intent.ACTION_BATTERY_OKAY -> Log.d("kkang", "ACTION_BATTERY_OKAY")
            intent.ACTION_BATTERY_LOW -> Log.d("kkang", "ACTION_BATTERY_LOW")
            intent.ACTION_BATTERY_CHANGED -> Log.d("kkang", "ACTION_BATTERY_CHANGED")
            intent.ACTION_POWER_CONNECTED -> Log.d("kkang", "ACTION_POWER_CONNTECTED")
            intent.ACTION_POWER_DISCONNECTED -> Log.d("kkang", "ACTION_POWER_DISCONNECTED")
        }
    }
}
```

```kotlin
// 2. 작성한 리시버 등록.
val filter = IntentFilter(Intent.ACTION_BATTERY_LOW).apply {
    addAction(Intent.ACTION_BATTERY_OKAY)
    addAction(Intent.ACTION_BATTERY_CHANGED)
    addAction(Intent.ACTION_POWER_CONNECTED)
    addAction(Intent.ACTION_POWER_DISCONNECTED)
}
registerReceiver(receiver, filter)
```

``` kotlin
// 3. 시스템 인텐트 없이 배터리 상태 파악하기. (이미 시스템이 보낸 배터리 상태 데이터를 인텐트로 가져온다).
val intentFilter = IntentFilter(Intent.ACTION_BATTERY_CHANGED)
val batteryStatus = registerReceiver(null, intentFilter)
```

```kotlin
// 4. 인텐트의 엑트스라를 이용해 배터리 상태 파악하기.
val status = batteryStatus!!.getIntExtra(BatteryManager.EXTRA_STATUS, -1)
if (status == BatteryManager.BATTERY_STATUS_CHARGING) {

    // 전원이 공급되고 있다면
    //   - BattertyManager.EXTRA_STATUS로 얻은 엑스트라값으로 배터리 상태를 알 수 있다.
    val chargePlug = batteryStatus.getIntExtra(BatteryManager.EXTRA_PLUGGED, -1)
    when (chargePlug) {
        
        // BatteryManager.BATTERY_PLUGGED_USB는 저속 충전 상태를 의미한다.
        BatteryManager.BATTERY_PLUGGED_USB -> Log.d("kkang", "usb charge")

        // BatteryManager.BATTERY_BLUGGED_AC는 고석 충전 상태를 의미한다.
        BatteryManager.BATTERY_PLUGGED_AC -> Log.d("kkang", "ac charge")
    } else {
        // 전원이 공급되고 있지 않다면
        Log.d("kkang", "not charging")
    }
}
```

```kotlin
// 5. 배터리 충전량을 퍼센트로 출력하기.
val level = batteryStatus.getIntExtra(BatteryManager.EXTRA_LEVEL, -1)
val scale = batteryStatus.getIntExtra(BatteryManager.EXTRA_SCALE, -1)
val batteryPct = level / scale.toFloat() * 100
Log.d("kkang", "batteryPct : $batteryPct")
```
