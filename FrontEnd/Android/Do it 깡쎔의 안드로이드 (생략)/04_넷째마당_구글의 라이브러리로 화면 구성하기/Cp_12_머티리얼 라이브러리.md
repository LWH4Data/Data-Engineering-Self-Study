<h1>1. 앱바 사용하기</h1>
<h2>1-1. 머티리얼 라이브러리란?</h2>
<ul>
  <li>
    구글의 머티리얼 디자인(material design)은 모바일과 데스크톱, 그리고 그 밖의 다양한 장치를 아우르는 일관된 <strong>애플리케이션 디자인 지침</strong>이다.
  </li>
  <li>
    머티리얼 라이브러리를 사용하려면 빌드 그래들 파일의 dependencies 항목에 선언을 해주어야 하지만 안드로이드 스튜디오 4.1 버전부터는 자동으로 라이브러리가 선언되며 그만큼 많이 사용된다.
  </li>
    <ul>
      <li>
        implementation(libs.material)
      </li>
    </ul>
</ul>

<br>

<h2>1-2. 앱바 레이아웃 - 화면 위쪽 영역 꾸미기</h2>
<ul>
  <li>
    앱바(AppBar)란 <strong>화면 위쪽</strong>의 꾸밀 수 있는 영역을 의미한다.
  </li>
  <li>
    앱바를 통해 단순히 위쪽 영역의 크기만 늘릴 수도 있고 툴바를 포함해 이미지나 문자열 등을 함께 출력할 수도 있다.
  </li>
  <li>
    머티리얼 라이브러리는 앱바를 구현하는 데 도움을 주는 앱바 레이아웃을 제공하며 앱바 레이아웃도 하나의 <strong>뷰</strong>이기 때문에 <strong>레이아웃 XML 파일에 등록</strong>한다.
  </li>
</ul>

<h3>1-2-1. 툴바 포함하기</h3>
<ul>
  <li>
    앱바를 사용할 때에는 대부분 앱바 레이아웃(AppBarLayout)에 툴바를 포함한다.
  </li>
</ul>

```xml
<!-- 1. 앱바 레이아웃에 툴바 포함. -->
<com.google.android.material.appbar.AppBarLayout
    android:id="@+id/appbar"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:theme="@style/ThemeOverlay.AppCompat.ActionBar">

    <androidx.appcompat.widget.Toolbar
        (... 생략 ...)
    />
</com.google.android.material.appbar.AppBarLayout>
```

<h3>1-2-2. 크기 확장하기</h3>
<ul>
  <li>
    앱바 레이아웃을 사용하면 툴바뿐만 아니라 여러 가지 콘텐츠를 함께 표시할 수 있다.
  </li>
</ul>

```xml
<!-- 1. 위쪽 영역 확장하기. -->
<com.google.android.material.appbar.AppBarLayout
    (... 생략 ...)
    android:layout_height="242dp">
</com.google.android.materital.appbar.AppBarLayout>
```

<h3>1-2-3. 이미지 넣기</h3>
<ul>
  <li>
    이미지를 포함한 다양한 콘텐츠도 함께 출력할 수 있다.
  </li>
</ul>

```xml
<!-- 1. 앱바에 이미지 표시. -->
<com.google.android.material.appbar.AppBarLayout ... 생략 ...>
    <androidx.appcompat.widget.Toolbar ... 생략 ... />
    <ImageView ... 생략 ... />
</com.google.android.material.appbar.AppBarLayout>
```

<h2>1-3. 코디네이터 레이아웃 - 뷰끼리 상호 작용하기</h2>
<ul>
  <li>
    코디네이터 레이아웃(CoordinatorLayout)은 머티리얼이 아닌 제트팩의 <strong>androidx 라이브러리</strong>에서 지공하지만 앱바 레이아웃에서 가장 많이 사용하기 때문에 여기서 설명한다.
  </li>
</ul>

<h3>1-3-1. 스크롤 연동하기</h3>
<ul>
  <li>
    코디네이터 레이아웃은 <strong>뷰끼리 상호 작용</strong>해야 할 때 사용한다.
  </li>
  <li>
    뷰 두 개를 코디네이터 레이아웃에 넣으면 뷰에서 발생한 스크롤 정보를 코디네이터 레이아웃이 받아서 <strong>다른 뷰에 전달</strong>한다.
  </li>
  <li>
    코디네이터 레이아웃에 뷰만 추가한다고 해서 모든 뷰의 스크롤 정보를 공유할 수 있는 것은 아니다.
  </li>
    <ul>
      <li>
        자식 뷰끼리 상호작용하기 위해서는 누군가는 <strong>코디네이터 레이아웃에 정보를 전달</strong>하고, 다른 누군가는 <strong>정보를 받을 수</strong> 있어야 한다. 이 역할을 <strong>비헤이비어(behavior)</strong>라고 한다.
      </li>
    </ul>
  <li>
    코디네이터 레이아웃으로 스크롤을 연동할 수 있는 뷰는 목록을 표현하기 위한 <strong>리사이클러 뷰</strong>와 앞서 소개한 <strong>앱바 레이아웃</strong>이 대표적이다.
  </li>
  <li>
    앱바 레이아웃으로 화면 위쪽 영역을 꾸미면 메인 콘텐츠의 영역이 작아진다.
    <br>→ 따라서 메인 콘텐트가 나오는 부분을 스크롤할 때 앱바 레이아웃도 함께 스크롤되도록 만든다.
    <br>→ 이때 코디네이터 레이아웃을 사용한다.
  </li>
</ul>

<h3>1-3-2. 증첩 스크롤 뷰 사용하기</h3>
<ul>
  <li>
    코디네이터 레이아웃은 텍스트 뷰, 이미지 뷰 등은 스크롤을 연동할 수 없으며 만약 연동이 필요하다면 <strong>androidx.core.widget.NestedScrollView</strong>를 이용한다.
  </li>
    <ul>
      <li>
        코디네이터 레이아웃에 <strong>중첩 스크롤 뷰(NestedScrollView)</strong>를 포함하고 여기에 텍스트 뷰나 이미지 뷰를 넣는 방식이다.
      </li>
        <ul>
          <li>
            <strong>app:layout_behavior 속성</strong>은 자신의 스크롤 정보를 <strong>어느 behavior 클래스</strong>가 받아 처리해야하는지를 의미한다.
          </li>
            <ul>
              <li>
                코디네이터 레이아웃이 중첩 스크롤 뷰 정보를 앱바 레이아웃의 <strong>ScrollingViewBehavior 클래스에 전달</strong>한다.
              </li>
            </ul>
          <li>
            <strong>app:layout_scrollFlags 속성</strong>이 설정된 뷰가 스크롤 정보를 <strong>수신</strong>해서 함께 스크롤된다.
          </li>
            <ul>
              <li>
                대부분은 개별 뷰보다 <strong>컬랩싱 툴바 레이아웃</strong>에 추가한다.
              </li>
            </ul>
        </ul>
    </ul>
</ul>

```xml
<!-- 1. 중첩 스크롤 뷰. -->
<androidx.coordinatorlayout.widget.CoordinatorLayout ... 생략 ...>
    <com.google.android.material.appbar.AppBarLayout ... 생략 ...>
        <androidx.appcompat.widget.Toolbar ... 생략 ...
            app:layout_scrollFlags="scroll|enterAlways" />
        <ImageView ... 생략 ...
            app:layout_scrollFlags="scroll|enterAlways" />
    </com.google.android.material.appbar.AppBarLayout>
    <androidx.core.widget.NestedScrollView ... 생략 ...
        app:layout_behavior="@string/appbar_scrolling_view_behavior">
        <TextView ... 생략 ... />
    </androidx.core.widget.NestedScrollView>
</androidx.coordinatorlayout.widget.CorrdinatorLayout>
```

<h2>1-4. 컬랩싱 툴바 레이아웃 - 앱바 접히는 형태 설정하기</h2>
<ul>
  <li>
    컬랩싱 툴바 레이아웃(CollapsingToolbarLayout)은 앱바 레이아웃 하위에 선언하여 <strong>앱바가 접힐 때 다양한 설정</strong>을 할 수 있는 뷰이다.
  </li>
  <li>
    가장 중요한 설명은 앱바를 <strong>어떻게 접을 것인가</strong>이다. 여러 개의 뷰를 추가한 경우 개별 뷰에 app:layout_scrollFlags 속성을 지정하는 것은 효율적이지 못하다.
    <br>→ 보통은 <strong>앱바 레이아웃 하위</strong>에 <strong>CollapsingToolbarLayout을 추가</strong>하여 앱바가 스크롤되거나 접힐 때 어떻게 동작해야 하는지 설정한다.
  </li>
</ul>

```xml
<!-- 1. 컬랩싱 툴바 레이아웃 등록. -->
<androidx.coordinatorlayout.widget.CoordinatorLayout ... 생략 ...>
    <com.google.android.material.appbar.AppBarLayout ... 생략 ...>
        <com.google.android.material.appbar.CollapsingToolbarLayout ... 생략 ...
            app:contentScrim="?attr/colorPrimary"
            app:expandedTitleMarginBottom="50dp"
            app:expandedTitleMarginStart="48dp"
            app:layout_scrollFlags="scroll|exitUntilCollapsed"
            app:title="AppBar Title">

            <ImageView ... 생략 ...
                app:layout_collapseMode="parallax" />
            
            <androidx.appcompat.widget.Toolbar ... 생략 ...
                app:layout_collapseMode="pin" />
            
        </com.google.android.material.appbar.CollapsingToolbarLayout>
    </com.google.android.material.appbar.AppBarLayout>
    <androidx.recyclerview.widget.RecyclerView ... 생략 ...
        app:layout_behavior="@string/appbar_scrolling_view_behavior" />
</androidx.coordinatorlayout.widget.CoordinatorLayout>
```

<h3>1-4-1. 스크롤 설정하기</h3>
<ul>
  <li>
    컬랩싱 툴바 레이아웃에서 앱바가 스크롤될지 설정할 때에는 <strong>layout_scroll_Flags</strong>를 사용한다.
  </li>
    <ul>
      <li>
        스크롤 되어야 한다면 <strong>scroll값</strong>을 지정하고 <strong>| 연산자</strong>를 추가해 스크롤 시 어떻게 움지역야 하는지 설정한다.
      </li>
        <ul>
          <li>
            <strong>scroll | enterAlways</strong>: 스크롤 시 완전히 사라졌다가 거꾸로 스크롤 시 <strong>처음부터 다시</strong> 나타난다.
          </li>
          <li>
            <strong>scroll | enterAlwaysCollapsed</strong>: 스크롤 시 완전히 사라졌다가 거꾸로 스크롤 시 <strong>메인 콘텐츠 부분이 끝까지</strong> 스크롤된 다음에 나타난다.
          </li>
          <li>
            <strong>scroll | exitUntilCollapsed</strong>: 스크롤 시 모두 사라지지 않고 <strong>툴바를 출력할 정보의 한 줄</strong>만 남을 때까지 스크롤된다.
          </li>
        </ul>
    </ul>
</ul>

<h3>1-4-2. 개별 뷰의 스크롤 설정하기</h3>
<ul>
  <li>
    <strong>layout_collapseMode</strong> 속성은 앱바를 스크롤할 때 앱바에 포함한 <strong>개별 뷰</strong>가 어떻게 움직여야 하는지를 설정한다.
  </li>
    <ul>
      <li>
        <strong>앱바 전체</strong>의 스크롤 설정은 컬랩싱 툴바 레이아웃의 <strong>layout_scrollFlags</strong> 속성으로, <strong>하위 뷰마다</strong> 스크롤 설정은 <strong>layout_collapseMode</strong> 속성으로 한다.
      </li>
      <li>
        layout_collapseMode 속성에는 <strong>pin</strong>과 <strong>parallax</strong>값을 지정한다.
      </li>
        <ul>
          <li>
            <strong>pin</strong>: 고정되어 스크롤되지 않는다. 즉, 툴바는 남아있는다.
          </li>
          <li>
            <strong>parallax</strong>: 함께 스크롤된다. 즉, 함께 스크롤되며 더 자연스러운 움직임 구현이 가능하다.
          </li>
        </ul>
    </ul>
</ul>

```xml
<!-- 1. 개별 뷰의 스크롤 설정. -->
<com.google.android.material.appbar.CollapsingToolbarLayout ... 생략 ...
    app:layout_scrollFlags="scroll|exitUntilCollapsed">

    <ImageView ... 생략 ...
        app:layout_collapseMode="parallax" />
    
    <androidx.appcompat.widget.Toolbar ... 생략 ...
        app:layout_collapseMode="pin" />
</com.google.android.material.appbar.CollapsingToolbarLayout>
```

<h1>2. 탭 레이아웃 - 탭 버튼 구성</h1>
<ul>
  <li>
    탭 레이아웃은 탭(tab)으로 구분하는 화면에서 <strong>탭 버튼을 배치</strong>하는 레이아웃이다. 
  </li>
  <li>
    탭 레이아웃의 탭 버튼을 동적으로 만들 필요가 없다면 탭 버튼을 코드에 정의하지 않고 레이아웃 XML 파일의 TabItem으로 정의해도 된다.
  </li>
  <li>
    사용자가 탭 버튼을 선택할 때 출력해야하는 내용은 코드에서 <strong>탭 버튼의 이벤트 핸들러</strong>에 명시한다.
  </li>
  <li>
    TabLayout은 기본적으로 탭 버튼을 <strong>가로 등분</strong>으로 배치한다.
  </li>
</ul>

```xml
<!-- 1. 탭 레이아웃 등록. -->
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android.layout_width="match_parent"
    android.layout_height="match_parent"
    android.orientation="vertical">

    <!-- 사용자가 TabLayout으로 구성한 탭 버튼을 선택하면 -->
    <com.google.android.material.tabs.TabLayout
        android:id="@+id/tabs"
        android:layout_width="match_parent"
        android:layout_height="wrap_content" />
    
    <!-- FrameLayout 위치에 탭의 내용을 출력한다. -->
    <FrameLayout
        android:id="@+id/tabContent"
        android:layout_width="match_parent"
        android:layout_height="match_parent" />
</LinearLayout>
```

```kotlin
// 2. 코드에서 탭 버튼 정의.
binding.tabs.run {

    // TabLayout.newTab() 함수로 만들어지는 TabLayout.Tab 객체가 탭 버튼이다.
    //   - text 속성으로 문자열을, icon 속성으로 이미지를 작성한다.
    val tab1: TabLayout.Tab = newTab()
    tab1.text="Tab1"
    // 생성한 버튼을 TabLayout.addTab() 함수의 매개변수로 지정하여 탭 버튼을 추가한다.
    addTab(tab1)

    val tab2: TabLayout.Tab = newTab()
    tab2.text="Tab2"
    addTab(tab2)

    val tab3: TabLayout.Tab = newTab()
    tab3.text="Tab3"
    addTab(tab3)
}
```

```xml
<!-- 3. XML 파일에서 탭 버튼 정의. -->
<com.google.android.material.tabs.TabLayout
    android:id="@+id/tabs"
    android:layout_width="match_parent"
    android:layout_height="wrap_content">

    <com.google.android.material.tabs.TabItem
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Tab1" />

    <com.google.android.material.tabs.TabItem
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Tab2" />

    <com.google.android.material.tabs.TabItem
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Tab3" />
</com.google.android.material.tabs.TabLayout>
```

```kotlin
// 4. 탭 버튼 이벤트 처리.
// TabLayout의 addOnTabSelectedListener() 함수를 이용해 이벤트 핸들러 객체를 지정한다.
//   - 탭 버튼의 이벤트 핸들러 객체는 TabLayout.OnTabSelectedListener를 구현한 객체여야 한다.
binding.tabs.addOnTabSelectedListener(object: TabLayout.OnTabSelectedListener {
    
    // 탭 버튼을 선택할 때 이벤트.
    override fun onTabSelected(tab: TabLayout.Tab?) {
        val transaction = supportFragmentManager.beginTransacton()
        when (tab?.text) {
            "Tab1" -> transaction.replace(R.id.tabContent, OneFragment())
            "Tab2" -> transaction.replace(R.id.tabContent, TwoFragment())
            "Tab3" -> transaction.replace(R.id.tabContent, ThreeFragment())
        }

        // 선택된 탭 버튼을 다시 선택할 때 이벤트.
        override fun onTabReselected(tab: TabLayout.Tab?) {

        }

        // 다른 탭 버튼을 눌러 선택된 탭 버튼이 해제될 때 이벤트
        override fun onTabUnselected(tab: TabLayout.Tab?) {

        }
    }
})
```

<h3>2-1-1. 탭 버튼 정렬하기</h3>
<ul>
  <li>
    tabGravity는 탭 버튼을 정렬하는 속성이다. 
  </li>
    <ul>
      <li>
        기본값은 fill이며 탭 버튼을 가로로 등분하여 배치한다.
      </li>
      <li>
        center는 탭 버튼을 가운데 정렬한다.
      </li>
    </ul>
</ul>

<h3>2-1-2. 스크롤 설정하기</h3>
<ul>
  <li>
    tabMode 속성은 탭 버튼을 스크롤할 수 있는지를 설정한다.
  </li>
    <ul>
      <li>
        기본값은 fixed이며 스크롤을 지원하지 않는다.
      </li>
      <li>
        scrollable로 설정하면 탭 버튼이 왼쪽부터 나열되고 모두 출력할 수 없다면 자동으로 가로 스크롤이 생긴다.
      </li>
    </ul>
</ul>

<h3>2-1-3. 뷰 페이저 연동하기</h3>
<ul>
  <li>
    탭 레이아웃과 뷰 페이저는 둘 다 화면을 여러 장 제공하는 용도이기 때문에 둘을 연동하는 일이 많다.
  </li>
    <ul>
      <li>
        탭 버튼으로 화면을 전환하는 기능을 제공하면서 동시에 뷰 페이저로 스와이프를 통해 화면을 전환하도록 기능을 제공할 수 있다.
      </li>
    </ul>
</ul>


<h1>3. 내비게이션 뷰 - 드로어 화면 구성</h1>
<ul>
  <li>
    <strong>내비게이션 뷰(NavigationView)</strong>는 <strong>드로어 레이아웃</strong>으로 열린 화면에 출력되는 내용을 구성할 때 사용한다. 보통은 <strong>메뉴 항목을 나열하는 형태</strong>로 구성한다.
  </li>
  <li>
    내비게이션 뷰로 화면을 구성할 때에는 <strong>위쪽에 적용할 레이아웃 XML 파일</strong>과 아래쪽에 적용할 <strong>메뉴 XML 파일</strong>을 만든다.
  </li>
  <li>
    항목 선택 이벤트는 DrawerLayout의 <strong>setNavigationItemSelectedListener() 함수</strong>로 이벤트 핸들러를 지정하여 처리한다.
  </li>
</ul>

```xml
<!-- 1. 내비게이션 뷰 등록. -->
<androidx.drawerlayout.widget.DrawerLayout ... 생략 ...>
    <LinearLayout ... 생략 ...>
        (... 생략 ...)
    </LinearLayout>
    
    <!-- headerLayout 속성으로 내비게이션 바의 위쪽 출력을, menu 속성으로 아래쪽 출력할 항목을 지정함. -->
    <com.google.android.material.navigation.NavigationView
        andorid:id="@+id/main_drawer_view"
        android:layout_width="wrap_content"
        android:layout_height="match_parent"
        nadroid:layout_gravity="start"
        app:headerLayout="@layout/navigation_header"
        app:menu="@menu/menu_navigation" />
</androidx.drawerlayout.widget.DrawerLayout>
```

```kotlin
// 2. 항목 선택 이벤트 핸들러.
```

<h1>4. 확장된 플로팅 액션 버튼</h1>
<ul>
  <li>
    확장된 플로팅 액션 버튼(ExtendedFloatingActionButton)은 <strong>화면에 떠 있는 듯한 버튼</strong>을 제공하는 뷰이다.
  </li>
  <li>
    확장된 플로팅 액션 버튼의 <strong>icon 속성</strong>으로 이미지를 지정하면 <strong>이미지를 포함</strong>해 화면에 떠 있는 듯한 버튼이 출력된다.
  </li>
    <ul>
      <li>
        <strong>text 속성</strong>으로 <strong>문자열</strong>을 지정하면 <strong>이미지와 함께</strong> 출력된다.
      </li>
      <li>
        text 속성을 지정하지 않고 icon 속성만 지정하면 <strong>둥근 모양</strong>이 출력된다.
      </li>
      <li>
        코드에서는 문자열까지 나오게 확장하거나 아이콘만 나오게 축소하는 등 <strong>버튼의 출력 형태</strong>를 조절할 수 있다.
      </li>
    </ul>
</ul>

```xml
<!-- 1. 확장된 플로팅 액션 버튼. -->
<com.google.android.material.floatingactionbutton.ExtendedFloatingActionButton
    (... 생략 ...)
    android:text="extended FAB"
    app:icon="@android:drawable/ic_input_add" />
```

```kotlin
// 2. 확장된 플로팅 액션 버튼 조절.
binding.extendedFab.setOnClickListener {
    
    // 아이콘과 문자열을 함께 표시했다면
    when (binding.extendedFab.isExtended) {
        
        // 아이콘만 표시.
        true -> binding.extendedFab.shrink()
        
        // 함께 표시.
        false -> binding.extendedFab.extend()
    }
}
```
