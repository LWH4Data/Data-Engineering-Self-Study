<h1>1. API 레벨 호환성 고려하기</h1>
<ul>
  <li>
    앱을 개발할 때에 minSdk보다 상위 버전에서 제공하는 API를 사용할 경우 <strong>호환성(compatibility)</strong>을 고려해야 한다.
  </li>
  <li>
    API 레벨 호환성에 문제가 있는 API를 사용할 때에는 <strong>@RequiresApi</strong> 혹은 <strong>@TargetApi</strong> 애너테이션을 추가해 <strong>오류가 발생하지 않도록</strong>할 수 있다.
  </li>
    <ul>
      <li>
        오류가 발생하지 않을 뿐 호환성 문제는 남아있기 때문에 이에 대해서는 <strong>분기처리</strong>가 필요하다.
      </li>
    </ul>
  <li>
    최신 디바이스에서 필요한 기능이나 혹은 특정 기능을 추가할 때 함께 추가해야할 기능들이 있기 때문에 API 호환성은 계속 고려가 될 수밖에 없다.
  </li>
</ul>

```kotlin
// 1. API 호환성 애너테이션 1 (@RequiresApi).
@RequiresApi(Build.VERSION_CODES.S)
fun noti() {
    (... 생략 ...)
    val builder: Notification.Builder = Notification.Builder(this, "1")
        .setStyle(
            Notification.CallStyle.forIncomingCall(caller, declineIntent, answerIntent)
        )
    (... 생략 ...)
}

// 2. API 호환성 애너테이션 2 (@TargetApi).
@TargetApi(Build.VERSION_CODES.S)
fun noti() {
    (... 생략 ...)
    val builder: Notification.Builder = Notification.Builder(this, "1")
        .setStyle(
            Notification.CallStyle.forIncomingCall(caller, declineIntent, answerIntent)
        )
    (... 생략 ...)
}

// 3. 호환성에 따른 분기처리.
// API 30레벨 이상에서만 addCallback() 함수 실행.
if(Build.VERSION.SDK_INT >= Build.VERSION_CODES.S) {
    val builder: Notification.Builder = Notification.Builder(this, "1")
        .setStyle(
            Notification.CallStyle.forIncomingCall(caller, declineIntent, answerIntent)
        )
}
```

<h1>2. 퍼미션 설정하기</h1>
<ul>
  <li>
    <strong>퍼미션(permission)</strong>이란 앱의 특정 기능에 부여하는 <strong>접근 권한</strong>을 의미한다.
  </li>
    <ul>
      <li>
        내가 개발하는 앱과 다른 앱들이 서로 권한을 얻은 앱에서만 사용할 수 있도록 할 때 활용한다.
      </li>
    </ul>
</ul>

<br>

<h2>2-1. 퍼미션 설정과 사용 설정</h2>
<ul>
  <li>
    <strong>permission</strong> 태그로 <strong>보호</strong>하려는 기능을 <strong>앱의 매니페스트 파일에 설정</strong>한다.
  </li>
    <ul>
      <li>
        매니페스트 파일에서 퍼미션을 설정할 때에는 permission 태그와 다음 속성을 활용한다.
      </li>
        <ul>
          <li> 
            name: 퍼미션의 이름. 퍼미션을 구별하는 식별자 역할을 한다.
          </li>
          <li>
            label, description: 퍼미션에 대한 설명. 해당 퍼미션을 이용하는 외부 앱에서 권한 인증 화면에 출력할 퍼미션 정보이다.
          </li>
          <li>
            protectionLevel: 보호 수준. 
          </li>
            <ul>
              <li>
                normal: 낮은 보호 수준. 사용자에게 권한 허용을 요청하지 않아도 된다.
              </li>
              <li>
                dangerous: 높은 보안 수준. 사용자에게 권한 허용을 요청해야 한다.
              </li>
              <li>
                signature: 같은 키로 인증한 앱만 실행한다.
              </li>
              <li>
                signatureOrSystem: 안드로이드 시스템 앱이거나 같은 키로 인증한 앱만 실행.
              </li>
            </ul>
        </ul>
    </ul>
  <li>
    <strong>uses-permission</strong> 태그로 퍼미션으로 <strong>보호된 기능을 사용</strong>하려는 앱의 매니페스트 파일에 설정한다.
  </li>
  <li>
    protectionLevel의 속성값은 주로 normal, dangerous, signature를 사용한다.
  </li>
    <ul>
      <li>
        normal의 경우 사용자에게 권한 요청을 하지 않아도 되기에 부담이 적으며 dangerous는 사용 설정과 사용자에게도 권한 요청을 해야한다.
      </li>
    </ul>
  <li>
    permission 설정을 한 다음에는 설정한 퍼미션으로 보호하려는 <strong>컴포넌트에 적용</strong>해야 한다. (danger는 적용이 되어 있다).
  </li>
    <ul>
      <li>
        퍼미션을 컴포넌트에 적용할 때에는 <strong>android:permission 속성</strong>을 이용한다.
      </li>
      <li>
        퍼미션이 적용된 컴포넌트는 보호되며 해당 컴포넌트를 이용하는 곳에서는 매니페스트 파일에 <strong>uses-permission</strong>을 선언해 주어야 한다.
      </li>
    </ul>
  <li>
    외부 앱과 연동할 때 퍼미션 사용을 설정하지만 <strong>안드로이드 시스템에서 보호하는 기능</strong>을 사용할 때에도 매니페스트 파일에 퍼미션을 설정해야 한다. 시스템이 보호하는 기능에는 대표적으로 다음이 있다.
  </li>
    <ul>
      <li>
        ACCESS_FINE_LOCATION: 위치 정보 접근
      </li>
      <li>
        ACCESS_NETWORK_STATE: 네트워크 정보 접근
      </li>
      <li>
        ACCESS_WIFI_STATE: 와이파이 네트워크 정보 접근
      </li>
      <li>
        BATTERY_STATS: 배터리 정보 접근
      </li>
      <li>
        BLUETOOTH: 블루투스 장치에 연결
      </li>
      <li>
        BLUETOOTH_ADMIN: 블루투스 장치를 검색하고 페어링
      </li>
      <li>
        CAMERA: 카메라 장치에 접근
      </li>
      <li>
        INTERNET: 네트워크 연결
      </li>
      <li>
        READ_EXTERNAL_STORAGE: 외부 저장소에서 파일 읽기
      </li>
      <li>
        WRITE_EXTERNAL_STORAGE: 외부 저장소에 파일 쓰기
      </li>
      <li>
        READ_PHONE_STATE: 전화기 정보 접근
      </li>
      <li>
        SEND_SNS: 문자 메시지 발신
      </li>
      <li>
        RECEIVE_SMS: 문자 메시지 수신
      </li>
      <li>
        RECEIVE_BOOT_COMPLETED: 부팅 완료 시 실행
      </li>
      <li>
        VIBRATE: 진동 울리기
      </li>
    </ul>
  <li>
    구글의 기본 앱들은 본보기가 되는 화면이나 기능을 많이 제공하기에 연동하는 경우가 종종 있다.
  </li>
  <li>
    시스템 기능에서 퍼미션 설정을 활용하는 이유는 사용자에게 민감한 기능에 대한 접근에 제한을 두기 위해서이다.
  </li>
</ul>

```xml
<!-- 1. 퍼미션 설정 예. -->
<permission android:name="com.example.permission.TEST_PERMISSION"
    android:label="Test Permission"
    ansroid:description="string/permission_desc"
    android:protectionLevel="dangerous" />
```

```xml
<!-- 2. 퍼미션 사용 설정. -->
<!-- 네트워크에 접근 -->
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
<!-- 사용자 위치 추적 -->
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
```

```xml
<!-- 3. 컴포넌트에 퍼미션 적용. -->
<activity android:name=".OneActivity" android:permission="com.example.TEST_PERMISSION">
    <intent-filter>
        <action android:name="android:intent.action.PICK" />
    </intent-filter>
</activity>
```

```xml
<!-- 4. 퍼미션 사용 설정 -->
<uses-permission android:name="com.exmaple.permission.TEST_PERMISSION" />
```

<h2>2-2. 퍼미션 허용 확인</h2>
<ul>
  <li>
    API 레벨 23 이전에는 uses-permission으로 선언만 하면 기능 사용이 가능했다. 하지만 23 버전 이후로는 <strong>허가제</strong>로 변경되었다.
  </li>
    <ul>
      <li>
        허가제란 uses-permission으로 선언했더라도 사용자가 권한 화면에서 이를 <strong>거부</strong>할 수 있다.
      </li>
      <li>
        사용자가 거부한 경우 uses-permission을 선언하지 않은 것과 같으며 앱에서 기능 사용이 불가하다.
      </li>
    </ul>
  <li>
    사용자가 퍼미션을 거부한 것을 기억하지 못하고 앱이 제대로 동작하지 않는다 불평할 수도 있다.
  </li>
    <ul>
      <li>
        앱을 처음 설치하면 기본적으로 퍼미션은 모두 거부 상태이며 따라서 <strong>사용자에게 퍼미션 사용 요청을 전달</strong>해야 한다.
      </li>
    </ul>
  <li>
    사용자가 퍼미션을 허용했는지 확인하려면 <strong>checkSelfPermission() 함수</strong>를 이용한다.
  </li>
    <ul>
      <li>
        두 번째 매개변수가 퍼미션을 구분하는 이름으로 결괏값은 다음 중 하나의 상수로 전달된다.
      </li>
        <ul>
          <li>
            PackageManeger.PERMISSION_GRANTED: 권한을 허용한 경우.
          </li>
          <li>
            PackageManager.PERMISSION_DENIED: 권한을 거부한 경우.
          </li>
        </ul>
    </ul>
  <li>
    사용자에게 퍼미션 허용을 요청할 경우 <strong>ActivityResultLauncher 클래스</strong>를 활용한다.
  </li>
    <ul>
      <li>
        액티비티에서 <strong>결과</strong>를 돌려 받아야할 때 사용한다.
      </li>
      <li>
        대표적으로 <strong>퍼미션 허용 요청</strong>과 다른 <strong>액티비티를 실행하고 결과를 돌려받을 때</strong>가 해당된다.
      </li>
      <li>
        ActivityResultLauncher 객체 <strong>registerForActivityResult() 함수</strong>를 호출해서 만든다.
      </li>
    </ul>
  <li>
    registerForActivityResult() 함수는 매개변수가 두 개이다.
  </li>
    <ul>
      <li>
        ActivityResultContract: 어떤 요청인지를 나타낸다. 다양한 요청에 대응하는 서브 클래스들이 있다.
      </li>
        <ul>
          <li>
            ActivityResultContract 타입 객체: 어떤 요청인지 나타낸다. 다양한 요청에 대응하는 서브 클래스들이 있다.
          </li>
          <li>
            StartActivityForResult: 다른 액티비티를 실행하고 결과를 돌려받을 때 사용한다.
          </li>
          <li>
            RequestPermission: 퍼미션을 허용할 때 사용한다.
          </li>
        </ul>
      <li>
        ActivityResultCallback: 결과를 받았을 때 호출되는 콜백이다.
      </li>
    </ul>
  <li>
    registerForActivityResult() 함수로 <strong>ActivityResultLauncher 객체</strong>를 만든 후 필요한 곳에서 ActivityResultLauncher 객체의 <strong>launch() 함수</strong>를 호출해 요청을 실행한다.
  </li>
</ul>

```kotlin
// 1. 퍼미션 허용 확인 함수.
open static fun checkSelfPermission(
    @NonNull context: Context,
    @NonNull permission: String
): int
```

```kotlin
// 2. 퍼미션 허용 확인 예.
val status = ContextCompat.checkSelfPermission(this, "android.permission.ACCESS_FINE_LOCATION")
if (status == PacakgeManager.PERMISSION_GRANTED) {
    Log.d("kkang", "permission granted")
} else {
    Log.d("kkang", "permission denied")
}
```

```kotlin
// 3. registerForActivityResult() 함수
public final <I, O> ActivityResultLauncher<I> register ForActivityResult(
    @NonNull ActivityResultContract<I, O> contract,
    @NonNull ActivityResultCallback<O> callback)
```

```kotlin
// 4. 퍼미션 허용 요청 확인.
val requestPermissionLauncher = registerForActivityResult(
    ActivityResultContracts.RequestPermission()
) { is Granted -> 
    if (isGranted) {
        Log.d("kkang", "callback, granted..")
    } else {
        Log.d("kkang", "callback, denied..")
    }
}
```

```kotlin
// 5. 퍼미션 허용 요청 실행.
// 요청 결과는 registerForActivityResult() 함수의 두 번째 매개변수로 등록
requestPermissionLauncher.launch("android.permission.ACCESS_FINE_LOCATION")
```

<h1>3. 다양한 다이얼로그</h1>
<ul>
  <li>
    다이얼로그(dialog)란 <strong>사용자와 상호 작용하는 대화상자</strong>이다.
  </li>
</ul>

<br>

<h2>3-1. 토스트 메시지 띄우기</h2>
<ul>
  <li>
    토스트(Toast)는 <strong>화면 아래쪽</strong>에 잠깐 <strong>보였다가 사라지는 문자열</strong>을 의미한다.
  </li>
    <ul>
      <li>
        일반적으로 사용자에게 간단한 메시지로 특정한 상황을 알릴 때 사용한다.
      </li>
    </ul>
  <li>
    토스트는 Toast의 <strong>makeText() 함수</strong>로 만든다.
  </li>
    <ul>
      <li>
        open static fun makeText(context: Context!, text: CharSequence!, duration: Int): Toast!
      </li>
      <li>
        open static fun makeText(context: Context!, resId: Int, duration: Int): Toast!
      </li>
    </ul>
  <li>
    makeText() 함수의 두 번째 매개변수는 <strong>출력할 문자열</strong>이고, 세 번째 매개변수는 <strong>토스트가 화면에 출력되는 시간</strong>이다.
  </li>
    <ul>
      <li>
        토스트가 화면에 출력되는 시간은 보통 Toast.LENGTH_SHORT 상수를 이용하며 일반적으로 3초 정도이다. Toast.LENGTH_LONG은 5초 정도의 시간을 의미한다.
      </li>
        <ul>
          <li>
            val LENGTH_LONG: Int
          </li>
          <li>
            val LENGTH_SHORT: Int
          </li>
        </ul>
    </ul>
      <li>
        makeText() 함수로 만든 토스트는 show() 함수로 화면에 출력한다.
      </li>
  <li>
    토스트는 makeText() 함수를 이용하지 않고 <strong>세터 함수</strong>로 만들 수도 있다. <strong>setDuration()과 setText() 함수</strong>를 이용해 <strong>문자열이나 화면에 보이지 않는 시간</strong>을 설정할 수 있고, <strong>setGravity()나 setMargin() 함수</strong>를 이용해 <strong>토스트가 뜨는 위치</strong>를 정할 수도 있다.
  </li>
    <ul>
      <li>
        open fun setDuration(duration: Int): Unit
      </li>
      <li>
        open fun setGravity(gravity: Int, xOffset: Int, yOffset: Int): Unit
      </li>
      <li>
        open fun setMargin(horizontalMargin: Float, verticalMargin: Float): Unit
      </li>
      <li>
        open fun setText(resId: Int): Unit
      </li>
    </ul>
  <li>
    토스트가 <strong>사라지는 순간</strong>을 콜백으로 감지해 특정 로직을 수행하게 할 수도 있다. 이는 <strong>API 30 버전</strong>에서 추가되었기에 호환성을 고려해야 한다.
  </li>
</ul>

```kotlin
// 1. 토스트 출력 예.
val toast = Toast.makeText(this, "종료하려면 한 번 더 누르세요", Toast.LENGTH_SHORT)
toast.show()
```

```kotlin
// 2. 콜백 기능 이용하기.
// 콜백 사용이 가능한지 우선 버전을 확인한다.
@RequiresApi(Build.VERSION_CODES.R)
fun showToast() {
    val toast = Toast.makeText(this, "종료하려면 한 번 더 누르세요.", Toast.LENGTH_SHORT)
    toast.addCallback(

        object : Toast.Callback() {

            // 화면에서 사라지는 순간 onToastHidden() 함수가 자동으로 호출된다.
            override fun onToastHidden() {
                super.onToastHidden()
                Log.d("kkang", "toast hidden")
            }
           
            // 화면에 뜨는 순간 자동으로 콜백 객체의 onToastShown() 함수가 호출된다.
            override fun onToastShown() {
                super.onToastShown()
                Log.d("kkang", "toast shown")
            }
        })
    toast.show()
}
```

<h2>3-2. 날짜 또는 시간 입력받기</h2>
<ul>
  <li>
    앱에서 사용자에게 날짜나 시간을 입력받는 데 사용하는 다이얼로그를 <strong>피커(Picker) 다이얼로그</strong>라고 한다.
  </li>
    <ul>
      <li>
        <strong>날짜</strong>를 입력받을 때는 <strong>데이트 피커 다이얼로그(DatePickerDialog)</strong>를 사용한다.
      </li>
      <li>
        <strong>시간</strong>을 입력 받을 때에는 <strong>타임 피커 다이얼로그(TimePickerDialog)</strong>를 사용한다.
      </li>
    </ul>
  <li>
    DatePickerDialog.OnDateSetListener 구현 객체를 등록하면 다이얼로 사용자가 설정한 날짜를 <strong>콜백 함수</strong>로 얻을 수 있다.
  </li>
</ul>


<h2>3-3. 알림 창 띄우기</h2>
<ul>
  <li>
    안드로이드 다이얼로그의 기본은 <strong>AlertDialog</strong>이다. 
  </li>
    <ul>
      <li>
        단순히 메시지만 출력할 수도 있고 다양한 화면을 출력할 수도 있다.
      </li>
      <li>
        앞서 다룬 데이트 피커와 타임 피커도 AlertDialog의 하위 클래스이다.
      </li>
    </ul>
  <li>
    알림 창은 크게 제목, 내용, 버튼 세 가지 영역으로 구분되며 항상 세 가지 영역이 모두 보이는 것은 아니다.
  </li>
  <li>
    알림 창의 생성자는 접근 제한자가 <strong>protected</strong>이기 때문에 객체를 직접 생성할 수 없으며 대신 제공하는 <strong>AlertDialog.Builder</strong>를 통해 알림창을 만든다.
  </li>
    <ul>
      <li>
        AlertDialog.Builder를 생성하고 빌더의 <strong>세터 함수</strong>로 알림 창의 정보를 지정한다.
      </li>
    </ul>
  <li>
    알림 창에 아이콘과 제목, 내용을 지정하는 함수는 다음과 같다. 함수 제목 영역에 아이콘을 출력할 때에는 setIcon() 함수, 제목 문자열을 출력할 때에는 setTitle() 함수, 간단한 문자열을 출력할 때에는 setMessage() 함수를 사용한다.
  </li>
    <ul>
      <li>
        open fun setItem(iconId: Int): AlertDialog.Builder!
      </li>
      <li>
        open fun setTitle(title: CharSequence!): AlertDialog.Builder!
      </li>
      <li>
        open fun setMessage(message: CharSequence!): AlertDialog.Builder!
      </li>
    </ul>
  <li>
    알림 창에 버튼을 지정하는 함수는 다음과 같다. 첫 번째 매개변수는 <strong>버튼의 문자열</strong>이고, 두 번째 매개변수는 버튼을 클릭했을 때 처리할 <strong>이벤트 핸들러</strong>이다.
  </li>
    <ul>
      <li>
        버튼을 클릭했을 때 처리할 내용이 없다면 <strong>null</strong>을 대입한다. 이 경우 버튼을 클릭하면 <strong>창이 닫힌다</strong>.
      </li>
      <li>
        알림창 버튼은 <strong>최대 3개</strong>까지만 추가할 수 있으며 같은 함수를 여러 번 사용하면 버튼은 <strong>중복되어 하나만</strong> 나타난다.
      </li>
      <li>
        open fun setPositiveButton(text: CharSequence!, listener: DialogInterface.OnClickListener!): AlertDialog.Builder!
      </li>
      <li>
        open fun setNegativeButton(text: CharSequence!, listener: DialogInterface.OnClickListener!): AlertDialog.Builder!
      </li>
      <li>
        open fun setNeutralButton(text: CharSequence!, listener: DialogInterface.OnClickListener!): AlertDialog.Builder!
      </li>
    </ul>
  <li>
    버튼 함수를 setPositiveButton(), setNegativeButton(), setNeutralButton() 세 개로 구분하는 이유는 이벤트 핸들러에서 <strong>어떤 버튼이 클릭되었는지 구분</strong>하기 위해서이다.
  </li>
    <ul>
      <li>
        셋 중 어떤 함수를 사용했는지에 따라 이벤트 핸들러에 전달되는 <strong>매개변숫값</strong>이 다르며 이 값으로 어떤 버튼이 클릭되었는지 구분한다.
      </li>
    </ul>
  <li>
    알림 창의 내용 영역에 간단히 문자열을 출력하는 setMessage() 외에도 setItems(), setMultiChoiceItems(), setSingleChoiceItems() 함수를 통해 <strong>목록</strong>을 제공할 수도 있다.
  </li>
    <ul>
      <li>
        함수의 첫 번째 매개변수는 배열 정보로 해당 문자열이 목록에 출력된다.
      </li>
      <li>
        open fun setItems(items: Array&ltCharSequence!&gt!, listener: DialogInterface.OnClickListener!): AlertDialog.Builder!
      </li>
      <li>
        open fun setMultiChoiceItems(items: Array&ltCharSequence!&gt!, checkedItems: BooleanArray!, listener: DialogInterface.OnMultiChoiceClickListener!): AlertDialog.Builder!
      </li>
      <li>
        open fun setSingleChoiceItems(items: Array&ltCharSequence!&gt!, checkedItem: Int, listener: DialogInterface.OnClickListener!): AlertDialog.Builder!
      </li>
    </ul>
  <li>
    setMultiChoiceItems() 함수는 <strong>다중 선택을 위한 체크박스</strong>가 함께 출력되는 항목을 만들어 준다. 두 번째 매개변수로 <strong>처음 체크 상태</strong>를 지정한다.
  </li>
  <li>
    setSingleChoiceItems() 함수는 <strong>하나만 선택</strong>할 수 있는 <strong>라디오 버튼</strong>으로 구성된 항목을 만든다. 두 번째 매개변수로 선택할 항목을 지정한다.
  </li>
  <li>
    <strong>setCancelable()</strong> 함수로 자용자가 기기의 <strong>뒤로 가기 버튼</strong>을 눌렀을 때, <strong>setCanceledTouchOutside()</strong> 함수로 <strong>알림 창의 바깥 영역을 터치</strong>했을 때를 처리할 수 있다.
  </li>
</ul>

<h3>4-2-1. 시간과 패턴을 지정해 진동 울리기(API 레벨 1부터 제공하는 함수)</h3>
<ul>
  <li>
    API 레발 26 버전에서 deprecated된 기능이지만 여기서는 호환성을 고려해 사용하는 상황을 가정한다. deprecated된 함수는 아래와 같다.
  </li>
    <ul>
      <li>
        open fun vibrate(milliseconds: Long): Unit
      </li>
        <ul>
          <li>
            매개변수는 Long 타입 하나로 진동이 울리는 시간을 의미한다.
          </li>
        </ul>
      <li>
        open fun vibrate(pattern: LongArray!, repeat: Int): Unit
      </li>
        <ul>
          <li>
            첫 번째 매개변수는 각 진동이 울리는 시간을, 두 번째 매개변수는 울리는 횟수를 의미한다.
          </li>
            <ul>
              <li>
                두 번째 매개변수를 -1로 지정하면 반복하지 않고 패턴대로 한 번만 진동이 울리고, 0으로 지정하며 코드에서 cancel() 함수로 진동을 끄기 전까지 울린다.
              </li>
            </ul>
        </ul>
    </ul>
</ul>

<h3>4-2-2. 진동의 세기까지 지정해 진동 울리기(API 레벨 26부터 제공하는 함수)</h3>
<ul>
  <li>
    API 레발 26부터는 지동 정보를 <strong>VibrationEffect 객체</strong>로 지정할 수 있는 함수를 제공한다.
  </li>
    <ul>
      <li>
        VibrationEffect 객체로는 <strong>진동이 울리는 시간</strong>과 <strong>진동의 세기</strong>까지 제어할 수 있으며 아래의 함수와 같다.
      </li>
        <ul>
          <li>
            open fun vibrate(vibe: VibrationEffect!): Unit
          </li>
        </ul>
    </ul>
  <li>
    <strong>vibrate() 함수의 매개변수</strong>에 VibrationEffect 객체를 지정하며 아래의 함수를 통해 진동 정보를 지정할 수 있다.
  </li>
    <ul>
      <li>
        open static fun createOneShot(milliseconds: Long, amplitude: Int): VibrationEffect!
      </li>
      <li>
        진동의 세기는 <strong>0 ~ 255 사이의 숫자</strong>로 표현하며 0으로 지정하면 울리지 않고 255로 지정하면 가장 센 강도로 울린다.
      </li>
      <li>
        <strong>VibrationEffect.DEFAULT_AMPLITUDE</strong>와 같이 <strong>상수</strong>를 지정하여 기기가 정의한 기본 세기로 진동이 울리게 할 수 있다.
      </li>
    </ul>
  <li>
    반복해서 진동을 울리는 함수로는 <strong>createWaveform()</strong> 함수가 있다.
  </li>
    <ul>
      <li>
        open static fun createWaveform(timings: LongArray!, amplitudes: IntArray!, repeat: Int): VibrationEffect!
      </li>
        <ul>
          <li>
            첫 번째 매개변수는 진동이 울리는 시간 패턴의 배열이다.
          </li>
          <li>
            두 번째 매개변수는 진동 세기 패턴의 배열이다.
          </li>
          <li>
            마지막 세 번째 매개변수는 패턴의 반복 횟수이다.
          </li>
        </ul>
    </ul>
</ul>

```kotlin
// 1. 기본 세기로 진도 울리기.

    vibrator.vibrate(
        VibrationEffect.createOneShot(500, VibrationEffect.DEFAULT_AMPLITUDE))

  vibrator.vibrate(500)
}
```

<h1>5. 알림 띄우기</h1>
<h2>5-1. 알림 채널</h2>
<ul>
  <li>
    화면 상단의 배터리, 네트워크, 시간 등 시스템의 상태 정보가 출력되는 한 줄을 상태 바라한다. <strong>상태 바에 앱의 정보를 출력</strong>하는 것을 알림(notification)이라 한다.
  </li>
    <ul>
      <li>
        원래 상태 바는 <strong>시스템</strong>에서 관리하는 곳이며 앱에서는 접근할 수 없다.
      </li>
      <li>
        따라서 사용자 이벤트를 처리하는 프로그래밍과는 구조가 다르며 <strong>알림을 위해 제공하는 API</strong>를 이용해야 한다.
      </li>
    </ul>
  <li>
    API Level 33버전부터는 앱의 알림을 띄우기 위해 사용자에게 <strong>퍼미션(허가)</strong>를 요청해야 한다.
  </li>
  <li>
    NotificationChannel 정보로 <strong>알림 채널</strong>을 생성한다.
    <br>→ 생성된 알림 채널 정보를 대입하여 <strong>NotificationCompat.Builder</strong>를 생성한다.
    <br>→ 생성된 NotificationCompat.Builder로 <strong>Notification 객체</strong>를 생성한다.
    <br>→ 생성된 Notification 객체를 NotificationManager의 <strong>notify() 함수에 대입</strong>한다.
  </li>
  <li>
    Notification을 만들려면 NotificationCompat.Builder가 필요한데 빌더를 만드는 방법이 <strong>API 레벨 26(Android 8)버전</strong>부터 변경되었다. 
  </li>
    <ul>
      <li>
        26 버전 이전까지는 <strong>Builder(context: Context!) 생성자</strong>를 이용하여 생성하였다.
      </li>
      <li>
        26 버전 이후에는 <strong>NotificationChannel</strong>을 만들고 채널의 식별값을 빌더의 <strong>생성자 매개변수</strong>에 지정해 주어야 한다.
      </li>
        <ul>
          <li>
            이는 API 레벨 26 버전에서 <strong>채널</strong>이라는 개념이 추가되고, 앱의 <strong>알림을 채널로 구분</strong>하겠다는 의도이다.
          </li>
          <li>
            채널로 구분을 하면 앱의 각종 알림을 <strong>채널로 구분</strong>하여 설정할 수 있어 특정 알림은 유지한 채 다른 알림은 끌 수 있는등 <strong>선택권</strong>이 주어진다.
          </li>
          <li>
            Builder(context: Context!, channelId: String!)
          </li>
        </ul>
    </ul>
  <li>
    채널의 각종 정보는 함수나 프로퍼티로 설정할 수 있다.
  </li>
    <ul>
      <li>
        fun setDescription(description: String!): Unit - 채널의 설명 문자열.
      </li>
      <li>
        fun setShowBadge(showBadge: Boolean): Unit - 홈 화면의 아이콘에 배지 아이콘 출력 여부.
      </li>
      <li>
        fun setSound(sound: Uri!, audioAttributes: AudioAttributes!): Unit - 알림음 재생.
      </li>
      <li>
        fun enableLights(lights: Boolean): Unit - 불빛 표시 여부.
      </li>
      <li>
        fun setLightColor(argb: Int): Unit - 불빛이 표시된다면 불빛의 색상.
      </li>
      <li>
        fun enableVibration(vibration: Boolean): Unit - 진동을 울릴지 여부.
      </li>
      <li>
        fun setVibrationPattern(vibrationPattern: LongArray!): Unit - 진동을 울린다면 진동의 패턴.
      </li>
    </ul>
</ul>


<h2>5-2. 알림 객체</h2>
<ul>
  <li>
    앞서 알림 빌더를 생성했기에 빌더를 이용해 <strong>Notification 객체</strong>를 생성해야 한다. 이때 객체에 출력할 <strong>이미지, 문자열 등의 정보</strong>를 담는다.
  </li>
    <ul>
      <li>
        앱에서 알림이 발생하며 상태 바에 이미지가 출력되며 이때 출력되는 이미지를 <strong>스몰 아이콘(small icon)</strong>이라 한다.
      </li>
    </ul>
  <li>
    알림은 스몰 아이콘과 발생 시각, 제목, 내용 등으로 구성되며 Notification 객체에 설정한다. 빌더의 <strong>세터 함수</strong>를 이용해 알림의 구성 정보를 설정한다.
  </li>
  <li>
    설정을 완료했다면 NotificationManager 클래스의 <strong>notify() 함수</strong>를 이용해 알림을 띄운다.
  </li>
  <li>
    사용자가 알림을 터치하면 이벤트가 발생할 수 있고 이때 알림은 화면에서 자동으로 사라진다(취소) 혹은 알림을 손가락으로 밀어서(스와이프) 취소할 수도 있다.
  </li>
    <ul>
      <li>
        터치나 스와이프를 하더라도 알림이 사라지지 않게 하려면 빌더의 빌더의 <strong>세터 함수</strong>로 지정해야 한다.
      </li>
      <li>
        <strong>setAutoCancel(false)</strong>로 지정하여 <strong>터치</strong>할 때 이벤트는 발생하지만 알림이 사라지지 않게하거나 <strong>setOngoing(true)</strong>로 지정하여 사용자가 알림을 <strong>스와이프</strong>해도 사라지지 않게 할 수 있다.
      </li>
        <ul>
          <li>
            둘 모두를 활성화하면 <strong>cancel()</strong> 함수로 알림을 취소해야 한다.
          </li>
        </ul>
    </ul>
</ul>

<h2>5-3. 알림 구성</h2>
<h3>5-3-1. 알림 터치 이벤트</h3>
<ul>
  <li>
    대부분의 앱은 사용자가 <strong>알림을 터치했을 때 앱의 액티비티 화면</strong>을 실행하며 이렇게 구성하기 위해서는 <strong>알림의 터치 이벤트</strong>를 구현해야 한다.
  </li>
  <li>
    알림은 앱이 관활하는 화면이 아니라 <strong>시스템</strong>에서 관리하는 <strong>상태 바</strong>에서 출력하는 정보이기 때문에 앱의 onTouchEvent()가 이전에 설명한 시스템의 <strong>Notification 객체</strong>에 정보를 담아두고 이벤트가 발생할 때 시스템이 실행되도록 한다.
  </li>
  <li>
    사용자가 알림을 터치하면 앱의 <strong>액티비티</strong> 또는 <strong>브로드캐스트 리시버</strong>를 실행해야 하는데 이를 실행하기 위해서는 <strong>인텐트(intent)</strong>를 이용해야 한다. 지금은 인텐트가 <strong>앱의 컴포넌트를 실행하는 데 필요한 정보</strong> 정도로 이해하면 된다.
  </li>
  <li>
    인텐트는 앱의 코드에서 준비하지만 <strong>실행 시점</strong>은 정할 수 없다. Notification 객체에 인텐트를 담아두면 <strong>이벤트 발생 시</strong>에 인텐트를 실행할 것을 <strong>시스템에 의뢰</strong>해야 한다.
  </li>
    <ul>
      <li>
        의뢰를 할 때에는 <strong>PendingIntent 클래스</strong>를 활용한다. PendingIntent 클래스는 <strong>컴포넌트별로 실행을 의뢰하는 함수</strong>를 제공한다.
      </li>
      <li>
        함수의 세 번째 매개변수는 flag 값으로 <strong>똑같은 알림이 발생</strong>했을 때 어떻게 처리해야 하는지를 나타낸다.
      </li>
        <ul>
          <li>
            static fun getActivity(context: Context!, requestCode: Int, intent: Intent!, flags: Int): PendingIntent!
          </li>
          <li>
            static fun getBroadcast(context: Context!, requestCode: Int, intent: Intent!, flags: Int): PendingIntent!
          </li>
          <li>
            static fun getService(context: Context!, requestCode: Int, intent: Intent!, flags: Int): PendingIntent!
          </li>
        </ul>
    </ul>
</ul>

<h3>5-3-3. 원격 입력</h3>
<ul>
  <li>
    원격 입력(remoteInput)이란 알림에서 <strong>사용자 입력을 직접 받는 기법</strong>이다.
  </li>
    <ul>
      <li>
        원래 사용자 입력은 에디트 텍스트 같은 뷰가 있는 화면을 제공하나 간단한 입력은 앱의 화면을 통하지 않고 <strong>원격으로 액션에서 직접 받아 처리</strong>할 수 있다.
      </li>
    </ul>
  <li>
    원격 입력도 <strong>액션</strong>의 한 종류이며 <strong>RemoteInput</strong>에 <strong>사용자 입력을 받는 정보</strong>를 설정한 후 액션에 추가하는 구조이다.
  </li>
  <li>
    RemoteInput은 API 레벨 20에서 추가되었다. 따라서 <strong>minSdk를 20 아래</strong>로 설정한 경우 API 레벨 호환성을 고려해서 작성해야 한다.
  </li>
    <ul>
      <li>
        앱의 API 레벨로 if ~ else 문을 작성해 처리해도 되지만 android.app.RemoteInput 대신 <strong>androidx.core.app.RemoteInput</strong>을 이용할 수도 있다. androidx.core.app.RemoteInput은 <strong>호환성을 돕는 라이브러리</strong>이다.
      </li>
    </ul>
  <li>
    RemoteInput도 액션이기 때문에 액션의 터치 이벤트를 처리하기 위한 <strong>PendingIntent</strong>를 준비한다.
  </li>
    <ul>
      <li>
        앞의 PendingIntent와 특별한 차이는 없지만 RemoteInput은 <strong>알림에서 사용자의 입력</strong>을 받기 때문에 이벤트로 액티비티를 실행해 앱의 화면에 출력하지 않고 <strong>브로드캐스트 리시버</strong>를 실행해 <strong>백그라운드에서 사용자의 입력을 처리</strong>하는 게 일반적이다.
      </li>
    </ul>
</ul>

<h3>5-3-4. 프로그래스</h3>
<ul>
  <li>
    앱에서 어떤 작업이 이루어지는 데 시간이 걸린다면 보통 알림을 이용해 일의 <strong>진행 상황</strong>을 <strong>프로그래스 바</strong>로 알려준다.
  </li>
    <ul>
      <li>
        알림의 프로그래스 바는 화면을 따로 준비하지 않고 빌더에 <strong>setProgress()</strong> 함수만 추가해 주면 자동으로 출력된다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 프로그래스 바 함수.
open fun setProgress(
    
    // 프로그래스 바의 최댓값.
    max: Int, 
    
    // 진행값: 처음에 현잿값을 지정한 후 스레드(thread) 같은 프로그램을 사용해 진행값을 바꾸면서 상황을 알려준다.
    progress: Int, 
    
    // 세 번째 매개변수가 true이면 프로그래스 바는 왼쪽에서 오른쪽으로 계속 흘러가듯이 표현된다.
    indeterminate: Boolean): Notification.Builder
```

```kotlin
// 2. 프로그래스 바의 진행값을 증가시키는 스레드.
builder.setProgress(100, 0, false)
manager.notify(11, builder.build())

thread {
    for ( i in 1..100) {
        builder.setProgress(100, i, false)
        manager.notify(11, builder.build())
        SystemClock.sleep(100)
    }
}
```

<h3>5-4-4. 메시지 스타일</h3>
<ul>
  <li>
    메시지 스타일 알림은 <strong>여러 사람이 주고받은 메시지를 구분</strong>해서 출력할 때 사용한다.
  </li>
    <ul>
      <li>
        메시지 스타일에 보일 메시지는 각각 Message 객체로 표현한다.
      </li>
    </ul>
  <li>
    하나의 메시지는 세 가지 정보로 표현된다.
  </li>
    <ul>
      <li>
        첫 번째 매개변수는 메시지의 <strong>내용</strong>이다.
      </li>
      <li>
        두 번째 매개변수는 메시지가 발생한 <strong>시각</strong>이다.
      </li>
      <li>
        세 번째 매개변수는 <strong>Person</strong>으로 알림에 출력될 <strong>한 사람의 정보를 담는 클래스</strong>이다
      </li>
    </ul>
  <li>
    Person은 <strong>API 레벨 28 버전</strong>에 추가된 클래스이기 때문에 API 레벨 호환성을 위해 <strong>androidx.core.app.Person</strong> 라이브러리를 임포트해야한다.
  </li>
</ul>

```kotlin
// 1. 메시지 객체.
Message(text: CharSequence, timestamp: Long, sender: Person?)
```

```kotlin
// 2. Person 객체 생성.
// 첫 번째 Person 객체.
val sender1: Person = Person.Builder()

    // setName()으로 사람의 이름을 등록.
    .setName("kkang")

    // setIcon()으로 프로필 사진 등의 아이콘 등록.
    .setIcon(IconCompat.createWithResource(this, R.drawable.person1))
    .build()

// 두 번째 Person 객체.
val sender2: Person = Person.Builder()
    .setName("kim")
    .setIcon(IconCompat.createWithResource(this, R.drawable.person2))
    .build()
```

```kotlin
// 3. 메시지 객체 생성.
val message1 = NotificationCompat.MessagingStyle.Message(
    "hello",
    System.currentTimeMiilis(),
    sender1
)

val message2 = NotificationCompat.MessagingStyle.Message(
    "world",
    System.currentTimeMillis(),
    sender2
)

// 4. Message 객체를 MessageStyle에 대입하여 메시지 스타일 생성.
val messageStyle = NotificationCompat.MessagingStyle(sender1)
    .addMessage(message1)
    .addMessage(message2)
```
