<h1>1. 안드로이드 스튜디오 설치하기</h1>
<ul>
  <li>
    안드로이드 슈튜디오(Android Studio)는 안드로이드 전용 앱을 개발하는 도구로 구글이 2013년에 처음 발표하였다.
  </li>
  <li>
    설치 과정은 도서의 p19부터 따라한다.ㅎ
  </li>
</ul>

<br><br>

<h1>2. 첫 번째 앱 만들기</h1>
<ul>
  <li>
    상세 과정은 도서를 따라간다.
  </li>
  <li>
    템플릿 목록 중에 <strong>[Empty Views Activity]</strong>는 <strong>뷰</strong>를 이용한 프로그램을 작성하고, <strong>[Empty Activity]</strong>는 <strong>컴포즈</strong>를 활용해 프로그램을 작성한다.
  </li>
    <ul>
      <li>
        뷰는 안드로이드 초창기부터 이용되면 방식이다.
      </li>
      <li>
        컴포즈는 제트팩에서 제공하는 방식이다.ㄴ
      </li>
    </ul>
  <li>
    프로젝트 생성시 주요 설정들
  </li>
    <ul>
      <li>
        Package name
      </li>
        <ul>
          <li>
            앱의 식별값이다.
          </li>
          <li>
            패키지명이 같은 앱은 스토어에 등록할 수 없으며 기기에 설치할 수 없어 <strong>고유한 이름</strong>으로 지어야 한다.
          </li>
          <li>
            보통은 <strong>도메인을 역순</strong>으로 입력하고 <strong>끝에 프로젝트명</strong>을 붙인다.
          </li>
        </ul>
      <li>
        Save location
      </li>
        <ul>
          <li>
            프로젝트 파일들이 저장되는 <strong>루트 디렉터리</strong>이다.
          </li>
          <li>
            "C"\" 경로는 <strong>쓰기 권한</strong>이 없을 수 있어 아래에 <strong>디렉터리를 하나</strong> 만들고 지정해야 한다.
          </li>
        </ul>
      <li>
        Language
      </li>
        <ul>
          <li>
            안드로이드 스튜디오는 개발 언어로 <strong>Java</strong>와 <strong>Kotlin</strong>을 지원한다.
          </li>
          <li>
            <strong>두 언어 모두</strong> 하나의 프로젝트에서 사용이 가능하다. 설정은 단순히 시작할 때 기본 소스를 만들 때만 적용된다.
          </li>
        </ul>
      <li>
        Minimum SDK
      </li>
        <ul>
          <li>
           Minimum SDK는 앱이 설치되는 <strong>최소 SDK 버전</strong>이다.
          </li>
          <li>
            즉, 지정한 <strong>버전 이상의 폰</strong>에만 설치된다.
          </li>
          <li>
            버전을 선택하면 앱 실행이 가능한 <strong>기기의 점유율</strong>을 보여준다.
          </li>
        </ul>
      <li>
        Build configuration language
      </li>
        <ul>
          <li>
            build.gradle 파일의 타입을 선택한다.
          </li>
          <li>
            안드로이드 앱은 Gradle 툴에 의해 빌드된다.
          </li>
            <ul>
              <li>
                전통적인 build.gradle, 코틀린 DSL 형식의 build.gradle.kts 등을 지정할 수 있다.
              </li>
              <li>
                권장되는 것은 build.gradle.kts 이다.
              </li>
            </ul>
        </ul>
    </ul>
</ul>

<h2>2-1. 안드로이드 스튜디오의 화면 구성</h2>
<ul>
  <li>
    왼쪽 아래 각종 도구 창이 있다. 이때 <strong>로그캣(Logcat)</strong>에는 <strong>앱이 실행 중일 때 발생하는 로그</strong>가 출력된다.
  </li>
</ul>

<br>

<h2>2-2. 안드로이드 SDk</h2>
<ul>
  <li>
    SDK는 안드로이드 앱 개발을 위한 <strong>툴킷</strong>이다.
  </li>
  <li>
    SDK의 버전을 확인하거나 다른 버전의 SDK를 추가로 설치할 때에는 <strong>[File → Settings]</strong>의 설정을 활용한다.
  </li>
  <li>
    <strong>[Languages & Framework → Android SDK]</strong>를 클릭해 SDK를 확인한다.
  </li>
  <li>
    <strong>[SDK Platforms]</strong>에서 SDK 목록을 확인할 수 있으며 <strong>체크를 표시하거나 해제</strong>하는 방법으로 SDK 설치 혹은 업데이트가 가능하다.
  </li>
  <li>
    <strong>[SDK Tools]</strong>에는 <strong>개발자 도구</strong>들이 표시되며 SDK와 마찬가지로 설치하거나 업데이트가 가능하다.
  </li>
  <li>
    에뮬레이터를 실행시켜 테스트를 하려면 <strong>하드웨어 가속기</strong>가 설치되어 있어야 한다.
  </li>
    <ul>
      <li>
        하드웨어 가속기를 설치하려면 먼저 <strong>CPU가 가상화기술을 지원</strong>해야 하고 이를 <strong>활성화(BIOS 설정)</strong> 해야 한다.
      </li>
      <li>
        설정은 컴퓨터 환경에 따라 달라지기에 안드로이드 개발자 사이트를 참고.
      </li>
    </ul>
</ul>

<br>

<h2>2-3. 기타 설정</h2>
<ul>
  <li>
    [File → Settings]의 [Appearance & Behavior] → [Appearance]에서 다른 테마로 변경할 수 있다.
  </li>
  <li>
    [Editor → Font]에서 소스 코드의 글꼴과 크기 등을 설정할 수 있다.
  </li>
  <li>
    [Editor → General → Auto Import]에서 Java와 Kotlin의 [Add unambiguous Imports on the fly]와 [Optimize imports on the fly]를 선택하고 Apply한다. 
  </li>
    <ul>
      <li>
        이 설정은 클래스를 사용할 때 자동으로 import를 해주는 기능을 한다.
      </li>
    </ul>
</ul>

<h1>3. 앱 실행하기</h1>
<ul>
  <li>
    안드로이드 스튜티오가 제공하는 <strong>가상 기기</strong>를 이용하는 방법과 <strong>실제 스마트폰</strong>을 사용하는 방법 두 가지가 있다.
  </li>
</ul>

<br>

<h2>3-1. 가상 기기에서 실행</h2>
<ul>
  <li>
    안드로이드 가상 기기는 <strong>AVD(Android virtual device)</strong>라고 하며 흔히 <strong>에뮬레이터(Emulator)</strong>라고 부른다.
  </li>
  <li>
    오른쪽 사이드바에서 [Device Manager]를 클릭하여 디바이스 매니저 창을 활성화 한다.
  </li>
    <ul>
      <li>
        "+" 버튼과 [Create Virtual Device]를 통해 AVD를 생성하고, "start" 버튼을 통해 실행한다.
      </li>
    </ul>
</ul>

<h3>3-1-1. 디바이스 선택</h3>
<ul>
  <li>
    도서에 있는 버전이 없어 Pixel4a로 진행함.
  </li>
</ul>

<h3>3-1-2. 시스템 이미지 선택</h3>
<ul>
  <li>
    안드로이드 운영체제 버전을 선택한다. (각 버전들이 적합하게 선택).
  </li>
</ul>

<h3>3-1-3. 에뮬레이터 실행</h3>
<ul>
  <li>
    실행 후에는 에뮬레이터 상단에 기기를 조작할 수 있는 설정들이 있다.
  </li>
</ul>

<h3>3-1-4. 제스처 내비게이션</h3>
<ul>
  <li>
    AVD 실행 후 내비게이션 바(back, home, app overview) 버튼이 없을 수 있는데 <strong>마우스를 통한 제스처</strong>로 이동이 가능하다.
  </li>
    <ul>
      <li>
        이는 시스템 내비게이션(system navigation)이 제스처 내비게이션(gesture navigation)으로 되어 있기 때문이다.
      </li>
      <li>
        시스템 내비게이션을 활용하려는 경우 [환경설정 → System → Navigation model]에서 설정 화면을 변경한다.
      </li>
    </ul>
</ul>

<br>

<h2>3-2. 실제 스마트폰에서 실행</h2>
<ul>
  <li>
    개발할 때에는 에뮬레이터를 주로 사용한다면 배포 전에느 실제 스마트 폰에서 테스트를 진행한다.
  </li>
  <li>
    이외에도 에뮬레이터 실행이 어려운 환경이거나 성능이 까다로운 앱의 경우에도 실제 기기에서 테스트를 진행한다.
  </li>
</ul>

<h3>3-2-1. 윈도우용 USB 드라이버 설치</h3>
<ul>
  <li>
    macOS는 컴퓨터가 스마트폰을 자동으로 인식하지만, 윈도우는 USB 드라이버를 다운 받아야 한다.
  </li>
  <li>
    설치는 도서의 p35를 따라할 것.
  </li>
</ul>

<h3>3-2-2. 스마트폰에 USB 디버깅 허용</h3>
<ul>
  <li>
    드라이버 설치 후에는 스마트폰에서 USB 디버깅을 허용해 주어야 한다. (갤럭시 기준 p35 참고).
  </li>
</ul>

<h3>3-2-3. 스마트폰에서 앱 실행</h3>
<ul>
  <li>
    스마트폰을 연결하면 안드로이드 스튜디오에는 연결된 기기의 이름이 표시된다.
  </li>
  <li>
    앱을 실행하면 해당 기기에 앱이 설치되고 실행된다.
  </li>
</ul>

<h3>3-2-4. 설치 문제 해결 팁</h3>
<ul>
  <li>
    각 프로그램을 설치할 때 환경에 따라 에러가 발생할 수 있는데 "안드로이드 스튜디오 문제 해결(developer.android.com/studio/troubleshoot?hl=ko)"와 "에뮬레이터 문제 해결("developer.android.com/studio/run/emulator-troubleshooting?hl=ko)"의 공식문서를 참고할 수 있다.
  </li>
</ul>

<h1>4. 앱 출시하기</h1>
<ul>
  <li>
    프로젝트의 파일을 묶어 배포 → 배포 파일을 스토어에 등록
  </li>
</ul>

<br>

<h2>4-1. 배포 파일</h2>
<ul>
  <li>
    안드로이드 앱의 배포 파일은 <strong>APK</strong>와 <strong>AAB</strong>가 있다.
  </li>
    <ul>
      <li>
        APK(Android application package)는 안드로이드의 전통적인 배포 파일이다.
      </li>
        <ul>
          <li>
            컴파일된 코드와 리소스를 묶어 <strong>키로 서명</strong>한다.
          </li>
        </ul>
      <li>
        AAB(Android app bundle) 파일은 Play 스토어에 올리면 사용자 기기에 맞게 <strong>최적화된 APK</strong>를 대신 만들어 준다.
      </li>
        <ul>
          <li>
            사용자가 APK를 설치하는 것은 동일하나 개발자가 APK를 만들지 않는다는 차이가 있다.
          </li>
          <li>
            사용자 기기에 필요한 파일만 받을 수 있기 때문에 APK 보다 <strong>크기가 줄어드는 효과</strong>가 있다.
          </li>
            <ul>
              <li>
                예를 들어 크기가 다른 기기를 위해 넣어둔 이미지는 특정 기기에는 사용되지 않기에 제외해도 문제가 되지 않는다.
              </li>
            </ul>
        </ul>
    </ul>
</ul>

<br>

<h2>4-2. 앱 서명</h2>
<ul>
  <li>
    안드로이드 배포 파일을 만들기 위해서는 <strong>키를 만들어 앱에 서명</strong>해야 한다.
  </li>
  <li>
    서명을 하는 방식은 크게 두 가지로 서명 키를 <strong>개발자가 직접 만들어 관리</strong>하는 방법과 <strong>구글 Play에서 관리</strong>하는 방법이 있다.
  </li>
    <ul>
      <li>
        <strong>개발자 서명 키</strong>는 출시용으로 앱을 빌드할 때 개발자가 만든 키로 앱을 서명하고 Play 스토어에 등록하여 사용자에게 전달한다.
      </li>
        <ul>
          <li>
            이 경우 개발자가 키를 <strong>분실</strong>하거나 <strong>도용</strong>되면 대처가 불가능하다는 문제가 있다.
          </li>
          <li>
            업데이트 시에도 <strong>이전 버전과 동일한 키</strong>를 사용해야 하며 완전 새로운 키를 쓰는 경우 <strong>완전 새로운 앱</strong>으로 등록이 되어버린다.
          </li>
        </ul>
      <li>
        Play 앱 서명은 위의 방식을 해결하기위해 나온 방법이다.
      </li>
        <ul>
          <li>
            <strong>구글 Play 앱 서명 방식</strong>은 키를 개발자가 만들고 관리하는 <strong>업로드 키(기존 서명 키)</strong>와 구글 Play가 만드는 <strong>앱 서명 키</strong> 두 개로 구분한다.
          </li>
          <ul>
            <li>
              업로드 키는 앱을 <strong>구글 Play에 등록</strong>할 때만 사용하고, 앱 서명 키는 Google play에서 <strong>사용자에게 전달</strong>할 때 사용한다.
            </li>
            <li>
              개발자가 키를 분실하거나 도용당해도 앱을 서명한 키는 안전하며, 업로드 키는 다시 만들어 Play 스토어에 등록하면 된다.
            </li>
          </ul>
        </ul>
    </ul>
</ul>

<br>

<h2>4-3. 구글 Play 스토어에 게시</h2>
<ul>
  <li>
    Play 스토어에 배포하기 위해 준비할 파일 목록 (p.40).
  </li>
    <ul>
      <li>
        <strong>AAB 파일</strong>: 완성한 앱을 서명한 배포 파일.
      </li>
      <li>
        <strong>앱 아이콘 이미지</strong>: Play 스토어에 표시할 앱 이미지. 
      </li>
      <li>
        <strong>그래픽 이미지</strong>: Play 스토어에서 앱을 프로모션할 때 이용할 이미지
      </li>
      <li>
        <strong>휴대전화 스크린샷</strong>: 스마트폰에서 앱을 실행한 스크린샷.
      </li>
      <li>
        <strong>7인치와 10인치 태블릿 스크린샷</strong>: 7인치나 10인치 태플릿에서 앱을 실행한 스크린 샷.
      </li>
    </ul>
  <li>
    Play 스토어 운영 정책은 수시로 바뀌기 때문에 배포할 때마다 Play 콘솔에서 안내하는 사항을 꼼꼼히 읽고 진행해야 한다.
  </li>
</ul>

<h1>5. Do it! 실습</h1>
<h2>5-1. AAB 배포 파일 만들기</h2>
<h3>5-1-1. 고유한 패키지명으로 바꾸기</h3>
<ul>
  <li>
    안드로이드 앱 배포를 위해 출시용 앱을 빌드하기 전 고유한 패키지명으로 변경한다.
  </li>
  <li>
    "Gradle Scripts/build.gradle.kts(Module :app)" 파일의 applicationId 항목을 고유한 이름으로 수정한다.
  </li>
  <li>
    수정을 완료했다면 편집 창 상단의 동기화 메뉴에서 [Sync now]를 클릭해 동기화를 하여 수정한 내용을 반영한다.
  </li>
</ul>

<h3>5-1-2. AAB 파일 생성 시작하기</h3>
<ul>
  <li>
    완성한 프로젝트가 안드로이드 스튜디오에 열린 상태로 [Build → Generate Signed App Bundle or API] → [Android App Bundle] → [Next]
  </li>
</ul>

<h3>5-1-3. 키 저장소와 서명 키(업로드 키) 만들기</h3>
<ul>
  <li>
    키 저장소(key store)는 일종의 인증서로 볼 수 있다. 이를 통해 만에 서명을 해야 앱을 출시할 수 있다.
  </li>
  <li>
    기존의 저장소가 있다면 [Choose existing]을 아니라면 [Create new]를 선택한다.
  </li>
  <li>
    이후 설정은 p42 참고.
  </li>
</ul>

<h3>5-1-4. 앱 서명하기</h3>
<ul>
  <li>
    생성한 키를 서명란에 입력한다.
  </li>
</ul>

<h3>5-1-5. 릴리즈용 빌드하기</h3>
<ul>
  <li>
    [release] → [Create]를 통해 AAB 파일로 빌드를 시작한다.
  </li>
</ul>

<h3>5-1-6. 빌드된 AAB 파일 확인하기</h3>
<ul>
  <li>
    빌드가 완료되면 프로젝트 루트 디렉터리에 "app/release/app-release.aab" 파일이 생성된다.
  </li>
  <li>
    app은 프로젝ㅌ으 모듈명에 해당하며 이 파일을 Play 스터어에 등록하여 앱을 출시한다.
  </li>
</ul>

<br>

<h2>5-2. 구글 플레이에 앱 등록하기</h2>
<ul>
  <li>
    많은 절차가 있으나 간단히 Play에 앱을 등록하는 절차만 살펴본다.
  </li>
  <li>
    자신의 구글 계정을 개발자 계정으로 등록해야 하며 25$의 비용이 든다.
  </li>
</ul>

<h3>5-2-1. 개발자 계정으로 등록하기</h3>
<ul>
  <li>
    구글 플레이 콘솔에 접속한 후 [Play Console로 이동] 후 구글 계정으로 로그인한다.
  </li>
  <li>
    과정을 잘 따라하고 25$를 지불하면 완료된다. (미래를 위한 투자...).
  </li>
  <li>
    각종 인증절차는... 등본이나 안드로이드 기기를 빌리거나 등등의 방법을 활용한다.
  </li>
</ul>

<h3>5-2-2. Play 콘솔에서 앱 만들기</h3>
<ul>
  <li>
    조금 힘든(?) 인증을 마치면 드디어 [앱 만들기]를 선택할 수 있다.
  </li>
  <li>
    기타 설정을 완료하고 [앱 만들기]를 선택한다.
  </li>
</ul>

<h3>5-2-3. Play 콘솔에서 앱 설정하기</h3>
<ul>
  <li>
    앱을 게시하는 데 필요한 각종 정보와 설정, 파일 등을 등록한다.
  </li>
  <li>
    [앱 설정 완료] 아래의 [할 일 보기]를 클릭하고 앱 콘텐츠에 관한 10가지 정보, 앱 분류와 표시되는 방식에 관한 두 가지 정보를 선택한다.
  </li>
  <li>
    구글이 요구하는 설정을 완료한다. (이미지 자료 포함).
  </li>
  <li>
    [스토어 등록정보 설정]을 선택하면 Play 스토어에서 사용자에게 보여질 앱에 대한 자세한 설명과 준비한 이미지를 등록한다.
  </li>
</ul>

<h3>5-2-4. 테스트 및 새 버전 만들기</h3>
<ul>
  <li>
    [새 버전 만들기] 이전에 <strong>[내부 테스트]</strong>와 <strong>[비공개 테스트]</strong> 중 하나를 선택해 테스터를 지정해야 한다.
  </li>
    <ul>
      <li>
        [내부 테스트]는 <strong>내부용 테스트 버전</strong>을 만들어 <strong>최대 10명의 테스터</strong>에게 앱을 빠르게 배포하여 테스트하는 방법이다.
      </li>
      <li>
        [비공개 테스트]는 내부 테스터보다 <strong>더 광범위한 테스터</strong>를 대상으로 앱을 테스트하는 방법이다.
      </li>
    </ul>
  <li>
    [테스터 선택]을 클릭하고 [이메일 목록 만들기]를 선택해 테스트에 참여할 테스터의 이메일을 등록하여 목록을 만든다.
  </li>
  <li>
    목록을 생성한 후에는 [새 버전 만들기]를 틀릭해 새 버전을 위한 정보를 입력한다.
  </li>
    <ul>
      <li>
        새 버전을 위한 App Bundle 파일을 업로드하고, 새 버전 출시명, 출시 노트 내용 등을 입력한다.
      </li>
    </ul>
</ul>
