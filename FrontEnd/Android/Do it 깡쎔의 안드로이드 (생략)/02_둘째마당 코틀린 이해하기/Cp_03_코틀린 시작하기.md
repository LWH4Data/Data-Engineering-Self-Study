<h1>1. 코틀린 언어 소개</h1>
<h2>1-1. 코틀린의 등장 배경</h2>
<ul>
  <li>
    코틀린은 젯브레인스(JetBrains)에서 오픈소스 그룹을 만들어 개말한 언어로 2011 공개되어 2017 구글에서 안드로이드 공식 언어로 지정하였다.
  </li>
    <ul>
      <li>
        Java는 인도네시아 섬 이름에서 이름을 차용했듯 Kotlin은 자바를 대체할 목적으로 러시아 섬 이름에서 유래하였다.
      </li>
    </ul>
  <li>
    코틀린도 <strong>JVM</strong>에서 실행할 수 있으며 이로 인해 안드로이드에서도 사용이 가능하다.
  </li>
  <li>
    코틀린의 확장자는 <strong>'.kt'</strong>이지만 코틀린 컴파일러(kotlinc)가 .kt 파일을 컴파일하면 <strong>자바 바이트 코드</strong>가 생성된다.
  </li>
    <ul>
      <li>
        자바 바이트 코드가 생성되기 때문에 <strong>JVM</strong>이 실행이 가능하며 자바를 대체할 목적으로 만든 언어임도 유추할 수 있다.
      </li>
    </ul>
</ul>

<h3>1-1-1. 표현력과 간결함(expressive and concise)</h3>
<ul>
  <li>
    같은 로직을 자바와 코틀린으로 작성해 보면 코틀린 코드가 훨씬 간결하다.
  </li>
</ul>

<h3>1-1-2. 안전한 코드(safer code)</h3>
<ul>
  <li>
    코틀린은 <strong>널 안전성(null safety)</strong>을 지원하는 언어이다.
  </li>
  <li>
    객체지향 프로그램에서 객체는 널 상태일 수 있고 이로인해 런타임 오류인 <strong>널 포인트 예외(NullPointException)</strong>가 발생할 수 있다.
  </li>
    <ul>
      <li>
        코틀린은 변수를 <strong>널 허용(nullable)</strong>과 <strong>널 불허용(not null)</strong>으로 구분해서 선언이 가능하며 이로인해 <strong>컴파일러</strong>가 관리를 하며 개발자가 작성하는 코드는 널 안정성을 확보한다..
      </li>
    </ul>
</ul>

<h3>1-1-3. 상호 운용성(interoperable)</h3>
<ul>
  <li>
    코틀린은 자바와 <strong>100% 호환</strong>하며 서로간에 클래스나 라이브러리를 사용할 수 있다.
  </li>
  <li>
    하나의 앱을 개발할 때에도 자바와 코틀린을 혼용해서 활용할 수 있다. 혹은 이미 자바로 개발을 하였어도 유지보수 때 추가 코드를 코틀린으로 작성할 수 있다.
  </li>
</ul>

<h3>1-1-4. 구조와 동시성(structured concurrency)</h3>
<ul>
  <li>
    코틀린이 제공하는 <strong>코루틴(coroutines)</strong> 기법을 이용하면 비동기 프로그래밍을 간소화할 수 있다.
  </li>
</ul>

<br>

<h2>1-2. 코틀린 파일 구성</h2>
<ul>
  <li>
    코틀린 파일의 확장자는 '.kt'이다.
  </li>
  <li>
    코틀린은 자바와 달리 <strong>파일명</strong>과 <strong>클래스명</strong>이 일치하지 않아도 된다.
  </li>
    <ul>
      <li>
        코틀린은 객체지향만 지원하는 언어가 아니기 때문이다. 따라서 <strong>변수, 함수 등</strong>을 클래스로 묶지 않고 <strong>최상위</strong>에 선언할 수 있다.
      </li>
      <li>
        자바는 객체지향 언어이기 때문에 모든 것이 <strong>클래스</strong>로 묶여야 한다.
      </li>
    </ul>
  <li>
    코틀린 컴파일러는 컴파일 시에 소스 파일 <strong>최상위에 선언한 변수와 함수를 포함</strong>하여 클래스로 만들기 때문에 자바에서도 코틀린으로 작성한 함수나 변수를 활용할 수 있다.
  </li>
</ul>

```kotlin
// 파일을 컴파일했을 때 만들어지는 클래스 파일의 위치. 
//   - 소스 파일의 맨 처음 한 줄로 선언한다.
//   - 위치와 상관없는 별도의 이름으로 선언할 경우 폴더를 생성하게 된다.
package com.example.test3

// 필요한 클래스를 가져온다.
//   - 같은 package에 선언한 클래스는 import 없이 사용할 수 있다.
import java.text.SimpleDateFormat
import java.util.*

// 변수와 함수는 클래스 안과 밖(최상위)에서도 선언이 가능하다.
var data = 10

fun formatDate(date: Date): String {
    val sdformat = SimpleDateFormat("yyyy-mm-dd")
    return sdformat.format(date)
}

class User {
    var name = "hello"

    fun sayHello() {
        println("name : $name")
    }
}
```

<h1>2. 변수와 함수</h1>
<h2>2-1. 변수 선언하기</h2>
<ul>
  <li>
    코틀린에서 변수는 <strong>val</strong>과 <strong>var</strong> 키워드로 선언한다.
  </li>
    <ul>
      <li>
        val은 value의 줄임말로 초깃값이 할당되면 <strong>바꿀 수 없는 변수</strong>를 선언할 때 사용한다.
      </li>
      <li>
        var는 variable의 줄임말로 초깃값이 할당된 후에도 값을 <strong>바꿀 수 있는</strong> 변수를 선언할 때 사용한다.
      </li>
    </ul>
  <li>
    val(혹은 var) 변수명: 타입=값
  </li>
</ul>

```kotlin
// 1. var와 val의 차이.
val data1 = 10
var data2 = 10

fun main() {
    data1 = 20  // 오류!
    data2 = 20  // 성공!
}
```

<h3>2-1-1. 타입 지정과 타입 추론</h3>
<ul>
  <li>
    변수명 뒤에 <strong>콜론(:)</strong>을 추가해 <strong>타입을 명시</strong>할 수 있다. <strong>추론</strong>이 가능한 경우에는 <strong>생략</strong>이 가능하다.
  </li>
</ul>

```kotlin
// 1. 변수에 타입 지정과 타입 추론.
// 1-1. 타입 지정.
val data1: Int = 10
// 1-2. 타입 추론
val data2 = 10
```

<h3>2-1-2. 초깃값 할당</h3>
<ul>
  <li>
    <strong>최상위</strong>에 선언한 <strong>변수나 클래스의 멤버 변수</strong>는 선언과 동시에 <strong>초깃값</strong>을 할당해야 한다.
  </li>
    <ul>
      <li>
        함수 내부에 선언한 변수는 선언시에 초깃값을 할당할 필요는 없으나 나중에 <strong>사용하기 전</strong>에는 할당해야 한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 초깃값 할당.
val data1: Int  // 오류! - 값을 초기화 안함.
val data2 = 10  // 성공!

fun  someFun() {
    val data3: Int
    println("data3: $data3")   // 오류! - 값을 초기화 안함.
    data3 = 10
    println("data3 : $data3")  // 성공! - 값을 10으로 초기화함.
}

class User {
    val data4: Int       // 오류! - 값을 초기화 안함.
    val data5: Int = 10  // 성공!
}
```

<h3>2-1-3. 초기화 미루기</h3>
<ul>
  <li>
    변수르 선언할 때 초깃값을 할당할 수 없는 경우가 있는데 이떄에는 <strong>lateinit</strong>이나 <strong>lazy 키워드</strong>를 통해 값을 이후헤 할당할 것을 <strong>컴파일러</strong>에게 알려야 한다.
  </li>
  <li>
    <strong>lateinit</strong>은 <strong>var 키워드</strong>로 선언한 변수에만 사용할 수 있다. 반면 Int, Long, Short, Double, Float, Boolean, Byte에는 사용이 불가하다.
  </li>
  <li>
    <strong>lazy 키워드</strong>는 변순 선언문 뒤에 <strong>"by lazy { }"</strong> 형식으로 선언한다.
  </li>
    <ul>
      <li>
        소스에서 변수가 최초로 이용되는 순간 <strong>중괄호 내의 코드</strong>가 실행되며 <strong>실행 결과값</strong>이 초깃값으로 할당된다.
      </li>
      <li>
        여러 중괄호가 있다면 <strong>가장 마지막 중괄호의 실행 결과</strong>가 변수의 초깃값이 된다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 초기화 미루기1 - lateinit 키워드.
lateinit var data1: Int     // 오류! - Int 선언.
lateinit val data2: String  // 오류! - val 선언.
lateinit var data3: String  // 성공!
```

```kotlin
// 2. 초기화 미루기2 - lazy 키워드.
val data4: Int by lazy {
    println("in lazy......")
    10
}

fun main() {
    println("in main......")
    println(data4 + 10)
    println(data4 + 10)
}

/*
<실행 결과>
in main......
in lazy......
20
20
*/
```

<h2>2-2. 데이터 타입</h2>
<ul>
  <li>
    코틀린의 모든 변수는 객체이고 따라서 <strong>모든 타입 또한 객체</strong>이다.
  </li>
    <ul>
      <li>
        예를 들어 정수를 다루는 데이터 타입인 Int도 기초 데이터 타입(primitive type)이 아니라 <strong>클래스</strong>이다.
      </li>
    </ul>
</ul>

```kotlin
// 1. int 타입에 null 대입과 메서드 이용.
fun someFun() {
    var data1: Int = 10
    var data2: Int? = null  // null 대입 가능.

    data1 = data1 + 10 
    data1 = data1.plus(10)  // 객체의 메서드 이용 가능.
}
```

<h3>2-2-1. Int, Short, Long, Double, Float, Byte, Boolean - 기초 타입 객체</h3>
<ul>
  <li>
    기초 데이터를 <strong>객체</strong>로 표현하는 타입이다.
  </li>
    <ul>
      <li>
        정수: Int, Short, Long
      </li>
      <li>
        실수: Double, Float
      </li>
      <li>
        2진수: Byte
      </li>
      <li>
        T/F: Boolean
      </li>
    </ul>
</ul>

<h3>2-2-2. Char, String - 문자와 문자열</h3>
<ul>
  <li>
    <strong>Char</strong>는 <strong>문자</strong>를 표현하는 타입으로 코틀린에서는 <strong>작은따옴표(')</strong>로 감싸서 표현하며 <strong>Number 타입</strong>으로 표현할 수 없다.
  </li>
  <li>
    <strong>String</strong>은 <strong>문자열</strong>을 표현하는 타입으로 <strong>큰따옴표(")</strong>나 <strong>삼중 따옴표(""")</strong>로 감싸서 표현한다.
  </li>
    <ul>
      <li>
        <strong>큰따옴표</strong>로 표현한 문자열에서 Enter 혹은 Tab을 활용한 <strong>줄 바꿈</strong>이나 <strong>들여쓰기</strong> 등을 유지하려면 <strong>역슬래시(\)</strong>로 시작하는 <strong>이스케이프 시퀀스(escape sequence)</strong>를 입력해야 한다.
      </li>
      <li>
        삼중 따옴표의 경우 자동 완성처럼 <strong>".trimIndent()"</strong>가 붙는데 이는 <strong>공통적으로 적용되는 문자 앞의 공백</strong>을 제거한다.
      </li>
    </ul>
  <li>
    String 타입의 데이터에 <strong>변숫값</strong>이나 어떤 <strong>연산식의 결과</strong>를 포함해야할 때에는 <strong>'$' 기호</strong>를 이용하며 이를 <strong>문자열 템플릿(string template)</strong>이라 한다.
  </li>
</ul>

```kotlin
// 1. 문자 표현.
val a: Char = 'a'
if (a == 1) {  // 오류! - Number 타입으로 표현 불가.

}

// 2. 문자열 표현 - 큰따옴표와 삼중 따옴표의 차이
fun main() {
    // 큰 따옴표는 이스케이프 시퀀스 필요.
    val str1 = "Hello \n World"
    // 삼중 따옴표는 이스케이프 시퀀스 없어도 됨.
    val str2 = """
        Hello
        World
    """
    println("str1: $str1")
    println("str2: $str2")
}

// 3. 문자열 템플릿 사용 예.
fun main() {
    fun sum(no: Int): Int {
        var sum = 0
        for (i in 1..no) {
            sum += i
        }
        return sum
    }

    val name: String = "kkang"
    println("name : $name, sum : ${sum(10)}, plus : ${10 + 20}")
    /*
    < 실행 결과 >
    name : kkang, sum : 55, plus : 30
    */
}
```

<h3>2-2-4. Unit - 반환문이 없는 함수</h3>
<ul>
  <li>
    <strong>Unit</strong>은 다른 타입과 다르게 데이터의 형식이 아니라 <strong>특수한 상황</strong>을 표현하려는 목적으로 사용한다.
  </li>
  <li>
    Unit 타입으로 선언한 변수에는 <strong>Unit 객체만 대입</strong>할 수 있다.
  </li>
    <ul>
      <li>
        따라서 Unit 타입으로 변수를 선언할 수는 있지만 의미는 없다.
      </li>
      <li>
        주로 <strong>함수에서 반환문이 없음</strong>을 명시적으로 나타낼 때 Unit 타입을 사용한다.
      </li>
      <li>
        함수를 선언할 때 반환 타입을 생략하면 <strong>자동으로 Unit이 적용</strong>된다.
      </li>
    </ul>
</ul>

```kotlin
// 1. Unit 타입 사용 예.
val data1: Unit = Unit

// 2. Unit 타입 사용 예 - 반환문이 없는 함수.
fun some(): Unit {
    println(10 + 20)
}

// 3. 반환 타입을 생략한 예. (2와 같음).
fun sum() {
    println(10 + 20)
}
```

<h3>2-2-5. Nothing - null이나 예외를 반환하는 함수</h3>
<ul>
  <li>
    Nothing도 특수한 상황에 사용하며 <strong>null만 대입</strong>할 수 있다. 즉, 데이터로서의 의미는 없다.
  </li>
  <li>
    Nothing은 주로 <strong>함수의 반환 타입</strong>에 사용하며 반환은 하지만 의미 있는 값은 아니라는 의미이다.
  </li>
    <ul>
      <li>
        <strong>항상 null</strong>만 반환하는 함수라든가 <strong>예외</strong>를 던지는 함수의 반환 타입으로 선언한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. nul 반환 함수와 예외를 던지는 함수.
fun some1(): Nothing? {
    return null
}
fun some2(): Nothing {
    throw Exception()
}
```

<h3>2-2-6. 널 허용과 불허용</h3>
<ul>
  <li>
    코틀린은 모든 타입이 <strong>객체</strong>이기 때문에 null을 대입할 수 있고, 이로인해 <strong>nullable(null 가능)</strong>인지 <strong>not null(null 불가능)</strong>인지 구분해서 선언해야 한다.
  </li>
  <li>
    타입 뒤에 <strong>물음표(?)</strong>를 추가하면 null 허용, 추가하지 않으면 null을 불허용한다는 의미이다.
  </li>
</ul>

```kotlin
// 1. 널 허용과 불허용.
var data1: Int = 10
data1 = null  // 오류!

var data2: Int? = 10
data2 = null  // 성공!
```

<h2>2-3. 함수 선언하기</h2>
<ul>
  <li>
    코틀린은 함수를 선언할 때 <strong>fun</strong> 키워드를 활용한다.
  </li>
    <ul>
      <li>
        fun 함수명(매개변수명: 타입): 반환 타입 { ... }
      </li>
    </ul>
  <li>
    함수에 반환 타입을 선언할 수 있고 생략하면 자동으로 Unit 타입으로 적용된다.
  </li>
  <li>
    함수 매개변수는 <strong>val</strong>이 자동으로 적용되며 var나 val 키워드를 사용할 수 없다.
  </li>
  <li>
    함수의 매개변수에는 <strong>기본값(default value)</strong>을 선언할 수 있다. 
  </li>
    <ul>
      <li>
        기본값을 선언한 경우 호출할 때 <strong>인자(argument)</strong>를 전달하지 않아도 된다. (기본값이 적용됨).
      </li>
    </ul>
  <li>
    어떤 함수의 매개변수가 여러 개면 호출할 때 전달할 인자를 <strong>순서대로 할당</strong>한다.
  </li>
    <ul>
      <li>
        <strong>매개변수명</strong>을 지정하여 호출하는 것을 <strong>명명된 매개변수(named parameter)</strong>라 하며 매개변수 순서에 맞춰 호출하지 않아도 된다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 반환 타입이 있는 함수 선언.
fun some(data1: Int): Int {
    return data1 = 10
}
```

```kotlin
// 2. 매개변숫값 변경 오류.
fun some(data1: Int) {
    data1 = 20  // 오류!
}
```

```kotlin
// 3. 기본값 활용.
fun main() {
    fun some(data1: Int, data2: Int = 10): Int {
        return data1 * data2
    }
    println(some(10))
    println(some(10, 20))
}

/* < 실행 결과 >
100
200
*/
```

```kotlin
// 4. 매개변수명 생략 - 매개변수 순서대로 할당.
fun some(data1: Int, data2: Int): Int {
    return data1 * data2
}
println(some(10, 20))

// 5. 매개변수명을 지정하여 호출.
```

<h3>2-4-1. Array - 배열 표현</h3>
<ul>
  <li>
    코틀린의 배열은 Array 클래스로 표현한다.
  </li>
  <li>

</ul>

<h3>2-4-3. List, Set, Map</h3>
<ul>
  <li>
    List, Set, Map은 <strong>Collection 인터페이스</strong>를 타입으로 표현한 클래스이며 통틀어 <strong>컬렉션 타입 클래스</strong>라고 한다.
  </li>
    <ul>
      <li>
        <strong>List</strong>: 순서 O, 중복 허용 O
      </li>
      <li>
        <strong>Set</strong>: 순서 X, 중복 허용 X
      </li>
      <li>
        <strong>Map</strong>: 키와 값으로 이루어진 데이터 집합. 순서 X, 키 중복 허용 X
      </li>
    </ul>
  <li>
    Collection 타입의 클래스는 <strong>가변(mutable) 클래스</strong>와 <strong>불변(immutable) 클래스</strong>로 나뉜다.
  </li>
    <ul>
      <li>
        가변 클래스와 불변 클래스는 <strong>초기 대입한 데이터를 변경할 수 있는가 여부</strong>를 기준으로 나뉜다.
      </li>
    </ul>
  <li>
    코틀린은 List, Set, Map에 있어 가변과 불변 <strong>두 가지 타입의 클래스</strong>를 제공한다.
  </li>
    <ul>
      <li>
        불변 클래스는 <strong>MutableList, MutableSet, MutableMap</strong> 이며 <strong>Of() 함수</strong> 또한 제공한다.
      </li>
      <li>
        불면 클래스는 <strong>size(), get() 함수</strong>만을, 가변 클래스는 여기에 <strong>add(), set() 함수</strong>까지 제공한다.
      </li>
    </ul>
  <li>
    Map 객체는 <strong>키와 값</strong>으로 이루어진 데이터 집합으로 Map 객체의 키와 값은 <strong>Pair 객체</strong>를 이용할 수도 있고 <strong>'키 to 값'</strong> 형태로 이용할 수도 있다.
  </li>
</ul>

<h1>3. 조건문과 반복문</h1>
<h2>3-1. 조건문 if~else와 표현식</h2>
<ul>
  <li>
    코틀린에서는 if~else를 <strong>표현식(expression)</strong>으로도 사용할 수 있다.
  </li>
    <ul>
      <li>
        표현식이란 <strong>결괏값</strong>을 반환하는 계산식을 의미한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. if~else 문 사용 예.
fun main() {
    var data = 10
    if (data > 0) {
        println("data > 0")
    } else {
      println("data <= 0")
    }
}

// 2. 조건을 여러 개 나열한 예.
fun main() {
  var data = 10
  if (data > 10) {
      println("data > 10")
  } else if (data > 0 && data <= 10) {
      println("data > 0 && data <= 10")
  } else {
    println("data <= 0")
  }
}

// 3. 표현식으로 사용 예.
//   - if-else문을 표현식으로 사용할 때 else를 생략할 수 없다.
//   - if~else 표현식이 반환하는 결괏값은 각 영역의 마지막 줄에 해당한다.
//     - if나 else 부분이 여럿이어도 마지막 줄의 실행 결과만 반환.
fun main() {
    var data = 10
    val result = if (data > 0) {
        println("data > 0")
        // 참일 때 반환.
        true
    } else {
        println("data <= 0")
        // 거짓일 때 반환.
        false
    }
    println(result)
}
```

<h2>3-1-1. 조건문 when</h2>
<ul>
  <li>
    코틀린은 if~else 문 이외에도 <strong>when</strong>이라는 조건문을 작성할 수 있다.
  </li>
  <li>
    when 조건문은 <strong>()안에 들어간 데이터</strong>가 조건이 되고 ()안의 값에 따라 각 구분을 실행한다.
  </li>
    <ul>
      <li>
        ()안에 들어가는 조건 데이터는 타입과 범위 등을 다양하게 명시할 수 있다.
      </li>
      <li>
        when 문도 표현식으로 사용할 때에는 else 문을 생략할 수 없다.
      </li>
    </ul>
  <li>
    <strong>is</strong>는 <strong>타입</strong>을 확인하는 연산자이고, <strong>in</strong>은 <strong>값의 범위</strong> 지정 연산자이다. 
  </li>
  <li>
    when 문을 이용하면 데이터를 명시하지 않고 조건만 명시할 수 있다.
  </li>
</ul>

```kotlin
// 1. when 문 사용 예.
fun main() {
    var data = 10
    when (data) {
        10 -> println("data is 10")
        20 -> println("data is 20")
        else -> {
            println("data is not valid data")
        }
    }
}

// 2. 문자열 타입을 조건으로 사용.
fun main() {
    var data = "hello"
    when(data) {
        "hello" -> println("data is hello")
        "world" -> println("data is world")
        else -> {
            println("data is not valid data")
        }
    }
}

// 3. 다양한 유형의 조건 제시.
fun main() {
    var data: Any = 10
    when (data) {
        is String -> println("data is String")  // data가 문자열 타입인 경우.
        20, 30 -> println("data is 20 or 30")   // data가 20 또는 30인 경우.
        in i..10 -> println("data is 1..10")    // data가 1~10의 값인 경우.
        else -> println("data is not valid")
    }
}

// 4. 조건만 명시한 예.
fun main() {
    var data = 10
    when {
        data <= 0 -> println("data is <= 0")
        data > 100 -> println("data is > 100")
        else -> println("data is valid")
    }
}

// 5. when 문을 표현식으로 사용.
fun main() {
    var data = 10
    // when 문을 표현식으로 사용.
    val result = when {
        data <= 0 -> "data is <= 0"
        data > 100 -> "data is > 100"
        // when 문을 표현식으로 사용하면 else 문 생략 불가.
        else -> "data is valid"
    }
    println(result)
}
```

<h2>3-2. 반복문 for와 while</h2>
<ul>
  <li>
    코틀린은 for 문의 조건에서 주로 <strong>범위 연산자 in</strong>을 사용한다.
  </li>
  <li>
    증감 조건을 활요하지 않고 <strong>indices 속성</strong>을 통해 <strong>컬렉션 타입의 데이터 개수</strong>만큼 반복할 수도 있다.
  </li>
  <li>
    for 문을 반복하면서 <strong>인덱스와 실제 데이터</strong>를 함께 가져오려면 <strong>whitIndex()</strong> 함수를 사용해야 한다.
  </li>
    <ul>
      <li>
        indices는 컬렉션 타입의 <strong>인덱스값</strong>을 의미하기 때문에 for 문을 반복하며 0, 1, 2 값을 i에 대입한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. for ~ in 반목문.
fun main() {
    var sum: Int = 0
    for (i in 1..10) {
        sum += i
    }
    println(sum)
}

// 2. 다양한 for 조건 표현.
for (i in 1..10) { println(i) }        // 1부터 10까지 1씩 증가.
for (i in 1 until 10) { println(i) }   // 1부터 9까지 1씩 증가 (10 미포함).
for (i in 2..10 step 2) { println(i) } // 2부터 10까지 2씩 증가.
for (i in 10 downTo 1) { println(i) }  // 10부터 1까지 1씩 감소.

// 3. 반복 조건에 컬렉션 타입 활용.
fun main() {
    var data = arrayOf<Int>(10, 20, 30)
    for (i in data.indices) {
        print(data[i])
        if (i !== data.size - 1) print(",")
    }
}

// 4. 인덱스와 데이터를 가져오는 withindex() 함수.
fun main() {
    var data = arrayOf<Int>(10, 20, 30)
    for ((index, value) in data.withIndex()) {
        print("$index : $value")
        if (index !== data.size - 1) print(", ")
    }
}

// 5. while 반복문.
fun main(args: Array<String>) {
    var x = 0
    var sum1 = 0
    while(x < 10) {
        sum1 += ++x
    }
    println(sum1)
}
```
