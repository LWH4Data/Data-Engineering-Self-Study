<h1>1. 화면을 구성하는 방법</h1>
<h2>1-1. 액티비티-뷰 구조</h2>
<ul>
  <li>
    안드로이드 앱의 기본 구조는 액티비티, 서비스, 브로드캐스트 리시버, 콘텐츠 프로바이더와 같은 컴포넌트를 적절히 조합해서 만든다.
  </li>
    <ul>
      <li>
        각 컴포넌트의 종류 중 <strong>화면</strong>을 출력하는 컴포넌트는 <strong>액티비티</strong>뿐이다.
      </li>
        <ul>
          <li>
            화면 없이 <strong>백그라운드</strong>에서 실행하도록 개발한다면 액티비티 없이 서비스를 만들 수도 있다.
          </li>
          <li>
            10개의 화면을 만들 때 10개 액티비티를 만들 수도 있지만, <strong>프래그먼트(Fragment)</strong> 혹은 <strong>컴포즈(Compose)</strong>를 이용해 만들 수도 있다.
          </li>
        </ul>
    </ul>
  <li>
    액티비티는 <strong>화면을 관리</strong>하는 컨트롤러이다. 따라서 별도 화면 구성 없이 액티비티만 실행하면 <strong>텅 빈 흰색 화면</strong>만 보인다.
  </li>
    <ul>
      <li>
        화면에 내용을 표시하기 위해서는 <strong>뷰(view) 클래스</strong>를 이용해 구성해야 한다.
      </li>
    </ul>
</ul>

<br>

<h2>1-2. 액티비티 코드로 화면 구성하기</h2>
<ul>
  <li>
    액티비티에서 뷰로 화면을 구성하는 방법은 <strong>액티비티 코드</strong>로 작성하는 방법과 <strong>레이아웃 XML 파일</strong>로 작성하는 방법 두 가지이다.
  </li>
    <ul>
      <li>
        액티비티 코드롤 작성할 때에는 화면을 구성하는 <strong>뷰 클래스를 액티비티 코드에서 직접 생성</strong>한다.
      </li>
    </ul>
  <li>
    호출하려는 함수가 고차 함수이고 마지막 인자가 람다 함수인 경우 소괄호를 생략할 수 있다.
  </li>
  <li>
    필기는 "app/kotlin+java/com.example.androidlap/MainActivity" 참고.
  </li>
</ul>

```kotlin
// 1. 호출할 고차 함수.
fun some(arg1: () -> Unit) {
} 

// 2. 위에 선언한 some을 호출할 때에는 고차 함수이면서 람다 함수 매개변수를 갖기에
..    () 생략 가능.
some ({ println("hello") })
some { println("hello") }

// 3. 매개변수가 2개 이상인 함수.
fun some(arg1: Int, arg2: () -> Unit, arg3: () -> Unit) {
}

// 4. 호출 예.
// 성공 - 모두 소괄호로 묶음.
some(10, { println("hello")}, {println("world")})
// 성공 - 마지막 람다 함수만 미포함.
some(10, { println("hello")}) {println("world")}
// 실패 - 마지막 람다 함수 외에도 () 미포함.
some 10, { pritln("hello")}, {println("world")}
// 실패 - 두 번째 람다 함수 인자까지 미포함.
some(10) {println("hello")} {println("world")}
```

<h1>2. 뷰 클래스</h1>
<h2>2-1. 뷰 클래스의 기본 구조</h2>
<ul>
  <li>
    액티비티를 실행하면 <strong>뷰 클래스</strong>를 이용해 <strong>화면을 구성</strong>한다.
  </li>
  <li>
    안드로이드는 TextView, ImageView, EditText, Button, Spinner, ListView 등 다양한 큐 클래스를 제공한다.
  </li>
</ul>

<h3>2-1-1. 뷰 객체의 계층 구조</h3>
<ul>
  <li>
    액티비티 화면을 구성할 때 사용하는 클래스는 모두 <strong>View의 하위 클래스</strong>이며 따라서 화면 구성과 관련한 클래스를 통칭하여 <strong>뷰 클래스</strong>라 한다.
  </li>
  <li>
    뷰 클래스는 <strong>계층 구조</strong>를 나타내며 각 클래스의 역할은 다음과 같다.
  </li>
    <ul>
      <li>
        <strong>View</strong>
      </li>
        <ul>
          <li>
            모든 뷰 클래스의 <strong>최상위 클래스</strong>이다.
          </li>
          <li>
            액티비티는 View의 <strong>서브 클래스</strong>만 화면에 출력한다.
          </li>
        </ul>
      <li>
        <strong>ViewGroup</strong>
      </li>
        <ul>
          <li>
            View의 하위 클래스지만 자체 UI가 없어 화면에 출력해도 <strong>아무것도 나오지 않는다</strong>.
          </li>
          <li>
            다른 <strong>뷰 여러 개를 묶어서 제어</strong>할 목적으로 사용하며 일종의 <strong>컨테이너 기능</strong>을 담당한다 볼 수 있다.
          </li>
          <li>
            실제로는 ViewGroup의 서브 클래스인 <strong>레이아웃 클래스</strong>를 사용한다.
          </li>
        </ul>
      <li>
        <strong>TextView</strong>
      </li>
        <ul>
          <li>
            특정 IO를 출력할 목적으로 사용하는 클래스로 <strong>문자열을 출력하는 뷰</strong>이다.
          </li>
          <li>
            TextView 이외에도 다양한 클래스가 존재한다.
          </li>
        </ul>
    </ul>
  <li>
    실습은 "app/res/layout/view_prac_01.xml"에서 진행한다.
  </li>
</ul>

<h3>2-1-2. 레이아웃 중첩</h3>
<ul>
  <li>
    뷰의 계층 구조는 <strong>레이아웃 객체를 중첩</strong>하여 복잡하게 구성할 수도 있다.
  </li>
  <li>
    계층 구조를 만들어 이용하는 패턴을 <strong>컴포지트 패턴(composite pattern)</strong> 또는 <strong>문서 객체 모델(document object model)</strong>이라고 한다.
  </li>
  <li>
    화면을 구현하는 프론트엔드 애플리케이션을 개발할 때에는 여러 객체를 계층으로 묶거나 레이아웃처럼 컨테이너 역할을 하는 클래스를 이용해 <strong>계층을 중첩하는 구조를 많이 사용</strong>한다.
  </li>
</ul>

<br>

<h2>2-2. 레이아웃 XML의 뷰를 코드에서 사용하기</h2>
<ul>
  <li>
    화면 구성을 <strong>레이아웃 XML 파일</strong>에 작성하고 액티비티에서 <strong>setContentView() 함수</strong>로 XML 파일을 지정하여 화면에 출력한다.
  </li>
  <li>
    XML에 선언한 객체를 코드에서 사용할 때에 이를 지칭할 수 있는 <strong>식별자</strong>가 필요하다. 식별자는 <strong>id 속성</strong>으로 지정한다.
  </li>
    <ul>
      <li>
        "@+id/text1" 형테로 id를 추가하며 text1이 아이디가 된다.
      </li>
      <li>
        id는 꼭 지정해야 하는 속성은 아니다.
      </li>
      <li>
        XML에 id 속성을 추가하면 자동으로 <strong>R.java 파일</strong>에 상수 변수로 추가된다.
      </li>
        <ul>
          <li>
            <strong>"@"</strong>로 시작하면 R.java 파일을 의미한다.
          </li>
        </ul>
    </ul>
  <li>
    id를 지정한 후에는 <strong>findViewById()</strong> 함수로 R.java에서 지정된 id의 상수 변수를 객체로 얻을 수 있다.
  </li>
    <ul>
      <li>
        findViewById() 함수로 얻은 뷰 객체의 타입을 <strong>제네릭</strong>으로 명시할 수도 있다.
      </li>
    </ul>
  <li>
    <strong>setContentView()</strong>는 액티비티의 <strong>화면을 출력</strong>하는 함수로 호출할 시에 XML의 내용이 액티비티 화면에 출력된다.
  </li>
</ul>

```xml
<!-- 
1. XML에 구현한 id 속성과 텍스트 뷰 객체 
  - ""@+id/text1"" 형태로 추가되며 "text1"이 id가 된다.
-->
<TextView
    android:id="@+id/text1"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="hello" />
```

```kotlin
// 2. 코드에서 XML에 입력한 객체 사용법.
// XML 화면 출력.
setContentView(R.layout.activity_main)
// id 값으로 뷰 객체 획득.
val textView1: TextView = findViewById(R.id.text1)

// 3. 제네릭으로 가져온 뷰 객체.
// XML 화면 출력, 뷰 객체 생성 완료.
setContentView(R.layouy.activity_main)
// id값으로 뷰 객체 획득.
val textView1 = findViewById<TextView>(R.id.text)
```

<h2>2-3. 뷰의 크기를 지정하는 방법</h2>
<ul>
  <li>
    크기 설정은 생략할 수 없는 속성이며 <strong>layout_width</strong>와 <strong>layout_height</strong>로 지정한다.
  </li>
  <li>
    크기를 나타내는 속성값에는 <strong>수치, match_parent, wrap_content</strong> 중 하나를 선언한다.
  </li>
    <ul>
      <li>
        <strong>수치</strong>로 선언할 경우 <strong>100px</strong> 등으로 선언하며 이때 <strong>단위(px, dp 등)는 생략할 수 없다</strong>.
      </li>
      <li>
        <strong>match_parent</strong>는 부모의 크기 전체, 즉 뷰의 계층 구조 중 <strong>자신보다 상위 계층의 크기</strong>를 뜻한다.
      </li>
      <li>
        <strong>wrap_content</strong>는 <strong>문자열</strong>이 출력될 정도의 크기를 의미한다.
      </li>
      <li>
        실습은 "app/res/layout/view_prac_size.xml"에 정리한다.
      </li>
    </ul>
  <li>
    일반적으로 안드로이드 <strong>기기의 크기가 다양</strong>하기 때문에 width와 height는 수치보다 wrap_content 혹은 match_parent로 작성한다.
  </li>
</ul>

```xml
<!-- 
1. 필수 속성인 크기
-->
<TextView
    android:id="@+id/text1"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="hello" />
```

<h2>2-4. 뷰의 간격 설정</h2>
<ul>
  <li>
    뷰의 간격은 <strong>margin</strong>과 <strong>padding</strong> 속성으로 설정한다.
  </li>
    <ul>
      <li>
        margin 속성은 <strong>뷰와 뷰 사이의 간격</strong>을 의미한다.
      </li>
      <li>
        padding 속성은 <strong>뷰의 콘텐츠와 테두리 사이의 간격</strong>을 의미한다.
      </li>
    </ul>
  <li>
    padding과 margin 속성은 <strong>네 방향의 간격이 모두 같은 크기</strong>로 설정된다.
  </li>
    <ul>
      <li>
        특정 방향의 간격만 설정하고 싶다면 paddingLeft, paddingRight, paddingTop, paddingBottom과 layout_marginLeft, layout_marginRight, layout_marginTop, layout_marginBottom 속성을 이용한다.
      </li>
    </ul>
  <li>
    실습은 "app/res/layout/view_prac_mp.xml"에 작성한다.
  </li>
</ul>

<h2>2-5. 뷰의 표시 여부 설정</h2>
<ul>
  <li>
    <strong>visibility 속성</strong>은 <strong>뷰가 화면에 출력되어야 하는지</strong>를 설정한다.
  </li>
  <li>
    visibility는 <strong>visible, invisible, gone</strong>으로 설정하며 visible이 기본값이다.
  </li>
    <ul>
      <li>
        <strong>visible</strong>인 경우만 뷰가 화면에 출력되고 <strong>invisible이나 gone</strong>인 경우 출력되지 않는다.
      </li>
        <ul>
          <li>
            <strong>invisible</strong>은 뷰가 화면에 보이지 않지만 <strong>자리를 차지</strong>하고, <strong>gone</strong>은 화면도 보이지 않고 <strong>자리도 차지하지 않는다</strong>.
          </li>
        </ul>
    </ul>
  <li>
    gone 옵션은 의미가 없어 보이지만 <strong>처음에는 화면에 보이지 않다가<strong> 어떤 순간이 되면 보이게 처리</strong>하게 할 때 사용한다.
  </li>
    <ul>
      <li>
        XML이 아닌 코드에서 뷰의 visibility 속성을 조정할 때에는 속성값을 <strong>View.VISIBLE</strong>이나 <strong>View.INVISIBLE</strong>로 설정한다.
      </li>
    </ul>
  <li>
    뷰의 속성을 조정할 때 <strong>setter 함수</strong>를 활용할 수도 있다.
  </li>
    <ul>
      <li>
        하지만 코틀린은 자바와 달리 필드(field)가 아니라 <strong>프로퍼티(property)</strong>이며 <strong>변수와 세터가 내장</strong>돼 있기에 변수처럼 이용해도 내부적으로 알아서 getter와 setter가 호출된다.
      </li>
    </ul>
  <li>
    실습 코드는 "app/res/layout/view_prac_visibility.xml"에서 진행한다.
  </li>
</ul>

```kotlin
// 1. 코드에서 visibility 속성값 변경.
visibleBtn.setOnClickListener {
    targetView.visibility = View.VISIBLE
}
invisibleBtn.setOnClickListener {
    targetView.visibility = View.INVISIBLE
}
```

```java
// 2. 코틀린의 프로퍼티와 자바의 필드 차이 
// 자바로 생성된 클래스.
public class User {
    private String name;
    public String getName() {
        return name;
    }
    public void setName(String name) {
        this.name = name;
    }
}
```

```java
// 2-1. 자반에서 객체 활용.
// 직접 getter와 setter를 활용.
user.setName("kkang")
println(user.getName())
```

```kotlin
// 2-2. 코틀린으로 객체 활용.
// 자동으로 getter와 setter가 호출됨.
user.name="kkang"
println(user.name)
```

<h1>3. 기본적인 뷰 살펴보기</h1>
<h2>3-1. 텍스트 뷰</h2>
<ul>
  <li>
    <strong>TextView</strong>: 화면에 문자열을 출력한다.
  </li>
    <ul>
      <li>
        <strong>android:text 속성</strong>
      </li>
        <ul>
          <li>
            TextView에 출력할 문자열을 지정한다.
          </li>
          <li>
            android:text="helloworld"와 같은 <strong>문자열</strong>도 대입이 가능하고, android:text="@string/hello" 처럼 <strong>문자열 리소스</strong>도 지정이 가능하다.
          </li>
        </ul>
      <li>
        <strong>android:textColor 속성</strong>
      </li>
        <ul>
          <li>
            문자열의 색상을 지정한다.
          </li>
          <li>
            android:textColor="#FF0000"과 같이 <strong>16진수 RGB 형식</strong>을 사용한다.
          </li>
        </ul>
      <li>
        android:textSize 속성
      </li>
        <ul>
          <li>
            문자열의 크기를 지정한다.
          </li>
          <li>
            android:textSize="20sp"와 같이 <strong>숫자</strong>를 사용하며 <strong>단위</strong>는 생략할 수 없다.
          </li>
          <li>
            단위에는 px, dp, sp 등을 사용한다.
          </li>
        </ul>
      <li>
        <strong>android:textStyle 속성</strong>
      </li>
        <ul>
          <li>
            문자열의 스타일을 지정한다.
          </li>
          <li>
            android:textStyle="bold"와 같이 활용하며 <strong>bold, italic, normal</strong> 중에 선택한다.
          </li>
        </ul>
      <li>
        <strong>android:autoLink 속성</strong>
      </li>
        <ul>
          <li>
            TextView에 출력할 문자열을 분석하여 특정 형태의 문자열에 <strong>자동 링크를 추가</strong>한다. 예를 들어 android:autoLink="web"으로 설정할 수 있다.
          </li>
          <li>
            autoLink의 속성값으로는 <strong>web, phone, email</strong> 등을 사용할 수 있으며 여러 개를 한 번에 설정할 때에는 <strong>"|" 기호</strong>로 연결한다.
          </li>
        </ul>
      <li>
        <strong>android:maxLines 속성</strong>
      </li>
        <ul>
          <li>
            TextView에 문자열을 출력할 때 긴 문자열은 자동으로 <strong>줄바꿈</strong>을 한다. 예를 들어 "android:maxLines=3"으로 설정하면 3행까지만 출력된다.
          </li>
        </ul>
      <li>
        <strong>android:ellipsize 속성</strong>
      </li>
        <ul>
          <li>
            maxLines 속성을 이용할 때 출력되지 않은 문자열이 더 있다는 표시할 때 <strong>줄임표(...)</strong>를 넣는데 이때 ellipsize 속성을 지정한다.
          </li>
          <li>
            eliipsize 속값으로는 end, middle, start 등을 설정한다.
          </li>
            <ul>
              <li>
                start와 middle은 줄임표가 앞과 중간에 추가되며 <strong>singleLine="true" 속성</strong>으로 문자열을 한 줄로 출력했을 때만 적용된다.
              </li>
              <li>
                end는 줄임표가 가장 마지막에 추가되며 singleLine="true"를 지정할 필요가 없다.
              </li>
            </ul>
        </ul>
    </ul>
</ul>

```xml
<!-- 1. 텍스트 뷰 속성 -->
<TextView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:text="helloworld"
    android:textColor="#FF0000"
    android:textSize="20sp"
    android:textStyle="bold" />

<!-- 2. 자동 링크 속성 -->
<TextView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:text="이지스퍼블리싱 - 웹페이지 : http://easyspub.com, 전화번호 : 325-1722, 이메일 : easy@easyspub.co.kr"
    android:autoLink="web|email|phone" />

<!-- 3. 줄임표 표시 -->
<LinearLayout xmlns:android:"http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">
    <TextView
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:text="@string/long_text"
        android:singleLine="true"
        android:ellipsize="middle" />
    <View
        android:layout_width="match_parent"
        android:layout_height="2dp"
        android:background="#000000" />
    <TextView
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:text="@string/long_text"
        android:maxLines="3"
        android:ellipsize="end" />
</LinearLayout>
```

<h2>3-2. 이미지 뷰</h2>
<ul>
  <li>
    ImageView는 이미지를 화면에 출력하는 뷰이며 다음과 같은 속성들이 있다.
  </li>
    <ul>
      <li>
        <strong>android:src 속성</strong>
      </li>
        <ul>
          <li>
            ImageView에 출력할 이미지를 설정하며 리소스 이미지, 파일 이미지, 네트워크 이미지 등을 출력할 수 있다.
          </li>
        </ul>
      <li>
        <strong>android:maxWidth, android:maxHeight, android:adjustViewBounds 속성</strong>
      </li>
        <ul>
          <li>
            ImageView가 출력하는 이미지의 최대 크기를 지정한다.
          </li>
          <li>
            layout_width과 layout_height는 속성의 크기가 고정기에 <strong>뷰에 넣을 이미지 크기가 다양한 경우</strong> 문제가 될 수 있다.
          </li>
          <li>
            <strong>이미지가 클 때</strong> wrap_content로 지정하면 뷰의 크기가 지나치게 커지는 문제가 발생할 수 있어 활용한다.
          </li>
          <li>
            maxWidth와 maxHeight 속성은 <strong>android:adjustViewBounds</strong> 속성과 함께 사용해야 하며 true로 지정하면 <strong>가로세로 길이와 비례</strong>해 뷰의 크기를 맞추어준다.
          </li>
        </ul>
    </ul>
</ul>

```xml
<!-- 1. 이미지 표시 -->
<ImageView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:background="#ff0000"
    android:src="@drawable/tesst" />

<!-- 2. 이미지 크기에 뷰의 크기 맞추기 -->
<ImageView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:maxWidth="100dp"
    android:maxHeight="100dp"
    android:adjustViewBounds="true"
    android:src="@drawable/test"
    android:background="#0000ff"/>
```

<h2>3-4. 에디트 텍스트</h2>
<ul>
  <li>
    EditText는 사용자가 <strong>글을 입력</strong>할 수 있는 뷰이다.
  </li>
    <ul>
      <li>
        <strong>android:lines, android:maxLines 속성</strong>
      </li>
        <ul>
          <li>
            EditEText는 <strong>처음에 한 줄 입력 크기</strong>로 출력되고 사용자가 키보드에 <strong>엔터</strong>를 누르면 아래로 늘어나며 <strong>여러 줄 입력 크기</strong>가 된다.
          </li>
          <li>
            처음부터 <strong>여러 줄 입력 크기</strong>로 나오게 할 때에는 <strong>android:lines 속성</strong>을 활용한다.
          </li>
          <li>
            maxLines 속성은 처음에는 <strong>한 줄 입력 크기</strong>로 나오지만 <strong>최대로 지정한 줄 수</strong>를 넘길 수는 없다.
          </li>
        </ul>
      <li>
        <strong>android:inputType 속성</strong>
      </li>
        <ul>
          <li>
            inputType은 EditText에 글을 입력할 때 나오는 <strong>키보드를 지정</strong>하는 속성이다. 설정하지 않으면 기본 문자 입력 모드가 나온다.
          </li>
          <li>
            <strong>android:inputType="phone"</strong>으로 설정하면 <strong>전화번호</strong> 입력 모드로 키보드가 나타나고 <strong>한 줄 입력 크기</strong>로 고정된다.
          </li>
        </ul>
    </ul>
  <li>
    다양한 inputType에 대해서는 p166 참고.
  </li>
  <li>
    기기에 따라 inputType 속성이 작동하지 않을 수 있다.
  </li>
</ul>

```xml
<!-- 1. 키보드 모드 설정 -->
<EditText
    android:layout_witdth="match_parent"
    android:layout_height="wrap_content"
    android:inputType="phone" />
```

<h1>4. 뷰 바인딩</h1>
<ul>
  <li>
    뷰 바인딩(view binding)은 레이아웃 XML 파일에 선언한 뷰 객체를 <strong>코드에서 쉽게 이용</strong>하는 방법이다.
  </li>
  <li>
    XML 파일에 등록한 뷰는 액티비티에서 <strong>findViewById() 함수</strong>로 얻어 사용한다.
  </li>
  <li>
    findViewById()를 매번 사용하기에는 번거롭다. 최근에는 그래들 파일에 뷰 바인딩 설정(viewBinding.isEnabled = true)을 하여 간단하게 코드작성 만으로도 활용이 가능하다.
  </li>
  <li>
    실습은 "app/res/layout/view_prac_binding.xml"에서 진행한다. 이때 그래들 파일에 뷰 바인딩 설정(viewBinding.isEnabled = true)를 지정한다.
  </li>
    <ul>
      <li>
        설정을 지정하면 레아이웃 <strong>XML 파일에 등록된 뷰 객체를 포함하는 클래스</strong>가 자동으로 만들어진다. 즉, 직접 코드에서 뷰를 선언하고 findViewById()를 호출할 필요가 없다.
      </li>
      <li>
        자동으로 만들어지는 클래스의 이름은 XML 파일명을 따른다.
      </li>
        <ul>
          <li>
            첫 글자를 대문자로 하고 밑줄(_)은 제거한 뒤 가장 마지막에 Binding을 추가한다.에를 들어 <strong>activity_main.xml → ActivityMainBinding</strong>이 된다.
          </li>
        </ul>
      <li>
        자동으로 만들어진 클래스의 <strong>inflate()</strong> 함수를 호출하면 바인딩 객체를 얻을 수 있다.
      </li>
        <ul>
          <li>
            인자로는 layoutInflater를 전달한다.
          </li>
          <li>
            바인딩 객체의 root 프로퍼티에는 <strong>XML의 루트 태그</strong>가 자동으로 등록되기에 액티비티 화면 출력은 <strong>setContentView() 함수에 binding.root</strong>로 전달한다.
          </li>
          <li>
            바인딩 객체에 등록된 뷰 객체명은 XML 파일에 등록한 뷰의 id 값을 따른다.
          </li>
        </ul>
    </ul>
  <li>
    build.gradle 파일에 뷰 바인딩을 추가하면 모든 레이아웃 XML 파일에 바인딩 클래스가 생성된다. 이를 방지하기 위해서는 <strong>해당되지 않는 XML 파일 루트 태그</strong>에 <strong>tools:viewBindingIgnore="true"</strong> 속성을 추가한다.
  </li>
</ul>

```xml
<!-- 1. 바인딩 생략 속성. -->
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    (... 생략 ...)
    tools.viewBindingIgnore="true">
```
