<h1>1. 선형으로 배치 - LinearLayout</h1>
<ul>
  <li>
    레이아웃 클래스는 화면을 독자적으로 출력하지 않고 다른 뷰 객체를 포함하는 일종의 <strong>그릇(배경)</strong>이다. 대표적인 레이아웃 5개를 살펴본다.
  </li>
</ul>

<br>

<h2>1-1. LinearLayout 배치 규칙</h2>
<ul>
  <li>
    LinearLayout은 뷰를 <strong>가로</strong>나 <strong>세로 방향</strong>으로 나열하는 레아이웃 클래스이다.
  </li>
    <ul>
      <li>
        <strong>orientation</strong>이라는 속성을 활용해 <strong>horizontal(가로)</strong>이나 <strong>vertical(세로)</strong> 방향으로 지정가능하다.
      </li>
    </ul>
  <li>
    LinearLayout은 방향만 설정하면 뷰를 <strong>추가한 순서대로</strong> 나열한다.
  </li>
    <ul>
      <li>
        복잡한 화면이나 가로세로가 중첩된 구조는 LinearLayout을 <strong>중첩</strong>하여 구현할 수 있다.
      </li>
      <li>
        레이아웃 클래스도 뷰이기에 <strong>다른 레이아웃 클래스를 포함</strong>할 수 있다.
      </li>
    </ul>
  <li>
    LinearLayout은 특정한 방향으로 뷰를 배치하는 단순한 레이아웃이지만 <strong>레이아웃을 중첩</strong>할 때 진가가 발휘되며 복잡한 화면도 구현할 수 있다.
  </li>
  <li>
    실습은 "app/res/layout/layout_linearlayout.xml에 작성한다.
  </li>
</ul>

```xml
<!-- 1. LinearLayout 중첩 -->
<LinearLayout (... 생략 ...)
    android:orientation="horizontal">
    
    <ImageView (... 생략 ...) />
    
    <LinearLayout (... 생략 ...)
        android:orientation="vertical">
        
        <TextView (... 생략 ...) />
        
        <TextView (... 생략 ...) />
    </LinearLayout>
    
    <TextView (... 생략 ...) />
</LinearLayout>
```

<h2>1-2. 여백을 채우는 layout_weight 속성</h2>
<h3>1-2-1. 뷰 1개로 전체 여백 채우기</h3>
<ul>
  <li>
    layout_weight 속성을 사용하면 수치를 따로 계산하지 않아도 <strong>각 뷰에 설정한 가중치</strong>로 여백을 채울 수 있다.
  </li>
  <li>
    실습은 "app/res/layout/fill_blank"에 작성한다.
  </li>
</ul>

<h3>1-2-2. 뷰 여러 개로 여백을 나누어 채우기</h3>
<ul>
  <li>
    "app/res/layout/fill_blank_01"의 BUTTON2에 "layout_weight="3""을 추가한다.
  </li>
  <li>
    BUTTON1은 layout_weight="1", BUTTON2는 layout_weight="3"이므로 <strong>1:3 비율</strong>로 여백이 채워진다.
  </li>
</ul>

<h3>1-2-3. 중첩된 레이아웃에서 여백 채우기</h3>
<ul>
  <li>
    "app/res/layout/fill_black_nested.xml"에 작성한다.
  </li>
</ul>

<h3>1-2-4. 여백 채우기로 뷰의 크기 설정하기</h3>
<ul>
  <li>
    뷰의 크기를 0으로 하고 layout_weight만 지정하면 <strong>지정한 값에 맞게 여백을 나누어 할당</strong>한다.
  </li>
    <ul>
      <li>
        기본적으로 뷰의 크기가 0이면 화면에 출력되지 않는다.
      </li>
    </ul>
  <li>
    뷰의 크기를 <strong>퍼센트(%)</strong>로 지정할 수도 있다. 뷰의 크기를 0으로 하고 layout_weight을 지정하면 자동적으로 <strong>퍼센트(%)</strong>를 적용한 것과 동일한 결과를 출력한다.
  </li>
  <li>
    실습은 "app/res/layout/fill_black_zerosize.xml"에 작성한다.
  </li>
</ul>

<br>

<h2>1-3. 뷰를 정렬하는 gravity, layout_gravity 속성</h2>
<ul>
  <li>
    뷰를 정렬할 때는 <strong>gravity</strong>와 <strong>layout_gravity</strong> 속성을 사용한다. 기본값은 <strong>left/top(좌측상단)</strong>이다.
  </li>
  <li>
    실습은 "app/res/layout/gravity.xml"에 작성한다.
  </li>
</ul>

<h3>1-3-1. 뷰에 gravity와 layout_gravity 속성 적용하기</h3>
<ul>
  <li>
    gravity는 <strong>콘텐츠</strong>를, layout_gravity는 <strong>뷰 자체</strong>를 정렬한다.
  </li>
  <li>
    실습은  "app/res/layout/gravity.xml"에 이어서 작성한다.
  </li>
</ul>

<h3>1-3-2. 레이아웃에 gravity 속성 적용하기</h3>
<ul>
  <li>
    LinearLayout 자체가 방향으로 뷰를 배치하는 레이아웃이기 때문에 LinearLayout의 <strong>orientation과 같은 방향</strong>으로는 layout_gravity 속성이 적용되지 않는다.
  </li>
  <li>
    뷰를 LinearLayout 영역의 정중앙에 정렬하기 위해서는 <strong>레이아웃</strong>에 <strong>gravity="center"</strong>로 설정한다.
  </li>
  <li>
    <strong>전체 자식 뷰</strong>를 같은 방식으로 정렬할 때에는 레이아웃 정렬(gravity)을 <strong>특정 자식 뷰</strong>를 정렬할 경우에는 해당 뷰에 layout_gravity를 사용한다.
  </li>
  <li>
    실습은 "app/res/layout/gravity.xml"에 이어서 작성한다.
  </li>
</ul>

<h1>2. 상대 위치로 배치 - RelativeLayout</h1>
<h2>2-1. RelativeLayout 배치 규칙</h2>
<ul>
  <li>
    RelativeLayout은 화면에 <strong>이미 출력된 뷰를 기준으로 방향을 지정(상대 뷰)</strong>하여 배치한다.
  </li>
  <li>
    다음과 같은 속성을 활용하며 속성에 입력하는 값의 기준은 <strong>뷰의 id</strong>이다.
  </li>
    <ul>
      <li>
        android:layout_above - 기준 뷰의 위쪽
      </li>
      <li>
        android:layout_below - 기준 뷰의 아래쪽
      </li>
      <li>
        andorid:layout_toLeftOf - 기준 뷰의 왼쪽
      </li>
      <li>
        android:layout_toRightOf - 기준 뷰의 오른쪽
      </li>
    </ul>
  <li>
    RelativeLayout은 LinearLayout과 달리 <strong>자동으로 가로세로 방향으로 배치되지 않아</strong> 이미지 뷰 위에 버튼이 겹쳐 나온다.
  </li>
  <li>
    실습은 "app/res/layout/layout_relative.xml"에 작성한다.
  </li>
</ul>

<br>

<h2>2-2. 맞춤 정렬하는 align 속성</h2>
<ul>
  <li>
    기준이 되는 상대 뷰의 <strong>어느 쪽</strong>에 맞추어 정렬할지 정하는 속성들에는 다음이 있다. 속성에 입력하는 값은 <strong>뷰의 id</strong>이다.
  </li>
    <ul>
      <li>
        android:layout_alignTop - 기준 뷰와 위쪽을 맞춤.
      </li>
      <li>
        android:layout_alignBotton - 기준 뷰와 아래쪽을 맞춤.
      </li>
      <li>
        android:layout_alignLeft - 기준 뷰와 왼쪽을 맞춤.
      </li>
      <li>
        android:layout_alignRight - 기준 뷰와 오른쪽을 맞춤.
      </li>
    </ul>
  <li>
    뷰를 <strong>부모 영역</strong>을 기준으로 정렬할 때에는 다음과 같은 속성을 사용할 수 있다. (p191).
  </li>
    <ul>
      <li>
        android:layout_alignParentTop - 부모의 위쪽에 맞춤.
      </li>
      <li>
        android:layout_alignParentBottom - 부모의 아래쪽에 맞춤.
      </li>
      <li>
        android:layout_alignParentLeft - 부모의 왼쪽에 맞춤.
      </li>
      <li>
        (... 생략 ...)
      </li>
    </ul>
  <li>
    어떤 레이아웃을 선택할지의 기준의 개발자의 판단에 달려있다.
  </li>
  <li>
    실습은 "app/res/layout/layout_relative.xml"에 이어서 작성한다.
  </li>
</ul>

<h1>3. 겹쳐서 배치 - FrameLayout</h1>
<ul>
  <li>
    자주 사용하는 일은 적으나 최근 카카오톡에서 대화탭을 여러번 누르면 "[현재 대화 방 - 안 읽은 메시지 - 그룹 챗 방]" 이런 식으로 터치할 때마다 달라지는 <strong>탭 화면</strong>을 만들 때 유용하다.
  </li>
  <li>
    FrameLayout은 뷰를 <strong>추가한 순서</strong>대로 지속적으로 위에 <strong>겹쳐서</strong> 출력한다.
  </li>
    <ul>
      <li>
        LinearLayout처럼 가로세로 방향으로 배치하지 않고, RelativeLayout처럼 상대 위치를 조절하는 속성도 없다.
      </li>
    </ul>
  <li>
    FrameLayout은 똑같은 위치에 여러 뷰를 겹쳐두고 <strong>필요한 순간 하나의 뷰</strong>만 출력하도록 할 때 사용하며 이로인해 대부분의 뷰의 표시 여부를 결정하는 <strong>visibility 속성</strong>을 함께 사용한다.
  </li>
    <ul>
      <li>
        <strong>액티비티</strong>에서 원하는 순간 visibility의 속성값을 바꾸어 어떤 뷰를 보이고 숨길지 결정한다.
      </li>
    </ul>
  <li>
    실습은 "app/res/layout/layout_frame.xml"에 작성한다.
  </li>
</ul>

<br><br>

<h1>4. 표 형태로 배치 - GridLayout</h1>
<h2>4-1. GridLayout 배치 규칙</h2>
<ul>
  <li>
    GridLayout은 <strong>행과 열</strong>로 구성된 테이블 화면을 만드는 레이아웃 클래스이다.
  </li>
  <li>
    GridLayout에는 다음과 같은 속성들이 있다.
  </li>
    <ul>
      <li>
        orientation: 방향 설정.
      </li>
      <li>
        rowCount: 세로로 나열할 뷰 개수.
      </li>
      <li>
        columnCount: 가로로 나열할 뷰 개수.
      </li>
    </ul>
  <li>
    LinearLayout처럼 orientation 속성을 사용하지만 <strong>줄바꿈을 자동</strong>으로 해준다는 차이가 있다.
  </li>
    <ul>
      <li>
        줄바꿈은 rowCount, columnCount에 지정한 속성의 개수를 기준으로 이루어진다.
      </li>
    </ul>
  <li>
    GridLayout에 추가한 <strong>뷰</strong>는 기본적으로 <strong>wrap_content</strong>로 지정되기 때문에 layout_width와 layout_height 속성을 지정하지 않아도 오류가 발생하지 않는다.
  </li>
  <li>
    실습은 "app/res/layout/layout_grid.xml"에서 진행한다.
  </li>
</ul>

<br>

<h2>4-2. GridLayout 속성</h2>
<h3>4-2-1. 특정 뷰의 위치 조정하기</h3>
<ul>
  <li>
    기본적으로 GriaLayout에 뷰를 배치하면 <strong>추가한 순서</strong>대로 배치된다. 이때 <strong>layout_row</strong>와 <strong>layout_column 속성</strong>을 활용하면 <strong>특정 뷰의 위치를 조정</strong>할 수 있다.
  </li>
    <ul>
      <li>
        layout_row: 뷰가 위치하는 세로 방향 인덱스
      </li>
      <li>
        layout_column: 뷰가 위치하는 가로 방향 인덱스
      </li>
    </ul>
  <li>
    실습은 "app/res/layout/layout_grid.xml"에 이어서 작성한다.
  </li>
</ul>

<h3>4-2-2. 특정 뷰의 크기 확장하기</h3>
<ul>
  <li>
    여백을 채우는 등 특정 뷰의 크기를 조정해야 한다면 layout_gravity 속성을 사용할 수도 있다.
  </li>
    <ul>
      <li>
        확장을 하지 않고 해당 위치에 뷰를 하나 더 추가하는 방법도 존재한다.
      </li>
    </ul>
  <li>
    실습은 "app/res/layout/layout_grid.xml"에 이어서 작성한다.
  </li>
</ul>

<h3>4-2-3. 열이나 행 병합하기</h3>
<ul>
  <li>
    뷰가 테이블에서 <strong>여러 칸을 차지</strong>하게 설정할 수 있으며 layout_columnSpan과 layout_rowSpan 속성을 활용한다.
  </li>
    <ul>
      <li>
        <strong>layout_columnSpan</strong>: 가로로 열 병합
      </li>
      <li>
        <strong>layout_rowSpan</strong>: 새로로 행 병합
      </li>
    </ul>
  <li>
    실습은 "app/res/layout/layout_grid.xml"에 이어서 작성한다.
  </li>
</ul>

<h3>4-2-4. TableLayout과 GridLayout</h3>
<ul>
  <li>
    TableLayout은 <strong>TableRow</strong>를 통해 <strong>행 단위</strong>로 뷰를 입력해야 한다.
  </li>
  <li>
    테이블에서 셀의 개수가 고정인 경우 TableLayout과 GridLayout은 큰 차이가 없다. 그러나 <strong>셀의 개수가 변하는 경우</strong> TableLayout은 <strong>행의 개수를 일일히 계산</strong>하여 TableRow를 지정해야 한다는 단점이 있다.
  </li>
    <ul>
      <li>
        반면 GridLayout은 자동으로 배치를 해준다.
      </li>
    </ul>
</ul>

<h1>5. 계층 구조로 배치 - ConstraintLayout</h1>
<ul>
  <li>
    ConstraintLayout은 안드로이드 플랫폼이 아니라 <strong>androidx</strong>에서 제공하는 라이브러리이다.
  </li>
  <li>
    ConstraintLayout을 사용하기 위해서는 build.gradle.kts의 dependencies에 의존성을 추가해야 한다.
  </li>
    <ul>
      <li>
        추가할 의존성: <strong>implementation(libs.androidx.constraintlayout)</strong>
      </li>
    </ul>
  <li>
    프로젝트를 만들 때 자동으로 만들어지는 레이아웃 XML 파일은 ConstraintLayout으로 설정된다. 다른 레이아웃을 이용하는 경우 수정을 하면 된다.
  </li>
</ul>

<br>

<h2>5-1. 레이아웃 편집기에서 레이아웃 구성하기</h2>
<ul>
  <li>
    ConstraintLayout은 뷰를 상대 위치로 배치하는 RelativeLayout과 비슷하지만 더 많은 속성을 제공한다.
  </li>
  <li>
    많은 속성은 XML 코드로 다루기에는 부담스러우며 안드로이드 스튜디오는 이를 위해 <strong>마우스 클릭</strong>으로 동작하는 <strong>레이아웃 편집기</strong>를 제공한다.
  </li>
  <li>
    레이아웃 편집기는 XML 파일을 <strong>디자인 모드</strong>로 열면 사용할 수 있다.
  </li>
  <li>
    디자인 모드에서 작업을 완료한뒤 다시 코드로 넘어가면 ConstraintLayout으로 만든 화면이 코드로 작성되어 있는 것을 볼 수 있다.
  </li>
  <li>
    실습은 "app/res/layout/layout_constraint.xml"에 작성한다.
  </li>
</ul>
