<h1>1. 클래스와 생성자</h1>
<h2>1-1. 클래스 선언</h2>
<ul>
  <li>
    코틀린에서 클래스는 <strong>class 키워드</strong>로 선언하며 <strong>중괄호({})</strong> 영역이 본문이 된다.
  </li>
    <ul>
      <li>
        클래스의 분문에 입력하는 내용이 없다면 <strong>중괄호({})를 생략</strong>할 수 있다.
      </li>
        <ul>
          <li>
            자바에서는 멤버가 없고 선언부만 있는 클래스는 의미가 없지만 코틀린에서는 <strong>클래스의 생성자</strong>를 본문이 아닌 <strong>선언부</strong>에 작성할 수 있어 의미가 있다.
          </li>
        </ul>
    </ul>
  <li>
    클래스의 멤버는 <strong>생성자, 변수, 함수, 클래스</strong>로 구성되며 이 중 코틀린 생성자는 <strong>constructor</strong>라는 키워드로 선언한다.
  </li>
  <li>
    클래스는 <strong>객체를 생성</strong>해 사용하고 객체로 <strong>클래스의 멤버에 접근</strong>한다.
  </li>
    <ul>
      <li>
        코틀린에서는 <strong>new 키워드</strong>를 사용하지 않고 클래스 이름과 같은 <strong>함수</strong>로 객체를 생성한다.
      </li>
        <ul>
          <li>
            객체를 생성할 때 <strong>생성자가 자동 호출</strong>되기 때문에 전달한 인자는 클래스에 선언된 <strong>생성자의 매개변수와 일치</strong>해야 한다.
          </li>
        </ul>
    </ul>
</ul>

```kotlin
// 1. 클래스 선언.
class User { }

// 2. 클래스의 멤버.
class User {
    // 변수
    var name = "kkang"
    // 생성자
    constructor(name: String) {
        this.name = name
    }
    // 함수
    fun someFun() {
        println("name : $name")
    }
    // 클래스
    class SomeClass { }
}

// 3. 객체 생성과 멤버 접근.
val user = User("kim")
user.someFun()
```

<h2>1-2. 주 생성자</h2>
<ul>
  <li>
    코틀린은 생성자를 <strong>주 생성자</strong>와 <strong>보조 생성자</strong>로 구분한다.
  </li>
    <ul>
      <li>
        한 클래스 안에 주 생성자 혹은 보조 생성자만 선언할 수도 있고 둘 모두 선언할 수도 있다.
      </li>
    </ul>
  <li>
    주 생성자는 <strong>constructor 키워드</strong>로 <strong>클래스 선언부</strong>에 선언한다.
  </li>
    <ul>
      <li>
        주 생성자 선언은 <strong>필수가 아니며</strong> 한 클래스에 <strong>하나</strong>만 생성 가능하다.
      </li>
      <li>
        주 생성자를 선언할 때 constructor 키워드는 생략할 수 있다.
      </li>
      <li>
        주 생성자를 선언하지 않으면 <strong>컴파일러</strong>가 <strong>매개변수가 없는 주 생성자</strong>를 자동으로 추가한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 주 생성자 선언.
class User constructor() {
}

// 2. constructor 키워드 생략 예.
class User() {
}

// 3. 매개변수가 없는 주 생성자 자동 선언.
class User {
}
```

<h3>1-2-1. 주 생성자의 매개변수</h3>
<ul>
  <li>
    주 생성자를 선언할 때 필요에 따라 <strong>매개변수</strong>를 선언할 수도 있다.
  </li>
    <ul>
      <li>
        매개변수를 선언하였다면 객체를 생성할 때 매개변수의 타입과 개수에 맞는 <strong>인자를 전달</strong>해야 한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 주 생성자의 매개변수
class User(name: String, count: Int) { 
}

// 2. 매개변수가 있는 생성자 호출.
val user = User("kkang", 10)
```

<h3>1-2-2. 주 생성자의 본문 - init 영역</h3>
<ul>
  <li>
    보통 클래스나 함수의 본문(실행 영역)을 중괄호({})로 감싸지만 주 생성자는 <strong>클래스 선언부</strong>에 있기 때문에 추가할 수 없다.
  </li>
    <ul>
      <li>
        주 생성자의 본문을 구현할 때에는 <strong>init 키워드</strong>를 이용한다.
      </li>
    </ul>
  <li>
    클래스에서 init 영역은 꼭 선언할 필요는 없고 <strong>주 생성자의 본문</strong>을 구현하고 싶을 때 사용한다.
  </li>
    <ul>
      <li>
        코틀린 클래스 안에서 init 키워드로 지정한 영역은 객체를 생성할 때 <strong>자동으로 실행</strong>된다.
      </li>
      <li>
        보조 생성자는 클래스 안에 선언하기에 본문 구현이 가능하며 따라서 init은 주로 주 생성자 본문을 구현할 때 사용한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 주 생성자 중괄호 때문에 오류 발생.
class User(name: String, count: Int) {
    // 주 생성자 본문
} { // 오류!
    // 클래스 본문
}

// 2. init 키워드로 주 생성자의 본문 지정.
class User(name: String, count: Int) {
    init {
        println("i am init...")
    }
}
fun main() {
    // User 객체를 생성할 때 init 본문도 실행된다.
    val user = User("kkang", 10)
}
```

<h3>1-2-3. 생성자의 매개변수를 클래스의 멤버 변수로 선언하는 방법</h3>
<ul>
  <li>
    생성자의 매개변수는 기본적으로 <strong>생성자에서만 사용</strong>할 수 있는 <strong>지역 변수</strong>이다.
  </li>
  <li>
    생성자의 매개변수를 다른 함수에서 사용해야 한다면 <strong>클래스 멤버로 선언</strong>하고 <strong>init 내부에서 초기화</strong>하는 방식이 있다.
  </li>
    <ul>
      <li>
        매개변수를 <strong>var나 val 키워드</strong>로 선언하면 손쉽게 클래스의 <strong>멤버 변수</strong>가 되고 클래스 <strong>외부</strong>에서도 사용이 가능해진다.
      </li>
        <ul>
          <li>
            원래 함수는 매개변수를 선언할 때 var이나 val 키워드를 추가할 수 없지만 <strong>주 생서자만 유일하게 선언이 가능</strong>하다.
          </li>
        </ul>
    </ul>
</ul>

```kotlin
// 1. 생성자의 매개변수를 init 영역에서 사용하는 예.
class User(name: String, count: Int) {
    // 생성자에서 사용할 수 있는 지역변수이기에 성공.
    init {
        println("name : $name, count : $count")
    }
    // init 본문 외에서 변수를 사용하려 하기에 오류가 발생한다.
    fun someFun() {
        println("name : $name, count : $count")
    }
}

// 2. 생성자의 매개변수를 다른 함수에서 사용하는 예.
class User(name: String, count: Int) {
    // 클래스 멤버 변수 선언.
    var name: String
    var count: Int
    init {
        // 클래스 멤버 변수에 생성자 매개변숫값 대입.
        this.name = name
        this.count = count
    }
    fun someFun() {
        println("name : $name, count : $count")
    }
}
fun main() {
    val user = User("kkang", 10)
    user.someFun()
}

// 3. 생성자의 매개변수를 클래스의 멤버 변수로 선언하는 방법.
class User(val name: String, val count: Int) {
    // val과 var로 선언했기에 외부에서 사용 가능.
    fun someFun() {
        println("name : $name, count: $count")
    }
}
fun main() {
    val user = User("kkang", 10)
    user.someFun()
}
```

<h2>1-3. 보조 생성자</h2>
<ul>
  <li>
    보조 생성자는 <strong>클래스의 본문</strong>에 constructor 키워드로 선언하는 함수이다.
  </li>
    <ul>
      <li>
        클래스 본무에 선언하기 때문에 <strong>여러 개</strong>를 추가할 수 있다.
      </li>
    </ul>
  <li>
    보조 생성자도 생성자이기 때문에 객체를 생성할 때 <strong>자동으로 호출</strong>된다.
  </li>
  <li>
    보조 생성자는 클래스 본문에 선언하기 때문에 생성자 본문을 <strong>중괄호({})</strong>로 묶어 <strong>객체 생성과 동시에 실행할 영역을 지정</strong>할 수 있다.
  </li>
</ul>

```kotlin
// 1. 보조 생성자 선언.
class User {
    // 본문 내부에 constructor를 통해 보조 생성자 생성.
    //   - 객체를 생성할 때 자동으로 호출된다.
    //   - 클래스 본문이기에 중괄호({})로 묶어 객체 생성과 동시에
    //     실행영역을 지정할 수 있다.
    constructor(name: String) {
        println("constructor(name: String) call...")
    }
    constructor(name: String, count: Int) {
        println("constructor(name: String, count: Int) call...")
    }
}
fun main() {
    val user1 = User("kkang")
    val user2 = User("kkang", 10)
}
```

<h3>1-3-1. 보조 생성자에 주 생성자 연결</h3>
<ul>
  <li>
    주 생성자와 보조 생성자를 모두 선언한다면 반드시 <strong>생성자끼리 연결</strong>을 해주어야 한다.
  </li>
    <ul>
      <li>
        둘 모두 존재할 경우 보조 생성자는 <strong>주 생성자를 호출</strong>해야 한다. 호출할 때에는 <strong>this() 구문</strong>을 활용한다.
      </li>
    </ul>
  <li>
    보조 생성자가 여럿일 경우 하나의 보조 생성자에서 <strong>다른 보조 생성자</strong>를 호출할 수 있다.
  </li>
    <ul>
      <li>
        단, 이때에 보조 생성자를 호출하는 <strong>호출 체인의 끝</strong>에는 <strong>주 생성자 호출</strong>이 있어야 한다.
      </li>
    </ul>
  <li>
    주 생성자와 보조 생성자를 나누는 큰 이유는 없지만 다음의 의도가 있다.
  </li>
    <ul>
      <li>
        객체를 <strong>여러 가지 형태</strong>로 생성할 수 있도록 매개변수를 다양하게 구성할 수 있다. 단, 이때 <strong>공통된 코드</strong>는 주 생성자에 작성하라는 의미이다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 주 생성자와 보조 생성자를 연결 없이 선언 시 오류.
class User(name: String) {
    constructor(name: String, count: Int) {
        (... 생략 ...)
    }
}

// 2. 보조 생성자에서 주 생성자 호출.
class User(name: String) {
    // this()를 통해 주 생성자를 호출하였으므로 성공.
    constructor(name: String, count: Int): this(name) {
        (... 생략 ...)
    }
}
fun main() {
    val user = User("kkang", 10)
}

// 3. 보조 생성자가 여럿일 때 생성자 연결.
class User(name: String) {
    constructor(name: String, count: Int): this(name) {
        (... 생략 ...)
    }
    constructor(name: String, count: Int, email.String): this(name, count) {
        (... 생략 ...)
    }
}
fun main() {
    val user = User("kkang", 10, "a@a.com")
}
```

<h1>2. 클래스를 재사용하는 상속</h1>
<h2>2-1. 클래스를 재사용하는 상속</h2>
<ul>
  <li>
    클래스를 선언할 때 <strong>다른 클래스를 참조</strong>하여 선언하는 것을 <strong>상속(inheritance)</strong>이라 한다.
  </li>
    <ul>
      <li>
        상속을 하면 기존 클래스를 <strong>재사용</strong>할 수 있다.
      </li>
    </ul>
  <li>
    코틀린에서는 어떤 클래스를 상속받을 때 선언부에 <strong>콜론(:)</strong>과 함께 상속 받을 <strong>클래스의 이름</strong>을 입력한다.
  </li>
  <li>
    상속 관계에서 상속 대상이 되는 클래스를 <strong>상위 클래스</strong>라 하고, 상속받는 클래스를 <strong>하위 클래스</strong>라 한다.
  </li>
  <li>
    코틀린은 기본적으로 다른 클래스를 상속할 수 없으며 상속이 가능하게 하려면 클래스를 선언할 때 <strong>open 키워드</strong>를 사용한다.
  </li>
  <li>
    <strong>매개변수</strong>가 있는 상위 클래스를 상속받을 경우 하위 클래스는 상위 클래스의 생성자를 호출할 때 매개변수 구성에 맞게 <strong>인자를 전달</strong>해야 한다.
  </li>
  <li>
    상위 클래스의 생성자 호출을 클래스 선언부에 작성할 필요는 없으며 하위 클래스에 보조 생성자만 있다면 <strong>보조 생성자에 작성</strong>할 수 있다.
  </li>
</ul>

```kotlin
// 1. 클래스 상속 형식.
// 상속할 수 있게 open 키워드를 이용한다.
open class Super {
}
// Super를 상속받아 Sub 클래스를 선언한다.
class Sub: Super() {
}

// 2. 매개변수가 있는 상위 클래스의 생성자 호출.
open class Super(name: String) {
}
// 상위 클래스 Super()의 name 매개변수를 인자로 전달한다.
class Sub(name: String): Super(name) {
}

// 2. 하위 클래스에 보조 생성자만 있는 경우 상위 클래스의 생성자 호출.
open class Super(name: String) {
}
class Sub: Super {
    constructor(name: String): super(name) {
    }
}
```

<br>

<h2>2-2. 오버라이딩 - 재정의</h2>
<ul>
  <li>
    상속이 주는 최고의 이점은 <strong>상위 클레스에 정의된 멤버(변수, 함수)</strong>를 하위 클래스에서 <strong>자신의 멤버</strong>처럼 사용할 수 있단는 점이다.
  </li>
  <li>
    때로는 상위 클래스에 정의된 <strong>멤버를 하위 클래스에서 재정의</strong> 해야할 수도 있는데 이를 <strong>오버라이딩(overriding)</strong>이라 한다.
  </li>
    <ul>
      <li>
        변수도 오버라이딩 기법으로 재정의가 가능하지만 주로 <strong>함수를 재정의</strong> 하는 데 사용한다. (하위 클래스에서 새로운 로직을 추가).
      </li>
    </ul>
  <li>
    코틀린에서 오버라이딩 규칙은 상위 클래스에서 오버라이딩을 허용할 <strong>변수나 함수 선언 앞</strong>에 <strong>open 키워드</strong>를 추가하는 것이다.
  </li>
    <ul>
      <li>
        open 키워드로 선언한 변수나 함수를 <strong>하위 클래스에서 재정의 할 때</strong>에는 반드시 선언문 앞에 <strong>override 키워드</strong>를 추가해야 한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 상속 관계인 두 클래스.
open class Super {
    var superData = 10
    fun superFun() {
        println("i am superFun: $superData")
    }
}
// Super 클래스를 상속 받는다.
class Sub: Super()
fun main() {
    val obj = Sub()
    // 상속받은 객체(Super)의 멤버를 활용이 가능하다.
    obj.superData = 20
    obj.superFun()
}

// 2. 오버라이딩 예.
open class Super {
    open var someData = 10
    open fun someFun() {
        println("i am super class function : $someData")
    }
}
class Sub: Super() {
    override var someData = 20
    override fun someFun() {
        println("i am sub class function : $someData)
    }
}
fun main() {
    val obj = Sub()
    obj.someFun()
}
```

<h2>2-3. 접근 제한자</h2>
<ul>
  <li>
    <strong>접근 제한자(visibility modifier)</strong>란 클래스의 멤버를 <strong>외부의 어느 범위까지</strong> 이용하게 할 것인지를 결정하는 키워드이다.
  </li>
    <ul>
      <li>
        코틀린의 접근 제한자에는 public, internal, protected, private이 있다. 
      </li>
        <ul>
          <li>
            <strong>public</strong>: <strong>접근 제한이 없음</strong>을 의미하고 어디서든 접근이 가능하다. (변수, 함수, 생성자 등을 선언할 때 기본값).
          </li>
          <li>
            <strong>internal</strong>: <strong>같은 모듈 내</strong>에서 접근이 가능하다. 모듈이란 그래들(Gradle) 혹은 메이븐(Maven) 같은 <strong>빌드 도구에서 프로젝트 단위 또는 같은 세트단위</strong>를 의미한다.
          </li>
          <li>
            <strong>protected</strong>: <strong>클래스의 멤버</strong>에서만 선언 가능하고 최상위에 선언되는 변수나 함수는 불가하다. 선언한 클래스와 <strong>해당 클래스</strong>를 상속받는 <strong>하위 클래스</strong>에서 접근 가능하다.
          </li>
          <li>
            <strong>private</strong>: <strong>클래스</strong>에서 사용할 때에는 <strong>해당 클래스 내부</strong>에서만 접근할 수 있다. <strong>최상위</strong>에서 선언할 때에는 <strong>해당 파일 내부</strong>에서만 접근할 수 있다.
          </li>
        </ul>
    </ul>
</ul>

```kotlin
// 1. 접근 제한자 사용 예.
open class Super {
    var publicData = 10
    protected var protectedData = 20
    private var privateData = 30
}
class Sub: Super() {
    fun subFun() {
        publicData++     // 성공!
        protectedData++  // 성공!
        privateData ++   // 오류! - 해당 클래스에서만
    }
}
fun main() {
    val obj = Super()
    obj.publicData++     // 성공!
    obj.protectedData++  // 오류! - 상속 받은 객체까지만
    obj.privateData++    // 오류! - 해당 클래스에서만
}
```

<h1>3. 코틀린의 클래스 종류</h1>
<h2>3-1. 데이터 클래스</h2>
<ul>
  <li>
    데이터 클래스는 <strong>data 키워드</strong>로 선언하며 자주 사용하는 데이터를 <strong>객체로 묶어</strong>준다.
  </li>
    <ul>
      <li>
        <strong>VO(value-object) 클래스</strong>를 편리하게 이용할 수 있게 해준다.
      </li>
        <ul>
          <li>
            VO 클래스는 <strong>값</strong>을 담고 있는 클래스를 의미한다.
          </li>
        </ul>
    </ul>
</ul>

```kotlin
// 1. 데이터 클래스 선언.
class NonDataClass(val name: String, val email: String, val age: Int)
data class DataClass(val name: String, val email: String, val age: Int)

// 2. 데이터 클래스 객체 생성.
fun main() {
    val non1 = NonDataClass("kkang", "a@a.com", 10)
    val non2 = NonDataClass("kkang", "a@a.com", 10)

    val data1 = DataClass("kkang", "a@a.com", 10)
    val data2 = DataClass("kkang", "a@a.com", 10)
}
```

<h3>3-1-1. 객체의 데이터를 비교하는 equals() 함수</h3>
<ul>
  <li>
    VO 클래스는 데이터를 주요하게 다루기에 데이터가 <strong>서로 같은지 비교</strong>할 때가 많다. (객체 비교가 아님).
  </li>
    <ul>
      <li>
        객체의 데이터가 같은지를 비교할 대에는 <strong>equals() 함수</strong>를 활용한다.
      </li>
    </ul>
  <li>
    데이터 클래스는 데이터를 다루는 데 편리한 기능을 제공하는 것이 주목적이기 때문에 <strong>주 생성자에 val과 var 키워드</strong>로 <strong>매개변수를 선언</strong>해 <strong>멤버 변수</strong>로 활용하는 것이 일반적이다.
  </li>
    <ul>
      <li>
        데이터 클래스 본문의 변수나 함수를 추가할 수도 있지만 equals()는 <strong>주 생성자</strong>에 선언한 <strong>멤버 변수의 데이터</strong>만 비교 대상으로 한다.
      </li>
    </ul>
  <li>
    데이터 클래스의 equals()는 <strong>주 생성자의 멤버 변수</strong>가 같은지만 판단한다.
  </li>
</ul>

```kotlin
// 1. 객체의 데이터를 비교하는 equals() 함수. (이전 코드 그대로 이어서).
println("non data class equals : ${non1.equals(non2)}")  // false
println("data class equals : ${data1.equals(data2)}")    // true

// 2. 데이터 클래스의 equals() 함수.
data class DataClass(val name: String, val email: String, val age: Int) {
    lateinit var address: String
    constructor(name: String, email: String, age: Int, address: String):
        this.(name, email, age) {
        this.address = address
    }
}
fun main() {
    val obj1 = DataClass("kkang", "a@a.com", 10, "seoul")
    val obj2 = DataClass("kkang", "a@a.com", 10, "busan")
    // address 변수값은 다르지만 주 생성자에 선언한 변수가 아니기에 true를 반환한다.
    println("obj1.equals(obj2) : ${obj1.equals(obj2)}")
}
```

<h3>3-1-2. 객체의 데이터를 반환하는 toString() 함수</h3>
<ul>
  <li>
    데이터 클래스에서 <strong>객체가 가지는 값</strong>을 확인해야 할 때에는 <strong>toString()</strong>을 사용한다.
  </li>
    <ul>
      <li>
        일반 클래스의 toString()은 값을 출력하지 않지만 <strong>데이터 클래스의 toString()은 값</strong>을 출력한다.
      </li>
    </ul>
  <li>
    마찬가지로 데이터 클래스의 toString() 또한 <strong>주 생성자의 매개변수에 선언된 데이터</strong>만 출력 가능하다.
  </li>
</ul>

```kotlin
// 1. 데이터 클래스의 toString() 함수.
fun main() {
    // 클래스 정의
    class NonDataClass(val name: String, val email: String, val age: Int)
    data class DataClass(val name: Sring, val email: String, val age: Int)
    
    // 클레스 생성.
    val non = NonDataClass("kkang", "a@a.com", 10)
    val data = DataClass("kkang", "a@a.com", 10)
    
    // toString() 비교
    // 데이터 클래스가 아닌 경우 값을 출력하지 않는다.
    //   → "com.example.test4.ch2.Test2Kt... (생략)"
    println("non data class toString : ${non.toString()}")
    // 데이터 클래스인 경우 값을 출력한다.
    //   → DataClass(name=kkang, email=a@a.com, age=10)
    println("data class toString : ${data.toString()}")
}
```

<h2>3-2. 오브젝트 클래스</h2>
<ul>
  <li>
    코틀린에서 오브젝트 클래스는 <strong>익명 클래스(anonymous class)</strong>를 만들 목적으로 사용한다.
  </li>
    <ul>
      <li>
        익명 클래스는 <strong>이름이 없는</strong> 클래스로 <strong>선언과 동시에 객체를 생성</strong>해야 한다.
      </li>
    </ul>
  <li>
    오브젝트 클래스는 선언과 동시에 객체를 생성한다는 의미에서 <strong>object 키워드</strong>를 사용한다.
  </li>
  <li>
    오브젝트 클래스는 보통 <strong>타입</strong>까지 함께 입력해서 선언하며 타입은 <strong>object 뒤에 콜론(:)</strong>을 입력하고 그 뒤에 <strong>클래스 상위</strong> 또는 <strong>인터페이스</strong>를 입력한다.
  </li>
</ul>

```kotlin
// 1. 오브젝트 클래스 사용 예.
val obj = object {
    var data = 10
    fun some() {
        println("data : $data")
    }
}
fun main() {
    // object 객체를 생성은 하였지만 타입이 없어 자동적으로 Any 타입이 된다.
    // → Any 타입에는 data와 some() 이라는 멤버가 없다.
    // → 오류가 발생한다.
    obj.data = 20
    obj.some() 
}

// 2. 타입을 지정한 오브젝트 클래스.
open class Super {
    open var data = 10
    open fun some() {
        println("i am super some() : $data")
    }
}
// 상속 받음과 동시에 Super() 타입이 된다.
val obj = object: Super() {
    override var data = 20
    override fun some() {
        println("i am object some() : $data")
    }
}
fun main() {
    // obj 객체를 통해 Super 클래스에 선언된 멤버에 접근.
    obj.data = 30
    obj.some();
}
```

<h2>3-3. 컴패니언 클래스</h2>
<ul>
  <li>
    컴패니언 클래스는 <strong>객체를 생성하지 않고 클래스 이름</strong>으로 특정 멤버를 접근할 때 활용한다.
  </li>
  <li>
    클래스 이름으로 멤버에 접근할 수 있게 하려면 <strong>companion</strong>이라는 키워드로 선언해야 한다.
  </li>
  <li>
    companion 키워드는 자바의 <strong>static 키워드</strong>를 대체한다고 할 수 있다.
  </li>
    <ul>
      <li>
        자바는 모든 것을 클래스 내부에 작성하기 때문에 static과 같은 키워드가 필수이다.
      </li>
      <li>
        반면 코틀린은 필요한 경우 클래스 내부가 아닌 <strong>최상단에 선언</strong>이 가능하기 때문에 static을 사용할 일이 많지 않아 대신 companion을 제공하는 것이다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 일반 클래스의 멤버 접근.
class MyClass {
    var data = 10
    fun some() {
        println(data)
    }
}
fun main() {
    // 객체를 생성하고 접근.
    val obj = MyClass()
    
    // 생성한 객체를 통한 접근은 성공한다.
    obj.data = 20
    obj.some()

    // 반면 클래스 이름으로 바로 접근하면 실패한다.
    MyClass.data = 20
    MyClass.some()
}

// 2. 컴패니언 클래스의 멤버 접근.
class MyClass {
    companion object {
        var data = 10
        fun some() {
            println(data)
        }
    }
}
fun main() {
    // companion으로 선언했기 때문에 클래스 이름으로 접근이 가능하다.
    MyClass.data = 20
    MyClass.some()
}
```
