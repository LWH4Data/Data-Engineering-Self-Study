<h1>1. 람다 함수와 고차 함수</h1>
<h2>1-1. 람다 함수</h2>
<ul>
  <li>
    람다 함수는 코틀린뿐만 아니라 많은 프로그래밍 언어에서 제공하는 <strong>익명 함수(anonymous unction) 정의 기법</strong>이다.
  </li>
    <ul>
      <li>
        람다 함수는 주로 <strong>함수를 간단하게 정의</strong>할 때 이용하며 람다식이라고도 한다.
      </li>
      <li>
        코틀린은 <strong>고차 함수</strong>를 지원하기 때문에 람다 함수를 많이 사용한다.
      </li>
        <ul>
          <li>
            고차 함수란 <strong>함수를 매개변수</strong>로 받거나 <strong>함수를 반환값</strong>으로 돌려주는 함수를 의미한다.
          </li>
        </ul>
    </ul>
</ul>

<h3>1-1-1. 람다 함수 선언과 호출</h3>
<ul>
  <li>
    람다 함수는 fun 키워드를 이용하지 않고 <strong>중괄호({})</strong>로 표현한다.
  </li>
    <ul>
      <li>
        람다 함수 선언 형식: <strong>{ 매개변수 -> 함수 본문 }</strong>
      </li>
    </ul>
  <li>
    코틀린에서 람다 함수를 사용하는 규칙은 다음과 같다.
  </li>
    <ul>
      <li>
        람다 함수는 <strong>{}</strong>로 표현한다.
      </li>
      <li>
        {} 안에 <strong>화살표(->)</strong>가 있으며 화살표 왼쪽은 <strong>매개변수</strong>, 오른쪽은 <strong>함수 본문</strong>이다.
      </li>
      <li>
        함수의 반환값은 함수 본문의 <strong>마지막 표현식</strong>이다.
      </li>
    </ul>
  <li>
    람다 함수는 이름이 없기 때문에 보통 <strong>변수에 대입</strong>해서 사용한다.
  </li>
    <ul>
      <li>
        변수에 대입하지 않고 람다 함수를 <strong>선언하자마자 호출</strong>할 수도 있다. 이때에 함수는 <strong>소괄호(())</strong>로 호출을 해야 실행되기에 람다 함수 뒤에 소괄호를 붙이고 <strong>인자를 전달</strong>한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 일반 함수 선언.
fun sum(no1: Int, no2: Int): Int {
    return no1 + no2
}

// 2. 람다 함수 선언.
val sum = {no1: Int, no2: Int -> no1 + no2}

// 3. 람다 함수 선언 (변수 대입)
val sum = {no1: Int, no2: Int -> no1 + no2}

// 4. 람다 함수 선언과 호출.
{no1: Int, no2: Int -> no1 + no2} (10, 20)
```

<h3>1-1-2. 매개변수 없는 람다 함수</h3>
<ul>
  <li>
    매개변수가 없는 람다 함수를 작성할 때에는 <strong>매개변수를 비우고, 화살표까지 생략</strong>할 수 있다.
  </li>
</ul>

```kotlin
// 1. 매개변수가 없는 람다 함수.
{-> println("function call")}

// 2. 화살표를 생략한 람다 함수.
{println("function call")}
```

<h3>1-1-3. 매개변수가 1개인 람다 함수</h3>
<ul>
  <li>
    매개변수가 하나인 경우 기존의 방식처럼 할 수 있지만 <strong>"(Int) -> Unit"</strong>로 매개변수가 한 개임을 알릴 수 있다.
  </li>
  <li>
    중괄호 안에서 매개변수 선언을 생략하고 print(it)처럼 <strong>it 키워드</strong>로 매개변수를 이용할 수도 있다.
  </li>
    <ul>
      <li>
        it 키워드는 해당 매개변수의 <strong>타입을 식별할 수 있을 때</strong>만 사용이 가능하다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 매개변수가 1개인 람다 함수.
fun main() {
    val some = {no: Int -> println(no)}
    some(10)
}

// 2. 매개변수가 1개인 람다 함수에서 it 키워드 사용.
fun main() {
    val some: (Int) -> Unit = {println(it)}
    some(10)
}

// 3. 타입을 식별할 수 없어 오류가 발생하는 경우.
val some = {println(it)}

// 4. 타입 식별이 가능한 경우 it 사용 가능.
val some: (Int) -> Unit = {println(it)}
```

<h3>1-1-4. 람다 함수의 반환</h3>
<ul>
  <li>
    람다 함수에서는 <strong>return문</strong>을 사용할 수 없다.
  </li>
  <li>
    람다 함수의 반환값은 <strong>마지막 줄의 실행 결과</strong>이다.
  </li>
</ul>

```kotlin
// 1. 람다 함수에서 return 문 사용 오류.
val some = {no1: Int, no2: Int -> return no1 * no2}

// 2. 람다 함수의 반환문.
fun main() {
    val some = {no1: Int, no2: Int ->
        println("in lambda function")
        // 마지막 줄인 아래의 결과가 반환값이 된다.
        no1 * no2
    }
    println("result : ${some(10, 20)}")
}
```

<h2>1-2. 함수 타입과 고차 함수</h2>
<ul>
  <li>
    코틀린에서 변수에 함수를 대입할 때에는 당연히 변수를 <strong>함수 타입</strong>으로 선언해야 한다.
  </li>
</ul>

<h3>1-2-1. 함수 타입 선언</h3>
<ul>
  <li>
    함수 타입이란 함수를 선언할 때 나타내는 <strong>매개변수 타입</strong>과 <strong>반환 타입</strong>을 의미한다.
  </li>
    <ul>
      <li>
        함수 타입을 먼저 선언하고 함수 내용을 구현한다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 일반 함수 선언.
fun some(no1: Int, no2: Int): Int {
    return no1 + no2
}

// 2. 함수 타입을 이용해 함수를 변수에 대입.
//   - "(Int, Int) -> Int"가 함수 타입에 해당한다.
val some: (Int, Int) -> Int = { no1, no2 -> no1 + no2 }
```

<h3>1-2-2. 타입 별칭 - typealias</h3>
<ul>
  <li>
    함수 타입을 <strong>typealias</strong>를 통해 선언할 수 있다. typealias는 <strong>타입의 별칭</strong>을 선언하는 키워드며 <strong>함수 타입</strong>과 <strong>데이터 타입</strong>을 선언할 때 사용할 수 있디.
  </li>
  <li>
    일반 타입에 typealias를 사용할 수 있지만 보통 <strong>함수 타입</strong>을 선언하는데 주로 활용한다.
  </li>
  <li>
    함수 타입 선언등으로 매개변수의 타입을 추정할 수 있는 경우 <strong>타입 생략</strong>이 가능하며 이는 람다 함수도 해당한다.
  </li>
    <ul>
      <li>
        반대로 함수 구현부에서 타입 추론이 가능하면 <strong>타입 선언을 생략</strong>할 수도 있다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 타입 별칭 선언과 사용.
// 타입 별칭 선언
typealias MyInt = Int
fun main() {
    val data1: Int = 10
    // 위에서 선언한 별칭을 사용.
    val data2: MyInt = 10
}

// 2. 함수 타입 별칭.
// 함수 타입을 별칭으로 선언.
typealias MyFunType = (Int, Int) -> Boolean

fun main() {
    // 위에서 선언한 타입을 활용하여 람다 함수 작성.
    val someFun: MyFunType = {no1: Int, no2: Int ->
        no1 > no2
    }
    println(someFun(10, 20))
    println(someFun(20, 10))
}

// 3. 매개변수 타입 선언 생략 예.
val someFun: (Int, Int) -> Boolean = {no1, no2 -> 
    no1 > no2
}

// 4. 변수 선언 시 타입 생략.
val someFun = {no1: Int, no2: Int ->
    no1 > no2
}
```

<h3>1-2-3. 고차 함수</h3>
<ul>
  <li>
    고차 함수(high order function)란 <strong>함수를 매개변수</strong>로 전달받거나 <strong>반환</strong>하는 함수를 의미한다.
  </li>
</ul>

```kotlin
// 1. 고차 함수.
//   - arg: (Int) -> Boolean
//     → Int 값을 하나 받아서 Boolean을 반환하는 함수 타입
//
//   - hofFun의 반환 타입: () -> String
//     → 매개변수는 없고 String을 반환하는 함수 타입
fun hofFun(arg: (Int) -> Boolean): () -> String {

    // arg(10)은 전달받은 람다 함수에 10을 넣어 실행한다는 뜻
    // main에서 전달한 { no -> no > 0 } 기준으로 보면
    // arg(10) == 10 > 0 이므로 true
    val result = if(arg(10)) {
        "valid"
    } else {
        "invalid"
    }
    // String을 바로 반환하는 것이 아니라,
    // String을 반환하는 람다 함수를 반환한다.
    return {"hofFun result : $result"}
}
fun main() {
    // { no -> no > 0 } 는 Int를 받아 Boolean을 반환하는 람다 함수
    // 따라서 hofFun의 매개변수 arg로 전달할 수 있다.
    val result = hofFun({no -> no > 0})

    // result는 String이 아니라 () -> String 타입의 함수이다.
    // 그래서 result()처럼 호출해야 최종 문자열을 얻을 수 있다.
    println(result())
}
```

<h1>2. 널 안정성</h1>
<h2>2-1. 널 안정성이란?</h2>
<ul>
  <li>
    널(null)이란 객체가 <strong>선언</strong>되었지만 <strong>초기화되지 않은 상태</strong>를 의미한다.
  </li>
    <ul>
      <li>
        객체란 데이터가 저장된 <strong>주소</strong>를 참조하기에 흔히 <strong>참조 변수</strong>라 한다.
        <br>→ null은 객체가 <strong>주소를 가지지 못한 상태</strong>를 의미한다.
      </li>
    </ul>
  <li>
    널인 샅애의 객체를 이용한면 <strong>널 포인트 예외(NullPointException)</strong>가 발생한다. 널 포인트 예외는 <strong>널인 상태의 객체를 이용할 수 없다는 오류</strong>이다.
  </li>
  <li>
    널 안전성이란 <strong>널 포인트 예외가 발생하지 않도록 코드를 작성</strong>하는 것을 의미한다.
  </li>
</ul>

```kotlin
// 1. 객체에 널 대입.
// "hello" 값에 대한 수조를 갖는다.
val data1: String = "hello"
// null이기에 주소를 갖지 못한다.
val data2: String? = null

// 2. 널 안전성을 개발자가 고려한 코드.
fun main() {
    var data: String? = null
    val length = if (data == null) {
        0
    } else {
        data.length
    }
    println("data length : $length")
}

// 2. 코틀린이 제공하는 널 안전성 연산자를 이용한 코드.
fun main() {
    var data: String? = null
    // 두 단계로 안전성을 보장한다.
    //   - ?.  → null이면 호출하지 않고 null 반환
    //   - ?:  → null이면 기본값 반환
    println("data length : ${data?.length ?: 0"})
}
```

<h2>2-2. 널 안전성 연산자</h2>
<h3>2-2-1. 널 허용 - ? 연산잔</h3>
<ul>
  <li>
    앞서 설명하였듯이 코틀린은 <strong>널 허용(nullable)</strong>과 <strong>널 불허(not null)</strong>로 구분한다.
  </li>
</ul>

```kotlin
// 1. 널 허용과 널 불허.
// 불허
var data1: String = "kkang"
data1 = null
// 허용
var data2: String? = "kkang"
data2 = null
```

<h3>2-2-2. 널 안전성 호출 - ?. 연산자</h3>
<ul>
  <li>
    널 허용인 경우 널인 변수에 접근할 수 있다. 이때에는 null로 인한 널 포인트 예외가 발생할 수 있기 때문에 반드시 <strong>?. 연산자</strong>를 사용해야 한다.
  </li>
  <li>
    .? 연산자는 변수가 null이면 멤버에 접근하지 않고 <strong>null을 반환</strong>한다.
  </li>
</ul>

```kotlin
// 1. 널 포인트 예외 오류.
var data: String/ = "kkang"
var length = data.length

// 2. 널 안전성 호출 연산자 이용.
var data: String? = "kkang"
var length = data?.length
```

<h3>2-2-3. 엘비스 - ?: 연산자</h3> 
<ul>
  <li>
    엘비스 연산자는 <strong>:? 기호</strong>이다. 변수가 <strong>널 일때 대입하거나 실행</strong>해야하는 구문이 있는 경우 사용한다.
  </li>
</ul>

```kotlin
// 1. 엘비스 연산자 사용.
fun main() {
    var data: String? = "kkang"
    println("data = $data : ${data?.length ?: -1}")
    data = null
    // .?에서 null 반환 → ?:에서 -1을 대입.
    println("data = $data : ${data?.length ?: -1}")
}
```

<h3>2-2-4. 예외 발생 - !! 연산자</h3>
<ul>
  <li>
    <strong>!!</strong>는 객체가 null일 때 <strong>예외(exception)</strong>를 일으키는 연산자이다.
  </li>
    <ul>
      <li>
        .?과 ?:를 통해 예외를 발생하지 않게 할 수도 있지만 어떤 경우에는 예외가 필요하다.
      </li>
    </ul>
</ul>

```kotlin
// 1. 예외 발생 연산자.
fun some(data: String?): Int {
    return data!!.length
}
fun main() {
    // 정상 작동.
    println(some("kkang"))
    // 예외 발생.
    println(some(null))
}
```
