<h1>1. UI/UX의 중요성</h1>
<ul>
  <li>
    알고리즘에 관한 정보는 많지만 아직 추천 시스템을 위한 UI/UX의 중요성에 대한 자료는 부족하다.
  </li>
  <li>
    추천 시스템을 활용하는 서비스의 특징적인 UI/UX에 관해 소개한다.
  </li>
</ul>

<br><br>

<h1>2. 서비스를 사용하는 사용자의 목적에 적합한 UI/UX 사례</h1>
<h2>2-1. 적합 아이템 발견</h2>
<ul>
  <li>
    <strong>적합 아이템 발견(find good items)</strong>은 사용자가 <strong>자신의 목적(기호)</strong>을 달성하는 데 적합한 아이템을 <strong>서비스상에서 최소 1개 이상 발견</strong>하는 경우를 의미한다.
  </li>
  <li>
    적합 아이템 발견은 사용자의 기호에 맞을 가능성이 높은 아이템부터 <strong>차례대로 정렬한 리스트</strong>를 사용자에게 제시하는 것이 효과적이다.
  </li>
  <li>
    리스트 형식으로 아이템을 표시할 때에는 아이템 하나당 표시할 수 있는 <strong>정보량이 제한</strong>된다. 따라서 사용자가 선호할만한 정보를 <strong>취사 선택</strong>하여 리스트에 표시해야 한다.
  </li>
  <li>
    같은 음식이라도 음식점은 음식점을 분위기를 함께 보여줄 수 있도록 <strong>음식점 사진</strong>을 포함하는 것이 좋을 수 있으며 배달이라면 <strong>음식을 메인</strong>으로 표시하는 게 나을 수 있다.
  </li>
    <ul>
      <li>
        한 가지 알고리즘으로 생성한 하나의 리스트만 제공한 경우 사용자에게 반드시 맞는 아이템을 보장하기 어렵기에 다른 측면에서 아이템을 추천하는 알고리즘에 따라 정렬된 <strong>다양한 리스트</strong>도 제시하는 것이 좋다.
      </li>
      <li>
        화면 위의 탭을 활용하는 방식을 사용하면 간단히 화면을 전환하여 다른 추천 알고리즘을 확인할 수 있다.
      </li>
      <li>
        아이템을 가로 방향으로 배열하여 여러 리스트를 표시하면 사용자는 다른 화면으로 이동 없이 자연스럽게 다른 추천 리스트를 탐색할 수 있다. 단, 개별 아이템의 정보를 넣을 공간이 부족해지기에 동영상 등 텍스트 정보가 적은 경우 적합하다.
      </li>
      <li>
        매칭 앱 같은 경우 좌우로 슬라이드를 하면서 선호 여부를 즉각 비드백 받는다. 이러한 경우 사용자로부터 명확한 피드백을 받을 수 있으며 부정적 피드백을 활용한 추천 시스템 구현을 고려해 볼 수 있다.
      </li>
    </ul>
</ul>

<br>

<h2>2-2. 적합 아이템 나열</h2>
<ul>
  <li>
    적합 아이템 나열(find all good items)은 사용자가 자신의 목적을 달성하는 데 적합한 아이템을 <strong>가능한 모든 서비스</strong>에서 발견하는 경우를 의미한다. 따라서 <strong>가능한 많은 후보군</strong>을 염두해야하는 주거지 선택 등이 해당한다. <strong>비용</strong>이 많이 드는 서비스도 해당한다.
  </li>
  <li>
    특허의 경우 추천 성능은 보장하지 못하여도 누락된 데이터가 없도록 구축하는 편이 낫다.
  </li>
  <li>
    비교적 많은 정보가 추천에 제공되고 사용자는 이들 대부분을 탐색하는 경우가 많기에 정보를 <strong>열람하는 부담</strong>을 낮추는 경우가 많다.
  </li>
  <li>
    현재 추천에 포함된 <strong>아이템의 수</strong>를 제공하여 사용자로 하여금 총 몇 개의 정보를 볼 수 있는지 알 수 있게하고 아이템 수가 너무 많으면 조건으로 필터링이 되게 하거나 반대로 적으면 조건을 삭제 하는 등 달성 목적에 맞게 조건을 수정하도록 하는 것이 좋다.
  </li>
  <li>
    사용자의 <strong>조건이 너무 까다로워 표시되는 아이템의 수가 너무 적다면</strong> 사용자가 입력한 조건에는 맞지 않지만 사용자의 목적을 달성할 가능성이 있는 조건의 아이템을 <strong>의도적으로 표시</strong>하는 것도 도움이 된다.
  </li>
  <li>
    사용자가 추천 받은 아이템을 위해 <strong>재방문</strong>을 할 가능성이 있기에 사용자가 <strong>입력한 조건을 저장</strong>해 두거나 조건에 일치하는 아이템을 새롭게 서비스 내에서 <strong>나타나는 대로 사용자에게 전달</strong>하는 것도 효과적이다.
  </li>
  <li>
    같은 아이템이라 하더라도 목적에 따라 다른 추천을 필요로 하기에 UI/UX를 다르게 고려할 필요가 있을 수 있다.
  </li>
</ul>

<br>

<h2>2-3. 아이템 계열 소비</h2>
<ul>
  <li>
    <strong>아이템 계열 소비</strong>는 <strong>열람 혹은 소비</strong>하는 동안 추천된 아이템 계열 전체에서 가치를 누리는 것을 목적으로 하는 경우이다.
  </li>
    <ul>
      <li>
        유튜브 뮤직의 경우 개별 음악도 매력적이지만 순서대로 플레이 되는 재생 목록도 매력적이라 느낀다. (크리스마스 테마 노래를 듣는데 갑자기 다른 장르가 추천되지 안도록).
      </li>
    </ul>
  <li>
    아이템 계열 소비의 독특한 특징은 <strong>같은 아이템을 여러 번 추천</strong>해도 사용자 만족도가 향상된다는 점이다.
  </li>
    <ul>
      <li>
        음악과 같은 소모품이 아닌 경우에는 해당하지 않는 경우가 많다.
      </li>
    </ul>
  <li>
    이미 선호하고 있는 아이템을 추천하되 해당 아이템을 기준으로 사용자가 <strong>좋아할만한 아이템</strong>을 추천하는 것을 고려하는 것이 중요하다.
  </li>
</ul>

<br>

<h2>2-4. 서비스 내 체류</h2>
<ul>
  <li>
    서비스 내 체류(Just Browsing)는 사용자가 사용하는 서비스 본래의 목적을 달성하는 것이 아니라 <strong>그저 아이템을 열람</strong>하는 것 자체를 목저긍로 서비스 안을 돌아다는 경우이다.
  </li>
  <li>
    구입 의사가 별로 없는 사용자에게 아이템 <strong>구입을 과도하게 강요</strong>하는 방식의 추천은 만족도를 낮출 수 있다.
  </li>
    <ul>
      <li>
        대신 개요 추천 혹은 다른 카테고리의 인기 아이템 등을 추천하여 <strong>재미있고 흥미를 가질 수 있는 아이템</strong>을 추천하는 것이 긍정적인 서비스 인식에 보다났다.
      </li>
    </ul>
  <li>
    당장 구매로 이어지지는 않지만 서비스 안을 돌아다니는 사용자의 만족도 향상을 노리고 탐색적인 UI를 제공하는 서비스가 늘고 있다.
  </li>
  <li>
    서비스를 탐색할 때의 <strong>행동 로그</strong>를 활용하여 새로운 서비스를 제공하거나 탐색한 아이템이 추후 <strong>구매</strong>로 이어지는 기대효과를 기대할 수 있다.
  </li>
</ul>

<br><br>

<h1>3. 서비스 제공자의 목적에 맞는 UI/UX 사례</h1>
<h2>3-1. 신규/사용 빈도가 낮은 사용자의 정착</h2>
<ul>
  <li>
    정착을 위해서는 사용자에게 좋은 경험을 제공하는 것뿐만 아니라 나쁜 경험을 하지 않도록 하는 것이 중요하며 이런 상황에는 개<strong>요 추천(broad recommendation)</strong>을 사용한다.
  </li>
  <li>
    개요 추천이란 <strong>서비스 내 데이터 통계 정보</strong> 혹은 서비스 지식이 풍부한 <strong>편집자의 선택</strong>을 기반으로 추천하는 것을 의미한다.
  </li>
  <li>
    서비스 신규 사용자나 사용 빈도가 낮은 사용자를 대상으로할 경우 적극적으로 찾지 않아도 서비스를 열람할 수 있도록 첫 페이지 등 <strong>눈에 띄기 쉬운 위치</strong>에 배치하는 것이 좋으며 간단한 <strong>필터링 기능</strong>도 효과적이다.
  </li>
  <li>
    가급적 많은 사용자가 서비스를 이탈하지 않는 것이 목적이라면 특정 사용자를 기준으로 하기 보다는 <strong>여러 사용자가 조금씩 연관된 아이템</strong>을 추천하는 것이 좋다.
  </li>
</ul>

<br>

<h2>3-2. 서비스에 대한 신뢰성 향상</h2>
<ul>
  <li>
    사용자는 서비스 제공의 아이템이 제공자의 이익을 가장 우선한다 생각하는 불신감을 갖고 있기에 이를 <strong>신뢰로 전환</strong>하는 것이 중요하다.
  </li>
  <li>
    신뢰성을 높이는 방법으로는 <strong>사용자 평가</strong>가 효과적인 경우가 많다.
  </li>
    <ul>
      <li>
        사용자 평가는 서비스 안의 <strong>사용자 사이에서 추천</strong>을 구현하는 것이다. (별점 등).
      </li>
    </ul>
  <li>
    사용자 평가는 서비스 운영자가 아닌 <strong>다른 사용자의 평가</strong>를 기반으로 추천을 제공할 수 있기에 <strong>신뢰도</strong>를 높이면서 <strong>서비스 차별화</strong>와 <strong>이탈률 감소</strong> 등의 효과를 기대할 수 있다.
  </li>
  <li>
    단, 리뷰를 사용할 경우 <strong>리뷰 어뷰징</strong>이나 <strong>스팸</strong>이 섞일 수 있기에 이에 대한 대응을 마련하는 것이 중요하다.
  </li>
  <li>
    추가적인 방안으로는 열람하려는 사용자와 <strong>비슷한 기호를 갖는 사용자 리뷰</strong>를 우선 제공하는 방법과 비판적인 평가를 선호한다는 점을 활용해 <strong>비판적 리뷰부터 노출</strong>되도록 하는 방법을 사용할 수도 있다.
  </li>
  <li>
    서비스를 오래 그리고 자주 사용하는 사용자의 경우 <strong>개인화</strong>가 적합하며 반대로 신규 사용자와 같은 경우 <strong>서비스 인기 아이템</strong>을 표기하는 방법이 적합하다.
  </li>
    <ul>
      <li>
        사용자가 프로필 정보를 변경하는 등의 상황이 발생하면 더욱 기호에 맞는 아이템을 추천하는 <strong>명확한 피드백</strong>을 보내거나 <strong>'현재 개인화 중입니다.' 등의 메시지</strong>를 전달하는 것도 효과적이다.
      </li>
      <li>
        메시지와 함께 추천되는 아이템이 변경된다면 사용자는 서비스가 자신의 행동을 충분히 반영한다는 신뢰를 가질 수 있다.
      </li>
    </ul>
</ul>

<br>

<h2>3-3. 사용 빈도 향상 및 이탈 사용자의 복귀</h2>
<ul>
  <li>
    사용자가 지속적으로 서비스를 사용하게 하는 것과 사용 빈도가 낮거나 이미 이탈한 사용자를 관리할 때에는 <strong>알림 서비스(notification services)</strong>가 효과적이다.
  </li>
    <ul>
      <li>
        단, 이 방법은 서비스와 관련된 정보를 보낸다는 강한 의지를 보이는 표현이기 때문에 사용자에게 <strong>불쾌감</strong>을 줄 수 있어 주의해서 설계해야 한다. 
      </li>
    </ul>
  <li>
    중요하게 고려할 점 중 하나는 <strong>알림을 보내는 빈도</strong>이다. 빈도는 서비스의 목적과 사용자의 유형에 따라 달라진다.
  </li>
    <ul>
      <li>
        바로 열람을 하지 못하는 경우에는 서비스 안에서 알림 내용을 다시 접근하는 기능을 염두해 볼 수 있다.
      </li>
    </ul>
  <li>
    푸시 알림은 작은 크기로 <strong>정보가 제한</strong>되기에 적합한 아이템을 추천하도록 노력해야한다.
  </li>
    <ul>
      <li>
        꼭 구매와 이어지지는 않더라도 사용자가 관심을 가질만한 유니크한 아이템을 추천하는 것 또한 만족도를 높일 수 있다.
      </li>
    </ul>
  <li>
    송신할 아이템 외적으로는 내용이 확실히 전달되도록 정보의 형태</strong>에도 주의를 기울이는 것이 좋다.
  </li>
</ul>

<br>

<h2>3-4. 교차 판매</h2>
<ul>
  <li>
    어떤 상품의 구매를 검토하고 있는 사용자에게 <strong>다른 상품을 함께 또는 개별적으로 구입</strong>하도록하여 단가를 높이는 방법을 <strong>교차 판매(cross selling)</strong>이라 부른다.
  </li>
  <li>
    교차 판매를 구현하는 효과적인 수단은 <strong>연관 아이템 추천(product-associated recommendations)</strong>이 있다.
  </li>
    <ul>
      <li>
        연관 아이템 추천이란 현재 사용자가 주목하고 있는 아이템과 <strong>연관된 아이템을 사용자에게 표시</strong>하는 것이다.
      </li>
      <li>
        연관 아이템이 <strong>왜 연관되어 있는지, 왜 함께 자주 구입되는지</strong> 등 근거를 사용자가 알 수 있도록 설명을 추가하거나 합계 금액을 제시한다면 더욱 효과적이다.
      </li>
      <li>
        혹은 연관이 있는 아이템을 하나로 묶어 <strong>패키지</strong>처럼 이벤트를 적용하여 판매하는 방법도 염두할 수 있다.
      </li>
    </ul>
  <li>
    교차 판매의 목적과는 다르지만 사용자가 주목하고 있는 아이템과 동등한 유사 아이템을 추천하여 사용자가 <strong>비교 후 결정</strong>하도록 유도할 수 있다.
  </li>
    <ul>
      <li>
        이때에는 비교 기준을 함께 제공하는 것이 효과적일 수 있다.
      </li>
    </ul>
</ul>

<br>

<h2>3-5. 장기적으로 사용자의 충성도 향상</h2>
<ul>
  <li>
    장기적인 관점에서 <strong>충성도(loyality)</strong>를 높여 서비스를 지속 사용하게 하는 것 또한 중요하다. (특히 구독 서비스 등은 더욱).
  </li>
  <li>
    <strong>개인화(personalize)</strong>는 서비스 내에서 수집한 사용자 정보나 행동 이력을 활용해 <strong>해당 사용자에 적합한 추천</strong> 결과를 제공하는 것을 의미한다.
  </li>
    <ul>
      <li>
        개인화 추천의 경우 사용자가 <strong>개인화 추천을 받고 있음을 인지</strong>할 수 있게 메시지 등을 띄워주는 것도 효과적이다.
      </li>
      <li>
        개인화 추천이지만 추천이 적합하지 않은 경우 크게 실망할 수 있기에 주의해서 설계해야하며 추천 서비스를 이용할 것인지 혹은 추천 아이템에 피드백을 받아 추천을 고도화하는 등 <strong>사용자의 피드백</strong>을 고려할 수 있다.
      </li>
      <li>
        단, 피드백을 남기지 않는 사용자도 있음을 고려해야 한다.
      </li>
      <li>
        개인 정보에 민감한 사용자가 있을 수 있음으로 사용자 정보 중 <strong>어떤 정보를 활용</strong>하는지 명시하거나 원치 않는 경우 <strong>개인화를 하지 않는 것</strong>도 중요하다.
      </li>
    </ul>
</ul>

<br><br>

<h1>4. 연관 주제</h1>
<h2>4-1. 아이템의 유사도</h2>
<ul>
  <li>
    유사도를 사용해 추천을 하더라도 <strong>이미 구매를한 제품</strong>은 추천하지 않고 <strong>보완 아이템</strong>이나 <strong>관련 아이템</strong>을 추천하는 것이 적합하다.
  </li>
  <li>
    <strong>유사도의 기준</strong>은 다양하며 서비스와 사용자를 고려하여 유사도 설계하는 것이 중요하다.
  </li>
</ul>

<br>

<h2>4-2. 신선함, 흥미로움, 다양성</h2>
<ul>
  <li>
    사용자에게 추천하는 아이템에 있어 <strong>신선함(novelty)</strong>은 중요한 요소이다. 즉, 사용자가 관심을 보이지만 아직 알지 못하는 아이템을 의미한다.
  </li>
    <ul>
      <li>
        사용자가 관심을 갖는 <strong>관심 분야</strong>와 아직 해당 분야에서 잘 모르는 아이템이 갖는 <strong>신규성</strong>을 적절히 활용한다.
      </li>
    </ul>
  <li>
    <strong>흥미로움(serendipity)</strong> 또한 중요한 기준이다. 흥미로움은 신선함에 <strong>의외성</strong>이라는 요소가 더해진 개념이다.
  </li>
    <ul>
      <li>
        예를 들면 사용자가 좋아하는 작가와 유사한 분위기의 신인 작가의 신작의 경우 기존 <strong>관심</strong>을 유히자면서 신인 작가라는 <strong>신규성</strong>과 추천되는 아이템을 알수 없기에 <strong>의외성</strong>까지 포함된다.
      </li>
    </ul>
  <li>
    단, 의외성이라는 감정적 요소를 측정하기는 어렵기에 <strong>다양성(diversity)</strong>이라는 관점에서 <strong>정량적으로 측정</strong>을 한다.
  </li>
    <ul>
      <li>
        다양성은 추천된 여러 아이템이 <strong>서로 비슷하지 않은 정도</strong>를 측정한다. 유사도를 측정한 후 아이템 간 유사도를 측정하여 집계한다.
      </li>
    </ul>
</ul>

<br>

<h2>4-3. 추천 아이템 선별</h2>
<ul>
  <li>
    제공할 추천 아이템들이 준비가 되었다면 <strong>아이템의 속성</strong>에 따라 필요 없는 추천 아이템을 선별 및 관리할 필요가 있다.
  </li>
    <ul>
      <li>
        예를 들어 도서의 경우 재구매할 가능성이 낮기에 이미 구매한 아이템은 제거해야 한다. 반면 생수와 같은 소모품은 정기적 구매로 이어지기에 다시 구입 등의 서비스로 제공하는 것이 좋다.
      </li>
    </ul>
  <li>
    고가의 아이템의 경우 의사결정까지 시간이 걸릴 수 있어 이러한 특징을 고려하여 열람 이력 서비스 등을 제공할 수도 있다.
  </li>
</ul>

<br>

<h2>4-4. 추천 이유 제시</h2>
<ul>
  <li>
    추천 시스템에서 아이템의 <strong>추천 이유(explanation of recommendations)</strong>를 제시하면 추천 효과, 추천의 투명성, 사용자 만족도를 향상 시킬 수 있다.
  </li>
    <ul>
      <li>
        추천의 투명성이란 사용자가 입력한 평가 또는 기타 정보와 출력된 추천 아이템 사이의 <strong>인과 관계</strong>를 명확하게 알 수 있는 상태를 의미한다.
      </li>
    </ul>
  <li>
    사용자가 납득할만한 추천 이유와 함께 추천 아이템을 제공하는 것이 UX 적으로 좋다.
  </li>
</ul>
